# 数据库技术局部规则

- 建模笔记区分现实对象、概念模型、逻辑模型和物理模型，明确实体、关系、键、约束及规范化目标。
- SQL 示例注明 DBMS、版本和方言；提供最小表结构、示例数据假设及预期结果，代码块标为 `sql`。
- 区分逻辑查询结果与执行计划；性能结论附数据规模、索引、统计信息、硬件或执行环境。
- 事务笔记明确并发场景、隔离级别、读写顺序、异常现象和一致性约束，不只罗列 ACID 定义。
- 数据库管理内容注明权限、备份、恢复、迁移和回滚前提；高风险命令必须醒目标注，不在非实验环境默认执行。
- Oracle 或其他产品特性不得描述为通用 SQL 或通用关系数据库规律。
- 示例避免使用真实个人数据、凭据或生产连接信息；敏感字段使用明确的虚构值。
