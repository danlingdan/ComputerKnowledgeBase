# 4.3.2 串的存储结构

与线性表类似，串也有两种基本存储结构：顺序存储和链式存储。但考虑到存储效率和算法的方便性，串多采用顺序存储结构。

## 1. 串的顺序存储

采用顺序存储结构存储的串称为**顺序串**，用一组地址连续的存储单元存储串值的字符序列。定长数组描述如下：

```c
// - - - - - 串的定长顺序存储结构 - - - - -
#define MAXLEN 255                 // 串的最大长度
typedef struct {
    char ch[MAXLEN + 1];           // 存储串的一维数组
    int length;                    // 串的当前长度
} SString;
```

这种定义方式是静态的，编译时刻即确定串空间大小。由于串操作中长度变化较大，固定大小不尽合理，因此最好根据需要在程序执行时动态分配/释放字符数组空间。C 语言的“堆”（Heap）自由存储区可为每个新串动态分配实际所需空间，约定串长作为存储结构的一部分，称为**堆式顺序存储结构**：

```c
// - - - - - 串的堆式顺序存储结构 - - - - -
typedef struct {
    char *ch;                      // 若是非空串，则按串长分配存储区，否则 ch 为 NULL
    int length;                    // 串的当前长度
} HString;
```

> [!definition] 顺序串
> 采用顺序存储结构存储的串称为**顺序串**。本章后续算法所用顺序串约定：从下标为 1 的数组分量开始存储，下标为 0 的分量闲置不用。定长形式用 `SString`（静态数组），堆式形式用 `HString`（动态分配指针）。

## 2. 串的链式存储

顺序串的插入和删除需移动大量字符，因此可采用单链表方式存储串，称为**链串**。由于串中每个数据元素是一个字符，存在“结点大小”问题：每个结点可存一个字符（结点大小 1），也可存多个字符（结点大小 >1）。图 4.3（a）为结点大小 4 的链表，图 4.3（b）为结点大小 1 的链表；当结点大小 >1 且串长不是其整倍数时，末结点常补 `"#"` 等非串值字符。

为便于操作，链表除头指针外可附设尾指针并给出当前串长，称为**块链结构**：

```c
// - - - - - 串的链式存储结构 - - - - -
#define CHUNKSIZE 80               // 可由用户定义的块大小
typedef struct Chunk {
    char ch[CHUNKSIZE];
    struct Chunk *next;
} Chunk;
typedef struct {
    Chunk *head, *tail;            // 串的头和尾指针
    int length;                    // 串的当前长度
} LString;
```

> [!definition] 链串与块链结构
> 采用链式存储结构存储的串称为**链串**。结点大小的选择影响存储密度与处理效率：结点大小为 1 时运算方便但占用空间大；结点大小 >1 时需补非串值字符。附带头/尾指针与串长的链式结构称为**块链结构**（`LString`）。

链式存储在连接等操作上较方便，但总体不如顺序存储灵活，占用存储量大且操作复杂，故本章模式匹配算法采用定长顺序存储实现。
