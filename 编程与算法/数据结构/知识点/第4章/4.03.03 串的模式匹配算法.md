子串的定位运算通常称为**串的模式匹配**或串匹配，应用极为广泛（搜索引擎、拼写检查、语言翻译、数据压缩等）。

设有两个字符串 $S$ 和 $T$，设 $S$ 为主串（正文串），$T$ 为子串（模式）。在主串 $S$ 中查找与模式 $T$ 相匹配的子串，若匹配成功，确定相匹配子串第一个字符在主串 $S$ 中出现的位置。著名的模式匹配算法有 **BF 算法**和 **KMP 算法**。

> [!definition] 模式匹配
> 子串的定位运算称为**串的模式匹配**。设主串 $S$、模式 $T$，在主串中查找与模式匹配的子串；成功则返回模式第一个字符在主串中的位置，失败返回 0。可指定主串中查找起始位置 $\text{pos}$。

## 1. BF 算法

最简单直观的模式匹配算法是 **BF（Brute-Force）算法**，也称朴素算法或简单匹配算法。基本思路：从主串 $S$ 的起始位置 $\text{pos}$ 起，第一个字符与模式 $T$ 第一个字符比较；相等则继续比较后续字符，不相等则从主串下一个字符重新与模式首字符比较；若模式比较完毕则匹配成功，否则失败返回 0。

## 算法 4.1 BF 算法

【算法步骤】
① 分别利用计数指针 $i$ 和 $j$ 指示主串 $S$ 和模式 $T$ 中当前正待比较的字符位置，$i$ 初值为 $\text{pos}$，$j$ 初值为 1。
② 若两个串均未比较到串尾（$i \leqslant S.\text{length}$ 且 $j \leqslant T.\text{length}$），循环：比较 $S.\text{ch}[i]$ 与 $T.\text{ch}[j]$；相等则 $i,j$ 各指向下一位置；不相等则指针后退（$i = i - j + 2$，$j = 1$）重新开始。
③ 若 $j > T.\text{length}$，匹配成功，返回 $i - T.\text{length}$；否则返回 0。

【例 4.1】 主串 $S = "\text{abaabaabcde}"$，模式 $T = "\text{abaabc}"$，起始位置 $\text{pos}=1$，用 BF 算法求 $T$ 在 $S$ 中的位置，过程如图 4.4 所示。

（图4.4 BF算法的匹配过程，共4趟比较，最终在位置4匹配成功）

【算法描述】

```c
int Index_BF(SString S, SString T, int pos)
{ // 返回模式 T 在主串 S 中第 pos 个字符开始第一次出现的位置。若不存在，则返回 0
  // 其中，T 非空，1 ≤ pos ≤ S.length
    i = pos; j = 1;                // 初始化
    while (i <= S.length && j <= T.length) // 两个串均未比较到串尾
    {
        if (S.ch[i] == T.ch[j]) { ++i; ++j; } // 继续比较后继字符
        else { i = i - j + 2; j = 1; }        // 指针后退，重新开始匹配
    }
    if (j > T.length) return i - T.length;    // 匹配成功
    else return 0;                            // 匹配失败
}
```

> [!tip] BF 算法的指针回溯过程
> ```mermaid
> flowchart LR
>     A["i=pos, j=1"] --> B{"S[i]==T[j]?"}
>     B -->|相等| C["i++, j++"]
>     B -->|不等| D["i=i-j+2, j=1<br/>主串指针回溯"]
>     C --> E{"j>T.length?"}
>     B -->|i>S.length| F["返回 0（失败）"]
>     D --> B
>     E -->|是| G["返回 i-T.length（成功）"]
>     E -->|否| B
> ```

【算法分析】
BF 算法直观、某些场合效率高。考虑两种极端：
- **最好情况**：每趟不成功都发生在模式首字符。设主串长 $n$、子串长 $m$，成功匹配起始位置 $i$ 由 1 到 $n-m+1$ 等概率，平均比较次数
  $$ \sum_{i=1}^{n-m+1} p_i (i-1+m) = \frac{1}{n-m+1} \sum_{i=1}^{n-m+1} i-1+m = \frac{1}{2}(n+m) $$
  即最好平均时间复杂度 $O(n+m)$。
- **最坏情况**：每趟不成功都发生在模式末字符。平均比较次数
  $$ \sum_{i=1}^{n-m+1} p_i (i \times m) = \frac{1}{n-m+1} \sum_{i=1}^{n-m+1} i \times m = \frac{1}{2} m \times (n-m+2) $$
  即最坏平均时间复杂度 $O(n \times m)$。

BF 算法思路简明，但匹配失败时主串指针 $i$ 总回溯到 $i-j+2$、模式指针 $j$ 回溯到 1，时间复杂度高。

## 2. KMP 算法

KMP 算法由 Knuth、Morris、Pratt 共同设计，可在较大程度上避免重复遍历，在 $O(n+m)$ 数量级完成匹配。其改进在于：**主串指针 $i$ 不需回溯**，直接从失败位置 $i$ 继续向右“滑动”；模式指针 $j$ 虽需回溯，但不必回到开头，而是尽可能向右“滑动”。理解 KMP 需先回答两个问题：（1）主串指针 $i$ 为何不回溯？（2）模式指针 $j$ 应回溯到何处？

（图4.5 BF算法匹配过程示例：主串$S=$"abcdefghi"，模式$T=$"abcdx"）

如图 4.5，模式首字符 “a” 与其后子串 “bcdx” 任一字符均不等，既然后4字符已相等，则 $i$ 取 2~5 时首字符比较必失败，第二、三、四趟判断多余。若后面子串也含 “a”，则需考虑 $j$ 的变化。回顾例 4.1：第一趟 $i=6,j=6$ 不等时，BF 让 $i$ 回溯到 2、$j$ 到 1；但观察可知 $i$ 前两个字符 “ab” 正与模式前两个字符相等（图 4.6），故 $i$ 不必回溯，只需让 $j$ 回溯到第 3 个位置（图 4.7）。

为何 $j$ 可回溯到第 3 个位置？因为 $i$ 前两个字符与模式前两个字符相等。而此性质只需在模式 $T$ 本身比较即可：设当前 $j$ 所指标符前的子串为 $T'$（如图 4.8 第一趟 $j=6$ 时 $T' = "\text{abaab}"$），比较 $T'$ 的前缀与后缀是否相等。前缀/后缀指从起始/末尾连续取出的若干字符，且不能取串本身。以 $T' = "\text{abaab}"$ 为例：

- 长度 1：前缀 `a`，后缀 `b`，不等；
- 长度 2：前缀 `ab`，后缀 `ab`，相等；
- 长度 3：前缀 `aba`，后缀 `aab`，不等；
- 长度 4：前缀 `abaa`，后缀 `baab`，不等。

相等前后缀最大长度为 2，则 $j$ 可回溯到 3。归纳：当 $i,j$ 所指字符不等时，若 $T'$ 相等前后缀最大长度为 $l_{\text{max}}$，则 $j$ 回溯到 $l_{\text{max}}+1$。用 $\text{next}[j]$ 表示 $j$ 需回溯的位置，$T' = "t_1 t_2 \cdots t_{j-1}"$，则

$$
\text{next}[j] =
\begin{cases}
0 & j = 1 \\
l_{\text{max}} + 1 & \text{$T'$ 的相等前缀和后缀的最大长度为 } l_{\text{max}} \\
1 & \text{没有相等的前缀和后缀}
\end{cases}
\tag{4-1}
$$

对模式串 $T = "\text{abaabc}"$ 求得 $\text{next}$ 函数值如图 4.9 所示。

![[Attachments/Pasted image 20260717171740.png]]
（图4.9 模式串的next函数值，$j:1\ 2\ 3\ 4\ 5\ 6$，$T:\text{a}\ \text{b}\ \text{a}\ \text{a}\ \text{b}\ \text{c}$，$\text{next}[j]: 0\ 1\ 1\ 2\ 2\ 3$）

一般地，设主串 $"s_1 s_2 \cdots s_n"$、模式串 $"t_1 t_2 \cdots t_m"$。失配（$s_i \neq t_j$）时，若模式应滑至第 $k\ (k<j)$ 个字符比较，则需满足式 (4-2) 且不存在 $k'>k$ 满足式 (4-2)：

$$ "t_1 t_2 \cdots t_{k-1}" = "s_{i-k+1} s_{i-k+2} \cdots s_{i-1}" \tag{4-2} $$

已得“部分匹配”结果：

$$ "t_{j-k+1} t_{j-k+2} \cdots t_{j-1}" = "s_{i-k+1} s_{i-k+2} \cdots s_{i-1}" \tag{4-3} $$

推得：

$$ "t_1 t_2 \cdots t_{k-1}" = "t_{j-k+1} t_{j-k+2} \cdots t_{j-1}" \tag{4-4} $$

令 $\text{next}[j] = k$，引出模式串 $\text{next}$ 函数定义：

$$
\text{next}[j] =
\begin{cases}
0 & j=1 \ (t_1 \text{与 } s_i \text{不等时，下一步进行 } t_1 \text{与 } s_{i+1} \text{的比较}) \\
\max \{ k \mid 1 < k < j \text{ 且有 } "t_1 t_2 \cdots t_{k-1}" = "t_{j-k+1} t_{j-k+2} \cdots t_{j-1}" \} \\
1 & k=1 \ (\text{不存在相同子串，下一步进行 } t_1 \text{与 } s_i \text{的比较})
\end{cases}
\tag{4-5}
$$

由此求得模式串 $\text{next}$ 函数值如图 4.10 所示。

![[Attachments/Pasted image 20260717171748.png]]
（图4.10 模式串的next函数值，$j: 1\ 2\ 3\ 4\ 5\ 6\ 7\ 8$，模式串: $\text{a}\ \text{b}\ \text{a}\ \text{a}\ \text{b}\ \text{a}\ \text{b}\ \text{c}$，$\text{next}[j]: 0\ 1\ 1\ 2\ 2\ 3\ 1\ 2$）

匹配步骤：指针 $i,j$ 分别指示主串、模式待比较字符，$i$ 初值 $\text{pos}$，$j$ 初值 1。若 $s_i = t_j$ 则 $i,j$ 各增 1；否则 $i$ 不变、$j$ 退到 $\text{next}[j]$ 再比较；直至 $j$ 退到某 $\text{next}$ 值字符相等，或 $j$ 退到 0（模式首字符失配）时 $i,j$ 同时增 1，从主串下一字符起重匹配。过程示例如图 4.11。

![[Attachments/Pasted image 20260717171755.png]]
（图4.11 利用模式的next函数进行匹配的过程示例，含4趟匹配）

```c
int Index_KMP(SString S, SString T, int pos)
{ // 利用模式串 T 的 next 函数求 T 在主串 S 中第 pos 个字符之后的位置
  // 其中，T 非空，1 ≤ pos ≤ S.length
    i = pos; j = 1;
    while (i <= S.length && j <= T.length) // 两个串均未比较到串尾
    {
        if (j == 0 || S.ch[i] == T.ch[j]) { ++i; ++j; } // 继续比较后继字符
        else j = next[j];                               // 模式串向右移动
    }
    if (j > T.length) return i - T.length;    // 匹配成功
    else return 0;                            // 匹配失败
}
```

> [!tip] KMP 算法的指针滑动（主串 $i$ 不回溯）
> ```mermaid
> flowchart LR
>     A["i=pos, j=1"] --> B{"S[i]==T[j]?"}
>     B -->|相等| C["i++, j++"]
>     B -->|不等且 j>0| D["j=next[j]<br/>主串 i 不回溯"]
>     B -->|不等且 j==0| H["i++, j++"]
>     C --> E{"j>T.length?"}
>     D --> B
>     H --> B
>     E -->|是| G["返回 i-T.length（成功）"]
>     B -->|i>S.length| F["返回 0（失败）"]
> ```

> [!example] 算法 4.2 KMP 算法
> 与算法 4.1（BF）形式极相似，区别在于失配时**指针 $i$ 不变，指针 $j$ 退回到 $\text{next}[j]$** 重新比较；当 $j$ 退至 0 时，$i$ 与 $j$ 同时增 1（即从主串第 $i+1$ 个字符起重匹配）。

如何求 $\text{next}$ 函数值？其值仅取决于模式串本身，可用递推法求得。由定义：
$$ \text{next}[1] = 0 \tag{4-6} $$
设 $\text{next}[j] = k$，则存在关系：
$$ "t_1 t_2 \cdots t_{k-1}" = "t_{j-k+1} t_{j-k+2} \cdots t_{j-1}" \tag{4-7} $$
此时 $\text{next}[j+1]$ 有两种情况：
（1）若 $t_k = t_j$，则：
$$ "t_1 t_2 \cdots t_k" = "t_{j-k+1} t_{j-k+2} \cdots t_j" \tag{4-8} $$
不可能存在 $k'>k$ 满足式 (4-8)，故 $\text{next}[j+1] = k+1$，即：
$$ \text{next}[j+1] = \text{next}[j] + 1 \tag{4-9} $$
（2）若 $t_k \neq t_j$，把求 $\text{next}$ 看成模式匹配问题（整个模式串既为主串又为模式），当前已有 $t_{j-k+1}=t_1, \dots, t_{j-1}=t_{k-1}$，当 $t_k \neq t_j$ 时模式右滑至第 $\text{next}[k]$ 个字符与第 $j$ 个字符比较。若 $\text{next}[k]=k'$ 且 $t_j = t_{k'}$，则：
$$ "t_1 t_2 \cdots t_{k'}" = "t_{j-k'+1} t_{j-k'+2} \cdots t_j" \quad (1 < k' < k < j) \tag{4-10} $$
即 $\text{next}[j+1] = k'+1$：
$$ \text{next}[j+1] = \text{next}[k] + 1 \tag{4-11} $$
同理若 $t_k \neq t_{k'}$ 继续右滑……直至匹配成功或不存在 $k'$ 满足式 (4-10)，则：
$$ \text{next}[j+1] = 1 \tag{4-12} $$
例：图 4.12 模式串已求得前 6 个 $\text{next}$，求 $\text{next}[7]$：$\text{next}[6]=3$，而 $t_6 \neq t_3$，比较 $t_6$ 与 $t_1$（$\text{next}[3]=1$），$t_6 \neq t_1$ 且 $\text{next}[1]=0$，故 $\text{next}[7]=1$；$t_7 = t_1$，故 $\text{next}[8]=2$。

![[Attachments/Pasted image 20260717171806.png]]
（图4.12 求next[7]的递推过程示意）

```c
void get_next(SString T, int next[])
{ // 求模式串 T 的 next 函数值并将其存入数组 next
    i = 1; next[1] = 0; j = 0;
    while (i < T.length)
    {
        if (j == 0 || T.ch[i] == T.ch[j]) { ++i; ++j; next[i] = j; }
        else j = next[j];
    }
}
```

> [!example] 算法 4.3 计算 $\text{next}$ 函数值
> 由式 (4-6)、(4-9)、(4-11)、(4-12) 仿照 KMP 算法递推求得。时间复杂度 $O(m)$。因模式长 $m$ 通常远小于主串长 $n$，增加这点运算是值得的。

> [!warning] next 函数的缺陷与修正
> 上述 $\text{next}$ 函数在某些情况下有缺陷。例如模式 `"aaaab"` 与主串 `"aaabaaaab"` 匹配，当 $i=4,j=4$ 失配时，按 $\text{next}$ 还需 $i=4$ 与 $j=3,2,1$ 比较 3 次。但模式中第 1~3 字符与第 4 字符均相等，无需再与第 4 字符比，可直接右滑 4 位进行 $i=5,j=1$ 比较。即若 $\text{next}[j]=k$ 且 $t_j = t_k$，则失配时不必与 $t_k$ 比，而直接与 $t_{\text{next}[k]}$ 比，此时 $\text{next}[j]$ 应与 $\text{next}[k]$ 相同。由此得修正算法 4.4，结果如图 4.13，匹配算法不变。

![[Attachments/Pasted image 20260717171816.png]]
（图4.13 next函数修正值，$j:1\ 2\ 3\ 4\ 5$，模式串: $\text{a}\ \text{a}\ \text{a}\ \text{a}\ \text{b}$，$\text{next}[j]: 0\ 1\ 2\ 3\ 4$，$\text{nextval}[j]: 0\ 0\ 0\ 0\ 4$）

```c
void get_nextval(SString T, int nextval[])
{ // 求模式串 T 的 next 函数修正值并将之存入数组 nextval
    i = 1; nextval[1] = 0; j = 0;
    while (i < T.length)
    {
        if (j == 0 || T.ch[i] == T.ch[j])
        {
            ++i; ++j;
            if (T.ch[i] != T.ch[j]) nextval[i] = j;
            else nextval[i] = nextval[j];
        }
        else j = nextval[j];
    }
}
```

> [!example] 算法 4.4 计算 $\text{next}$ 函数修正值
> 在 `get_next` 基础上，当 $T.\text{ch}[i] == T.\text{ch}[j]$ 时，若 $\text{nextval}[j]$ 已等于 $j$ 对应值则直接继承 $\text{nextval}[j]$，避免与相等字符重复比较。

> [!note] 两种算法对比与 KMP 特点
> - BF 时间复杂度 $O(n \times m)$，但一般情况下实际执行时间近似 $O(n+m)$，至今仍被采用。
> - KMP 仅当主串与模式存在许多“部分匹配”时才显著快于 BF；其最大特点是**主串指针不需回溯**，对整个匹配只需从头至尾查一遍，适合边读入边匹配（如处理庞大外存文件）。
> - 模式匹配在现代生物学（基因序列分析、生物标志物识别、蛋白质序列分析）与网络安全（入侵检测系统 Snort 规则匹配）中均至关重要。KMP 还有 BM、Horspool 等改进版本。
