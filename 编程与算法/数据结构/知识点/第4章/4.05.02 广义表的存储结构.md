# 4.5.2 广义表的存储结构

> [!nav] 导航
> 上一知识点：[[4.05.01 广义表的定义]] · [[MOC - 第4章 串、数组和广义表|本章目录]] · [[MOC - 数据结构|课程总览]] · 下一知识点：[[4.06 案例分析与实现]]

> [!topic] 所属主题
> [[MOC - 第4章 串、数组和广义表#4.5 广义表|4.5 广义表]]

由于广义表元素结构不同（原子或列表），难以用顺序存储，通常采用链式存储。常用两种：头尾链表的存储结构和扩展线性链表的存储结构。

> [!definition] 广义表的两种链式存储结构
> - **头尾链表存储结构**：表结点（tag=1，含 hp 表头指针、tp 表尾指针）表示子表，原子结点（tag=0，含值域 atom）表示原子。
> - **扩展线性链表存储结构**：表结点与原子结点均由 3 个域组成（tag、数据/表头、tp 尾指针），原子结点也带 tp 指针。

## 1. 头尾链表的存储结构

![[Attachments/Pasted image 20260717171934.png]]
（图4.19 头尾链表结点结构：表结点(tag=1|hp|tp)；原子结点(tag=0|atom)）

一对确定的表头和表尾可唯一确定广义表。表结点由 3 个域组成（标志域、表头指针域、表尾指针域），原子结点由两个域组成（标志域、值域）。`tag=1` 表示子表，`tag=0` 表示原子。

```c
// - - - - - 广义表的头尾链表存储表示 - - - - -
typedef enum {ATOM, LIST} ElemTag;   // ATOM == 0 : 原子; LIST == 1 : 子表
typedef struct GLNode {
    ElemTag tag;                     // 公共部分，用于区分原子结点和表结点
    union {                          // 原子结点和表结点的联合部分
        AtomType atom;               // atom 是原子结点的值域，AtomType 由用户定义
        struct { struct GLNode *hp, *tp; } ptr; // ptr 是表结点的指针域，ptr.hp 和 ptr.tp 分别指向表头和表尾
    };
} *GList;                            // 广义表类型
```

4.5.1 的例子的存储结构如图 4.20 所示。该结构有 3 个特点：![[Attachments/Pasted image 20260717171942.png]]
（1）除空表头指针为空外，非空广义表的表头指针均指向一个表结点，`hp` 指向表头（原子或表结点），`tp` 指向表尾（除非表尾为空）；
（2）易分清原子和子表层次（如 $D$ 中 $a,e$ 同层，$b,c,d$ 低一层，$B,C$ 同层子表）；
（3）最高层表结点个数即广义表长度。
