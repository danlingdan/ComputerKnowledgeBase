**【算法练习题 3.1】 LeetCode 150 逆波兰表达式求值 ★★**

> [!example] 练习题 3.1　逆波兰表达式求值
> 【问题描述】根据逆波兰表示法求表达式的值。有效算符包括 `+`、`-`、`*`、`/`，每个运算对象可以是整数或另一个逆波兰表达式。给定逆波兰表达式总是有效，整数除法只保留整数部分。
> 【输入输出示例】输入：`tokens = ["10","6","9","3","+","-11","*","/","*","17","+","5","+"]`；输出：`22`。该算式转化后的常见中缀表达式为：
> $$ ((10 * (6 / ((9 + 3) * -11))) + 17) + 5 $$
> $$ = ((10 * (6 / (12 * -11))) + 17) + 5 $$
> $$ = ((10 * (6 / -132)) + 17) + 5 $$
> $$ = ((10 * 0) + 17) + 5 $$
> $$ = (0 + 17) + 5 $$
> $$ = 17 + 5 $$
> $$ = 22 $$
> 【问题分析】逆波兰表达式（Reverse Polish Notation，RPN）由波兰逻辑学家卢卡西维兹提出，特点是无括号、运算符总在相关操作数之后，故也称后缀表达式。计算时用一个栈存储操作数，从左到右遍历：遇操作数入栈；遇运算符则将栈顶两个操作数出栈（先出栈为右操作数，后出栈为左操作数）运算，结果入栈。遍历结束栈内唯一元素即表达式的值。具体实现步骤如图 3.17 所示。
> ![[Attachments/Pasted image 20260717163447.png]]
> （图 3.17 LeetCode 150 具体实现步骤）
> 【算法步骤】① 遍历逆波兰表达式：若 `token` 为操作数则转化为整型入栈；若为运算符则出栈两操作数运算后入栈。② 遍历结束，栈内元素即表达式的值 `res`。③ 返回 `res`。
> 【算法描述】
> ```c
> bool isNumber(char* token)
> {// 判断 token 是否为操作数
>     return strlen(token) > 1 || ('0' <= token[0] && token[0] <= '9');
> }
>
> int evalRPN(char** tokens, int tokensSize)
> {// 求逆波兰表达式的值
>     int n = tokensSize;
>     int* stk = (int*)malloc(10000 * sizeof(int));
>     int top = 0;
>     for (int i = 0; i < n; i++)                     // 遍历 tokens
>     {
>         char* token = tokens[i];
>         if (isNumber(token))                        // 遇到操作数，字符转化为整型数据，入栈
>             stk[top++] = atoi(token);
>         else                                        // 遇到运算符，将两个操作数出栈
>         {
>             long long num2 = stk[--top];            // num2 记录右操作数
>             long long num1 = stk[--top];            // num1 记录左操作数
>             switch (token[0])                       // 运算得到的新操作数入栈
>             {
>                 case '+': stk[top++] = num1 + num2; break;
>                 case '-': stk[top++] = num1 - num2; break;
>                 case '*': stk[top++] = num1 * num2; break;
>                 case '/': stk[top++] = num1 / num2; break;
>             }
>         }
>     }
>     int res = stk[top - 1];
>     return res;                                     // 返回逆波兰表达式的值
> }
> ```
> 【算法分析】遍历 `tokens` 一次，时间复杂度 $O(n)$；栈内元素个数不超过表达式长度，空间复杂度 $O(n)$。

**【算法练习题 3.2】 LeetCode 1249 移除无效的括号 ★★**

> [!example] 练习题 3.2　移除无效的括号
> 【问题描述】给定由 `(`、`)` 和小写字母组成的字符串 `s`，删除最少数目的括号使剩下的字符串有效。有效括号字符串指：空串或仅小写字母；可写作 `AB`（`A`、`B` 均有效）；或写作 `(A)`（`A` 有效）。
> 【输入输出示例】输入：`s = "lee(t(c)o)de)"`；输出：`"lee(t(c)o)de"`。
> 【问题分析】遍历字符串，利用栈统计需删除的括号并在原串中标记为 `-1`：遇 `(` 则下标入栈；遇 `)` 且栈空则标记为 `-1`，否则与栈顶 `(` 匹配并弹出。遍历结束后将栈中剩余 `(` 标记为 `-1`；再遍历修改后的串，将非 `-1` 的有效字符存入 `news` 返回。具体实现步骤如图 3.18 所示。
> ![[Attachments/Pasted image 20260717163505.png]]
> （图 3.18 LeetCode 1249 具体实现步骤）
> 【算法步骤】① 遍历 `s`：小写字母跳过；`(` 下标入栈；`)` 时栈空则标记 `-1`，否则弹出栈顶。② 将栈中剩余字符标记 `-1`。③ 遍历修改后的 `s`，将非 `-1` 元素存入 `news`。④ 返回 `news`。
> 【算法描述】
> ```c
> char* minRemoveToMakeValid(char* s)
> {// 返回有效括号字符串
>     int n = strlen(s);
>     int* stk = (int*)malloc(n * sizeof(int));
>     int top = 0;
>     for (int i = 0; i < n; i++)                      // 遍历 s
>     {
>         if (s[i] != '(' && s[i] != ')')
>             continue;
>         else if (s[i] == '(')                       // 遇到 "("，将其下标入栈
>             stk[top++] = i;
>         else                                        // 遇到 ")"
>         {
>             if (top == 0)                           // 栈为空，将其在原字符串中标记为 -1
>                 s[i] = -1;
>             else                                    // 栈不为空，弹出栈顶元素
>                 stk[--top];
>         }
>     }
>     for (int i = 0; i < top; i++)                   // 处理栈中剩余的不合理括号
>         s[stk[i]] = -1;
>     int i = 0, j = 0;
>     char* news = (char*)calloc(n+1, sizeof(char));
>     while (j < n)                                   // 遍历字符串，生成有效括号字符串
>     {
>         if (s[j] != -1)
>             news[i++] = s[j++];
>         else
>             j++;
>     }
>     return news;
> }
> ```
> 【算法分析】整体时间复杂度 $O(n)$；最坏情况下栈存储 $n$ 个元素且 `news` 长度也为 $n$，空间复杂度 $O(n)$。

**【算法练习题 3.3】 LeetCode 1823 找出游戏的获胜者 ★★**

> [!example] 练习题 3.3　找出游戏的获胜者
> 【问题描述】$n$ 名小伙伴围成一圈顺时针从 1 到 $n$ 编号。游戏规则：从小伙伴 1 开始，顺时针数 $k$ 名（含起始者），最后一名离开圈子；若圈中仍不止一名，则从下一位开始重复；只剩一名即获胜。给定 $n$ 和 $k$，返回获胜者编号。
> 【输入输出示例】输入：$n=5, k=2$；输出：3。
> 【问题分析】将 $n$ 个小伙伴编号存入无头结点链队。每轮将队头出队并在队尾重新入队，重复 $k-1$ 次；之后队头元素即本轮第 $k$ 名小伙伴，将其取出（离开圈子）。每轮后新队头为下一轮起始，圈中减少一人；当只剩 1 个元素时即获胜者。游戏过程如图 3.19 所示。
> ![[Attachments/Pasted image 20260717163514.png]]
> 游戏过程如图 3.19 所示。
> （图 3.19 游戏过程）
> 具体实现步骤如图 3.20 所示。
> ![[Attachments/Pasted image 20260717163521.png]]
> （图 3.20 LeetCode 1823 具体实现步骤）
> 【算法步骤】① 定义 `head`、`tail` 指针初始为空。② 将 $n$ 个编号存入无头结点链队。③ 当队列元素个数大于 1 时循环：重复 $k-1$ 次将队头出队并在队尾重新入队；再将队头出队。④ 返回获胜者编号 `res`。
> 【算法描述】
> ```c
> int findTheWinner(int n, int k)
> {// 返回游戏的获胜者，本题链队的存储结构同第 2 章给出的链表结构
>     struct ListNode* head = NULL;
>     struct ListNode* tail = NULL;
>     for (int i = 1; i <= n; i++)
>     {
>         struct ListNode* node = (struct ListNode*)malloc(sizeof(struct ListNode));
>         node->val = i;
>         node->next = NULL;
>         if (!head) { head = node; tail = node; }
>         else { tail->next = node; tail = node; }
>     }
>     while (head != tail)                            // 队列不止一个元素
>     {
>         for (int i = 1; i < k; i++)                 // 循环 k-1 次
>         {
>             struct ListNode* node = head;
>             head = head->next;                      // 队头元素出队
>             tail->next = node;                      // 队尾处重新入队
>             tail = tail->next;
>             tail->next = NULL;
>         }
>         struct ListNode* node = head;
>         head = head->next;                          // 队头元素出队
>         free(node);
>     }
>     int res = head->val;
>     free(head);
>     return res;                                     // 返回获胜者编号
> }
> ```
> 【算法分析】初始入队 $O(n)$，每轮将 $k$ 个元素出/入队、共 $n-1$ 轮，时间复杂度 $O(nk)$；队列最多 $n$ 个元素，空间复杂度 $O(n)$。

**【算法练习题 3.4】 LeetCode 32 最长有效括号 ★★★**

> [!example] 练习题 3.4　最长有效括号
> 【问题描述】给定只含 `(` 和 `)` 的字符串，找出最长有效括号子串的长度（格式正确且连续的括号子串）。
> 【输入输出示例】输入：`s = ")()())"`；输出：`4`（最长子串为 `()()`）。
> 【问题分析】借助一个栈追踪未匹配的右括号位置以计算最长有效括号子串长度。遍历时始终确保栈底为已遍历元素中最后一个未匹配右括号的下标，栈中其余元素为左括号下标。初始压入 `-1`；遇 `(` 下标入栈；遇 `)` 弹出栈顶，若弹出后栈空则将其下标入栈（更新栈底），否则以当前下标减去栈顶元素（下一个未匹配右括号下标）得以该右括号结尾的最长子串长度。每步更新 `maxans`，遍历结束返回 `maxans`。具体实现步骤如图 3.21 所示。
> ![[Attachments/Pasted image 20260717163530.png]]
> （图 3.21 LeetCode 32 具体实现步骤）
> 【算法步骤】① 栈 `stk` 初始化并存入 `-1`。② 遍历 `s`：遇 `(` 下标入栈；遇 `)` 弹出栈顶，若弹出后栈空则将其下标入栈，否则以其下标减栈顶得最长长度。③ 返回 `maxans`。
> 【算法描述】
> ```c
> int longestValidParentheses(char* s)
> {// 最长有效括号子串长度
>     int maxans = 0, n = strlen(s);
>     int* stk = (int*)malloc((n + 1) * sizeof(int));
>     int top = 0;
>     stk[top++] = -1;
>     for (int i = 0; i < n; i++)
>     {
>         if (s[i] == '(')                           // 遇到 "("，下标入栈
>             stk[top++] = i;
>         else
>         {
>             --top;                                 // 遇到 ")"，弹出栈顶元素与之匹配
>             if (top == 0)
>                 stk[top++] = i;                    // 栈为空，下标入栈
>             else                                   // 栈不为空，返回最长有效括号子串长度
>                 maxans = fmax(maxans, i - stk[top - 1]);
>         }
>     }
>     return maxans;
> }
> ```
> 【算法分析】只需遍历字符串一次，时间复杂度 $O(n)$；最坏情况下栈大小达 $n$，空间复杂度 $O(n)$。

**【算法练习题 3.5】 LeetCode 1106 解析布尔表达式 ★★★**

> [!example] 练习题 3.5　解析布尔表达式
> 【问题描述】给定字符串形式的布尔表达式 `expression`，返回运算结果。约定：`"t"`→true；`"f"`→false；`"!(expr)"`→NOT；`"&(expr1,expr2,...)"`→AND；`"|(expr1,expr2,...)"`→OR。
> 【输入输出示例】输入：`expression = "!(f)"`；输出：`true`。
> 【问题分析】利用栈结构解析和计算布尔表达式。逐字符遍历：非右括号字符入栈；遇右括号则弹出栈中相应子表达式，统计 `t` 和 `f` 的数量，根据操作符计算结果再入栈。最终栈顶元素即整个表达式的布尔值。具体实现步骤如图 3.22 所示。
> ![[Attachments/Pasted image 20260717163545.png]]
> （图 3.22 LeetCode 1106 具体实现步骤）
> 【算法步骤】① 初始化字符栈 `stack`，栈顶指针 `top` 为 0。② 遍历每个字符：逗号跳过；非 `)` 则入栈；是 `)` 则统计括号内 `t`、`f` 数量。③ 根据操作符 `op` 计算布尔结果。④ 遍历完成返回栈顶元素。
> 【算法描述】
> ```c
> bool parseBoolExpr(char* expression)
> {// 计算布尔表达式
>     int n = strlen(expression);
>     char stack[n];                                 // 用于存储字符的栈
>     int top = 0;                                   // 栈顶指针
>     for (int i = 0; i < n; i++)
>     {
>         char c = expression[i];
>         if (c == ',')                              // 跳过逗号
>             continue;
>         if (c != ')')                              // 如果当前字符不是右括号，则将其入栈
>         {
>             stack[top++] = c;
>             continue;
>         }
>         int num_t = 0, num_f = 0;
>         while (stack[top - 1] != '(')              // 统计括号内的 "t" 和 "f" 的数量
>         {
>             char val = stack[--top];
>             if (val == 't') num_t++;
>             else if (val == 'f') num_f++;
>         }
>         top--;                                     // 弹出左括号
>         char op = stack[--top];                    // 获取括号前的操作符
>         switch (op)                                // 根据操作符计算结果并入栈
>         {
>             case '!': stack[top++] = (num_f == 1) ? 't' : 'f'; break;   // 非
>             case '&': stack[top++] = (num_f == 0) ? 't' : 'f'; break;   // 与
>             case '|': stack[top++] = (num_t > 0) ? 't' : 'f'; break;   // 或
>             default: break;
>         }
>     }
>     return stack[top - 1] == 't';                  // 返回最终结果
> }
> ```
> 【算法分析】每个字符只被访问一次，时间复杂度 $O(n)$；最坏情况下栈高接近表达式长度一半，空间复杂度 $O(n)$。
