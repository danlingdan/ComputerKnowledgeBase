> [!definition] 递归（Recursion）
> 若在一个函数、过程或数据结构定义的内部又直接（或间接）出现定义本身的应用，则称其是递归的，或称递归定义的。以下 3 种情况常使用递归。

**1. 定义是递归的**

很多数学函数是递归定义的，如阶乘函数：

$$ Fact(n) = \begin{cases} 1 & n=0 \\ n \cdot Fact(n-1) & n>0 \end{cases} \quad (3-1) $$

二阶斐波那契数列：

$$ Fib(n) = \begin{cases} 1 & n=1 \text{ 或 } n=2 \\ Fib(n-1)+Fib(n-2) & \text{其他情形} \end{cases} \quad (3-2) $$

> [!example] 阶乘与斐波那契的递归程序
> 对于 (3-1) 的阶乘函数，可用递归过程求解：
> ```c
> long Fact(long n)
> {
>     if (n==0) return 1;                 // 递归终止的条件
>     else return n*Fact(n-1);            // 递归步骤
> }
> ```
> ![[Attachments/Pasted image 20260717163240.png]]
> 图 3.7 为主程序调用 `Fact(4)` 的执行过程：参数依次为 3、2、1、0 递归调用，最后一次因 $n=0$ 执行 `if` 分支终止，逐步返回时依次计算 $1\times1$、$2\times1$、$3\times2$、$4\times6$，最终将 24 返回主程序。
> 类似地，斐波那契数列的递归程序为：
> ```c
> long Fib(long n)
> {
>     if (n==1 || n==2) return 1;         // 递归终止的条件
>     else return Fib(n-1)+Fib(n-2);      // 递归步骤
> }
> ```

对于类似复杂问题，若分解成几个相对简单且解法相同或类似的子问题来求解，称为递归求解。例如计算 $4!$ 时先计算 $3!$，再分解求解，这种“分解-求解”策略称为**分治法**。采取分治法求解递归问题需满足 3 个条件：
（1）能将问题转变成解法相同或类似的新问题，仅处理对象更小且变化有规律；
（2）可通过上述转化使问题简化；
（3）必须有明确的递归出口（递归边界）。

分治法求解递归问题算法的一般形式为：
```c
void p(参数表)
{
    if (递归结束条件成立) 可直接求解;            // 递归终止的条件
    else p(较小的参数);                         // 递归步骤
}
```
阶乘函数和斐波那契数列的递归过程均与此对应。

**2. 数据结构是递归的**

某些数据结构本身具有递归特性，其操作可递归地描述。例如链表的结点 `LNode` 由数据域 `data` 和指针域 `next` 组成，而 `next` 是指向 `LNode` 类型的指针，即 `LNode` 的定义中又用到自身，故链表是一种递归的数据结构。

> [!example] 算法 3.9 遍历输出链表中各个结点的递归算法
> 【算法步骤】
> ① 如果 `p` 为 `NULL`，递归终止。
> ② 否则输出 `p->data`，`p` 指向后继结点继续递归。
> 【算法描述】
> ```c
> void TraverseList(LinkList p)
> {
>     if(p==NULL) return;                 // 递归终止
>     else
>     {
>         cout<<p->data<<endl;             // 输出当前结点的数据域
>         TraverseList(p->next);          // p 指向后继结点继续递归
>     }
> }
> ```
> 当递归结束条件成立只执行 `return` 时，可简化为：
> ```c
> void TraverseList(LinkList p)
> {
>     if (p)
>     {
>         cout<<p->data<<endl;
>         TraverseList(p->next);
>     }
> }
> ```
> 广义表、二叉树等也是典型的递归数据结构，相应算法也可采用递归实现。

**3. 问题的解法是递归的**

还有一类问题本身无明显递归结构，但用递归求解比迭代更简单，如 Hanoi（汉诺）塔问题、八皇后问题、迷宫问题等。

> [!example] 例 3.1　n 阶 Hanoi 塔问题
> 【问题描述】有 3 个命名为 A、B、C 的塔座，塔座 A 上插有 $n$ 个直径不同、依小到大编号为 $1,2,\cdots,n$ 的圆盘（见图 3.8）。要求将 A 上的 $n$ 个圆盘移至 C 上并按同样顺序叠排，移动须遵循：① 每次只能移动一个圆盘；② 圆盘可插在 A、B、C 任一塔座上；③ 任何时刻都不能将较大圆盘压在较小圆盘之上。
> 【问题分析】用分治递归法。设 A 上最初盘子总数为 $n$：当 $n=1$ 时，直接将 1 号盘从 A 移到 C；否则执行三步：① 用 C 作过渡，将 A 上 $(n-1)$ 个盘移到 B；② 将 A 上最后一个盘直接移到 C；③ 用 A 作过渡，将 B 上 $(n-1)$ 个盘移到 C。图 3.8 展示了 $n=4$ 的移动过程。
> 搬动操作定义为 `move(A, n, C)`，指将 $n$ 号盘从 A 移到 C；设全局变量 `m` 初值为 0 对搬动计数：
> ```c
> int m=0;
> void move(char A, int n, char C)
> {
>     cout<<++m<<", "<<n<<", "<<A<<", "<<C<<endl;
> }
> ```
> 【算法步骤】
> ① 若 $n=1$，直接将 1 号盘从 A 移到 C，递归结束。
> ② 否则：递归将 A 上 1 至 $(n-1)$ 号盘移到 B（C 作辅助）；直接将 $n$ 号盘从 A 移到 C；递归将 B 上 1 至 $(n-1)$ 号盘移到 C（A 作辅助）。
> 【算法描述】
> ```c
> void Hanoi(int n, char A, char B, char C)
> {// 将塔座A上的n个圆盘按规则搬到塔座C上，塔座B作为辅助塔座
>     if (n==1) move(A, 1, C);             // 将编号为1的圆盘从塔座A移到塔座C
>     else
>     {
>         Hanoi(n-1, A, C, B);             // 将A上编号为1至n-1的圆盘移到塔座B，塔座C作为辅助塔座
>         move(A, n, C);                   // 将编号为n的圆盘从塔座A移到塔座C
>         Hanoi(n-1, B, A, C);             // 将B上编号为1至n-1的圆盘移到塔座C，塔座A作为辅助塔座
>     }
> }
> ```
