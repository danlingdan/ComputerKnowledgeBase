# 3.8 小结

> [!nav] 导航
> 上一知识点：[[3.07 LeetCode 算法练习题]] · [[MOC - 第3章 栈和队列|本章目录]] · [[MOC - 数据结构|课程总览]]

> [!summary] 本章要点
> 本章介绍两种特殊的线性表：栈和队列。
> 1. **栈**：限定仅在表尾插入或删除的线性表，又称后进先出（LIFO）线性表。有顺序表示（顺序栈）和链式表示（链栈）；主要操作是进栈和出栈，顺序栈需注意判断栈满/栈空。
> 2. **队列**：先进先出（FIFO）线性表，只允许在一端插入、另一端删除。有顺序表示（循环队列）和链式表示（链队）；主要操作是进队和出队，循环队列需注意判断队满/队空，凡涉及头/尾指针修改都要对 `MAXQSIZE` 求模。
> 3. **栈与队列的应用**与其表示方法和运算规则相联系，表 3.3 从逻辑结构、存储结构、运算规则三方面比较二者。
> 4. **栈的重要应用——递归**：递归是程序设计的重要方法，结构清晰、形式简洁；但执行时需系统提供隐式工作栈保存调用过程中的参数、局部变量和返回地址，故占用内存较多、运行效率较低。

表 3.3 栈和队列的比较
| 比较项目 | 栈 | 队列 |
| :--- | :--- | :--- |
| 逻辑结构 | 和线性表一样，数据元素之间存在一对一的关系 | 和线性表一样，数据元素之间存在一对一的关系 |
| 存储结构 | 顺序存储：存储空间预先分配，可能出现空间闲置或栈满溢出；元素个数不能自由扩充<br><br>链式存储：动态分配，不会出现闲置或栈满溢出；元素个数可自由扩充 | 顺序存储（常设计成循环队列形式）：存储空间预先分配，可能出现空间闲置或队满溢出；元素个数不能自由扩充<br><br>链式存储：动态分配，不会出现闲置或队满溢出；元素个数可自由扩充 |
| 运算规则 | 插入和删除在表的一端（栈顶）完成，后进先出 | 插入在表的一端（队尾）进行，删除在另一端（队头）进行，先进先出 |
