> [!definition] 最短路径、源点、终点
> 带权有向网中，路径长度定义为路径上各边权值之和（不再只是边数）。路径第一个顶点称**源点（Source）**，最后一个顶点称**终点（Destination）**。

本节讨论两类：单源点到其余各顶点的最短路径、每一对顶点间的最短路径。

## 1. 从某个源点到其余各顶点的最短路径（Dijkstra）

![[Attachments/Pasted image 20260717192609.png]]

> [!example] 算法 6.10 迪杰斯特拉算法（Dijkstra）
> **【思想】** 将顶点分两组：$S$（已确定最短路径的终点集）、$V-S$（未确定集）。按路径长度递增次序，逐个将 $V-S$ 中顶点加入 $S$。
> **【辅助数组】** `S[i]`（是否已确定）、`Path[i]`（当前最短路径上 $v_i$ 的直接前驱）、`D[i]`（当前最短路径长度）。
> **【算法步骤】**
> ① 初始化：$S[v_0]=\text{true}$；$D[i]=G.arcs[v_0][v_i]$；有弧则 `Path[i]=v0`，否则 $-1$。
> ② 循环 $n-1$ 次：选 $D[k]=\min\{D[i]\mid v_i\in V-S\}$，将 $v_k$ 加入 $S$；若 $D[k]+G.arcs[k][i]<D[i]$ 则更新 $D[i]$ 与 `Path[i]=k`。
> **【算法描述】**
> ```c
> void ShortestPath_DIJ(AMGraph G, int v0)
> {
>     n=G.vexnum;
>     for(v=0; v<n; ++v)
>     {
>         S[v]=false;
>         D[v]=G.arcs[v0][v];
>         if(D[v]<MaxInt) Path[v]=v0; else Path[v]=-1;
>     }//for
>     S[v0]=true; D[v0]=0;
>     for(i=1; i<n; ++i)
>     {
>         min=MaxInt;
>         for(w=0; w<n; ++w)
>             if(!S[w]&&D[w]<min){v=w; min=D[w];}
>         S[v]=true;
>         for(w=0; w<n; ++w)
>             if(!S[w]&&(D[v]+G.arcs[v][w]<D[w]))
>             { D[w]=D[v]+G.arcs[v][w]; Path[w]=v; }
>     }//for
> }
> ```
> **【算法分析】** 主循环 $n-1$ 次，每次 $O(n)$，时间复杂度 $O(n^2)$。

## 2. 每一对顶点之间的最短路径（Floyd）

> [!example] 算法 6.11 弗洛伊德算法（Floyd）
> **【思想】** 逐步在 $v_i,v_j$ 间加入中间顶点 $v_0,v_1,\dots,v_{n-1}$，取较短者。对应方阵序列：
> $$D^{(-1)}, D^{(0)}, D^{(1)}, \dots, D^{(n-1)}$$
> 其中 $D^{(-1)}[i][j]=G.arcs[i][j]$，且
> $$D^{(k)}[i][j] = \min \{ D^{(k-1)}[i][j], D^{(k-1)}[i][k] + D^{(k-1)}[k][j] \}$$
> $D^{(n-1)}[i][j]$ 即 $v_i$ 到 $v_j$ 的最短路径长度。
> **【辅助数组】** `D[i][j]`（最短路径长度）、`Path[i][j]`（$v_j$ 的前驱）。
> **【算法描述】**
> ```c
> void ShortestPath_Floyd(AMGraph G)
> {
>     for(i=0; i<G.vexnum; ++i)
>         for(j=0; j<G.vexnum; ++j)
>         {
>             D[i][j]=G.arcs[i][j];
>             if(D[i][j]<MaxInt && i!=j) Path[i][j]=i;
>             else Path[i][j]=-1;
>         }//for
>     for(k=0; k<G.vexnum; ++k)
>         for(i=0; i<G.vexnum; ++i)
>             for(j=0; j<G.vexnum; ++j)
>                 if(D[i][k]+D[k][j] < D[i][j])
>                 {
>                     D[i][j]=D[i][k]+D[k][j];
>                     Path[i][j]=Path[k][j];
>                 }//if
> }
> ```
> **【算法分析】** 时间复杂度 $O(n^3)$；形式上比 $n$ 次调用 Dijkstra 更简洁。
