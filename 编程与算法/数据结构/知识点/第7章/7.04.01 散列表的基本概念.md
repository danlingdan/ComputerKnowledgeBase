# 7.4.1 散列表的基本概念

> [!topic] 所属主题
> [[MOC - 第7章 查找#7.4 散列表的查找|7.4 散列表的查找]]

前面方法都以关键字比较为基础，记录在存储结构中的位置与其关键字无直接关系，查找时间与表长有关。若能建立元素存储位置与关键字的**直接关系**，则无需（或很少）比较即可由关键字直接找到记录，这就是**散列查找法**（Hash Search，又称杂凑法/散列法）的思想。

> [!definition] 散列法基本术语
> 1. **散列函数和散列地址**：在记录存储位置 $p$ 与关键字 $key$ 间建立确定对应关系 $H$，使 $p = H(key)$，称 $H$ 为**散列函数**，$p$ 为**散列地址**。
> 2. **散列表**：一个有限连续的地址空间（通常是一维数组），散列地址是数组下标，用以存储按散列函数算得相应地址的数据记录。
> 3. **冲突和同义词**：对不同关键字得到同一散列地址，即 $key_1 \neq key_2$ 而 $H(key_1) = H(key_2)$，称**冲突**；具有相同函数值的关键字互称**同义词**。

> [!example] 散列函数与冲突示例
> 对 C 语言部分关键字集合 $S_1 = \{\text{main, int, float, while, return, break, switch, case, do}\}$ 建散列表，长度 26：
> ```c
> char HT[26][8];
> ```
> 取散列函数为关键字首字母在字母表中的序号：
> $$H(key) = \text{key}[0] - \text{'a'}$$
> 所得散列表如表 7.1 所示。
>
> **表 7.1 关键字集合 $S_1$ 对应的散列表**
> | 0 | 1 | 2 | 3 | 4 | 5 | ... | 8 | ... | 12 | ... | 17 | 18 | ... | 22 | ... | 25 |
> |---|---|---|---|---|---|---|---|---|---|---|---|---|---|---|---|---|
> | break | case | do | | float | | | int | | main | | return | switch | | while | | |
>
> 扩充 $S_2 = S_1 + \{\text{short, default, double, static, for, struct}\}$，散列函数不变时：$H(\text{short}) = H(\text{static}) = H(\text{struct}) = 18$，$H(\text{default}) = H(\text{double}) = 3$，$H(\text{for}) = 5$，均发生冲突。switch/short/static/struct 互为同义词；do/default/double 互为同义词；float/for 互为同义词。
>
> 理想的无冲突散列函数极少存在，因为关键字取值集合通常远大于表空间地址集。例如标识符（长度 ≤ 8 的字母数字串）取值集合大小约：
> $$C_{52}^1 \times C_{62}^7 \times 7! = 1.09 \times 10^{12}$$
> 而源程序标识符有限，散列表长度设为 1000 足矣。将 $10^{12}$ 个可能标识符映射到有限地址，难免冲突。散列函数多为多对一映射，冲突不可避免，只能选“好”的散列函数减少冲突，并在冲突发生时及时处理。

> [!summary] 散列查找法研究的两大问题
> 1. 如何构造散列函数；
> 2. 如何处理冲突。
