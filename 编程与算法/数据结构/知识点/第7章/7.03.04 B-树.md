前面方法适用于内存中较小的文件（**内查找法**）。若文件很大存于外存，反复内外存交换很费时。1970 年 Bayer 与 McCreight 提出适用于外查找的平衡多叉树——**B- 树**，磁盘目录管理与数据库索引多采用之。

## 1. B-树的定义

> [!definition] m 阶 B-树
> 一棵 $m$ 阶 B- 树或是空树，或是满足下列特性的 $m$ 叉树：
> 1. 每个结点至多有 $m$ 棵子树；
> 2. 若根结点不是叶子，则至少有两棵子树；
> 3. 除根外所有非终端结点至少有 $\lceil m/2 \rceil$ 棵子树；
> 4. 所有叶子结点出现在同一层次，不带信息，称**失败结点**（实际不存在，指针为空，仅为分析查找性能引入）；
> 5. 所有非终端结点最多有 $m-1$ 个关键字，结点结构如图 7.47 所示。
> ![[Attachments/Pasted image 20260718194152.png]]
> 其中 $K_i\ (i=1,\cdots,n)$ 为关键字且 $K_i<K_{i+1}$；$P_i\ (i=0,\cdots,n)$ 为子树指针，$P_{i-1}$ 所指子树关键字均小于 $K_i$，$P_n$ 所指子树关键字均大于 $K_n$；$n$ 满足 $\lceil m/2 \rceil - 1 \leqslant n \leqslant m-1$。

> [!info] B-树的特点（以 4 阶 B-树为例）
> ![[Attachments/Pasted image 20260718194201.png]]
> 1. 所有叶子同层 → **平衡**；
> 2. 结点内关键字有序，左子树关键字小于 $K_i$、右子树大于 $K_i$ → **有序**；
> 3. 结点关键字/子树数可变（4 阶最多 3 关键字 4 子树）→ **多路**。
> 为记录双亲，结点存储结构常增加 `parent` 指针（图 7.49）。
> ![[Attachments/Pasted image 20260718194208.png]]

## 2. B-树的查找

> [!example] 算法 7.8 B-树的查找
> 【算法步骤】将 `key` 与根结点各关键字 $K_1,\cdots,K_j$ 顺序或折半比较：
> ① 若 `key == K_i`，查找成功；
> ② 若 `key < K_1`，沿 $P_0$ 向下；
> ③ 若 $K_i < key < K_{i+1}$，沿 $P_i$ 向下；
> ④ 若 `key > K_j`，沿 $P_j$ 向下。
> 直到找到或到达叶子（失败）。
>
> 【算法描述】
> ```c
> Result SearchBTree(BTree T, KeyType key)
> { // 在m阶B-树T上查找关键字key，返回结果(pt, i, tag)
>   // 若查找成功，则特征值tag=1，指针pt所指结点中第i个关键字等于key
>   // 否则特征值tag=0，等于key的关键字应插入在指针pt所指结点中第i个和第i+1个关键字之间
>     p=T; q=NULL; found=FALSE; i=0;      // 初始化，p指向待查结点，q指向p的双亲
>     while (p&&!found)
>     {
>         i=Search(p, key);
>         // 在p->K[1..keynum]中查找i，使得p->K[i]<=key<p->K[i+1]
>         if (i>0&&p->K[i]==key) found=TRUE;  // 找到待查关键字
>         else {q=p; p=p->ptr[i];}
>     }
>     if (found) return(p, i, l);             // 查找成功
>     else return(q, i, 0);                   // 查找不成功，返回key的插入位置信息
> }
> ```
> 【算法分析】B-树查找含两类操作：①在磁盘上找结点；②在内存中结点内找关键字。决定效率的首要因素是**待查关键字所在结点在 B-树上的层次数**。含 $N$ 个关键字的 $m$ 阶 B-树最大深度：
> $$h \leqslant \log_{\lceil m/2 \rceil} \left( \frac{N+1}{2} \right) + 1$$
> 故从根到目标结点的路径上结点数不超过该值。

## 3. B-树的插入

插入不是在树中加叶子，而是先在最低层某非终端结点加关键字；若结点关键字数不超过 $m-1$ 则完成，否则结点“分裂”：以中间关键字为界一分为二，中间关键字上插入双亲，双亲满则继续分裂，最坏分裂到根使树高增 1。

> [!example] 算法 7.9 B-树的插入
> 【算法步骤】
> ① 查找应插入的叶子上一层结点（由 `q` 指向），若成功则失败。
> ② 插入后若 `q` 结点关键字数未超 $m-1$，成功；
> ③ 否则以第 $\lceil m/2 \rceil$ 个关键字为拆分点分裂：左部留原结点，右部存新结点，中间关键字插入双亲，对双亲重复 ②③；
> ④ 若 `q` 为根，由其分裂产生的两结点与中间关键字构成新根，树高增 1。
>
> 【算法描述】
> ```c
> Status InsertBTree(BTree &T, KeyType key, BTree q, int i)
> { // 在m阶B-树T上结点*q的K[i]与K[i+1]之间插入关键字key
>   // 若引起结点过大，则沿双亲链进行必要的结点分裂调整，使T仍是m阶B-树
>     x=key; ap=NULL; finished=FALSE;        // x表示新插入的关键字，ap为一个空指针
>     while(q&&!finished)
>     {
>         Insert(q, i, x, ap);                // 将x和ap分别插入q->key[i+1]和q->ptr[i+1]
>         if(q->keynum<m) finished=TRUE;      // 插入完成
>         else                                // 分裂结点
>         {
>             s=(m+1)/2; split(q, s, ap); x=q->K[s];
>             // 将q->K[s+1..m], q->ptr[s..m]和q->recptr[s+1..m]移入新结点*ap
>             q=q->parent;
>             if(q) i=Search(q, x);           // 在双亲结点*q中查找x的插入位置
>         }                                   // else
>     }                                       // while
>     if(!finished)                           // T是空树(参数q初值为NULL)或者根结点已分裂为结点*q和*ap
>         NewRoot(T, q, x, ap);               // 生成含信息(T, x, ap)的新的根结点*T，原T和ap为子树指针
>     return OK;
> }
> ```

## 4. B-树的删除

删除指定关键字及其邻近指针后须调整，保证每个结点关键字数在 $[\lceil m/2 \rceil - 1,\ m-1]$。若删除后关键字数 $< \lceil m/2 \rceil - 1$，做“合并”操作。非最下层非终端结点的关键字，可用其右（左）子树中最小（大）关键字替换，归结为在最下层非终端结点删除。有 3 种可能（以 3 阶 B-树为例）：

> [!info] B-树删除的 3 种情况
> 1. 被删结点关键字数 $\geqslant \lceil m/2 \rceil$：直接删去 $K_i$ 与 $P_i$ 即可。
> 2. 被删结点关键字数 $= \lceil m/2 \rceil - 1$，但右（左）兄弟关键字数 $> \lceil m/2 \rceil - 1$：兄弟最小（大）关键字上移至双亲，双亲中紧邻关键字下移至被删结点。
> 3. 被删结点与兄弟关键字数均 $= \lceil m/2 \rceil - 1$：合并——被删结点剩余信息与双亲中 $K_i$ 一起并入兄弟；若双亲因此不足，则递归向上合并。
