# 7.2.2 折半查找

> [!nav] 导航
> 上一知识点：[[7.02.01 顺序查找]] · [[MOC - 第7章 查找|本章目录]] · [[MOC - 数据结构|课程总览]] · 下一知识点：[[7.02.03 分块查找]]

> [!topic] 所属主题
> [[MOC - 第7章 查找#7.2 线性表的查找|7.2 线性表的查找]]

**折半查找**（Binary Search）又称**二分查找**，是一种效率较高的查找方法。它要求线性表必须采用顺序存储结构，且表中元素按关键字有序排列。以下均假设有序表是递增有序的。

折半查找的查找过程：从表的中间记录开始，若给定值与该记录关键字相等则查找成功；否则若给定值小于（或大于）中间记录的关键字，则在前（或后）半个子表中继续查找。每步使查找区间缩小一半。

用 `low`、`high` 分别表示当前查找区间的下界和上界，`mid` 为区间中间位置。

> [!example] 算法 7.3 折半查找
> 【算法步骤】
> ① 置查找区间初值：`low = 1`，`high = 表长`。
> ② 当 `low <= high` 时循环：
> - `mid = (low + high) / 2`；
> - 若 `key == ST.R[mid].key`，查找成功，返回 `mid`；
> - 若 `key < ST.R[mid].key`，令 `high = mid - 1`；否则令 `low = mid + 1`。
> ③ 循环结束，查找区间为空，查找失败，返回 0。
>
> 【算法描述】
> ```c
> int Search_Bin(SSTable ST, KeyType key)
> { // 在有序表ST中折半查找其关键字等于key的数据元素。若找到，则函数值为该元素在表中的位置，否则为0
>     low=1; high=ST.length;          // 置查找区间初值
>     while (low<=high)
>     {
>         mid=(low+high)/2;
>         if(key==ST.R[mid].key) return mid;      // 找到待查元素
>         else if (key<ST.R[mid].key) high=mid-1; // 继续在前一子表进行查找
>         else low=mid+1;                         // 继续在后一子表进行查找
>     }
>     return 0;                                   // 表中不存在待查元素
> }
> ```
> 【注意】循环条件必须是 `low <= high` 而非 `low < high`，因为 `low == high` 时区间内还有一个结点需要比较。

> [!example] 例 7.1 折半查找过程
> 已知包含 11 个数据元素的有序表：`(5, 16, 20, 27, 30, 36, 44, 55, 60, 67, 71)`。给出查找关键字 `27` 和 `65` 的折半查找过程。
>
> 设 `mid = floor((low+high)/2)`。`low`、`high` 初值分别为 1 和 11，`mid` 初值为 6。
> ![[Attachments/Pasted image 20260718193248.png]]
> 查找 `key = 27` 的过程：先与 `ST.R[6].key = 36` 比较，$36 > 27$，故 `high = 5`，新 `mid = floor((1+5)/2) = 3`；与 `ST.R[3].key = 20` 比较，$20 < 27$，故 `low = 4`，`mid = 4`；与 `ST.R[4].key = 27` 比较相等，查找成功，返回 4。
> 查找 `key = 65` 的过程类似；最后一趟因 `low > high`，查找区间不存在，查找失败，返回 0。

> [!info] 折半查找的判定树
> 折半查找过程可用二叉树描述：树中每结点对应表中一个记录（结点值为记录在表中的位置序号），以当前查找区间的中间位置为根，左、右子表分别为其左、右子树，称此树为**判定树**。
> ![[Attachments/Pasted image 20260718193301.png]]
> 例 7.1 的有序表对应判定树如上。成功的折半查找恰是走了一条从根到被查结点的路径，比较次数即该结点在树中的层次。由图可知比较 1 次的只有根结点，比较 2 次的有 2 个结点，比较 3 次和 4 次的各有 4 个结点。假设各记录查找概率相同，则长度为 11 的有序表折半查找的 ASL 为：
> $$ASL = \frac{1}{11}(1 + 2 \times 2 + 3 \times 4 + 4 \times 4) = 3$$

> [!tip] 判定树示例
> **例 7.1 的折半查找判定树（n = 11）**
> ```mermaid
> graph TD
>     R((36)) --> L20((20))
>     R --> R60((60))
>     L20 --> L5((5))
>     L20 --> L27((27))
>     R60 --> L44((44))
>     R60 --> R67((67))
>     L5 --> R16((16))
>     L27 --> R30((30))
>     L44 --> R55((55))
>     R67 --> R71((71))
> ```

> [!info] 不成功查找与判定树深度
> 在判定树所有结点的空指针域上加指向方形“外部结点”的指针（圆形为内部结点），则查找失败的过程即走了一条从根到外部结点的路径，比较次数等于路径上内部结点个数。
> ![[Attachments/Pasted image 20260718193314.png]]
> 判定树的形态只与记录个数 $n$ 有关，与关键字取值无关；具有 $n$ 个结点的判定树深度为 $\lfloor \log_2 n \rfloor + 1$。因此折半查找成功与不成功时比较的关键字个数最多均不超过 $\lfloor \log_2 n \rfloor + 1$。

> [!example] 折半查找 ASL 的推导
> 设有序表长度 $n = 2^h - 1$，则判定树是深度为 $h = \log_2(n+1)$ 的满二叉树。假设各记录查找概率相等（$P_i = 1/n$），则查找成功时 ASL 为：
> $$
> \begin{aligned}
> ASL &= \sum_{i=1}^{n} P_i C_i \\
> &= \frac{1}{n} \sum_{j=1}^{h} j \cdot 2^{j-1} \\
> &= \frac{n+1}{n} \log_2(n+1) - 1
> \end{aligned} \tag{7-2}
> $$
> 当 $n$ 较大时：
> $$ASL \approx \log_2(n+1) - 1 \tag{7-3}$$
> 故折半查找时间复杂度为 $O(\log_2 n)$。

> [!summary] 折半查找的优缺点
> - **优点**：比较次数少，查找效率高。
> - **缺点**：对表结构要求高，只能用于顺序存储的有序表；插入/删除需维持有序性，平均要移动约一半元素，费时。
> 因此折半查找适用于**不常变动的静态有序表**，不适用于数据元素经常变动的线性表。
