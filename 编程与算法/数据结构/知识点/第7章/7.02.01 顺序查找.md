# 7.2.1 顺序查找

> [!nav] 导航
> 上一知识点：[[7.01 查找的基本概念]] · [[MOC - 第7章 查找|本章目录]] · [[MOC - 数据结构|课程总览]] · 下一知识点：[[7.02.02 折半查找]]

> [!topic] 所属主题
> [[MOC - 第7章 查找#7.2 线性表的查找|7.2 线性表的查找]]

**顺序查找**（Sequential Search）的查找过程为：从表的一端开始，依次将记录的关键字和给定值进行比较，若某个记录的关键字和给定值相等，则查找成功；反之，若查完整个表后仍未找到关键字和给定值相等的记录，则查找失败。

顺序查找既适用于线性表的顺序存储结构，又适用于链式存储结构。下面只介绍以顺序表作为存储结构时的实现。

数据元素类型定义如下：

```c
typedef struct {
    KeyType key;            // 关键字域
    InfoType otherinfo;     // 其他域
} ElemType;
```

顺序表的定义同第 2 章：

```c
typedef struct {
    ElemType *R;            // 存储空间基地址
    int length;             // 当前长度
} SSTable;
```

在此定义下，顺序查找算法与第 2 章的算法 2.3 相同。假设元素从 `ST.R[1]` 开始顺序向后存放，`ST.R[0]` 闲置不用，查找时从表的最后开始比较，如算法 7.1 所示。

> [!example] 算法 7.1 顺序查找
> 【算法描述】
> ```c
> int Search_Seq(SSTable ST, KeyType key)
> { // 在顺序表ST中顺序查找其关键字等于key的数据元素。若找到，则函数值为该元素在表中的位置，否则为0
>     for (i=ST.length; i>=1; --i)
>         if (ST.R[i].key==key) return i;    // 从后往前查找
>     return 0;
> }
> ```

算法 7.1 在查找过程中每步都要检测整个表是否查找完毕（循环变量是否满足 $i \geqslant 1$）。改进方法是查找前先对 `ST.R[0]` 的关键字赋值 `key`，此时 `ST.R[0]` 起到**监视哨**的作用，免去每步的越界检测，如算法 7.2 所示。

> [!example] 算法 7.2 设置监视哨的顺序查找
> 【算法描述】
> ```c
> int Search_Seq(SSTable ST, KeyType key)
> { // 在顺序表ST中顺序查找其关键字等于key的数据元素。若找到，则函数值为该元素在表中的位置，否则为0
>     ST.R[0].key=key;                 // “监视哨”
>     for (i=ST.length; ST.R[i].key!=key; --i);  // 从后往前找
>     return i;
> }
> ```
> 【算法分析】算法 7.2 仅作了程序设计技巧上的改进：通过设置监视哨，免去查找过程中每一步对“是否查完”的检测。实践证明，当 `ST.length` 较大时，该改进能使一次查找的平均时间几乎减少一半。其时间复杂度与算法 7.1 相同，均为 $O(n)$。

> [!summary] 顺序查找的优缺点
> - **优点**：算法简单，对表结构无任何要求，既适用于顺序结构也适用于链式结构，无论记录是否按关键字有序均可应用。
> - **缺点**：平均查找长度较大，查找效率较低，因此当 $n$ 很大时不宜采用。
> 其查找成功时的平均查找长度为 $ASL = \dfrac{1}{n}\sum_{i=1}^{n} i = \dfrac{n+1}{2}$。
