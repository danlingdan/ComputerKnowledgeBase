**二叉排序树**（Binary Sort Tree）又称**二叉查找树**，是一种对排序和查找都很有用的特殊二叉树。

## 1. 二叉排序树的定义

> [!definition] 二叉排序树
> 二叉排序树或是空树，或是具有下列性质的二叉树：
> 1. 若左子树不空，则左子树上所有结点的值均小于根结点的值；
> 2. 若右子树不空，则右子树上所有结点的值均大于根结点的值；
> 3. 左、右子树也分别为二叉排序树。
>
> 由定义可得重要性质：**中序遍历一棵二叉排序树可得到一个键值递增的有序序列**。

> [!info] 中序遍历验证
> 图 7.5 所示为两棵二叉排序树。
> ![[Attachments/Pasted image 20260718193337.png]]
> 中序遍历图 (a) 得：`3, 12, 24, 37, 45, 53, 61, 78, 90, 100`；中序遍历图 (b) 得：`CAO, CHEN, DING, DU, LI, MA, WANG, XIA, ZHAO`。

二叉排序树采用二叉链表存储，结点数据域含关键字项与其他数据项：

```c
// - - - - 二叉排序树的二叉链表存储表示 - - - -
typedef struct {
    KeyType key;            // 关键字项
    InfoType otherinfo;     // 其他数据项
} ElemType;                 // 每个结点的数据域的类型

typedef struct BSTNode {
    ElemType data;          // 每个结点的数据域包括关键字项和其他数据项
    struct BSTNode *lchild, *rchild;  // 左右孩子指针
} BSTNode, *BSTree;
```

## 2. 二叉排序树的查找

> [!example] 算法 7.4 二叉排序树的递归查找
> 【算法步骤】
> ① 若树为空，查找失败，返回空指针。
> ② 若树非空，将 `key` 与根结点关键字 `T->data.key` 比较：
> - 相等 → 查找成功，返回根结点地址；
> - 小于 → 递归查找左子树；
> - 大于 → 递归查找右子树。
>
> 【算法描述】
> ```c
> BSTree SearchBST(BSTree T, KeyType key)
> { // 在根指针T所指二叉排序树中递归地查找某关键字等于key的数据元素
>   // 若查找成功，则返回指向该数据元素结点的指针，否则返回空指针
>     if ((!T) || key==T->data.key) return T;              // 查找结束
>     else if (key<T->data.key) return SearchBST(T->lchild, key); // 在左子树中继续查找
>     else return SearchBST(T->rchild, key);               // 在右子树中继续查找
> }
> ```

> [!info] 查找性能与树形有关
> 在二叉排序树上查找的过程，是走一条从根到目标结点的路径，比较次数 = 路径长度 + 1（或结点所在层数），不超过树的深度。
> ![[Attachments/Pasted image 20260718193407.png]]
> 图 7.6 两棵树结点值相同但插入序列不同：`(45, 24, 53, 12, 37, 93)` 深度为 3，`(12, 24, 37, 45, 53, 93)` 深度为 6。设 6 个记录查找概率均为 $1/6$：
> $$ASL_{(a)} = \frac{1}{6} [1 + 2 + 2 + 3 + 3 + 3] = 14/6,\qquad ASL_{(b)} = \frac{1}{6} [1 + 2 + 3 + 4 + 5 + 6] = 21/6$$
> 当插入关键字有序时，树蜕变为单支树，深度为 $n$，ASL 退化为 $(n+1)/2$（与顺序查找相同），这是最坏情况；最佳情况形态近似折半判定树，ASL 与 $\log_2 n$ 成正比。平均而言，二叉排序树 ASL 仍与 $\log_2 n$ 同数量级。

## 3. 二叉排序树的插入

插入以查找为基础：从根向下查找，仅当树中不存在关键字等于 `key` 的结点时才插入。新插入结点一定是新叶子结点，位于查找不成功时路径上最后访问结点的左/右孩子。

> [!example] 算法 7.5 二叉排序树的插入
> 【算法描述】
> ```c
> void InsertBST(BSTree &T, ElemType e)
> { // 当二叉排序树T中不存在关键字等于e.key的数据元素时，则插入该元素
>     if (!T)                     // 找到插入位置，递归结束
>     {
>         S=new BSTNode;          // 生成新结点*S
>         S->data=e;              // 新结点*S的数据域置为e
>         S->lchild=S->rchild=NULL;// 新结点*S作为叶子结点
>         T=S;                    // 把新结点*S链接到已找到的插入位置
>     }
>     else if (e.key<T->data.key)
>         InsertBST(T->lchild,e); // 将*S插入左子树
>     else if (e.key>T->data.key)
>         InsertBST(T->rchild,e); // 将*S插入右子树
> }
> ```
> 例如，在图 7.5 (a) 中插入关键字 55：依次与 45、53 比较后，因 $55 < 61$ 且 61 的左子树为空，将 55 作为 61 的左孩子插入。
> ![[Attachments/Pasted image 20260718193418.png]]
> 【算法分析】插入的基本过程是查找，时间复杂度同查找，为 $O(\log_2 n)$。

## 4. 二叉排序树的创建

从空树开始，每输入一个结点，经查找后插入合适位置。

> [!example] 算法 7.6 二叉排序树的创建
> 【算法描述】
> ```c
> void CreatBST(BSTree &T)
> { // 依次读入关键字为key的结点，将相应结点插入二叉排序树T中
>     T=NULL;                     // 将二叉排序树T初始化为空树
>     cin>>e;
>     while (e.key!=ENDFLAG)      // ENDFLAG为自定义常量，作为输入结束标志
>     {
>         InsertBST(T,e);         // 将此结点插入二叉排序树T中
>         cin>>e;
>     }
> }
> ```
> 【算法分析】$n$ 个结点需 $n$ 次插入，每次 $O(\log_2 n)$，故创建时间复杂度为 $O(n\log_2 n)$。
> 例如，输入序列 `45, 24, 53, 45, 12, 24, 90` 生成二叉排序树的过程如图 7.8 所示。
> ![[Attachments/Pasted image 20260718193427.png]]
> 无序序列通过构造二叉排序树可变为有序序列；且每次插入的新结点都是叶子结点，插入时只需改动某个指针，相当于在有序序列上插入记录而不移动其他记录。

## 5. 二叉排序树的删除

被删结点可能是任何结点，删除后须修改双亲及相关结点指针以保持二叉排序树特性。

> [!example] 算法 7.7 二叉排序树的删除
> 【算法步骤】先查找关键字为 `key` 的待删结点 `*p`（双亲 `*f`），设 $P_L$、$P_R$ 为其左右子树。设 `*p` 是 `*f` 的左孩子（右孩子类似），分 3 种情况：
> 1. **叶子结点**（$P_L$、$P_R$ 均为空）：只需 `f->lchild = NULL;`
> 2. **只有左子树或只有右子树**：令 $P_L$（或 $P_R$）直接成为 `*f` 的左子树：`f->lchild = p->lchild;`（或 `f->lchild = p->rchild;`）
> 3. **左右子树均不空**：有两种处理。常用方案是以 `*p` 的**直接前驱**（或直接后继）替代 `*p`，再删去该前驱（或后继）结点。由于前驱结点一定没有右子树，不会增加树高。
> ![[Attachments/Pasted image 20260718193446.png]]
>
> 【算法描述】
> ```c
> void DeleteBST(BSTree &T, KeyType key)
> { // 从二叉排序树T中删除关键字等于key的结点
>     p=T; f=NULL;                // 初始化
>     /*----------- 下面的while循环从根开始查找关键字等于key的结点*p -----------*/
>     while(p)
>     {
>         if (p->data.key==key) break;      // 找到关键字等于key的结点*p，结束循环
>         f=p;                              // *f为*p的双亲结点
>         if (p->data.key>key) p=p->lchild; // 在*p的左子树中继续查找
>         else p=p->rchild;                 // 在*p的右子树中继续查找
>     }                                     // while
>     if (!p) return;                       // 找不到被删结点则返回
>     /*--- 考虑3种情况实现p所指子树内部的处理：*p左右子树均不空、无右子树、无左子树 ---*/
>     q=p;
>     if ((p->lchild) && (p->rchild))       // 被删结点*p左右子树均不空
>     {
>         s=p->lchild;
>         while (s->rchild)                 // 在*p的左子树中继续查找其前驱结点，即最右下结点
>         {
>             q=s; s=s->rchild;             // 向右到尽头
>         }
>         p->data=s->data;                  // s指向被删结点的“前驱”
>         if (q!=p) q->rchild=s->lchild;    // 重接*q的右子树
>         else q->lchild=s->lchild;         // 重接*q的左子树
>         delete s;
>         return;
>     }
>     else if (!p->rchild)                  // 被删结点*p无右子树，只需重接其左子树
>     {
>         p=p->lchild;
>     }
>     else if (!p->lchild)                  // 被删结点*p无左子树，只需重接其右子树
>     {
>         p=p->rchild;
>     }
>     /*------------ 将p所指的子树挂接到其双亲结点*f相应的位置 ------------*/
>     if (!f) T=p;                          // 被删结点为根结点
>     else if (q==f->lchild) f->lchild=p;   // 挂接到*f的左子树位置
>     else f->rchild=p;                     // 挂接到*f的右子树位置
>     delete q;
> }
> ```
> 【算法分析】删除基本过程也是查找，时间复杂度仍为 $O(\log_2 n)$。图 7.10 给出删除的 3 种情况。
