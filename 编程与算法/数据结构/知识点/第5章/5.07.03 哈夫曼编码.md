## 1. 哈夫曼编码的主要思想

为使压缩后文件尽可能短，可采用不定长编码（出现次数多的字符编以较短编码）。为确保压缩与正确解码，利用哈夫曼树设计二进制编码：左分支标 0、右分支标 1，根到叶子的路径 0、1 序列即该字符编码。

> [!definition] 前缀编码
> 若编码方案中任一编码都不是其他任何编码的前缀（最左子串），则称该编码是**前缀编码**（如表 5.1 b 的 0、10、110、111 是前缀编码，表 5.1 c 的 0、01、010、111 不是）。前缀编码保证解码无二义性。

> [!definition] 哈夫曼编码
> 对含 $n$ 个叶子的哈夫曼树，每个左分支赋 0、右分支赋 1，从根到每个叶子的路径上各分支赋值构成的二进制串，称为该叶子的**哈夫曼编码**。

> [!note] 哈夫曼编码的两个性质
> **性质 1（前缀性）**：哈夫曼编码是前缀编码。*证明*：编码对应根到叶子的路径，若路径 A 是路径 B 的最左部分则 B 经过 A，A 的终点必非叶子；而哈夫曼编码终点必为叶子，故任一编码不会与另一编码的前缀完全重叠。
> **性质 2（最优性）**：哈夫曼编码是最优前缀编码。*证明*：文件总长 $\sum_{i=1}^n w_i l_i$ 恰为二叉树带权路径长度；以字符出现频率为权构造哈夫曼树，出现次数多的字符编码短，故编码后文件最短。

## 2. 哈夫曼编码的算法实现

求编码的主要思想：从叶子出发向上回溯至根，走左分支生成 0、走右分支生成 1。因编码变长，用指针数组存放各字符编码串首地址。

```cpp
//------ 哈夫曼编码表的存储表示 ------
typedef char **HuffmanCode;  // 动态分配数组存储哈夫曼编码表
```

各字符编码存于 `HC`（$n+1$ 行，0 号不用）；因编码长度不定，临时用长度 $n$ 的数组 `cd` 存放，求解完再为 `HC[i]` 分配空间并复制。

> [!example] 算法 5.11 根据哈夫曼树求哈夫曼编码
> **【算法步骤】** ① 分配 HC（长 $n+1$）与临时数组 `cd`（`cd[n-1]='\0'`）；② 循环 $n$ 次：设 `start` 指向 `cd` 末、`c=i`、`f=HT[i].parent`，当 `f≠0` 时 `--start`，`c` 是 `f` 左孩子生成 '0' 否则 '1'，回溯；按 `cd` 长度为 `HC[i]` 分配空间并复制；③ 释放 `cd`。
> **【算法描述】**
> ```cpp
> void CreateHuffmanCode(HuffmanTree HT, HuffmanCode &HC, int n)
> { // 从叶子到根逆向求每个字符的哈夫曼编码，存储在编码表 HC 中
>     HC = new char *[n + 1];                 // 分配存储 n 个字符编码的编码表空间
>     cd = new char [n];                      // 分配临时存放每个字符编码的动态数组空间
>     cd[n - 1] = '\0';                       // 编码结束符
>     for (i = 1; i <= n; ++i)                // 逐个字符求哈夫曼编码
>     {
>         start = n - 1;                      // start 开始时指向编码结束符位置
>         c = i; f = HT[i].parent;            // f 指向结点 c 的双亲结点
>         while (f != 0)                      // 从叶子向上回溯至根
>         {
>             --start;                        // 回溯一次 start 向前指一个位置
>             if (HT[f].lchild == c) cd[start] = '0'; // c 是 f 的左孩子，生成代码 0
>             else cd[start] = '1';                   // c 是 f 的右孩子，生成代码 1
>             c = f; f = HT[f].parent;        // 继续向上回溯
>         }
>         HC[i] = new char [n - start];       // 为第 i 个字符编码分配空间
>         strcpy(HC[i], &cd[start]);          // 将求得的编码复制到 HC[i]
>     }
>     delete cd;                              // 释放临时空间
> }
> ```

> [!example] 例 5.3
> 某系统通信只出现 8 种字符，概率 0.05、0.29、0.07、0.08、0.14、0.23、0.03、0.11，权值取 $w=(5,29,7,8,14,23,3,11)$。对应哈夫曼树如图 5.28，左分支 0、右分支 1，得哈夫曼编码表（图 5.29）：1:0001、2:10、3:1110、4:1111、5:110、6:01、7:0000、8:001。

## 3. 文件的编码和译码

- **编码**：据字符在 HC 中找到编码串，依次转换。
- **译码**：借助哈夫曼树，从根（`HT[m]`）出发，读入 0 走向左孩子、1 走向右孩子，到达叶子 `HT[i]` 即译出 `HC[i]`，再从根继续，直至文件结束。
