## 1. 线索二叉树的基本概念

遍历二叉树将结点排列成线性序列（先序/中序/后序），使每个结点（除首尾）有唯一前驱和后继。但以二叉链表为存储结构时，只能得到左右孩子信息，前驱后继只能在遍历动态过程中得到，故引入线索二叉树来保存这些信息。
![[Attachments/Pasted image 20260717185630.png]]
由于 $n$ 个结点的二叉链表有 $n+1$ 个空链域，可充分利用它们存放前驱/后继。规定：若结点有左子树，则 `lchild` 指示左孩子，否则指示前驱；若结点有右子树，则 `rchild` 指示右孩子，否则指示后继。需增加两个标志域（图 5.15）：

$$
\text{LTag} = \begin{cases} 0 & \text{lchild 域指示结点的左孩子} \\ 1 & \text{lchild 域指示结点的前驱} \end{cases}
\qquad
\text{RTag} = \begin{cases} 0 & \text{rchild 域指示结点的右孩子} \\ 1 & \text{rchild 域指示结点的后继} \end{cases}
$$

> [!definition] 线索二叉树（Threaded Binary Tree）
> 以含 `LTag`/`RTag` 标志的结点构成的二叉链表作存储结构，称为**线索链表**；指向结点前驱/后继的指针称为**线索**；加上线索的二叉树称为**线索二叉树**；对二叉树按某种次序遍历使其变为线索二叉树的过程称为**线索化**。

```cpp
//------ 二叉树的二叉线索存储表示 ------
typedef struct BiThrNode
{
    TElemType data;
    struct BiThrNode *lchild, *rchild;  // 左右孩子指针
    int LTag, RTag;                     // 左右标志
} BiThrNode, *BiThrTree;
```

中序线索化示例（图 5.16）：实线为指针（指向左右子树），虚线为线索（指向前驱/后继）。仿照线性表，在线索链表上加一头结点，其 `lchild` 指向根、`rchild` 指向中序最后一个结点；首结点 `lchild` 与末结点 `rchild` 均指向头结点，形成双向线索链表。

## 2. 构造线索二叉树

线索化的实质是将空指针改为指向前驱/后继的线索，前驱后继信息仅在遍历时得到，故线索化即遍历中修改空指针，可用递归算法。

> [!example] 算法 5.7 以结点 p 为根的子树中序线索化
> **【算法步骤】** ① p 非空则左子树递归线索化；② p 左孩子为空则加左线索（LTag=1，lchild 指 pre），否则 LTag=0；③ pre 右孩子为空则加右线索（RTag=1，rchild 指 p），否则 RTag=0；④ pre=p；⑤ 右子树递归线索化。
> **【算法描述】**
> ```cpp
> void InThreading(BiThrTree p)
> { // pre 是全局变量，初始化时其右孩子指针为空
>     if (p)
>     {
>         InThreading(p->lchild);          // 左子树递归线索化
>         if (!p->lchild)                  // p 的左孩子为空
>         {
>             p->LTag = 1;                 // 给 p 加上左线索
>             p->lchild = pre;             // p 的左孩子指针指向 pre(前驱)
>         }
>         else p->LTag = 0;
>         if (!pre->rchild)                // pre 的右孩子为空
>         {
>             pre->RTag = 1;               // 给 pre 加上右线索
>             pre->rchild = p;             // pre 的右孩子指针指向 p(后继)
>         }
>         else pre->RTag = 0;
>         pre = p;                         // pre 指向前驱结点
>         InThreading(p->rchild);          // 右子树递归线索化
>     }
> }
> ```

> [!example] 算法 5.8 带头结点的二叉树中序线索化
> **【算法描述】**
> ```cpp
> void InOrderThreading(BiThrTree &Thrt, BiThrTree T)
> { // 中序遍历二叉树 T，并将其中序线索化，Thrt 指向头结点
>     Thrt = new BiThrNode;                // 建头结点
>     Thrt->LTag = 0;                      // 头结点有左孩子（树非空时为树根）
>     Thrt->RTag = 1;                      // 头结点的右孩子指针为右线索
>     Thrt->rchild = Thrt;                 // 初始化时右指针指向自己
>     if (!T) Thrt->lchild = Thrt;         // 若树为空，则左指针也指向自己
>     else
>     {
>         Thrt->lchild = T; pre = Thrt;    // 头结点的左孩子指向根，pre 初值指向头结点
>         InThreading(T);                  // 调用算法 5.7 对 T 中序线索化
>         pre->rchild = Thrt;              // 结束后 pre 为最右结点，右线索指向头结点
>         pre->RTag = 1;
>         Thrt->rchild = pre;              // 头结点的右线索指向 pre
>     }
> }
> ```

## 3. 遍历线索二叉树

有了前驱/后继信息，在线索二叉树中查找前驱后继、遍历都变简单。不同次序下查找规则不同（中序最直观，先序找前驱/后序找后继较复杂，必要时可直接建含 4 个指针的线索链表）。
![[Attachments/Pasted image 20260717185704.png]]
> [!example] 算法 5.9 遍历中序线索二叉树
> **【算法步骤】** ① p 指向根；② p 非空或遍历未结束时循环：沿左孩子下行至最左下结点（中序首结点）→ 访问 p → 沿右线索反复访问后继直至右线索为 0 或结束 → 转向 p 的右子树。
> **【算法描述】**
> ```cpp
> // 中序遍历二叉线索树 T 的非递归算法，对每个数据元素直接输出
> p = T->lchild;                           // p 指向根结点
> while (p != T)                           // 空树或遍历结束时，p == T
> {
>     while (p->LTag == 0) p = p->lchild;  // 沿左孩子向下
>     cout << p->data;                     // 访问其左子树为空的结点
>     while (p->RTag == 1 && p->rchild != T)
>     {
>         p = p->rchild; cout << p->data;  // 沿右线索访问后继结点
>     }
>     p = p->rchild;                       // 转向 p 的右子树
> }
> ```
> **【算法分析】** 遍历线索二叉树时间复杂度 $O(n)$，空间复杂度 $O(1)$（无需栈，避免频繁进栈出栈）。
