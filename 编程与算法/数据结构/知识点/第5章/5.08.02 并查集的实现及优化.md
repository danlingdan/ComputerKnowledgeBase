## 1. 并查集的实现

常用树的**双亲表示法**：每个集合表示为一棵树，结点代表单元素，全集合构成森林；数组下标既代表元素名也代表树根，根结点下标即集合名，其双亲指针设为 -1（负数表示集合规模）。

```cpp
//------ 并查集的存储结构 ------
#define MAXSIZE 100
int UFSet[MAXSIZE];                        // 集合元素数组 (双亲指针数组)
```

> [!example] 算法 5.12 并查集的初始化
> ![[Attachments/Pasted image 20260717185942.png]]
> **【算法描述】**
> ```cpp
> void Init_UFSet(int UFSet[], int size)
> { // 初始化并查集
>     for (int i = 0; i < size; i++)
>         UFSet[i] = -1;                     // 每个由单个元素组成的集合
> }
> ```

> [!example] 算法 5.13 并查集的查找
> **【算法步骤】** 从 $x$ 出发沿双亲指针向上，直至双亲为负（即根）。
> **【算法描述】**
> ```cpp
> int Find_UFSet(int UFSet[], int x)
> { // 查找元素 x 所属的集合
>     while (UFSet[x] >= 0)                  // 循环寻找元素 x 的根
>         x = UFSet[x];
>     return x;
> }
> ```

> [!example] 算法 5.14 并查集的合并
> ![[Attachments/Pasted image 20260717185954.png]]
> ![[Attachments/Pasted image 20260717190001.png]]
> **【算法步骤】** ① 查找两结点根 Root1、Root2；② 若同集则不合并，否则将 Root2 链接到 Root1 下（并更新规模）。
> **【算法描述】**
> ```cpp
> void Union_UFSet(int UFSet[], int Root1, int Root2)
> { // 将两个子集合并
>     Root1 = Find(UFSet, Root1);
>     Root2 = Find(UFSet, Root2);
>     if (Root1 == Root2) return;
>     else
>     {
>         UFSet[Root1] += UFSet[Root2];
>         UFSet[Root2] = Root1;              // 将根 Root2 链接到另一根 Root1 下面
>     }
> }
> ```

## 2. 并查集的优化
![[Attachments/Pasted image 20260717190013.png]]

查找与合并的时间复杂度主要取决于树的深度，平均 $O(\log n)$。但最坏情况下 $n-1$ 次合并可使树退化为深度 $n$ 的链，此时 $n$ 次查找总时间复杂度达：

$$
O\left( \sum_{i=1}^{n} i \right) = O(n^2) \tag{5-6}
$$

> [!definition] 路径压缩（path compression）
> 在找到根结点后，返回时将路径上各元素的双亲指针均直接指向根结点，从而缩短查找路径。对图 5.35（a）的树采用折叠规则查找 6 后，路径上各结点双亲改为根 0（图 5.35 b），下次查找 6 一步到位。路径压缩可用递归或非递归实现，使查找时间复杂度降为 $O(1)$。

> [!example] 算法 5.15 路径压缩的递归算法
> **【算法步骤】** ① 找 $x$ 的树根并对所在子树路径压缩，返回调整后的双亲；② 若 $x$ 为根则返回自身；③ 找到根后原路返回，逆序压缩中间结点。
> **【算法描述】**
> ```cpp
> int CompressionFind(int UFSet[], int x)
> { // 递归实现路径压缩
>     if (UFSet[x] < 0)                      // 找到根结点
>         return x;
>     return UFSet[x] = CompressionFind(UFSet, UFSet[x]); // 路径压缩，提高查找效率
> }
> ```
