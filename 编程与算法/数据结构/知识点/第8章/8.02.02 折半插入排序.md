> [!definition] 折半插入排序（Binary Insertion Sort）
> 直接插入排序用顺序查找找插入位置；该"查找"可用"折半查找"实现，由此得到的插入排序称为折半插入排序。

## 算法 8.2 折半插入排序

> [!info] **算法步骤**
> ① 设记录存于数组 `r[1..n]`，`r[1]` 为有序序列。
> ② 循环 $n-1$ 次，用折半查找法找 `r[i]` 在 `r[1..i-1]` 中的插入位置并插入，最终得到表长为 $n$ 的有序序列。

**【算法描述】**

```c
void BInsertSort(SqList &L)
{ // 对顺序表 L 进行折半插入排序
    for(i=2; i<=L.length; ++i)
    {
        L.r[0]=L.r[i];                      // 将待插入的记录暂存到监视哨中
        low=1; high=i-1;                    // 置查找区间的初值
        while(low<=high)                    // 在 r[low..high] 中折半查找插入的位置
        {
            m=(low+high)/2;                 // 折半
            if(L.r[0].key < L.r[m].key) high=m-1; // 插入点在前一子表
            else low=m+1;                   // 插入点在后一子表
        } // while
        for(j=i-1; j>=high+1; --j) L.r[j+1]=L.r[j]; // 记录后移
        L.r[high+1]=L.r[0];                 // 将 r[0] 即原 r[i]，插入正确位置
    } // for
}
```

> [!abstract] **算法分析**
> **（1）时间复杂度**：折半查找比顺序查找快，平均性能优于直接插入排序。插入第 $i$ 个记录需 $\lfloor \log_2 i \rfloor + 1$ 次比较，与初始排列无关；但记录移动次数与直接插入排序相同。平均仅减少比较次数，移动次数不变，时间复杂度仍为 $O(n^2)$。
> **（2）空间复杂度**：只需一个记录辅助空间 `r[0]`，为 $O(1)$。

> [!tip] **算法特点**
> （1）稳定排序。（2）因需折半查找，只能用于顺序结构，不能用于链式结构。（3）适合初始记录无序、$n$ 较大的情况。
