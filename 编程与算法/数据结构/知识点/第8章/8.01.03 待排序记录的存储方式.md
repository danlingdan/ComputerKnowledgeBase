# 8.1.3 待排序记录的存储方式

> [!nav] 导航

> [!topic] 所属主题
> [[MOC - 第8章 排序#8.1 基本概念和排序方法概述|8.1 基本概念和排序方法概述]]

- **（1）顺序表**：记录次序由存储位置决定，排序需要移动记录。
- **（2）链表**：记录次序由指针指示，排序不需移动记录，仅需修改指针（链表排序）。
- **（3）地址排序**：记录存于连续单元，另设地址向量指示存储位置；排序中不移动记录本身，只移动地址向量中的"地址"，结束后按向量值调整存储位置。

> [!note] 本章的数据类型约定
> 除基数排序外，待排序记录均按顺序表存储，且关键字设为整数。记录数据类型定义如下：
>
> ```c
> #define MAXSIZE 20                // 顺序表的最大长度
> typedef int KeyType;              // 定义关键字类型为整型
> typedef struct{
>     KeyType key;                  // 关键字项
>     InfoType otherinfo;           // 其他数据项
> }RedType;                         // 记录类型
> typedef struct{
>     RedType r[MAXSIZE+1];         // r[0] 闲置或用做哨兵单元
>     int length;                   // 顺序表长度
> }SqList;                          // 顺序表类型
> ```
