> [!definition] 直接插入排序（Straight Insertion Sort）
> 一种最简单的插入排序方法，基本操作是将一条记录插入已排好序的表，得到记录数增 1 的新有序表。

## 算法 8.1 直接插入排序

> [!info] **算法步骤**
> ① 设记录存于数组 `r[1..n]`，`r[1]` 是一个有序序列。
> ② 循环 $n-1$ 次，用顺序查找法找 `r[i] (i = 2, \cdots, n)` 在 `r[1..i-1]` 中的插入位置，将其插入，直到 `r[n]` 插入后得到表长为 $n$ 的有序序列。

> [!example] **例 8.1** 待排序关键字序列 $\{49, 38, 65, 97, 76, 13, 27, 49\}$ 的直接插入排序过程。

实现时采用从后向前比较，并在 `r[0]` 设监视哨避免下标越界，同时后移记录。

**【算法描述】**

```c
void InsertSort(SqList &L)
{ // 对顺序表 L 进行直接插入排序
    for(i=2; i<=L.length; ++i)
        if(L.r[i].key < L.r[i-1].key)      // "<"，需将 r[i] 插入有序子表
        {
            L.r[0]=L.r[i];                  // 将待插入的记录暂存到监视哨中
            L.r[i]=L.r[i-1];                // r[i-1] 后移
            for(j=i-2; L.r[0].key<L.r[j].key; --j) // 从后向前查找插入位置
                L.r[j+1]=L.r[j];            // 记录逐个后移，直到找到插入位置
            L.r[j+1]=L.r[0];                // 将 r[0] 即原 r[i]，插入正确位置
        } // if
}
```

> [!abstract] **算法分析**
> **（1）时间复杂度**：基本操作为比较关键字与移动记录。最好情况（正序）比较 1 次、不移动；最坏情况（逆序）比较 $i$ 次、移动 $i+1$ 次。总比较次数 $KCN$ 与移动次数 $RMN$ 最大值分别为：
> $$ KCN = \sum_{i=2}^n i = \frac{(n+2)(n-1)}{2} \approx \frac{n^2}{2} $$
> $$ RMN = \sum_{i=2}^n (i+1) = \frac{(n+4)(n-1)}{2} \approx \frac{n^2}{2} $$
> 平均情况下比较与移动次数均约 $\frac{n^2}{4}$。时间复杂度为 $O(n^2)$。
> **（2）空间复杂度**：只需一个记录辅助空间 `r[0]`，为 $O(1)$。

> [!tip] **算法特点**
> （1）稳定排序。（2）算法简便、易实现。（3）也适用于链式存储结构（只需改指针，不移动记录）。（4）更适合初始记录基本有序（正序）；当无序且 $n$ 较大时时间复杂度高，不宜采用。
