> [!definition] 堆（Heap）
> $n$ 个元素的序列 $\{k_1, k_2, \cdots, k_n\}$ 称之为堆，当且仅当满足：
> （1）$k_i \ge k_{2i}$ 且 $k_i \ge k_{2i+1}$，或 （2）$k_i \le k_{2i}$ 且 $k_i \le k_{2i+1}$（$1 \le i \le \lfloor n/2 \rfloor$）。
>
> 对应完全二叉树中，所有非终端结点的值均不大于（或不小于）其左右孩子的值。堆顶（根）必为最大值（或最小值），分别称**大根堆**（或**小根堆**）。

例如序列 $\{96, 83, 27, 38, 11, 09\}$ 与 $\{12, 36, 24, 85, 47, 30, 53, 91\}$ 分别满足条件（1）（2），均为堆（见图 8.9）。

堆排序步骤（以构造大根堆为例）：
① 将 `r[1..n]` 调整为大根堆（**建初堆**），交换 `r[1]` 与 `r[n]`，`r[n]` 为最大记录。
② 将 `r[1..n-1]` 重新调整为堆，交换 `r[1]` 与 `r[n-1]`，`r[n-1]` 为次大。
③ 循环 $n-1$ 次，直至交换 `r[1]` 与 `r[2]`，得非递减有序序列。也可构造小根堆得非递增序列。

实现需解决两个问题：（1）建初堆；（2）调整堆（去掉堆顶后重新成堆）。建初堆用到调整堆，故先讨论调整堆。

## 算法 8.7 筛选法调整堆

> [!info] **算法步骤**
> 从 `r[2s]` 与 `r[2s+1]` 中选关键字较大者（设 `r[2s]` 较大），比较 `r[s]` 与 `r[2s]`：
> ① 若 `r[s].key >= r[2s].key`，以 `r[s]` 为根的子树已是堆，不必调整。
> ② 若 `r[s].key < r[2s].key`，交换二者；若以 `r[2s]` 为根的子树不是堆，重复将其调整为堆，直至叶子。

> [!example] 图 8.10（a）是大根堆，堆顶 97 与最后元素 38 交换后（b），除根外均满足堆性质，只需自上而下一条路径调整：38 与左子树根 76 交换得（c）；38 破坏左子树，继续调整至叶子得（d）。重复将 76 与 27 交换并调整得（d）新堆。
>
> ![[Attachments/Pasted image 20260718224432.png]]
> *图 8.10 堆顶改变后调整堆的过程（"筛选法"：小关键字逐层筛下，大关键字逐层选上）*

**【算法描述】**

```c
void HeapAdjust(SqList &L, int s, int m)
{ // 假设 r[s+1..m] 已经是堆，将 r[s..m] 调整为以 r[s] 为根的大根堆
    rc=L.r[s];
    for(j=2*s; j<=m; j*=2)                  // 沿 key 较大的孩子结点向下筛选
    {
        if(j<m && L.r[j].key<L.r[j+1].key) ++j;  // j 为 key 较大的记录的下标
        if(rc.key>=L.r[j].key) break;       // rc 应插入在位置 s 上
        L.r[s]=L.r[j]; s=j;
    } // for
    L.r[s]=rc;                              // 插入
}
```

## 算法 8.8 建初堆

> [!info] **算法步骤**
> 对无序序列 `r[1..n]`，从 $i=n/2$ 开始反复调用 `HeapAdjust(L, i, n)`，依次将以 `r[i]`, `r[i-1]`, $\cdots$, `r[1]` 为根的子树调整为堆（所有序号大于 $\lfloor n/2 \rfloor$ 的结点都是叶子，已是堆）。

**【算法描述】**

```c
void CreatHeap(SqList &L)
{ // 把无序序列 L.r[1..n] 建成大根堆
    n=L.length;
    for(i=n/2; i>0; --i)                    // 反复调用 HeapAdjust
        HeapAdjust(L, i, n);
}
```

## 算法 8.9 堆排序

**【算法描述】**

```c
void HeapSort(SqList &L)
{ // 对顺序表 L 进行堆排序
    CreatHeap(L);                           // 建成大根堆
    for(i=L.length; i>1; --i)
    {
        x=L.r[1];                           // 堆顶与当前未排序子序列 L.r[1..i] 末记录互换
        L.r[1]=L.r[i];
        L.r[i]=x;
        HeapAdjust(L, 1, i-1);              // 将 L.r[1..i-1] 重新调整为大根堆
    } // for
}
```

> [!tip] **堆排序主循环流程（大根堆）**
>
> ```mermaid
> flowchart TD
>     S[建初堆 CreatHeap] --> L{i > 1?}
>     L -->|是| X[交换 r[1] 与 r[i]]
>     X --> H[HeapAdjust L,1,i-1 重新调整堆]
>     H --> D[i = i - 1]
>     D --> L
>     L -->|否| E[得到非递减有序序列 r[1..n]]
> ```

> [!abstract] **算法分析**
> **（1）时间复杂度**：主要耗费在反复筛选。建初堆时每个非终端结点自上而下筛选，关键字总比较次数不超过 $4n$；重建堆 $n-1$ 次筛选的比较次数不超过 $2n\lfloor \log_2 n \rfloor$。最坏情况时间复杂度也为 $O(n \log_2 n)$，且平均性能接近最坏性能。
> **（2）空间复杂度**：仅一个记录大小的交换辅助空间，为 $O(1)$。

> [!tip] **算法特点**
> （1）**不稳定**。（2）只能用于顺序结构。（3）初始建堆比较次数较多，记录少时不宜采用；但最坏情况 $O(n \log_2 n)$，相对快排最坏 $O(n^2)$ 更有优势，记录多时高效。
