> [!definition] 简单选择排序（Simple Selection Sort）
> 也称直接选择排序。

## 算法 8.6 简单选择排序

> [!info] **算法步骤**
> ① 第一趟从 `r[1]` 起经 $n-1$ 次比较选最小者 `r[k]`，交换 `r[1]` 与 `r[k]`。
> ② 第二趟从 `r[2]` 起经 $n-2$ 次比较选最小者，交换 `r[2]` 与 `r[k]`。
> ③ 第 $i$ 趟从 `r[i]` 起经 $n-i$ 次比较选最小者，交换 `r[i]` 与 `r[k]`。
> ④ 经 $n-1$ 趟完成。

**【算法描述】**

```c
void SelectSort(SqList &L)
{ // 对顺序表 L 进行简单选择排序
    for(i=1; i<L.length; ++i) {          // 在 L.r[i..L.length] 中选择关键字最小的记录
        k=i;
        for(j=i+1; j<=L.length; ++j)
            if(L.r[j].key < L.r[k].key) k=j;    // k 指向此趟关键字最小的记录
        if(k!=i)
        { t=L.r[i]; L.r[i]=L.r[k]; L.r[k]=t; }  // 交换 r[i] 与 r[k]
    } // for
}
```

> [!abstract] **算法分析**
> **（1）时间复杂度**：移动次数少，最好（正序）不移动，最坏移动 $3(n-1)$ 次。但无论初始排列如何，比较次数恒为：
> $$ KCN = \sum_{i=1}^{n-1} n-i = \frac{n(n-1)}{2} \approx \frac{n^2}{2} $$
> 时间复杂度为 $O(n^2)$。
> **（2）空间复杂度**：仅交换时需要辅助空间，为 $O(1)$。

> [!tip] **算法特点**
> （1）本节采用“选出最小记录后与首个无序记录直接交换”的标准实现，长距离交换可能改变相等关键字的相对次序，因此**不稳定**；若改用删除最小记录并顺次后移其他记录的策略，可以设计稳定变体。（2）可改写为适用于链式存储的版本。（3）移动次数少，记录较大时可减少数据移动开销。

> [!info] 减少比较次数的思路——树形选择排序
> $n$ 个关键字选最小值至少需 $n-1$ 次比较，但继续选次小值不必重做 $n-2$ 次——可利用前 $n-1$ 次比较所得信息减少后续比较。体育锦标赛即一种选择排序（如 8 人决前三名至多 11 场而非 $7+6+5=18$ 场）。
