# 8.6.2 链式基数排序

> [!topic] 所属主题
> [[MOC - 第8章 排序#8.6 基数排序|8.6 基数排序]]

基数排序思想类似 LSD 洗牌，借助分配和收集对单逻辑关键字排序。逻辑关键字可看成若干关键字复合：若关键字为 $0 \le K \le 999$ 的数值，可看成由 3 个关键字 $(K^0, K^1, K^2)$ 组成（$K^0$ 百位、$K^1$ 十位、$K^2$ 个位）；若由 5 字母单词组成，则看成 5 个关键字。每个 $K^j$ 取值范围相同（数字 $0 \le K^j \le 9$，字母 'A'$\le K^j \le$'Z'），故可分配收集排序。

设记录逻辑关键字由 $d$ 个关键字组成，每个可取 $rd$ 个值。从最低位起，按不同值分配到 $rd$ 个队列后收集，重复 $d$ 次完成排序，称**基数排序**，其中"基" $rd$ 为取值范围（十进制为 10，字母为 26）。具体实现一般采用链式基数排序（静态链表）。

**【数据类型定义】**

```c
#define MAXNUM_KEY 8               // 关键字项数的最大值
#define RADIX 10                   // 关键字基数，此处为十进制整数
#define MAX_SPACE 10000
typedef struct
{
    KeyType keys[MAXNUM_KEY];      // 关键字
    InfoType otheritems;           // 其他数据项
    int next;
}SLCell;                           // 静态链表的结点类型
typedef struct
{
    SLCell r[MAX_SPACE];           // 静态链表可利用空间，r[0] 为头结点
    int keynum;                    // 记录的当前关键字个数
    int recnum;                    // 静态链表的当前长度
}SLList;                           // 静态链表类型
typedef int ArrType[RADIX];        // 数组类型
```

## 算法 8.12 基数排序

**【算法描述】**

```c
void Distribute(SLCell &r, int i, ArrType &f, ArrType &e)
{ // 静态链表 L 的 r 域中记录已按 (keys[0], ..., keys[i-1]) 有序
  // 本算法按第 i 个关键字 keys[i] 建立 RADIX 个子表
    for(j=0; j<RADIX; ++j) f[j]=0;          // 各子表初始化为空表
    for(p=r[0].next; p; p=r[p].next)
    {
        j=ord(r[p].keys[i]);                // ord 将第 i 个关键字映射到 [0..RADIX-1]
        if(!f[j]) f[j]=p;
        else r[e[j]].next=p;
        e[j]=p;                             // 将 p 所指结点插入第 j 个子表
    } // for
}

void Collect(SLCell &r, int i, ArrType f, ArrType e)
{ // 按 keys[i] 自小至大把 f[0..RADIX-1] 各子表依次链接成链表
    for(j=0; !f[j]; j=succ(j));             // 找第一个非空子表，succ() 为求后继函数
    r[0].next=f[j]; t=e[j];                 // r[0].next 指向第一个非空子表首结点
    while(j<RADIX)
    {
        for(j=succ(j); j<RADIX-1 && !f[j]; j=succ(j)); // 找下一个非空子表
        if(f[j]) { r[t].next=f[j]; t=e[j]; } // 链接两个非空子表
    } // while
    r[t].next=0;                            // t 指向最后一个非空子表的末结点
}

void RadixSort(SLList &L)
{ // L 为静态链表表示的顺序表；排序后 L 成为按关键字自小到大有序的静态链表
    for(i=0; i<L.recnum; ++i) L.r[i].next=i+1;
    L.r[L.recnum].next = 0;                 // 将 L 改造为静态链表
    for(i=0; i<L.keynum; ++i)               // 按最低位优先依次对各关键字分配和收集
    {
        Distribute(L.r, i, f, e);           // 第 i 趟分配
        Collect(L.r, i, f, e);              // 第 i 趟收集
    } // for
}
```

> [!abstract] **算法分析**
> **（1）时间复杂度**：每趟分配 $O(n)$、收集 $O(rd)$，共 $d$ 趟，故 $O(d(n+rd))$。
> **（2）空间复杂度**：需 $2rd$ 个队列指针，另增加 $n$ 个指针域，为 $O(n+rd)$。

> [!tip] **算法特点**
> （1）稳定排序。（2）可用于链式结构，也可用于顺序结构。（3）时间复杂度可突破基于比较方法的下界 $O(n \log_2 n)$，达到 $O(n)$。（4）使用条件严格：需知各级关键字的主次关系与取值范围。
