## LeetCode 769 最多能完成排序的块 ★★

> [!abstract] **问题描述**
> 给定长度为 $n$ 的整数数组 `arr`，元素在 $[0, n-1]$ 范围内且互不相同。将 `arr` 分割成若干块，每块单独排序后连接的结果与升序排序原数组相同。返回能分成的最多块数。

> [!example] **输入输出示例**
> 输入：`arr = [1, 0, 2, 3, 4]`
> 输出：`4`
> 解释：可分成 `[1,0],[2,3,4]`；但分成 `[1,0],[2],[3],[4]` 块数最多，各块排序后为 `[0,1],[2],[3],[4]`。

> [!info] **问题分析**
> 元素在 $[0, n-1]$ 且互不相同，升序排序后 `arr[i] = i`。分块最小依据：当前块中最大值 `max` 等于其在数组中的对应下标。遍历时保存 `max`，当 `max == i` 即到最小分块位置，`chunknum` 加 1（图 8.24）。

> [!example] 具体实现步骤（图 8.24，`arr = [1, 0, 2, 3, 4]` 从 `i=0` 到 `i=4` 遍历，每次更新 `max` 并与当前下标比较）。
>
> ![[Attachments/Pasted image 20260718224738.png]]
> *图 8.24 LeetCode 769 具体实现步骤*

> [!note] **算法步骤**
> ① 定义 `chunknum`（当前块数）与 `max`（当前块最大值），初值 0。
> ② 遍历 `arr`：将 `max` 与 `arr[i]` 的较大者赋给 `max`；若 `max == i`，`chunknum` 加 1。
> ③ 返回 `chunknum`。

**【算法描述】**

```c
int maxChunksToSorted(int* arr, int arrSize)
{ // 计算数组能分成的最多块数
    int chunknum = 0, max = 0;
    for (int i = 0; i < arrSize; ++i)
    {
        max = fmax(max, arr[i]);        // 将当前元素 arr[i] 和 max 中的较大值赋值给 max
        if (max == i)                    // 如果 max 等于 i，将可分块数加 1
            chunknum++;
    }
    return chunknum;
}
```

> [!abstract] **算法分析**
> 遍历数组一次，时间复杂度 $O(n)$；仅需常量额外空间，空间复杂度 $O(1)$。

## LeetCode 215 数组中的第 k 大元素 ★★

> [!abstract] **问题描述**
> 给定整数数组 `nums` 与整数 `k`，返回第 $k$ 大的元素。请设计时间复杂度 $O(n)$ 的算法。

> [!example] **输入输出示例**
> 输入：`nums = [3, 2, 1, 5, 6, 4]`, `k = 2`
> 输出：`5`

> [!info] **问题分析**
> 可先排序再取倒数第 $k$ 个（平均 $O(n\log_2 n)$），但可改进为"快速选择"：任取元素作枢轴，一趟快排后小于者前移、大于者后移，枢轴落分界处不再变。若枢轴恰在倒数第 $k$ 个位置即为答案；否则只递归含目标的子区间（而非两区间），避免全排序（图 8.25）。

> [!example] 具体实现步骤（图 8.25，数组 `[3,2,1,5,6,4]` 选倒数第 2 大即第 5 小元素，快速选择最终定位到 5）。
>
> ![[Attachments/Pasted image 20260718224746.png]]
> *图 8.25 LeetCode 215 具体实现步骤*

> [!note] **算法步骤**
> ① 取 `nums` 首元素作枢轴 `pivotkey`。
> ② 遍历划分：小于等于者移至左、大于者移至右。
> ③ 判断枢轴下标 $j$ 与目标下标 $k$：若 $j < k$ 递归右子区间，否则递归左子区间。

**【算法描述】**

```c
int quickselect(int *nums, int l, int r, int k)
{ // 快速选择函数
    if (l == r) return nums[k];          // 区间内只有一个元素，直接返回
    int pivotkey = nums[l];              // 选择第一个元素作为枢轴
    int i = l - 1, j = r + 1;
    while (i < j)
    {
        do i++; while (nums[i] < pivotkey); // 从左向右找第一个 >= pivotkey 的数
        do j--; while (nums[j] > pivotkey); // 从右向左找第一个 <= pivotkey 的数
        if (i < j)                          // 交换这两个元素
        {
            int tmp = nums[i];
            nums[i] = nums[j];
            nums[j] = tmp;
        }
    }
    if (j < k)
        return quickselect(nums, j + 1, r, k); // 在右子区间中查找
    else
        return quickselect(nums, l, j, k);     // 在左子区间中查找
}

int findKthLargest(int *nums, int numsSize, int k)
{ // 寻找第 k 大元素
    return quickselect(nums, 0, numsSize - 1, numsSize - k);
}
```

> [!abstract] **算法分析**
> 最坏情况每次选到最大/最小元素作枢轴，每趟仅排除一个，递归深度 $O(n)$，时间复杂度 $O(n^2)$；平均情况每次近似均匀二分，递归深度对数级，平均 $T(n) = T(n/2) + O(n)$，得 $O(n)$。递归需 $\log_2 n$ 栈空间，空间复杂度 $O(\log_2 n)$。

## LeetCode 23 合并 k 个升序链表 ★★★

> [!abstract] **问题描述**
> 给定链表数组，每个链表已升序排列，要求合并到一个升序链表并返回。

> [!example] **输入输出示例**
> 输入：`lists = [[1,4,5],[1,3,4],[2,6]]`
> 输出：`[1,1,2,3,4,4,5,6]`
> 即 `1->4->5`, `1->3->4`, `2->6` 合并为 `1->1->2->3->4->4->5->6`。

> [!info] **问题分析**
> 每个链表升序，可用归并排序将各链表合并。递归将 $k$ 个链表两两一组合并；第一轮合为 $k/2$ 个，重复至一个有序链表。合并两有序链表同第 2 章算法 2.17。

> [!note] **算法步骤**
> ① 递归两两合并，当 `low == high` 返回 `lists[low]`，`low > high` 返回 `NULL`。
> ② 取 `mid = (low + high)/2` 分为两部分，分别递归左右子表后合并。

**【算法描述】**

```c
struct ListNode* MergeTwoLists(struct ListNode* list1, struct ListNode* list2)
{ // 归并两个有序链表
    struct ListNode* node = (struct ListNode*) malloc(sizeof(struct ListNode));
    struct ListNode* list3 = node;
    while(list1 && list2)
    {
        if(list1->val > list2->val)
        {   // 链表 1 当前结点值大于链表 2 当前结点值
            node->next = list2;             // 将链表 2 的结点接到新结点后面（后插法）
            list2 = list2->next;            // 链表 2 工作指针后移
        }
        else
        {   // 链表 1 当前结点值 <= 链表 2 当前结点值
            node->next = list1;
            list1 = list1->next;
        }
        node = node->next;
    } // while
    node->next = list1 != NULL ? list1 : list2; // 多余部分接到新链表尾部
    return list3 -> next;                   // 返回新链表
}

struct ListNode* Merge(struct ListNode** lists, int low, int high)
{ // 递归实现链表的合并
    if (low == high) return lists[low];
    if (low > high) return NULL;
    int mid = (low + high) / 2;
    return MergeTwoLists(Merge(lists, low, mid), Merge(lists, mid + 1, high));
}

struct ListNode* mergeKLists(struct ListNode** lists, int listsSize)
{ // 将 k 个链表合并到一个升序链表中
    return Merge(lists, 0, listsSize - 1);
}
```

> [!abstract] **算法分析**
> 递归回升过程：第一轮合 $k/2$ 组、每组代价 $O(2n)$；第二轮 $k/4$ 组、每组 $O(4n)$……总代价 $\sum_{i=1}^{\log_2 k} \frac{k}{2^i} \times 2n = kn \log_2 k$，整体时间复杂度 $O(kn \log_2 k)$；递归需 $\log_2 n$ 栈空间，空间复杂度 $O(\log_2 n)$。
