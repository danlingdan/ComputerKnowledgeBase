> [!topic] 所属主题
> [[MOC - 第8章 排序#8.7 外部排序|8.7 外部排序]]

由式（8-5），归并趟数 $s$ 与 $k$、$m$ 均反比，减少 $m$ 是另一途径。$m = \lceil n/l \rceil$，$l$ 为初始归并段长度，取决于内部排序可用内存工作区大小。要减小 $m$（增 $l$）需新排序法。

**置换-选择排序**（Replacement-Selection Sorting）基于树形选择排序，特点是：选择最小（大）关键字与输入、输出交叉/平行进行。

例：初始文件 24 个记录，关键字为 `51,49,39,46,38,29,14,61,15,30,1,48,52,3,63,27,4,13,89,24,46,58,33,76`，内存工作区容 6 个记录。普通选择排序得 4 个初始归并段（RUN1～RUN4）；置换-选择排序得 3 个（更长）：RUN1：29,38,39,46,49,51,61；RUN2：1,3,14,15,27,30,48,52,63,89；RUN3：4,13,24,33,46,58,76。

设输入文件 FI、输出文件 FO、内存工作区 WA（容量 $w$），操作过程：
① 从 FI 输出 $w$ 个记录到 WA。② 从 WA 选关键字最小者记 MINIMAX。③ 将 MINIMAX 输出 FO。④ 若 FI 不空，从 FI 输出下一记录到 WA。⑤ 从 WA 中所有比 MINIMAX 大的记录中选最小关键字者作新 MINIMAX。⑥ 重复 ③～⑤ 至 WA 选不出新 MINIMAX，得一初始归并段，输入结束标志到 FO。⑦ 重复 ②～⑥ 至 WA 空。

> [!note] **说明**
> （1）内存工作区中的记录作为败者树的外部结点，根结点的双亲结点指示工作区中关键字最小的记录。
> （2）为便于选 MINIMAX，为每个记录附设所在归并段序号；比较时先比段号（小者为胜），段号相同再比关键字。
> （3）败者树建立可从工作区所有记录段号均为 0 开始，从 FI 逐个输入 $w$ 个记录时自上而下调整；这些记录段号为"1"，对段号 0 者均为败者，逐个填充到败者树各结点。

下图 8.19（a）～（g）显示败者树建立过程，最小关键字记录为 `wa[0]`；输出 `wa[0].rec` 并从 FI 输入下一记录至 `wa[0]`，因其关键字小于刚输出者，段号设为 2（图（h））；输出 `wa[1]` 后新输入者关键字较大，段号仍为 1（图（i））；图（j）输出 6 个记录后选得 MINIMAX 为 `wa[1]`；图（k）新输入记录关键字较小、段号亦为 2，工作区所有记录段号均为 2，新 MINIMAX 段号大于当前段号，说明该段结束、新记录属下一归并段。

**【内存工作区存储结构】**

```c
// - - - - 内存工作区的存储结构 - - - -
typedef struct
{
    RedType rec;                // 记录
    KeyType key;                // 从记录中抽取的关键字
    int rnum;                   // 所属归并段的段号
}RedNode, WorkArea[w];          // 内存工作区，容量为 w
WorkArea wa;
```

*图 8.19 置换-选择过程中的败者树；（图 8.20 环形路上路面积雪状态）*

置换-选择所得初始归并段长度不等，且当输入关键字为随机数时，平均长度为工作区大小 $w$ 的两倍（E.F. Moore 1961 年以扫雪机类比证明）。扫雪机在环形路等速扫雪、下雪均衡，达到平衡后积雪总量不变，紧靠机前最深（深 $h$）、刚扫过处为 0；伸展开如图 8.20，积雪总体积 $w$、路长 $l$，机扫一圈扫掉体积 $lh = 2w$。类比：工作区记录如积雪，输出 MINIMAX 如扫走雪，新输入如新下雪；关键字随机时新记录比 MINIMAX 大/小概率相等，大则属当前段（机前，本圈扫走），小则属下一段（机后，下圈扫走）。得一初始归并段如机走一圈，工作区容量 $w$ 时期望段长即为 $2w$。不计输入输出，生成所有初始归并段时间为 $O(n \log_2 w)$。
