# 编程与算法局部规则

- 算法笔记说明问题、输入、输出、前置条件、核心思想、正确性依据、时间复杂度和空间复杂度。
- 数据结构笔记说明抽象接口、表示方式、不变量、典型操作、复杂度和适用场景；区分抽象数据类型与具体实现。
- 先用数学描述或伪代码表达通用算法，再按需提供语言实现；不要让某种语言的语法掩盖算法本身。
- 代码示例注明语言标准或版本、依赖、构建命令、运行环境、样例输入输出和是否已验证。
- 比较算法时使用相同计算模型和输入假设；平均、最坏、均摊和期望复杂度不得混写。
- 记录边界条件、失败情况、数值溢出、索引约定和可变性等常见错误。
- 编程语言笔记区分语法、语义、类型系统、运行时和工程惯例；版本相关行为写入 `version` 或正文。
- 课程作业与实验保留题目要求和个人实现边界；不要把参考答案冒充个人验证结果。
