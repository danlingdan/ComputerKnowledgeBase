## 第一章 计算机网络概述

> [!abstract]- 本章高频考点

### 1.1 计算机网络在信息时代的作用

- **信息时代特征**：数字化、网络化、信息化。
- **三类基础网络**：电信网络、有线电视网络、计算机网络。
- **互联网（Internet）核心价值**：联通性与资源共享。

### 1.2 因特网概述

#### 1.2.1 网络的网络

计算机网络由**节点（node）**和**链路（link）**组成。节点可以是计算机、集线器、交换机、路由器等设备。

#### 1.2.2 互联网基础结构发展的三个阶段

1. 从单个网络ARPANET向互连网发展。
2. 建成了三级结构的互联网：主干网、地区网、校园网（企业网）。
3. 全球范围的多层次ISP结构的互联网。

#### 1.2.3 互联网的标准化工作

标准化组织关系可概括为：**互联网协会（ISOC）→ 互联网体系结构委员会（IAB）→ IETF 与 IRTF**。

标准化材料通常从 **Internet-Draft** 开始，经评审后可能发布为 RFC；RFC 不都属于标准。标准轨 RFC 可继续成熟，并由 STD 编号索引正式互联网标准。

### 1.3 互联网的组成

- **边缘部分**：由连接到互联网的主机组成，是==用户直接使用的部分==。
- **核心部分**：由大量网络及连接这些网络的路由器组成，负责==为边缘部分提供分组交换服务==。

#### 1.3.1 边缘部分

边缘部分常见的两种通信方式：

1. ==客户—服务器方式（C/S）==。
2. ==对等方式（P2P）==。

#### 1.3.2 核心部分

##### 1.3.2.1 电路交换

主叫端与被叫端先建立==一条专用的物理通路==，过程为：**建立连接 → 通信 → 释放资源**。资源在通信期间持续占用，因此对突发数据而言利用率较低。

##### 1.3.2.2 分组交换

分组交换采用==存储转发==技术。整块待发送数据称为==报文==；报文被划分为数据段，每段前加上==首部（header）==后形成==分组（packet）==，也常称为包。

分组交换具有线路利用率高、路由灵活、适合突发数据等优点；它本身不保证端到端可靠，可靠性需要具体链路层或运输层机制提供。

分组转发需要排队，会有**时延**，分组也需要携带控制信息，有额外**开销**

##### 1.3.2.3 总结

- **电路交换**：整个报文的比特流连续地从源点直达终点，如同在一条管道中传送。
- **报文交换**：完整报文到达相邻节点并全部存储后，节点查找转发表并转发到下一跳。
- **分组交换**：报文被拆成若干分组，各分组逐跳存储、查表和转发。

> [!compare] 三种交换方式怎么区分
> - **电路交换**：先建立专用通路，连续传输，资源预留但突发数据利用率低。
> - **报文交换**：以完整报文为单位逐跳存储转发，节点缓存需求和等待时延较大。
> - **分组交换**：报文拆成分组后逐跳存储转发，可流水并发，是互联网的核心方式。

$$
\text{时延带宽积} = \text{传播时延} \times \text{带宽}
$$

#### 1.6.6 往返时间 RTT

往返时间（RTT）是从发送端发出数据到收到相应确认所经历的时间，包含双向传播、发送、处理和排队时延。仅在路径近似对称且忽略其他时延时，才可近似写成：

$$
\mathrm{RTT} \approx 2 \times \text{单向传播时延}
$$

#### 1.6.7 利用率

$$
D = \frac{D_0}{1 - U}
$$

- $D$ 表示当前网络的平均时延。
- $D_0$ 表示网络空闲时的基准时延。
- $U$ 表示网络利用率，且 $0 \le U < 1$。

> [!warning] 模型边界
> $D=D_0/(1-U)$ 是说明利用率接近 1 时排队时延急剧增加的简化模型，不应脱离其排队假设直接套用于所有网络。

### 1.7 计算机网络体系结构

#### 分层体系的好处

- ==各层之间是独立的。==
- ==灵活性好。==
- ==结构上可分割开。==
- ==易于实现和维护。==
- ==能促进标准化工作。==

#### 课程采用的五层体系结构

![[assets/image-20260605180300884.png]]

1. **应用层（第 5 层）**

	- 负责为应用程序提供网络服务接口。
	- 典型协议：HTTP、FTP、SMTP、DNS。
	- 数据单位：报文（message）。

2. **运输层（第 4 层）**

	- 负责端到端的通信和数据可靠传输。
	- 典型协议：TCP（可靠）、UDP（快速）。
	- 数据单位：报文段（segment）。

3. **网络层（第 3 层）**

	- 负责分组转发和路由选择，实现跨网络通信。
	- 核心协议：IP协议（IPv4/IPv6）。
	- 数据单位：数据报（datagram）。

4. **数据链路层（第 2 层）**

	- 负责相邻节点之间的帧传输和差错控制。
	- 典型协议：以太网、Wi-Fi（802.11）。
	- 数据单位：帧（frame）。

5. **物理层（第 1 层）**

	- 负责原始比特流在物理介质上的传输。
	- 涉及：电压、线缆、接口、光纤、无线电波等。
	- 数据单位：比特（bit）。

#### 2.2.3 信道的极限容量

信道容量受两个重要因素限制：

##### 带宽受限：信道能够通过的频率范围

奈氏准则

$$
\text{最高码元速率}=2W\quad(\text{Baud})
$$

在带宽为 $W\ \mathrm{Hz}$ 的理想低通信道中，若不考虑噪声，码元传输的最高速率为 $2W$ Baud。超过此上限会产生严重的==码间串扰==，使接收端难以正确判决码元。

##### 噪声影响：信噪比与香农公式

香农公式

$$
C=W\log_2\left(1+\frac{S}{N}\right)\quad(\mathrm{bit/s})
$$

信道带宽或信噪比越大，理论极限信息率越高。

> [!formula] 两个容量公式的适用条件
> - **奈氏准则**讨论理想低通信道的码元速率上限；若每个码元有 $M$ 种状态，理想数据率上限为 $2W\log_2M$。
> - **香农公式**讨论有噪声信道的理论信息率上限，其中 $S/N$ 必须使用线性比值；若题目给出分贝，先用 $\mathrm{SNR}_{dB}=10\log_{10}(S/N)$ 换算。

### 2.3 物理层下的传输媒体

传输媒体分为两大类：导引型传输媒体和非导引型传输媒体。

#### 2.3.1 导引型传输媒体

| 媒体     | 特点                             | 优点                                   | 局限                             |
| -------- | -------------------------------- | -------------------------------------- | -------------------------------- |
| 双绞线   | 两根绝缘铜线绞合而成             | 便宜、易安装、应用广泛                 | 带宽和距离有限，抗干扰能力一般   |
| 同轴电缆 | 内导体、绝缘层、外导体屏蔽层组成 | 抗干扰能力比双绞线强，带宽较大         | 成本较高，铺设不如双绞线灵活     |
| 光纤     | 利用光脉冲传输数据               | 带宽极大、损耗低、抗电磁干扰、安全性好 | 接续和施工要求较高，设备成本较高 |

##### 双绞线分类

| 分类方式             | 主要类型             | 关键特点与适用场景                                           |
| :------------------- | :------------------- | :----------------------------------------------------------- |
| **按屏蔽性**         | **非屏蔽 (UTP)**     | 无金属屏蔽层，靠线对绞合抵消干扰。价格便宜、安装方便，适用于普通家庭和办公网络。 |
|                      | **屏蔽 (STP/FTP等)** | 带铝箔或金属编织网屏蔽层，抗干扰能力强，且有更好保密性。适用于工厂、机房等电磁干扰强的环境。**注意，屏蔽系统必须正确接地才能发挥作用**。 |
| **按性能等级 (Cat)** | **Cat 5e 及以下**    | Cat 5e 通常支持最长 100 m 的千兆以太网；更低类别的能力和适用范围不同。 |
|                      | **Cat 6 / Cat 6a**   | Cat 6支持**千兆**，短距离可达**万兆**。Cat 6a是前者增强版，稳定支持**万兆**（100米）。 |
|                      | **Cat 7 / Cat 8**    | Cat 7为支持**万兆**的高规格屏蔽线。Cat 8是专为数据中心设计的**高速（25/40Gbps）** 线。 |

#### 2.3.2 非导引型传输媒体

非导引型传输媒体就是无线传输媒体，信号不沿着固定导线传播，而是在自由空间传播。

常见类型：

1. 无线电波：传播能力强，可绕射，适合广播和移动通信。
2. 微波：频率高，容量大，方向性强，常用于地面微波和卫星通信。
3. 红外线：常用于短距离通信，不能穿透墙壁。
4. 激光：方向性很强，适合点到点通信，但受天气影响明显。

### 2.4 信道复用技术

> [!important] 复用的目的
> 让多个用户共享同一条高速传输线路，从而提高信道利用率、降低通信成本。

#### 2.4.1 频分复用、时分复用、统计时分复用

##### 频分复用 FDM

频分复用把总带宽划分为多个互不重叠的频带，每个用户占用一个固定频带。

特点：

1. 用户在同一时间通信。
2. 不同用户使用不同频率。
3. 需要保护频带，避免相邻频带干扰。

##### 时分复用 TDM

时分复用把时间划分为周期性的时隙，每个用户在自己的时隙中发送数据。

特点：

1. 用户共享同一频带。
2. 不同用户在不同时间发送。
3. 普通 TDM 即使用户没有数据，也会占用固定时隙，因此可能浪费资源。

##### 统计时分复用 STDM

统计时分复用按需分配时隙，只有有数据的用户才占用时隙。

优点：

1. 比普通 TDM 更灵活。
2. 信道利用率更高。
3. 适合突发式数据通信。

#### 2.4.2 波分复用

波分复用本质上是光纤中的频分复用。不同用户或不同信号使用不同波长的光在同一根光纤中传输。

| 类型 | 含义                                         |
| ---- | -------------------------------------------- |
| WDM  | 波分复用                                     |
| DWDM | 密集波分复用，波长间隔更小，可复用更多光信道 |

#### 2.4.3 码分复用

##### CDM / CDMA

码分复用让多个用户在同一时间、同一频带通信，但每个用户使用不同的码片序列。

CDMA 的关键思想：

1. 每个站分配一个互相正交的码片序列。
2. 发送 1 时发送自己的码片序列。
3. 发送 0 时发送码片序列的反码。
4. 接收端用对应码片序列做内积运算，提取目标用户的数据。
5. ==其他用户的码片序列与目标用户正交，相关结果为 0，因此不会互相干扰。==

$$
S \bullet T \equiv \frac{1}{m} \sum_{i=1}^{m} S_i T_i = 0
$$


## 第三章 数据链路层

> [!abstract]- 本章高频考点
> [[#3.1.2 三个基本问题|成帧、透明传输、差错检测]] · [[#3.2 点对点协议 PPP|PPP]] · [[#3.3.2 CSMA/CD 协议|CSMA/CD]] · [[#2. 自学习算法|交换机自学习]] · [[#3.4.3 虚拟局域网 VLAN|VLAN]]

数据链路层面对两类基本信道：

- **点对点信道**：链路连接两个节点。
- **广播信道**：多个节点共享同一传输媒体，需要介质访问控制。

### 3.1 数据链路层的几个共同问题

#### 3.1.1 数据链路和帧

**链路**是相邻结点之间的一段物理线路，例如一段双绞线、一段光纤、一个无线信道。它强调的是物理连接。

**数据链路**是在链路的基础上，加上实现通信所需的硬件和协议。例如网卡、驱动程序、数据链路层协议等。

**帧**是数据链路层传送的数据单位。网络层交下来的 IP 数据报会被封装到帧的数据部分，再加上首部和尾部。

#### 3.1.2 三个基本问题

##### 1. 封装成帧

封装成帧就是给网络层交下来的数据加上帧首部和帧尾部，让接收方能够判断一帧从哪里开始、到哪里结束。

![[assets/image-20260606144827465.png|720]]

常用的帧定界方法有：

| 方法           | 思想                                   |
| -------------- | -------------------------------------- |
| 字符计数法     | 在帧首部写明帧的长度                   |
| 特殊字符定界法 | 用特殊字符表示帧开始和帧结束           |
| 比特填充法     | 用特殊比特串作标志，并在数据中进行填充 |
| 违规编码法     | 利用物理层不会正常出现的编码表示边界   |

##### 2. 透明传输

透明传输的意思是：**无论数据部分出现什么比特组合，都应该被当作普通数据传过去，而不应被误认为控制信息。**

例如 PPP 帧中使用 `7E` 作为帧定界符。如果数据中真的出现了 `7E`，就必须进行转义，否则接收方会误以为这一帧结束了。

![[assets/image-20260606144850293.png|720]]

常见处理方式：

| 场景           | 方法                   |
| -------------- | ---------------------- |
| 面向字节的协议 | 字节填充，也叫字符填充 |
| 面向比特的协议 | 零比特填充             |

##### 3. 差错检测

物理链路可能受到噪声干扰，导致比特 0 变 1 或 1 变 0。因此数据链路层需要进行差错检测。

最常用的方法是 **CRC，循环冗余检验**。

CRC 的基本思路：

1. 发送方和接收方约定一个生成多项式 `P(x)`。
2. 发送方用待发送数据除以生成多项式，得到余数。
3. 把余数作为 FCS 加在数据后面。
4. 接收方收到后再次除以同一个生成多项式。
5. 如果余数为 0，认为没有检测到差错；如果余数不为 0，认为有差错。

> [!warning] CRC 的能力边界
> CRC 用于**检错**，不能纠正错误，也不等价于可靠传输。可靠交付还需要确认、序号、超时重传等机制。

![[assets/image-20260606144919981.png|500]]

### 3.2 点对点协议 PPP

#### 3.2.1 PPP 协议的特点

| 特点               | 说明                               |
| ------------------ | ---------------------------------- |
| 简单               | 不追求复杂的可靠传输，只做必要功能 |
| 封装成帧           | 能把网络层数据报封装成 PPP 帧      |
| 透明性             | 支持字节填充或零比特填充           |
| 支持多种网络层协议 | 通过协议字段区分 IP、IPv6 等       |
| 差错检测           | 使用 FCS 检测帧差错                |
| 多种类型链路       | 串行的，并行的。光的，电的等       |
| 检测连接状态       | 自动检测是否正常工作               |
| 最大传送单元       | 设置默认的MTU                      |
| 网络层地址协商     | 能够使网络层相互协商地址           |
| 数据压缩协商       | 协商使用的压缩算法                 |

PPP协议的组成：

1 将IP数据报封装到串行链路的方法。

2 一个建立、配置和测试数据链路连接的**链路控制协议(LCP)**

3 一套**网络控制协议(NCP)**

#### 3.2.2 PPP 协议的封装格式

​	透明传输方法

##### 1. 异步传输中的字节填充

异步链路按字节传输。PPP 中：

- `7E` 是帧定界符；
- `7D` 是转义字符。

##### 2. 同步传输中的零比特填充

#### 3.3.2 CSMA/CD 协议

> [!note] 协议过程
> **先听后发，边发边听，碰撞停止，随机退避。** CSMA/CD 适用于传统共享式、半双工以太网；交换式全双工以太网不存在碰撞，不使用该碰撞检测过程。

##### 1. 多点接入

多个站点共享同一条总线或同一个碰撞域。

##### 2. 载波监听

发送前先听信道：

```text
信道空闲 → 可以发送
信道忙 → 暂不发送
```

##### 3. 碰撞检测

发送过程中继续监听信道。如果检测到碰撞，就立即停止发送，并发送人为干扰信号，通知其他站点发生碰撞。

![[assets/image-20260606150953325.png|720]]

##### 4. 争用期

以太网规定最短帧长，是为了保证发送站在帧发送完之前能够检测到最远端可能发生的碰撞。

争用期也叫碰撞窗口，传统 10 Mbit/s 以太网的争用期是：

```text
512 比特时间 = 51.2 μs
```

##### 5. 最短帧长

传统以太网最短 MAC 帧长是：

```text
64 字节 = 512 bit
```

如果帧太短，发送方可能已经发送完毕，却还没有收到远端碰撞信号，就会误以为发送成功。

##### 6. 截断二进制指数退避

发生碰撞后，第 `i` 次重传时，从下面范围随机选一个整数 `r`：

```text
0 ≤ r < 2^k，k = min(i, 10)
```

等待时间为：

```text
r × 512 比特时间
```

当重传次数太多时，以太网会放弃该帧并报告错误。

#### 3.3.3 使用集线器的星型拓扑

使用集线器的以太网在逻辑上仍是一个总线网,各站共享逻辑上的总线,使用的还是CSMA/CD 协议。网络中的各站必须竞争对传输媒体的控制,并且在==同一时刻至多只允许一个站发送数据==。

![[assets/image-20260606151228174.png|720]]

#### 3.3.4 以太网的信道利用率

定义参数 $a$ 为**以太网单程端到端时延 $\tau$ 与帧的发送时间 $T_0$ 之比**：

$$
a=\frac{\tau}{T_0}
$$

*   **当 $a \to 0$ 时**：表示一发生碰撞就能立即检测并停止发送，信道资源浪费极少，利用率高。
*   **当 $a$ 较大时**：表明争用期占比大，发生碰撞后浪费较多的信道资源，导致信道利用率明显降低。
*   **核心结论**：**参数 $a$ 的值应当尽可能小些。**

要让 $a$ 变小，根据公式 $a=\frac{\tau}{T_0}$，需要：
1.  **减小分子 $\tau$**：限制以太网连线的长度（不能太长）。
2.  **增大分母 $T_0$**：限制帧的长度（不能太短）。

在完全理想（无碰撞、调度效率最高）的情况下，**极限信道利用率 $S_{\max}$** 为：

$$
S_{\max}=\frac{T_0}{T_0+\tau}=\frac{1}{1+a}
$$

前24比特表示厂商，后24位厂商自定义

### 4.1 网络层的几个重要概念

#### 4.1.1 网络层提供的两种服务

网络层服务可采用面向连接的虚电路思路或无连接的数据报思路；互联网的 IP 层采用**尽最大努力交付的数据报服务**，虚电路思路仍存在于其他网络技术中。

网络层要设计得尽量简单,向其上层只提供简单灵活的、无连接的、==尽最大努力交付==的数据报服务

#### 4.1.2 网络层的两个层面

1. **数据层面**：负责实际转发分组。路由器收到分组后，根据转发表查找下一跳并转发出去。
2. **控制层面**：负责生成和维护转发表。它通过路由选择算法和路由协议决定到各个目的网络应走哪条路径。

传统路由器中，数据层面和控制层面通常都在路由器内部实现。SDN 把二者分离：交换机主要负责数据层面，控制器负责控制层面。

### 4.2 网际协议 IP

#### 4.2.1 虚拟互连网络

真实互联网由许多异构网络组成，不同网络可能使用不同的链路层协议、帧格式、地址形式、MTU 和接入方式。IP 的作用是把这些异构网络在网络层上统一起来，使它们看起来像一个逻辑上的统一网络。称为**虚拟互连网络**。

#### 4.2.2 IPv4 地址

##### 1 表示方法

IPv4 地址长度为 32 位，通常用点分十进制表示。

![[assets/image-20260606221817093.png|720]]

![[assets/image-20260606221831149.png|720]]

##### 2 分类

![[assets/image-20260606221944601.png|720]]

![[assets/Untitled.png|600]]

特殊地址：

`127.0.0.0/8` 用于 IPv4 回环。传统子网中，主机位全 0 表示网络地址，主机位全 1 表示定向广播地址，例如 `192.168.1.0/24` 与 `192.168.1.255/24` 通常不能分配给普通主机接口。

全0给网关，全1是进行广播

##### 3. 无分类编址 CIDR

1. ##### 网络前缀

	在 CIDR 中，IP 地址写成 `192.168.1.1/24` 这种形式，`/24` 表示前 **24 位**是网络前缀。

	==实际是掩码有效位数==

![[assets/image-20260606223110962.png|720]]

	这里就应该写成192.168.0.xxx/26，即(24+2)

2. ##### 地址块

	具有**相同网络前缀**的一组 IP 地址的集合

	- 例如 `192.168.1.0/24` 这个地址块，包含了从 `192.168.1.0` 到 `192.168.1.255` 共 256 个地址。

3. ##### 地址掩码

	用来“遮住” IP 地址，只露出网络前缀的“筛子”。

	- 掩码也是一个 32 位的数字：前缀部分全为 **1**，主机部分全为 **0**。
	- 将 IP 地址和掩码做“按位与”运算，就能得到网络前缀（即该 IP 所属的地址块）。
		- 示例：IP=`192.168.1.100`，掩码=`255.255.255.0`，按位与后得到 `192.168.1.0` → 这就是它所属的地址块。

![[assets/image-20260606223350144.png|720]]

##### 4. IP 地址的特点

###### 1. IP 地址的分层结构（网络前缀 + 主机号）

- **结构**：IP地址分为**网络前缀**（代表所在网络）和**主机号**（代表具体设备）。
- **好处**：路由器转发分组时只根据“网络前缀”进行转发，而不关心具体的主机号，这样能大大减小路由表的规模，加快转发速度。

###### 2. IP 地址标识的是“接口”，而不是“设备”

- 一个主机或路由器如果连接了多个网络（如路由器），它就有多个IP地址，被称为**多归属主机**。IP地址实际上标识的是该设备在特定网络上的**“接口（网卡）”**，而不是设备本身。

###### 3. 网络的本质与路由器的连接

- **网络的定义**：具有相同网络前缀的主机或路由器的接口集合，构成一个网络（或子网）。
- **路由器的角色**：路由器负责连接不同的网络。因此，**路由器至少有两个不同的IP地址**，且这两个地址的网络前缀必然是不同的（因为它们属于不同的网络）。

###### 4. 重要细节

- **相同网络前缀**：在同一个局域网内，所有主机或路由器的接口IP，其“网络前缀”必须是完全一样的。
- **网络地址**：一个网络的地址（Network Address）就是主机号全为0的IP地址。例如图中 `LAN1` 的网络地址是 `1.1.1.0/29`。
- **MAC地址与IP地址的区别**：
	- **以太网交换机**工作在数据链路层（第二层），它**只有MAC地址，没有IP地址**。
	- 主机或路由器的具体接口不仅有MAC地址，还会有分配的具体IP地址。
- **点对点链路（租用线路）的特殊处理**：
	- 传统的点对点链路（如PPP）只有两端接口，如果按常规分配IP，会浪费IP地址。为了节约IP资源，点对点链路常使用 **/31 的子网掩码**（此时主机位只有1位，分别为0和1，恰好可以分配给两端接口）。还有一种极端的节约方式叫**无编号网络**（Unnumbered network），即点对点链路不需要分配具体的IP地址。

### 4.3 IP 层转发分组过程

#### 4.3.1 基于终点的转发

传统 IP 转发是**基于终点的转发**。路由器只根据目的 IP 地址查找转发表，不关心这个分组来自哪里，也不关心应用层内容。

转发表中通常包含：

```text
目的网络前缀 → 下一跳 / 转发接口
```

例如：

```text
145.13.128.0/18 → 接口 m2
0.0.0.0/0       → 默认路由
```

#### 4.3.2 最长前缀匹配

CIDR 环境下，一个目的地址可能匹配多个前缀。路由器必须选择**最长前缀匹配**的那一项。

例如目的地址 `11.1.2.5` 同时匹配：

```text
11.0.0.0/8
11.1.0.0/16
11.1.2.0/24
```

应选择 `/24`，因为 `/24` 最具体，前缀最长。

#### 4.3.3 默认路由

```text
0.0.0.0/0
```

它可以匹配任何目的地址。只有当没有更具体的路由项匹配时，才使用默认路由。

### 4.4 网际控制报文协议 ICMP

IP 只负责尽力交付，但网络中可能出现目的不可达、TTL 过期、分片失败等问题。ICMP 用来报告差错和提供控制信息。

ICMP 报文分为两类：

1. **差错报告报文**：目的不可达、时间超过、参数问题、改变路由等。`ping`
2. **询问报文**：回送请求和回答、时间戳请求和回答等。`traceroute` / `tracert`

### 4.5 IPv6

IPv6 不是简单地把地址从 32 位扩展到 128 位，它还改进了首部结构、扩展首部、自动配置、安全性和多播支持。

### 4.6 互联网的路由选择协议

#### 4.6.1 自治系统 AS

自治系统 AS 是在同一管理机构控制下的一组网络和路由器。互联网路由分为：

- AS 内部路由：使用内部网关协议 IGP。
- AS 之间路由：使用外部网关协议 EGP，现在主要是 BGP。

#### 4.6.2 RIP

RIP 是内部网关协议，属于距离向量协议。

特点：

1. 度量为跳数。
2. 最大有效距离为 15，16 表示不可达。
3. 只和相邻路由器交换路由信息。
4. 周期性发送整张路由表。
5. 简单，但收敛慢，适合小型网络。

#### 4.6.3 OSPF

OSPF 是内部网关协议，属于链路状态协议。

特点：

1. 使用洪泛法发送链路状态信息。
2. 每个路由器最终形成全网拓扑数据库。
3. 使用 Dijkstra 算法计算最短路径。
4. 收敛快，支持分层区域，适合较大网络。
5. 直接封装在 IP 数据报中。

#### 4.6.4 BGP

BGP 是自治系统之间的路由协议。

特点：

1. 属于路径向量协议。
2. 重点不是找“最短路径”，而是根据策略选择可达路径。
3. 使用 TCP 连接传输路由信息，可靠性由 TCP 保证。
4. 支持丰富的路径属性，例如 AS_PATH。
5. 可避免 AS 级路由环路。

### 4.7 IP 多播

IP 多播用于一对多通信。发送方只发送一份数据报，网络负责把它复制并交付给多个组成员。

#### 4.7.1 多播地址

IPv4 多播地址属于 D 类地址

```text
224.0.0.0 ~ 239.255.255.255
```

#### 4.7.2 IGMP

IGMP 的作用是让本地多播路由器知道：本局域网上是否有主机加入了某个多播组。

IGMP 并不负责整个互联网的多播路由。多播路由器之间还需要使用多播路由选择协议。

### 4.8 VPN 和 NAT

#### 4.8.1 VPN

VPN 是虚拟专用网。它利用公共互联网，通过隧道、加密、认证等技术，让分散的主机或网络像在一个专用网络中通信。

#### 4.8.2 NAT 与 NAPT

NAT 的作用是把内部专用地址转换为全球可路由地址。

NAPT 又称端口地址转换，它把多个内部主机的连接映射到同一个公网 IP 的不同端口上。

### 4.9 MPLS

MPLS 是多协议标签交换。思想：分组进入 MPLS 域时打上固定长度标签，之后路由器根据标签而不是完整 IP 前缀快速转发。

### 4.10 软件定义网络 SDN

SDN 的核心思想是：控制层面和数据层面分离，由集中式或逻辑集中式控制器统一计算转发规则，交换机只按流表执行转发动作。

### 4.11 本章计算题常用公式和方法

> [!example] 网络层计算题统一步骤
> 1. 写出前缀长度、主机位数、MTU 和首部长度等已知量。
> 2. CIDR 题先求掩码与地址块边界；转发题对所有匹配项选择最长前缀。
> 3. 分片题先求每片最大数据长度，并确保除最后一片外的数据长度是 8 B 的整数倍。
> 4. 最后检查网络地址、广播地址、片偏移单位和 MF 标志。
#### 4.11.1 CIDR 地址块范围

==对地址 `A/B`：==

```text
网络前缀长度 = B
主机位数 = 32 - B
地址总数 = 2^(32-B)
```

==地址块首地址：把主机位全置 0。==

==地址块末地址：把主机位全置 1。==

#### 4.11.2 子网划分

==若一个地址块要划分为 `m` 个等长子网，需要借用 `k` 位：==

```text
2^k ≥ m
新前缀长度 = 原前缀长度 + k
每个子网地址数 = 2^(32 - 新前缀长度)
```

#### 4.11.3 主机数需求对应前缀

==若一个子网需要容纳 `H` 台主机，通常要满足：==

```text
2^h - 2 ≥ H
前缀长度 = 32 - h
```

==其中减去 2 是因为传统 IPv4 子网中全 0 地址表示网络地址，全 1 地址表示广播地址。==

#### 4.11.4 IP 分片

==设：==

- ==MTU：链路层最大数据字段长度；==
- ==IP 首部长度：通常 20 字节；==
- ==原 IP 数据报数据部分长度：D。==

==则：==

```text
每片最大数据长度 = MTU - IP首部长度
片偏移 = 当前片数据起始位置 / 8
总发送字节数 = 原数据部分长度 + 分片数 × IP首部长度
```

#### 4.11.5 最长前缀匹配

```text
能匹配多个前缀 → 选择前缀最长者
没有具体匹配 → 使用默认路由 0.0.0.0/0
仍没有 → 丢弃并可能返回 ICMP 目的不可达
```

### 5.1 运输层协议概述

#### 5.1.1 进程之间的通信

网络层负责把 IP 数据报从源主机送到目的主机，但到达目的主机后，数据还必须交给正确的应用进程。==运输层正是为应用进程之间提供通信服务。==运输层的通信单位是：

- UDP：用户数据报；
- TCP：报文段。

运输层使用端口号进行复用与分用。一个套接字端点通常表示为“IP 地址 + 端口号”；一条 TCP 连接由源/目的 IP、源/目的端口和运输层协议共同区分，即常说的五元组。

常见端口：分配给常见服务器进程，如 HTTP 80、DNS 53、FTP 21、SSH 22、HTTPS 443

#### 5.1.2 运输层的两个主要协议

| 对比项   | UDP                                     | TCP                               |
| -------- | --------------------------------------- | --------------------------------- |
| 连接方式 | 无连接                                  | 面向连接                          |
| 可靠性   | 不保证可靠交付                          | 可靠交付                          |
| 数据形式 | 面向报文                                | 面向字节流                        |
| 首部长度 | 固定 8 字节                             | 最小 20 字节，可变长              |
| 流量控制 | 无                                      | 有                                |
| 拥塞控制 | 无                                      | 有                                |
| 适合场景 | DNS、DHCP、实时语音、视频、简单请求响应 | Web、文件传输、电子邮件、远程登录 |

UDP 的优点是简单、开销小、时延低；TCP 的优点是可靠、有序、面向连接，但开销较大。

#### 5.1.3 运输层的端口

端口号的作用是实现运输层的==复用和分用==。

- 复用：发送方多个应用进程的数据都可以交给运输层，由运输层统一交给网络层发送。
- 分用：接收方运输层收到数据后，根据目的端口号把数据交给正确的应用进程。

### 5.2 用户数据报协议 UDP

#### 5.2.1 UDP 概述

UDP 是无连接的运输层协议。发送数据前不需要建立连接，发送后也不需要释放连接。支持一对一、一对多、多对一、多对多通信。

#### 5.2.2 UDP 的首部格式

UDP 首部固定为 8 字节，包含 4 个字段，每个字段 2 字节：

| 字段     |   长度 | 含义                                     |
| -------- | -----: | ---------------------------------------- |
| 源端口   | 16 bit | 发送方应用进程端口号；可选，若不用则为 0 |
| 目的端口 | 16 bit | 接收方应用进程端口号；必须使用           |
| 长度     | 16 bit | UDP 用户数据报总长度，即首部 + 数据      |
| 校验和   | 16 bit | 检验 UDP 首部、数据和伪首部              |

UDP 校验和的特点：

- UDP 校验和不仅检验 UDP 首部和数据，还要加入 IP 伪首部。
- 伪首部不真正传输，只参与校验和计算。
- 伪首部包含源 IP 地址、目的 IP 地址、协议字段、UDP 长度等，用来检查是否送错主机或送错协议。
- 若发送方不使用 UDP 校验和，则校验和字段置为全 0。
- 若计算出的校验和本身为全 0，则必须写成全 1，避免和“不使用校验和”混淆。

### 5.3 传输控制协议 TCP 概述

#### 5.3.1 TCP 最主要的特点

TCP 是面向连接的可靠运输层协议。它的主要特点如下：

1. 面向连接的运输层协议：通信前要建立 TCP 连接，通信结束后要释放连接。
2. 每一条 TCP 连接只能有两个端点，因此是点对点的。
3. 提供可靠交付：保证无差错、不丢失、不重复、按序到达。
4. 提供全双工通信：连接双方可以同时发送和接收数据。
5. 面向字节流：TCP 把应用层数据看作连续的字节流，而不是一条条报文。

### 5.6 TCP 可靠传输的实现

#### 5.6.1 以字节为单位的滑动窗口

![[assets/image-20260608161146547.png|600]]

TCP 的滑动窗口以字节为单位，而不是以报文段为单位。

窗口中的字节大致分为四类：

1. 已发送且已确认。
2. 已发送但未确认。
3. 允许发送但尚未发送。
4. 暂不允许发送。

#### 5.6.2 超时重传时间的选择

TCP 必须为已发送但未确认的报文段设置重传计时器。若超时仍未收到确认，则重传。

TCP 根据往返时间 RTT 的测量值动态估算 RTO。

$$
\text{新的 RTT}_S=(1-\alpha)\times(\text{旧的 RTT}_S)+\alpha\times(\text{新的 RTT 样本}) \tag{5-4}
$$

$$
\mathrm{RTO}=\mathrm{RTT}_S+4\times\mathrm{RTT}_D \tag{5-5}
$$

$$
\text{新的 RTT}_D=(1-\beta)\times(\text{旧的 RTT}_D)+\beta\times\left|\mathrm{RTT}_S-\text{新的 RTT 样本}\right|
$$

α委员会推荐为0.125，β0.25

#### 5.6.3 选择确认 SACK

传统 TCP 使用累积确认。若中间某个报文段丢失，后续已收到的数据也可能无法被充分利用。

SACK 允许接收方明确告诉发送方“哪些不连续的数据块已经收到”。这样发送方可以只重传缺失部分，提高效率。

### 5.7 TCP 的流量控制

#### 5.7.1 利用滑动窗口实现流量控制

流量控制的目的：防止发送方发送太快，超过接收方接收能力。==发送方的发送窗口不能超过接收方给出的接收窗口==

若接收窗口变为 0，发送方不能继续发送普通数据，但应启动坚持计时器，定期发送窗口探测报文段，防止窗口更新报文丢失后双方永久等待。

---

### 5.8 TCP 的拥塞控制

#### 5.8.1 拥塞控制的一般原理

拥塞控制的目的：防止过多数据注入网络，导致路由器队列溢出、时延增大、丢包严重。

流量控制和拥塞控制不同：

> [!warning] 两种控制不要混答
> **流量控制保护接收方，拥塞控制保护网络。** TCP 实际可发送窗口受二者共同限制：$\min(\mathrm{rwnd},\mathrm{cwnd})$。

| 项目         | 流量控制             | 拥塞控制         |
| ------------ | -------------------- | ---------------- |
| 关注对象     | 接收方是否来得及接收 | 网络是否发生拥塞 |
| 控制依据     | 接收窗口 rwnd        | 拥塞窗口 cwnd    |
| 主要目的     | 保护接收方           | 保护整个网络     |
| 实际发送窗口 | 与 rwnd 有关         | 与 cwnd 有关     |

==发送窗口 = min(rwnd, cwnd)==

![[assets/image-20260608162137736.png|600]]

#### 5.8.2 TCP 的拥塞控制方法

##### 慢开始

刚开始发送时，cwnd 从较小值开始，按指数规律增长。通常每经过一个 RTT，cwnd 约加倍。

##### 拥塞避免

当 cwnd 达到慢开始门限 ssthresh 后，进入拥塞避免阶段。此时 cwnd 按线性规律增长，通常每经过一个 RTT，cwnd 增加 1 个 MSS。

##### 乘法减小

当判断网络发生拥塞时，门限 ssthresh 通常设置为当前 cwnd 的一半。

![[assets/image-20260608162219477.png|600]]

以上为 **TCP Tahoe** 的典型处理思路，以下为 **TCP Reno**。

##### 快重传

发送方收到三个重复确认时，认为某个报文段很可能丢失，不必等超时，立即重传丢失的报文段。

##### 快恢复

快重传后，不把 cwnd 降到 1，而是把 cwnd 调整到较小值，继续拥塞避免。

![[assets/image-20260608162631698.png|720]]

![[assets/image-20260608162727726.png|720]]

#### 5.8.3 主动队列管理 AQM

AQM 的思想是在路由器队列还没有完全溢出前，主动丢弃或标记部分分组，提前通知发送方减小发送速率。

### 5.9 TCP 的运输连接管理

#### 5.9.1 TCP 的连接建立

TCP 建立连接使用三报文握手。

![[assets/image-20260608162846874.png|720]]

> [!question] 为什么需要三报文握手
> - 双方都要确认自己的发送能力和对方的接收能力正常。
> - 双方需要确认并同步初始序号。
> - 可以避免失效的旧连接请求使服务器误建连接。

#### 5.9.2 TCP 的连接释放

TCP 连接释放通常使用四报文挥手，因为 TCP 是全双工通信，一方关闭发送方向后，另一方仍可能继续发送数据。

![[assets/image-20260608162953510.png|600]]

> [!question] TIME-WAIT 为什么等待 2MSL
> - 若最后一个 ACK 丢失，主动关闭方仍能收到对方重传的 FIN 并再次确认。
> - 等待足够时间让旧连接的迟到报文段从网络中消失，避免污染后续同一四元组连接。

#### 5.9.3 TCP 的有限状态机

| 状态        | 含义                                      |
| ----------- | ----------------------------------------- |
| CLOSED      | 关闭状态                                  |
| LISTEN      | 服务器等待连接请求                        |
| SYN-SENT    | 主动打开后等待 SYN+ACK                    |
| SYN-RCVD    | 收到 SYN 后等待 ACK                       |
| ESTABLISHED | 连接已建立，可传输数据                    |
| FIN-WAIT-1  | 主动关闭方已发送 FIN                      |
| FIN-WAIT-2  | 主动关闭方收到对 FIN 的 ACK，等待对方 FIN |
| CLOSE-WAIT  | 被动关闭方收到 FIN，等待本地应用关闭      |
| LAST-ACK    | 被动关闭方已发送 FIN，等待 ACK            |
| CLOSING     | 双方几乎同时关闭，等待 ACK                |
| TIME-WAIT   | 等待 2MSL 后关闭                          |

### 6.1 域名系统 DNS

DNS 的作用是把用户容易记忆的域名转换为网络层能够使用的 IP 地址。例如 `www.example.com` 需要解析成对应的 IPv4 或 IPv6 地址后，主机才能真正通信。

#### 6.1.1 DNS 的作用

DNS 是一个分布式、层次化的命名系统。它不是由一台服务器集中保存全部域名，而是由根域名服务器、顶级域名服务器、权限域名服务器和本地域名服务器共同完成解析。

DNS 通常使用 UDP，端口号为 53。当报文较大或进行区域传送时，也可以使用 TCP。

#### 6.1.2 域名结构

域名采用层次结构，从右到左范围逐渐缩小。例如：

```text
www.tsinghua.edu.cn
```

其中 `cn` 是国家顶级域名，`edu` 是教育类域名，`tsinghua` 是组织名，`www` 是主机名。

常见顶级域名：

| 类型         | 示例                                   |
| ------------ | -------------------------------------- |
| 国家顶级域名 | `.cn`、`.us`、`.jp`                    |
| 通用顶级域名 | `.com`、`.org`、`.net`、`.edu`、`.gov` |
| 基础结构域名 | `.arpa`                                |

#### 6.1.3 域名服务器

| 类型           | 作用                          |
| -------------- | ----------------------------- |
| 根域名服务器   | 知道顶级域名服务器的位置      |
| 顶级域名服务器 | 管理某类顶级域名              |
| 权限域名服务器 | 保存某个区域的权威解析记录    |
| 本地域名服务器 | 用户主机首先查询的 DNS 服务器 |

DNS 查询有两种方式：

1. **递归查询**：主机把请求交给本地域名服务器，本地域名服务器负责查到底再返回。
2. **迭代查询**：服务器只告诉查询方下一步应当查询谁。

DNS 高速缓存可以减少查询次数、降低网络负载、加快解析速度。

> [!tip] DNS 过程题答题链
> 应用先检查本地缓存，再把查询交给本地域名服务器；本地域名服务器通常代表客户进行后续查询，并通过根、顶级域与权限服务器逐步获得权威结果。递归与迭代描述的是**谁继续向下查询**，不要与缓存是否命中混淆。

### 6.2 文件传送协议 FTP

FTP 用于在互联网上传送文件，采用客户-服务器方式，使用 TCP 实现可靠传输。

#### 6.2.1 FTP 概述

FTP 的主要功能是文件上传、下载、目录查看和文件管理。它能够屏蔽不同操作系统文件格式差异。

#### 6.2.2 FTP 的基本工作原理

FTP 使用两个 TCP 连接：

> [!important] FTP 的核心特征
> 控制连接与数据连接彼此独立：控制连接通常贯穿会话，数据连接按传输任务建立。主动/被动模式的主要差异是**数据连接由哪一方发起以及使用什么端口**。

| 连接     | 作用                   | 常用端口          |
| -------- | ---------------------- | ----------------- |
| 控制连接 | 传送命令和响应         | TCP 21            |
| 数据连接 | 传送文件内容或目录列表 | TCP 20 或临时端口 |

控制连接在整个会话期间保持，数据连接只在传送数据时建立，用完关闭。

FTP 有主动模式和被动模式。现在被动模式更常见，因为它更容易穿过 NAT 和防火墙。

#### 6.2.3 简单文件传送协议 TFTP

TFTP 是简化版文件传送协议，使用 UDP。它实现简单，不提供复杂认证，常用于无盘启动、路由器配置文件传送等场景。

| 对比项 | FTP      | TFTP           |
| ------ | -------- | -------------- |
| 传输层 | TCP      | UDP            |
| 功能   | 复杂     | 简单           |
| 可靠性 | TCP 保证 | 应用层简单确认 |
| 认证   | 支持     | 通常不支持     |
