# 1.3 互联网的组成

从工作方式看，互联网可以分为网络边缘、接入网和网络核心：边缘运行应用，接入网建立入口，核心负责把分组送往目的方向。这个划分关注功能位置，而不是地理位置或所有权。

```mermaid
flowchart LR
    A["源应用进程"] --> H1["源端系统"]
    H1 --> AN1["接入网"]
    AN1 --> C["网络核心：路由器与互连链路"]
    C --> AN2["接入网"]
    AN2 --> H2["目的端系统"]
    H2 --> B["目的应用进程"]
```

> 端系统位于边缘，路由器和互连网络构成核心；接入网连接二者。

## 网络边缘

网络边缘由连接到互联网的端系统（end system）组成，例如个人计算机、手机、传感器和服务器。设备规模和用途可以不同，但它们都运行应用进程，并通过协议栈使用网络服务。

### 网络通信的对象

> [!definition] 严格表述
> “主机 A 与主机 B 通信”是简化说法。严格地说，通信双方是运行在不同端系统上的**应用进程**；主机提供运行环境，[[1.7 计算机网络体系结构|协议栈]]负责数据交付。
> ^process-communication

端系统之间常见的应用组织方式是客户—服务器（Client–Server, C/S）与对等（Peer-to-Peer, P2P）。它们描述的是**进程角色**，并不直接规定底层采用哪种网络。

### 客户—服务器方式

- **客户进程**主动发起请求，因此必须知道如何定位服务器。
- **服务器进程**持续或按需等待请求，并可并发服务多个客户。
- 请求建立后，数据仍可双向传输；“客户”不等于只能接收，“服务器”也不等于只能发送。

![[Pasted image 20260715204031.png]]

> [!example] Web 请求
> 浏览器进程根据服务器地址主动请求网页，服务器进程接收请求并返回响应。这里的客户和服务器首先是软件角色；“客户端设备”“服务器主机”只是根据所运行角色对机器的简称。

### 对等方式

P2P 中，每个对等方都可以请求资源，也可以向其他对等方提供资源。例如，节点 C 从 D 下载数据时，C 扮演客户、D 扮演服务器；C 同时向 F 上传时，又对 F 扮演服务器。

![[Pasted image 20260715204041.png]]

> [!note] P2P 并未消除客户—服务器关系
> P2P 消除的是固定、中心化的角色分配。观察一次具体交互时，仍然存在请求方和服务方。

| 维度 | 客户—服务器 | P2P |
| --- | --- | --- |
| 角色 | 通常相对固定 | 同一节点可同时承担两种角色 |
| 资源位置 | 倾向集中于服务器 | 可分布在多个对等方 |
| 扩展方式 | 扩充服务器能力 | 新节点既带来需求，也可能带来资源 |
| 典型挑战 | 服务器瓶颈与集中故障 | 节点动态加入、离开与管理复杂度 |

## 接入网

接入网把端系统连接到本地 ISP 的第一个路由器。它是边缘进入互联网的“桥梁”，可采用以太网、无线局域网、光纤接入或蜂窝网络等技术。接入网的**功能位置**不同于按覆盖范围定义的 LAN、MAN、WAN，详见[[1.5 计算机网络的类别#接入网是功能分类]]。

## 网络核心

网络核心由路由器和互连链路组成。路由器根据分组首部与转发表逐跳转发数据，为边缘提供连通性。核心为何采用分组交换、不同交换方式有何取舍，见[[1.3.2 互联网的核心部分]]。

## 本节小结

- 网络边缘运行应用进程，接入网连接边缘与核心，网络核心转发分组。
- 网络通信的实际对象是应用进程，而不只是抽象的“主机”。
- C/S 与 P2P 描述进程角色；P2P 节点可以同时请求和提供服务。
