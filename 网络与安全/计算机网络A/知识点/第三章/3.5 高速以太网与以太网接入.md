# 3.5 高速以太网与以太网接入

以太网从共享总线演进为交换式全双工网络，并把速率扩展到多个数量级。它成功的关键不只是“更快”，而是在升级物理层和交换结构时持续保留 MAC 帧与上层接口的兼容性。

## 演进中保持不变的部分

- 基本 Ethernet II / IEEE 802.3 MAC 帧结构；
- 48 bit MAC 地址和 EtherType 上层协议标识；
- 常见 1500 B MTU 与 64 B 标准最短帧；
- 交换机按 MAC 地址转发；
- 上层协议通常无需感知具体物理媒体与线路速率。

## 发生变化的部分

- 物理编码、收发器、双绞线或光纤规格；
- 线路速率和可达距离；
- 从共享半双工转为交换式全双工；
- 并行通道、波分复用和更复杂的信号处理；
- 交换机背板、缓存与端口容量。

```mermaid
flowchart LR
    E10["10 Mbit/s<br/>共享或交换"] --> E100["100 Mbit/s<br/>Fast Ethernet"]
    E100 --> E1G["1 Gbit/s"]
    E1G --> E10G["10 Gbit/s<br/>仅全双工"]
    E10G --> EN["更高速率<br/>保持 MAC 兼容"]
```

## 命名方式

常见名称形如：

```text
速率 + BASE + 物理媒体/距离变体
```

例如：

- `100BASE-TX`：100 Mbit/s、基带、双绞线变体；
- `1000BASE-T`：1 Gbit/s、基带、双绞线；
- `10GBASE-SR`：10 Gbit/s、短距离光纤变体；
- `10GBASE-LR`：10 Gbit/s、较长距离单模光纤变体。

具体距离取决于标准变体、光纤/铜缆类别、连接器和链路预算，不应只按速率名称推断。

## 半双工兼容机制

快速以太网和早期吉比特以太网标准曾保留半双工 CSMA/CD 兼容模式。

### 100 Mbit/s

最短帧仍为 64 B，但速率提高 10 倍，发送时间缩短为原来的十分之一。为了仍在发送完最短帧前检测碰撞，半双工网络直径和时延预算必须相应减小。

### 1 Gbit/s

半双工吉比特以太网曾定义：

- **载波延伸**：短帧之后加入特殊扩展符号，把发送时间延长到碰撞检测所需窗口；
- **分组突发**：连续发送多个短帧，降低每帧都进行载波延伸的开销。

这些机制主要用于历史兼容。现实部署以交换式全双工为主，不使用碰撞检测、载波延伸或分组突发。

## 10 Gbit/s 及以上

10GbE 及更高速以太网只定义全双工工作方式：

- 不使用 CSMA/CD；
- 不受碰撞窗口对网络直径的限制；
- 仍使用兼容的 MAC 帧结构；
- 可通过多通道并行或[[2.4 信道复用技术#波分复用 WDM|波分复用]]聚合容量；
- 可运行在铜缆、多模光纤、单模光纤或背板等媒体上。

> [!note] 速率不是端到端吞吐量
> 端口线速率只是单段链路能力。实际吞吐还受最慢链路、共享上联、协议开销、交换机缓存、拥塞和端系统处理能力限制，见[[1.6 计算机网络的性能#速率、带宽与吞吐量]]。

## 以太网接入

以太网也可用作用户接入网：终端通过以太网链路连接楼宇交换机、ONU 或运营商设备，再进入汇聚和城域网络。

用户到第一个交换端口的链路可能独享，但交换机上联通常由多个用户共享。因此“接入口是 1 Gbit/s”不代表互联网端到端吞吐始终为 1 Gbit/s。

## PPPoE：在以太网上承载 PPP

PPPoE（PPP over Ethernet）把 PPP 会话封装到以太网中，使共享以太网环境可以使用 PPP 的链路协商、用户识别和网络层配置能力。

```mermaid
flowchart LR
    H["用户主机 / CPE"] -->|"Ethernet + PPPoE + PPP"| AC["接入集中器"]
    AC --> ISP["运营商网络"]
```

### 协议关系

```text
以太网帧
└── PPPoE 头部与会话标识
    └── PPP 帧内容
        └── IP 等网络层数据
```

PPPoE 增加额外封装，因此可承载的网络层 MTU 通常要相应考虑，避免把标准以太网 1500 B 载荷机械等同于 PPPoE 内层 IP 可用 MTU。

### 工作阶段

1. **发现阶段**：用户侧寻找接入集中器并建立会话标识；
2. **PPP 会话阶段**：运行 LCP、可选鉴别和 NCP，随后承载网络层数据；
3. **终止**：PPP 或 PPPoE 机制结束会话并释放状态。

PPP 状态与失败处理见[[3.2 点对点协议 PPP#链路状态机]]。

> [!warning] PPPoE 不是所有宽带接入的必需条件
> 它是一种常见接入方案，但以太网、FTTx 或 PON 并不天然要求 PPPoE。运营网络也可以采用 DHCP、静态配置或其他用户管理方式。

## 本节小结

- 高速以太网主要通过更新物理层和交换能力扩展速率，同时保持 MAC 帧兼容。
- 半双工快速/吉比特以太网的碰撞兼容机制具有历史意义；10GbE 及以上只工作在全双工模式。
- 物理端口速率不等于端到端吞吐，共享上联和拥塞仍会形成瓶颈。
- PPPoE 在以太网上承载 PPP 会话，但不是所有以太网或光纤接入的必选方案。
