# 网络与安全局部规则

- 网络协议笔记注明层次、参与方、报文方向、状态转换、端口或标识、可靠性与失败处理。
- 区分协议规范、常见实现、部署惯例和抓包观察；版本相关内容注明 RFC、标准版本或软件版本。
- 性能讨论说明带宽、时延、吞吐、丢包、并发量和测试拓扑，禁止在条件不同的结果间直接比较。
- 安全笔记明确资产、信任边界、攻击者能力、威胁模型、安全目标、前置条件和缓解措施。
- 区分认证、授权、机密性、完整性、可用性和不可否认性；不要用“安全”替代具体保证。
- 攻击或扫描命令只用于明确授权的实验环境，写明目标范围、风险和恢复方法；不得提供针对真实第三方的操作步骤。
- 密码学内容优先引用标准构造，注明算法、模式、密钥或随机数要求；不要自行设计密码方案或把编码、哈希和加密混同。
- 漏洞、产品和协议状态可能变化时记录核实日期并标注受影响版本。
