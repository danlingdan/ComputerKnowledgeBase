# 模板使用教程

## 📋 概述

模板是 Obsidian 中提高笔记效率的重要工具。通过预定义的模板，可以快速创建结构一致、内容完整的笔记。本教程将介绍如何配置和使用模板系统。

## 🔧 Templater 配置

### 1. 设置模板文件夹路径

1. 打开 **Settings → Templater**
2. 在 **Template folder location** 中设置为：
   ```
   Templates
   ```

### 2. 设置快捷键

推荐设置以下快捷键：

| 操作 | 快捷键 |
|------|--------|
| 插入模板 | `Ctrl/Cmd + T` |
| 新建笔记并应用模板 | `Ctrl/Cmd + N` |

---

## 🚀 使用模板

### 方法一：使用快捷键

1. 按 `Ctrl/Cmd + T` 打开模板选择菜单
2. 使用方向键选择所需模板
3. 按 Enter 确认

### 方法二：右键菜单

1. 在文件浏览器中右键点击文件夹
2. 选择 **New note from template**
3. 选择模板并输入文件名

### 方法三：命令面板

1. 按 `Ctrl/Cmd + P` 打开命令面板
2. 输入 `Template` 或模板名称
3. 选择 **Insert template**

# 数组越界错误排查

## 📋 问题信息

| 项目 | 内容 |
|------|------|
| **问题类型** | 运行时错误 |
| **严重程度** | 🔴 紧急 |
| **状态** | ✅ 已解决 |

---

## 🐛 问题描述

> [!error] 问题概述
> 
> **现象**：程序运行时崩溃，报 Segmentation fault
> 
> **复现步骤**：
> 1. 输入数组长度为 10
> 2. 访问 arr[10]
> 3. 程序崩溃

**错误信息**：

```
Segmentation fault (core dumped)
```

---

## 🔍 排查过程

### 第一步：分析可能原因

| 可能原因 | 排查方法 | 结果 |
|----------|----------|------|
| 数组下标越界 | 检查循环条件 | ✅ 确认 |

### 第二步：定位问题

**关键代码段**：

```c
for (int i = 0; i <= n; i++) {
    printf("%d\n", arr[i]);  // i == n 时越界
}
```

### 第三步：解决方案

> [!success] 解决方案
> 
> **方案**：将循环条件改为 `i < n`
> 
> **修改内容**：

```c
for (int i = 0; i < n; i++) {
    printf("%d\n", arr[i]);
}
```

---

## 💡 经验总结

> [!tip] 经验教训
> 
> - 学到的知识：数组下标从 0 开始，最大下标为 n-1
> - 以后如何避免：使用边界检查，编译器开启越界警告
> - 相关知识点：数组

## 💡 使用建议

1. **统一命名规范**：使用清晰的文件名，如 `二叉搜索树.md`
2. **及时更新状态**：定期更新笔记的 `status` 字段

4. **使用标签分类**：合理使用标签（如 `#知识点`、`#数据结构`）
