# Obsidian 文件组织与管理

## 一、Vault 概念

### 什么是 Vault

Vault（库）是 Obsidian 对笔记文件夹的称呼。一个 Vault 就是一个本地文件夹，包含所有 Markdown 文件和附件。

### 创建 Vault

1. 打开 Obsidian
2. 点击 **Open folder as vault**
3. 选择或创建一个文件夹

### Vault 位置建议

- 放在云同步目录下（如 iCloud、OneDrive、Dropbox）
- 便于备份和跨设备访问
- 注意同步冲突问题

## 二、文件结构设计

### 1. 按主题组织

```
Vault/
├── 编程/
│   ├── Python.md
│   ├── JavaScript.md
│   └── 算法/
│       └── 排序算法.md
├── 学习/
│   ├── 英语.md
│   └── 数学.md
└── 生活/
    ├── 日记.md
    └── 待办事项.md
```

### 2. 按项目组织

```
Vault/
├── Project-A/
│   ├── 需求文档.md
│   ├── 设计方案.md
│   └── 进度跟踪.md
├── Project-B/
│   └── ...
```

### 3. PARA 方法

- **Projects**：项目（有明确目标和截止日期）
- **Areas**：领域（持续关注的领域）
- **Resources**：资源（可复用的参考材料）
- **Archives**：归档（已完成或不再需要的内容）

```
Vault/
├── Projects/
│   ├── 网站开发/
│   └── 论文写作/
├── Areas/
│   ├── 健康/
│   └── 财务/
├── Resources/
│   ├── 书籍笔记/
│   └── 教程/
└── Archives/
    └── 2023/
```

### 4. Zettelkasten 方法

每个笔记都是一个独立的知识单元，通过链接而非文件夹组织。

```
Vault/
├── 20240101-笔记标题.md
├── 20240102-另一个笔记.md
└── ...
```

## 三、文件命名规范

### 1. 使用清晰的标题

```markdown
# 好：Python函数定义.md
# 好：机器学习入门指南.md
# 坏：笔记1.md
# 坏：temp.md
```

### 2. 添加日期前缀

```markdown
20240101-每日总结.md
20240115-项目周报.md
```

### 3. 使用连字符分隔

```markdown
Python-函数定义.md
机器学习-监督学习.md
```

### 4. 统一语言

保持文件名语言一致，避免混用中英文。

## 四、文件夹管理

### 1. 不要过深嵌套

```markdown
# 好：最多 3 层
编程/Python/基础语法.md

# 坏：嵌套过深
编程/语言/Python/基础/语法/变量.md
```

### 2. 使用 Tags 补充分类

文件夹用于宏观分类，Tags 用于微观分类。

```markdown
编程/Python.md
#标签：#编程/Python #学习/入门
```

### 3. 定期清理

- 删除不再需要的笔记
- 合并重复内容
- 更新过时信息

## 五、附件管理

### 1. 附件存放位置

#### 方案一：统一附件文件夹

```
Vault/
├── 附件/
│   ├── images/
│   ├── pdf/
│   └── 其他/
└── 笔记.md
```

#### 方案二：笔记同级附件文件夹

```
Vault/
├── 笔记.md
└── 笔记_assets/
    └── image.png
```

#### 方案三：按主题存放附件

```
Vault/
├── 编程/
│   ├── Python.md
│   └── 附件/
│       └── code.png
└── 生活/
    ├── 日记.md
    └── 附件/
        └── photo.jpg
```

### 2. 附件命名

```markdown
# 好：描述性名称
python-code-example.png
machine-learning-workflow.jpg

# 坏：无意义名称
image1.png
IMG_20240101.jpg
```

### 3. 插入附件

```markdown
![图片描述](附件/images/python.png)
[[附件/pdf/report.pdf]]
```

## 六、搜索功能

### 1. 基本搜索

```markdown
搜索关键词：python
搜索标题：path:编程
```

### 2. 高级搜索

```markdown
# 搜索包含特定标签的笔记
tag:#编程

# 搜索包含特定链接的笔记
link:[[Python]]

# 搜索特定类型的文件
file:pdf

# 组合搜索
tag:#编程 file:md
```

### 3. 搜索快捷键

- `Ctrl/Cmd + Shift + F`：全局搜索
- `Ctrl/Cmd + F`：当前笔记搜索

## 七、标签管理

### 1. 创建标签

```markdown
#编程 #学习 #Python
```

### 2. 嵌套标签

```markdown
#编程/Python
#学习/数学/线性代数
#生活/健康/运动
```

### 3. 标签面板

在左侧边栏查看所有标签，点击标签筛选相关笔记。

### 4. 标签最佳实践

- 不要创建过多标签，保持精简
- 使用嵌套标签建立层次结构
- 定期清理和重命名标签

## 八、模板功能

### 1. 创建模板

在 `.obsidian/templates/` 文件夹中创建模板文件。

```markdown
---
标题：
日期：{{date}}
标签：
---

## 内容

## 参考链接
```

### 2. 使用模板

- 点击 **Command Palette** (`Ctrl/Cmd + P`)
- 输入 **Insert Template**
- 选择模板

### 3. 常用模板示例

- **日记模板**：日期、天气、心情、事件
- **会议记录模板**：参会人员、议程、决议、待办
- **书籍笔记模板**：书名、作者、摘要、心得

## 九、快捷键

### 文件操作

- `Ctrl/Cmd + N`：新建笔记
- `Ctrl/Cmd + S`：保存笔记
- `Ctrl/Cmd + Shift + N`：新建文件夹
- `Ctrl/Cmd + Delete`：删除笔记

### 导航

- `Ctrl/Cmd + O`：快速切换笔记
- `Ctrl/Cmd + G`：打开图谱视图
- `Ctrl/Cmd + Shift + B`：切换侧边栏

### 编辑

- `Ctrl/Cmd + B`：加粗
- `Ctrl/Cmd + I`：斜体
- `Ctrl/Cmd + K`：插入链接

## 十二、最佳实践总结

### 1. 保持简单

不要过度设计文件夹结构，从简单开始，根据需要调整。

### 2. 多用链接

不要依赖文件夹组织一切，通过双链建立知识网络。

### 3. 定期整理

每周或每月花时间整理笔记，删除冗余，更新链接。

### 4. 使用模板

为重复性内容创建模板，提高效率。

### 5. 备份重要内容

定期备份，防止数据丢失。

## 常见问题

**Q：文件夹和标签应该怎么配合使用？**

A：文件夹用于宏观分类（如按主题、项目），标签用于微观分类（如按技能、难度、状态）。

**Q：如何处理跨文件夹的笔记？**

A：使用双链和标签来建立跨文件夹的关联，不要为了分类而复制笔记。

**Q：笔记太多找不到怎么办？**

A：使用搜索功能、标签筛选、图谱视图等多种方式定位笔记。
