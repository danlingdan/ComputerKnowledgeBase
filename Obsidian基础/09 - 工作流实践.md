# Obsidian 工作流实践

## 一、工作流概述

工作流是指在 Obsidian 中完成特定任务的一系列步骤。建立适合自己的工作流可以提高效率和生产力。

## 二、笔记写作工作流

### 1. 快速笔记

```
打开命令面板 → 创建新笔记 → 使用模板 → 快速记录
```

**适用场景**：临时记录想法、灵感

**快捷键**：`Ctrl/Cmd + N`

### 2. 深度写作

```
创建新笔记 → 设置标题 → 编写大纲 → 填充内容 → 添加链接 → 优化格式
```

**适用场景**：撰写文章、报告、论文

**技巧**：
- 使用大纲视图规划结构
- 实时预览查看效果
- 添加相关笔记链接

### 3. 每日笔记

```
打开每日笔记 → 填写日期 → 记录计划 → 记录完成事项 → 总结反思
```

**适用场景**：日常记录、时间管理

**推荐插件**：
- Calendar：日历视图
- Templater：模板系统

## 三、知识整理工作流

### 1. 收集阶段

```
阅读文章/书籍 → 摘录要点 → 创建笔记 → 添加标签 → 建立链接
```

**适用场景**：学习新知识、阅读书籍

**技巧**：
- 使用双链建立关联
- 添加标签分类
- 记录来源和引用

### 2. 整理阶段

```
回顾笔记 → 合并重复内容 → 更新链接 → 删除冗余 → 创建 MOC
```

**适用场景**：定期整理知识库

**技巧**：
- 使用图谱视图发现关联
- 查看反向链接了解引用
- 创建 MOC 汇总主题

### 3. 输出阶段

```
选择主题 → 创建大纲 → 整合笔记 → 撰写文章 → 发布分享
```

**适用场景**：输出文章、分享知识

**技巧**：
- 使用搜索功能查找相关笔记
- 使用 Dataview 汇总信息
- 导出为 PDF/HTML

## 四、项目管理工作流

### 1. 项目初始化

```
创建项目文件夹 → 创建项目笔记 → 定义目标 → 规划任务 → 设置截止日期
```

**适用场景**：开始新项目

**推荐插件**：
- Kanban：看板视图
- Dataview：数据查询

### 2. 任务管理

```
创建任务 → 分配优先级 → 设置截止日期 → 标记完成 → 回顾进度
```

**适用场景**：管理项目任务

**任务列表格式**：

```markdown
## 待办事项
- [ ] 完成需求文档（截止：2024-01-20）
- [ ] 设计系统架构（截止：2024-01-25）

## 进行中
- [ ] 编写代码（截止：2024-02-01）

## 已完成
- [x] 确定项目范围（完成：2024-01-15）
```

### 3. 项目总结

```
回顾项目 → 记录经验 → 整理文档 → 归档项目 → 创建总结笔记
```

**适用场景**：项目完成后

**总结笔记模板**：

```markdown
---
date: 2024-02-15
tags:
  - 项目总结
---

# 项目总结 - 项目名称

## 项目概况
- **项目周期**：2024-01-01 ~ 2024-02-15
- **团队成员**：张三、李四、王五
- **项目目标**：完成网站开发

## 完成情况
- ✅ 需求文档：已完成
- ✅ 系统设计：已完成
- ✅ 代码开发：已完成
- ✅ 测试上线：已完成

## 经验教训
- 需求变更管理不足
- 测试时间不够充分
- 沟通效率有待提高

## 改进计划
- 建立需求变更流程
- 增加测试时间
- 定期同步进度
```

## 五、学习工作流

### 1. 学习计划

```
确定学习目标 → 创建学习笔记 → 规划学习路径 → 设置里程碑 → 记录进度
```

**适用场景**：学习新技能、新语言

**学习笔记模板**：

```markdown
---
date: 2024-01-15
tags:
  - 学习计划
---

# 学习计划 - Python

## 学习目标
- 掌握 Python 基础语法
- 学习数据结构和算法
- 完成一个实战项目

## 学习路径
1. 基础语法（1-2周）
2. 数据结构（2-3周）
3. 算法（3-4周）
4. 实战项目（4-6周）

## 进度记录
- [x] 基础语法：已完成（2024-01-20）
- [ ] 数据结构：进行中
- [ ] 算法：待开始
- [ ] 实战项目：待开始
```

### 2. 知识巩固

```
复习笔记 → 做练习题 → 总结错题 → 更新笔记 → 创建思维导图
```

**适用场景**：巩固所学知识

**推荐插件**：
- Excalidraw：绘制思维导图
- Flashcards：制作卡片复习

### 3. 知识应用

```
寻找应用场景 → 动手实践 → 记录问题 → 解决问题 → 总结经验
```

**适用场景**：将知识应用到实际项目

## 常见问题

**Q：如何建立适合自己的工作流？**

A：根据自己的需求和习惯，从简单开始，逐步优化，定期回顾和调整。

**Q：工作流太复杂怎么办？**

A：简化工作流，去除不必要的步骤，使用插件自动化重复任务。

**Q：如何分享工作流？**

A：可以创建工作流笔记，记录步骤和技巧，分享到社区。
