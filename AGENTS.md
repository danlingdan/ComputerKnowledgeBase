# 计算机专业知识库总规则

## 目标与作用域

- 将本库维护为准确、可检索、可关联、可复习且可持续扩展的本科计算机知识体系。
- 默认采用主动执行策略：只要用户目标和处理对象可以合理确定，就直接检查、判断、编辑并验证，不因一般性不确定、改动数量或可逆的结构调整停下来等待确认。
- 以完成用户目标为优先。允许在明确任务的自然边界内主动处理依赖项、反向链接、相关 MOC、Frontmatter 和明显冲突；不要把“最小改动”当作遗漏必要工作的理由。
- 本文件适用于全库；进入含有下级 `AGENTS.md` 的目录时，同时遵守其局部规则，冲突时以更具体的规则为准。
- 目录表示主要归属，Frontmatter 表示结构化属性，标签表示跨目录主题，MOC 表示人工组织的知识路径；不要让四者重复承担同一分类职责。
- 课程目录保存学习语境，但不要把课程划分直接当作最终知识分类。跨课程稳定概念可独立成笔记，并由课程笔记链接过去。

## 文件与命名

- 让一篇知识笔记围绕一个相对稳定的主题；课程原始记录、综合复习资料、术语表和 MOC 可以保持综合性。
- 使用准确、可辨识的中文概念名；必要时把英文名或缩写放入 `aliases`，不要在文件名中堆叠同义词。
- 课程章节可沿用现有编号；同一课程内保持既有编号、空格和连字符风格。不要仅为统一格式批量重命名旧文件。
- MOC 使用 `MOC - 名称.md`；保留已有合理命名，不制造仅有标题的占位笔记。
- 创建文件前搜索同名、别名和近义主题，优先补充已有笔记或建立明确链接。

## Frontmatter

- 新建知识笔记至少使用以下字段：

```yaml
---
domain: 计算机系统
type: 概念
status: 整理中
created: YYYY-MM-DD
---
```

- `domain` 使用稳定领域；`type` 从“概念、课程笔记、例题、习题、实验、项目、MOC、参考资料、术语集”中选择；`status` 优先使用“草稿、整理中、待核实、已完成、持续更新”。
- 按需使用 `course`、`aliases`、`tags`、`updated`。发生实质内容变化时更新 `updated`，纯格式调整不必更新。
- 仅在适用时使用 `source`、`version`、`prerequisites` 等字段。不要为追求完整而添加空字段。
- 使用 YAML 列表保存多个值。标签不加 `#`，保持少量、稳定、可复用。
- 允许旧笔记保留 `创建时间`、`更新时间`、`date`、`category` 等既有字段；不要批量迁移。实际编辑旧笔记时才提出兼容性调整。

## 正文与技术表达

- 在开头用一至数句说明该知识解决什么问题、为什么需要它。
- 按主题需要区分定义、原理、推导、示例、应用、限制和易错点；不要强制所有笔记套用完全相同的标题结构。
- 首次出现的重要中英文术语时给出英文或缩写；后文保持术语一致。
- 数学公式使用 LaTeX：行内用 `$...$`，独立公式用 `$$...$$`。说明符号、单位、定义域、假设和公式适用条件。
- 代码块必须标明语言。代码示例说明输入、输出和关键前提；不要把未经运行或核实的示例描述为可用代码。
- 命令注明操作系统、Shell、工作目录、权限要求和关键版本；不要混用 PowerShell、CMD、Bash 或不同平台命令而不说明。
- 软件、语言、标准、协议和工具知识记录适用版本或核实日期；可能随时间变化的结论先核实来源。
- 表格只用于真正需要逐项比较的内容；复杂流程或结构可用 Mermaid，简单关系使用正文或列表。
- 图片必须有解释性说明和来源。使用相对库路径引用附件；任务需要时可同步移动、重命名或替换附件，并立即修复库内引用。
- 无法确认的内容明确标记“待核实”，说明疑点；禁止猜测、补写不存在的来源或伪造实验结果。

## 链接、标签与 MOC

- 只添加能帮助理解、导航或复习的 `[[双向链接]]`。优先连接先修知识、上下位概念、理论与实现、算法与数据结构、软件与硬件、原理与工程应用、相似与易混概念、课程依赖。
- 链接到章节时使用 `[[笔记名#标题]]`；显示文字与文件名不同时使用别名。链接目标有歧义时使用足以消除歧义的路径。
- 不要为了增加链接数量添加弱相关链接，也不要自动创建大量尚无内容的链接目标。
- 标签用于跨目录检索，不重复文件路径和全部课程层级。优先沿用已有标签，新增近义标签前先搜索。
- 为拥有多个稳定主题、明确学习顺序或跨课程入口的领域维护 MOC。MOC 应包含范围、入口、先修关系、学习路径和关键分支，而不是机械罗列全部文件。

## 来源、引用与原始记录

- 摘录、转述、公式、图表和外部代码应记录可定位来源；优先写作者、标题、版本或版次、章节或页码、URL 和访问日期中的适用项。
- 清楚区分原文、转述、个人理解和推导。短引文使用引用格式，不把大段原文伪装成原创总结。
- 导入外部材料时记录原始路径或来源。先保留原始记录，再提炼稳定知识；不得用整理稿覆盖唯一原件。
- 来源冲突时并列记录差异并标记待核实，不擅自选择看似合理的结论。

## 重复与分类冲突

- 发现重复笔记时先比较范围、来源、粒度和链接。可明确判断时指定主笔记并建议其他笔记链接过去；存在语义差异时保持分立。
- 对高度重复且主从关系明确的笔记，可主动合并或重定向并修复链接；存在实质语义差异时保持分立。永久删除仍需用户明确授权，未获授权时将冗余内容移入可恢复的归档位置或仅报告。
- 一篇笔记只选择一个主要目录；用 `course`、标签、链接和 MOC 表达其他关系，不复制同一正文到多个目录。
- 对职责重叠且归属可以可靠判断的目录，记录归属理由后可直接重构并修复链接；只有多种方案会显著改变用户的组织方式且无法从现有结构判断偏好时才询问。

## 原始 Markdown 到 Obsidian 知识库的彻底重构专用规则

### 触发与优先级

- 当用户要求把原始 Markdown、课程材料、长文、摘录、讲义或外部导出内容整理、导入、知识库化或 Obsidian 化时，自动启用本节；本节是该类任务的专用规则，与一般的“保留原格式”“最小改动”发生冲突时，以本节为准。
- 默认目标不是润色原文或统一排版，而是把原始材料彻底重构为可独立理解、可链接、可查询、可组合、可长期维护的 Obsidian 知识节点。允许大幅改写、重排、拆分、合并、去重、补全上下文和重建标题体系。
- 不要求保留原文段落顺序、标题层级、文件边界或表达方式。只保留有知识价值、来源价值或推导价值的内容；目录、套话、重复总结、教学流程提示和低信息密度内容可直接压缩或移除。
- 在不永久删除唯一原件的前提下直接完成重构，不把方案说明、文件数量、跨目录调整或结构变化作为等待确认的理由。

### 内容优先与知识建模

- 先识别材料中的稳定概念、命题、机制、算法、数据结构、公式、术语、例题、实验、工程实践、限制、争议和来源，再决定笔记边界；不得按原文页码或段落机械切割。
- 每篇重构后的笔记必须有清晰的知识对象和自包含语境：开头直接说明“它是什么、解决什么问题、为什么成立或如何工作”，随后按内容需要组织定义、原理、推导、示例、实现、复杂度、适用条件、限制和易错点。
- 激进消除重复：同一知识只保留一个权威主叙述，其他位置改为带语义的内部链接或嵌入；相互冲突的内容并列呈现证据、适用条件与待核实项，不做无来源猜测。
- 主动补齐原文省略但理解所必需的前提、符号定义、中英文术语、上下位关系和关键过渡。可使用可靠来源核实并补充内容，新增信息必须可区分于原始材料并记录来源。
- 公式统一为规范 LaTeX，解释全部关键符号、定义域、单位、假设和结论边界；代码必须标注语言，修复明显错误并尽可能运行验证，说明输入、输出、环境、复杂度和关键前提。
- 用例子服务概念，不堆砌相似例题。保留能够揭示边界、反例、典型过程或工程取舍的示例；低价值重复例题合并为模式总结。
- 以内容本身为中心。默认不生成周计划、日程、打卡表、学习阶段、复习周期、背诵任务、泛化的“学习建议”或大段自测题。MOC 只负责知识导航与结构呈现；仅在用户明确要求时添加学习计划。

### Obsidian 原生能力与插件利用

- 开始重构前读取 `.obsidian/community-plugins.json` 与 `.obsidian/core-plugins.json`，确认当前实际启用的能力和版本配置。必须逐项评估与当前材料相关的 Obsidian 核心功能和插件，并充分使用能显著增强表达、检索、复用或维护性的功能；不得为了“用了插件”而添加无信息价值的装饰。
- 必须系统使用 Obsidian Properties：为新笔记建立准确 Frontmatter，使用 `aliases` 承载英文名、缩写和常见异名，使用 `domain`、`type`、`status`、`source`、`version`、`prerequisites`、`tags` 等适用字段形成可查询属性；多值字段使用 YAML 列表。
- 必须系统建立内部链接：连接先修、组成、上下位、相似与易混、理论与实现、结论与证明、概念与例题、技术与应用；链接到精确标题或块时使用标题链接、块引用与显示别名。需要复用同一内容时优先使用 `![[笔记#标题]]` 或块嵌入，而不是复制正文。
- 使用 Callout / Admonition 表达具有明确语义的定义、定理、证明、警告、反例、示例、实现提示、来源冲突和待核实内容。Callout 必须承载信息结构，禁止把普通段落全部包装成彩色框。
- 对可由属性自动汇总的内容使用 Dataview 或 Bases 构建动态索引、分类视图和缺失字段检查；不得手工维护容易过期的长列表。查询必须限定范围、字段稳定且在当前库中可运行。
- 对多分支关系、系统组成、调用链、状态变化和算法流程，优先在 Markdown 内使用 Mermaid；当空间布局、自由连线或视觉推演明显更合适时，使用 Canvas 或 Excalidraw，并从相关笔记嵌入或链接。图必须有文字解释，不能替代关键正文。
- 数学内容充分利用 LaTeX Suite 的书写约定；需要展示交换图、几何构造、函数图形或复杂数学结构且 TikZ 能明显提升精度时，使用 TikZJax。简单公式不得为了插件化而转成图片或复杂图形。
- 重复性结构可使用 Templates / Templater 固化，但模板只抽取稳定字段和骨架，不把所有主题强塞进同一标题集合。长文拆分和段落提取可利用 Note Composer，并在拆分后修复嵌入、链接与上下文。
- 使用 Tag Wrangler 既有标签体系统一近义标签；使用反向链接、出链、图谱和 Smart Connections 检查遗漏的强关联与潜在重复。语义相似结果只能作为发现线索，必须阅读内容后再决定链接或合并。
- Obsidian Git、File Recovery 与同步类插件用于重构前后的可恢复性和版本追踪；不得把同步状态当作内容正确性的证据。除非任务确实需要，不修改插件配置或引入新插件。
- 完成时实际检查笔记在 Obsidian 中的可解析性：Frontmatter、Callout、Dataview、Bases、嵌入、标题链接、块引用、Mermaid、数学公式、Canvas / Excalidraw / TikZ 引用均不得只写语法而不验证目标、路径和渲染前提。

### 文件拆分、归档与交付标准

- 原始文件过长、混合多个稳定主题或同时承担资料、概念和练习职责时，主动拆成多篇知识笔记，并创建或更新一个内容型 MOC 作为入口。MOC 展示概念结构、依赖关系与关键分支，不写学习日程。
- 拆分后的文件名采用准确概念名；课程编号只保留在确有课程导航价值的位置。主动搜索同名、别名和近义笔记，将内容并入最合适的现有主笔记，避免制造平行知识孤岛。
- 唯一原始材料不得永久丢失。若直接改写原文件，应通过 Git 历史或明确的原始资料归档保证可恢复，并在重构稿中保留可定位的 `source`；不得把完整原文重复塞入每篇知识笔记。
- 重构完成的最低标准是：内容结构已重建、重复已处理、关键上下文已补齐、属性可查询、强关联已链接、必要视图或图示已建立、来源可追溯、主要语法和链接已验证。仅添加 Frontmatter、Callout 和若干链接不算完成重构。
- 交付时按“新建、重写、拆分、合并、移动、归档、动态视图或图示、待核实”汇报实际结果，并说明启用了哪些 Obsidian 能力及其具体作用；不要输出冗长学习计划或把下一步整理工作推回给用户。

## 安全边界

- 采用“可逆优先、直接执行”的安全策略。移动、重命名、合并、大幅改写和批量修改不再因规模本身等待确认；执行前检查影响范围，保留映射或备份，执行后验证链接、引用和内容完整性。
- 永久删除笔记或原始记录、覆盖唯一原件、清空目录等不可逆操作，仍须用户明确授权。未获授权时使用可恢复方案完成其余工作。
- `.obsidian`、`.git`、`.smart-env` 和 `.trash` 默认不修改；任务明确涉及这些目录时可以处理。模板和附件可在任务需要时同步修改，但必须检查其引用方并避免丢失原件。
- 不创建无实际内容的占位笔记。可以把处理范围扩展到完成当前目标所必需的全库关联项，但不得自行启动与目标无关的全库美化或重写。
- 批量操作不设 10 个文件上限。超过 10 个文件、跨目录移动、字段迁移或批量重命名时，直接生成文件清单、变换规则和回退方式后执行，并在结果中汇报；无需仅因数量等待确认。
- 保留用户已有且合理的内容、命名和格式；同时主动修复会妨碍当前目标的明显错误、断链、重复和结构冲突。对目标外但低风险且高度确定的问题，可一并处理并在结果中说明。

## Codex 工作流

1. 读取当前路径生效的全部 `AGENTS.md`，确认任务范围和安全边界。
2. 主动检查目标目录、相关 MOC、同名或近义笔记、链接、Frontmatter 和必要的引用方；为可靠完成任务可以读取全部相关文本，不以抽样替代关键验证。
3. 根据现有结构和上下文作出合理判断。目标明确时立即执行；只有不可逆操作缺少授权，或关键歧义会导致实质不同的结果且无法从库内证据消除时才询问用户。
4. 以完整解决问题为准编辑；保留原意、原始记录和有效链接，同时同步处理必要的关联修改。发现会直接影响任务质量的范围外问题时主动修复，其他问题简要报告。
5. 检查 YAML、标题层级、LaTeX、代码语言、链接目标、附件引用、来源及“待核实”标记。
6. 汇报新增、修改、移动、合并和未处理的文件，概述关键判断、验证结果、回退方式与遗留疑点；对已执行的批量操作给出范围和结果，不把汇报当作执行前置条件。
