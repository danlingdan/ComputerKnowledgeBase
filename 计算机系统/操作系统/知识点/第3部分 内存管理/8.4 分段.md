# 8.4 分段

本节聚焦于**分段**，是[[第八章 内存管理策略]]中的独立知识节点。

## 8.4.1 基本方法

1. **程序员的“内存视图”与物理内存的差异**
    
    - **物理内存**在硬件上是一个线性的字节数组。
        
    - **程序员视角**：内存是由一组逻辑上独立、长度不一、没有特定顺序要求的**段**组成的。程序员通过名称访问不同的模块（如“主程序”、“堆栈”、“数学库”、“对象数组”等），并不关心它们在物理内存中的具体位置。
        
2. **分段（Segmentation）管理的核心概念**
    
    - **定义**：分段是一种内存管理方案，其核心任务是支持并实现上述“程序员视角”的逻辑内存结构。
        
    - **组成**：一个进程的逻辑地址空间由多个具有**名称**和**长度**的段组成。
        
    - **定位方式**：程序内部通过**段名称**加上**段内偏移**来指定具体的逻辑地址。
        
3. **逻辑地址的结构（硬件视角）**
    
    - 为了便于硬件实现和快速寻址，系统为每个段分配唯一的**段号**。
        
    - 因此，内存管理单元（MMU）识别到的逻辑地址实际上是一个**有序对**：
        
        $$
        <\text{段号}, \text{偏移}>
        $$
        
          
        
    - 这个 `偏移` 表示该指令或数据在对应段内的相对位置。
        
4. **编译与加载过程**
    
    - **编译器职责**：在编译源代码时，编译器会根据程序结构自动划分并构建出不同的逻辑段。
        
    - **典型分段（以 C 语言为例）**：
        
        - **代码段**（存放程序指令）
            
        - **全局变量段**（存放全局数据）
            
        - **堆**（用于动态内存分配）
            
        - **栈**（每个线程独立的栈帧）
            
        - **标准 C 库**（通常作为独立段处理）
            
    - **加载器职责**：程序加载时，加载器将上述所有段装入物理内存，并为它们分配具体的段号，为后续的地址映射做准备。
        

## 8.4.2 分段硬件

> [!important] 从二维地址到一维物理内存的转换
> 
> **用户视角**：应用程序使用"二维"逻辑地址（段号 s，段内偏移 d）来引用对象。
> 
> **物理现实**：硬件物理内存是连续的一维字节序列。
> 
> **核心任务**：需要通过内存管理单元（MMU）和段表，将这种二维逻辑视图映射到一维物理地址空间。
> 
> ```mermaid
> graph TD
>     CPU[CPU] -->|VA &lt;s=2, d=53&gt;| MMU[MMU]
>     ST["Seg Table<br/>Seg0: Base=1000, Lim=500<br/>Seg1: Base=2000, Lim=800<br/>Seg2: Base=4300, Lim=400"] --> MMU
>     MMU -->|PA=4300+53=4353| MEM[(Memory)]
>     
>     style CPU fill:#FF6B6B,color:white
>     style ST fill:#4CAF50,color:white
>     style MEM fill:#9E9E9E,color:white
> ```
        
2. **段表的组成与作用**
    
    - 段表（Segment Table）是分段内存管理的核心数据结构。
        
    - 段表的每一个条目都包含了两个关键参数：
        
        - **段基地址（Segment Base）**：该段在物理内存中的起始地址。
            
        - **段界限（Segment Limit）**：该段的大小（长度），用于防止访问越界。
            
    - 从结构上看，段表实际上就是一组**基址寄存器与界限寄存器值对的数组**。
        
3. **地址转换与内存保护机制** 转换流程如下：
    
    1. **索引查询**：CPU 生成逻辑地址 <段号 s , 偏移 d> 。系统用段号 s 作为索引去查找段表。
        
    2. **界限检查（关键的保护步骤）**：系统检查偏移量 d 是否满足条件 $0 \le d < \text{界限}$。
        
    3. **异常处理**：若偏移量 d **超出**界限（即试图访问段外地址），硬件会触发**陷入（Trap）**，交由操作系统处理非法内存访问。
        
    4. **物理地址计算**：若偏移合法，则物理地址为 **段基地址 + 偏移量 d**。
        
4. **实例验证** 根据教材中的假设示例：
    
    - 段 2（基址 4300，长度 400）：访问 `<2, 53>` \rightarrow 物理地址 = 4300 + 53 = 4353 （合法）。
        
    - 段 3（基址 3200）：访问 `<3, 852>` \rightarrow 物理地址 = 3200 + 852 = 4052 （合法）。
        
    - 段 0（长度 1000）：访问 `<0, 1222>` \rightarrow 访问越界，**陷入操作系统**（非法）。
