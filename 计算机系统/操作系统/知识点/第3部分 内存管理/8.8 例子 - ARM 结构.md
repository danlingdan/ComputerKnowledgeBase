# 8.8 例子：ARM 结构

本节聚焦于**例子：ARM 结构**，是[[第八章 内存管理策略]]中的独立知识节点。

1. **商业对比**：Intel 采取“设计+制造”的垂直整合模式，而 ARM 采取“仅设计、对外授权”的商业模式，其处理器广泛应用于移动设备（如 iPhone 和 Android 手机）。
    
2. **页面与段支持**：32 位 ARM 内存管理支持两种粒度的块：**4KB/16KB 的页面**，以及 **1MB/16MB 的段（Section）**。
    
3. **分页层级**：系统根据引用对象采用不同的分页方式：
    
    - 对于 **1MB/16MB 的段**，采用 **一级分页**。
        
    - 对于 **4KB/16KB 的页**，采用 **二级分页**。
        
4. **两级 TLB 结构及查找流程**：ARM 架构利用两级 TLB 加速地址转换，具体流程为：
    
    - **第一级**：查询外部的两个 **微 TLB（Micro TLB）**，分别用于数据和指令缓存（支持 ASID 机制）。
        
    - **第二级**：若微 TLB 未命中，则查询内部的 **主 TLB（Main TLB）**。
        
    - **最终步骤**：若两级 TLB 均未命中，硬件最终执行页表遍历（Walk）来查找映射关系。
