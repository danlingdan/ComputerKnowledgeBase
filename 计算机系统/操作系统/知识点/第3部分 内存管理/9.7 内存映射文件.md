# 9.7 内存映射文件

本节聚焦于**内存映射文件**，是[[第九章 虚拟内存管理]]中的独立知识节点。

> [!abstract] 内存映射文件的核心价值
> 
> 通过虚拟内存技术将磁盘文件直接映射到进程地址空间，将传统的文件 I/O 转化为内存访问，避免系统调用开销，显著提升文件操作性能。

> [!info] 传统 I/O 的开销
> 
> 使用标准的文件系统调用（如 `open()`、`read()`、`write()`）读取磁盘文件时，每一次操作都涉及**用户态与内核态的切换**（系统调用）以及**实际的磁盘 I/O**，导致频繁的系统开销。

> [!important] 内存映射文件的核心机制
> 
> - 引入虚拟内存技术来优化文件访问。
> - 通过将磁盘文件的一部分**直接映射到进程的虚拟地址空间**中，使得应用程序无需调用传统的文件读写函数，而是直接对虚拟内存地址进行读写。
> - 映射完成后，应用程序对文件数据的访问变成了**常规的内存访问**。
> - 文件的读写由操作系统的**虚拟内存管理单元（MMU）和请求调页机制**在底层自动完成（按需加载页面）。

> [!tip] 性能提升的本质
> 
> 避免了显式的系统调用和磁盘寻道开销，从而能够带来**显著的性能提高**。
        

## 9.7.1 基本机制

> [!important] 基础运作机制
> 
> - 内存映射文件本质上是通过**虚拟内存技术**将磁盘上的文件块直接映射到进程的虚拟地址空间。
> - 初始访问时，通过**请求调页**（Demand Paging）机制按需加载，产生缺页错误后从文件系统读取相应物理页面；一旦加载，后续读写就像操作常规内存一样。
> - **避免了系统调用开销**：不再需要通过 `read()` 和 `write()` 等系统调用进行显式的数据搬运，简化了 I/O 操作。

> [!note] 数据写回策略
> 
> - 内存映射文件的写入不一定同步更新到磁盘（除非显式指定）。
> - 普遍采用**定期检查修改状态**或**在文件关闭时**集中将脏页（修改过的页面）写回磁盘并从虚拟内存中清除的策略，以此平衡性能与数据持久性。

> [!info] 操作系统的实现差异（以 Solaris 为例）
> 
> - 某些操作系统只在应用显式调用 `mmap()` 时才提供内存映射支持。
> - 但 Solaris 系统将**所有文件 I/O**（无论是否经由标准的 `open/read/write` 调用）都通过**内存映射**实现：对于标准调用，它将文件映射到**内核地址空间**进行操作；以此统一并优化所有文件访问路径。
        
4. **进程间共享与通信 (IPC)**

## 9.7.2 共享内存 Windows API

Windows 可用命名文件映射对象让多个进程映射同一组物理页。创建方通过 `CreateFileMappingW()` 建立对象并用 `MapViewOfFile()` 获得本进程视图；使用方通过 `OpenFileMappingW()` 打开同名对象。映射只解决数据共享，不解决并发同步，真实程序仍需事件、互斥体或其他协议。

## 9.7.3 内存映射I/O

> [!important] 核心机制与原理
> 
> 内存映射 I/O 技术将一段专门的内存地址空间与 I/O 设备的内部寄存器直接对应。在这种架构下，CPU 对这段特定内存地址进行**读取或写入操作**，在硬件层面会被自动转化为对该设备寄存器的数据传输。

> [!example] 典型应用场景
> 
> - **快速外设（如视频控制器）**：此技术非常适合需要高速响应的设备。例如在 IBM PC 中，屏幕上的每一个像素或字符位置都直接映射到一个特定的内存地址，程序员向这些内存位置写入数据，相当于直接在屏幕上渲染显示文本或图形。
> - **其他外设（串口、并口）**：也广泛应用于调制解调器、打印机等设备的数据通信。

> [!note] 通信与操作流程示例（串口发送数据）
> 
> 1. CPU 将数据字节写入映射的**数据寄存器**。
> 2. CPU 将**控制寄存器**的特定状态位置 1，表示"数据已准备好发送"。
> 3. 硬件设备读取数据并清零控制寄存器的指示位，表示"已准备好接收下一个字节"。
> 4. CPU 检测到就绪标志，循环执行以上操作。

> [!tip] CPU 与设备交互的两种控制模式
> 
> | 模式 | 机制 | 特点 |
> |------|------|------|
> | **PIO（程序 I/O）** | CPU 轮询设备控制位 | 效率低，占用大量 CPU 周期 |
> | **中断驱动** | 设备就绪时发送硬件中断 | 高效，CPU 可执行其他任务 |
