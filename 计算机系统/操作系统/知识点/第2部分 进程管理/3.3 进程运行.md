# 3.3 进程运行

本节聚焦于**进程运行**，是[[第三章 进程]]中的独立知识节点。

> [!summary] 进程的生命周期
> 进程从创建到终止的完整生命周期，包括进程创建、进程终止以及特殊的僵尸进程和孤儿进程状态。

## 3.3.1 进程创建

> [!summary] 父子关系与进程树
> 进程由其他进程创建，创建者称为**父进程**，被创建的称为**子进程**。这种父子关系构成了树状的**进程树**。

**基础概念**

- **进程标识符（PID）**：每个进程拥有一个唯一的整型 PID，用于系统识别。以 Linux 为例，`init` 进程（PID=1）是所有用户进程的祖先。使用 `ps` 命令可查看系统的进程树结构。
- **资源分配**：子进程需要的资源（CPU、内存、文件等）可以从操作系统获取，或者从父进程处继承资源子集。合理限制资源可以防止系统超载。

**UNIX/Linux 的进程创建机制（经典模型）**

> [!mechanism] fork—exec 两阶段模型
> 父进程调用 `fork()` 后，父子进程从同一返回点继续执行：父进程获得子进程 PID，子进程获得 0。子进程通常关闭不需要继承的文件描述符，再调用 `exec()` 以新程序映像替换自身；父进程可继续并发执行，或通过 `wait()` / `waitpid()` 回收退出状态。`fork()` 或 `exec()` 失败时必须检查返回值并结束错误分支。

**Windows 的进程创建机制（一步到位）**

> [!mechanism] CreateProcess 单阶段模型
> Windows 的 `CreateProcess()` 在一次调用中创建进程对象、初始线程和新的地址空间，并返回两者的句柄与标识符。调用者需要明确命令行、环境、工作目录、句柄继承和启动信息；成功后应关闭不再使用的进程与线程句柄，失败时依据返回值和错误码清理资源。

## 3.3.2 进程终止

> [!summary] 两种终止方式
> 进程可以主动终止（`exit()`）或被动终止（父进程强制终止）。

**进程终止的方式**

| 终止方式 | 触发方式 | 返回值 |
| --- | --- | --- |
| **主动终止** | 进程执行完最后一条语句，通过系统调用 `exit()` 自行结束 | 可返回一个整数退出状态给父进程 |
| **被动/强制终止** | 父进程通过系统调用（如 Windows 的 `TerminateProcess()`）强制终止某个子进程 | 无 |

**父进程终止子进程的原因**

- 子进程消耗了超过分配的资源。
- 子进程原先肩负的任务已经完成或不再需要。
- 父进程即将退出，且系统禁止无父进程的子进程继续运行。这会产生**级联终止（Cascade Termination）**，即父进程终止时，所有子进程也会被系统一并终止。

**两种特殊的进程状态**

> [!warning] 僵尸进程（Zombie Process）
> 子进程已经终止（资源已释放），但其在**进程表（Process Table）**中的条目依然存在，原因是父进程尚未调用 `wait()` 来获取其退出状态。这是一个短暂的状态，一旦父进程执行 `wait()`，僵尸进程就会被彻底清理。

> [!warning] 孤儿进程（Orphan Process）
> 父进程在没有调用 `wait()` 的情况下提前终止，导致子进程变成"孤儿"。在 Linux/UNIX 系统中，此类进程会被系统根进程 **`init`**（PID 为 1）自动领养。`init` 进程会周期性地调用 `wait()` 来收集这些孤儿进程的退出状态并彻底回收资源，防止系统进程表溢出。
