# 第三章 进程

> [!abstract] 本章解决什么问题？
> 进程是操作系统最核心的概念之一。本章回答：什么是进程？进程如何被调度和管理？进程之间如何通信？从进程状态、PCB、调度队列到 IPC 机制，全面揭示进程管理的原理。

## 学习目标

- [ ] 能区分程序（被动）与进程（主动）的本质区别。
- [ ] 能描述进程在内存中的组成部分（文本段、数据段、堆栈、堆）。
- [ ] 能画出进程的五种基本状态及其转换关系。
- [ ] 能说明 PCB 的作用和包含的七大类信息。
- [ ] 能描述三种调度队列和调度程序的分工。
- [ ] 能解释上下文切换的过程和性能开销。
- [ ] 能区分共享内存和消息传递两种 IPC 模型的优劣。
- [ ] 能说明套接字、RPC 和管道的适用场景。
