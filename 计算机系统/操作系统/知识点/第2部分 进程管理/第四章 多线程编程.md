# 第四章 多线程编程

> [!abstract] 本章解决什么问题？
> 第三章介绍了进程的基本概念和进程间通信（IPC）机制，但进程模型存在创建开销大、上下文切换成本高的问题。本章引入**线程**概念——作为进程内的执行单元，线程共享进程资源但拥有独立的执行上下文，大幅降低并发编程的开销。同时探讨多核架构下的并行编程挑战、线程模型设计、线程库使用以及多线程带来的问题与解决方案。

## 学习目标

完成本章学习后，你应该能够：
- 解释线程与进程的区别，说明多线程的优势和适用场景。
- 区分并发（Concurrency）与并行（Parallelism），理解多核架构的并行计算原理。
- 对比三种线程模型（多对一、一对一、多对多）的优缺点。
- 描述主流线程库的特点，理解异步线程与同步线程的创建策略。
- 掌握隐式多线程技术（线程池、OpenMP、GCD）的工作原理。
- 分析多线程环境下的特殊问题（fork/exec、信号处理、线程撤销等）。
- 理解 Windows 和 Linux 操作系统的线程实现机制。
