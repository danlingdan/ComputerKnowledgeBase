# 15.9 例子：Windows 7

本节聚焦于**例子：Windows 7**，是[[第十五章 系统安全]]中的独立知识节点。

> [!warning] 历史平台案例
> Microsoft 已于 2020-01-14 结束 Windows 7 支持。本节仅用于理解 Windows NT 系列的 SID、访问令牌和访问检查思路，不是现代 Windows 的部署或加固指南。

> [!important] 基于用户账户与安全 ID (SID) 的模型
> 
> Windows 7 采用基于用户账户的安全模型。用户通过唯一的 **安全 ID (SID)** 向系统标识自己，并可自由分组以分配权限。

> [!tip] 安全访问令牌 (Access Token)
> 
> - 用户登录时，系统会生成一个 **安全访问令牌**。该令牌包含用户的 SID、所属组的 SID 以及拥有的特权列表（如备份文件、关闭系统等）。
> - 进程执行时会获得其发起用户的令牌副本。每当访问对象时，系统利用令牌中的身份信息进行权限验证。

> [!note] 主题 (Subject) 与安全上下文
> 
> 为限制用户程序的权限范围，Windows 7 引入了"主题"概念。主题用于管理用户运行的程序的权限，包含令牌和运行的程序实体。
> 
> | 类型 | 描述 |
> |------|------|
> | **简单主题** | 基于用户的默认安全上下文执行（如普通应用程序） |
> | **服务器主题** | 充当服务代理，在代替客户执行任务时采用客户的安全上下文 |

> [!example] 强制完整性控制 (Mandatory Integrity Control)
> 
> - Windows 为对象和主题分配 **完整性标签**：不可信、低、中、高、系统。
> - **访问规则**：进程的完整性级别必须**等于或高于**目标对象，才能执行写入操作（`NoWriteUp` 自动强制）。

> [!important] 用户账户控制 (UAC) 与双令牌机制
> 
> 为了实施最小特权原则，UAC 为管理员账户提供**双令牌**：
> - **标准令牌**：用于日常操作，禁用管理员权限，完整性等级为"中等"。
> - **提升令牌**：用于执行需要特权的管理任务，启用管理员组，完整性等级为"高"。

> [!tip] 安全描述符 (Security Descriptor) 与 ACL
> 
> - 对象的安全属性通过 **安全描述符** 描述，包含所有者 SID、组 SID、定制的**访问控制列表 (ACL)** 和系统审计控制列表 (SACL)。
> - ACL 条目由特定用户或组的允许/拒绝 **访问掩码** 组成。文件拥有非常精细的访问粒度（如 `ReadData`, `WriteData`, `Execute`, `ReadAttributes` 等）。

> [!note] 权限继承与容器对象
> 
> - 目录等**容器类对象**中的新对象默认继承父对象的权限，文件复制到新目录时会继承目标目录的权限。
> - **注意**：若修改父目录的权限，**不会**自动应用到现有的子对象和文件，需要用户显式应用。
        
> [!warning] 实际部署的安全挑战与解决
> 
> - Windows 7 提供了丰富的安全工具和功能，但默认情况下许多重要保护措施**未开启**，加之大量的开机服务和预装应用，成为了安全漏洞的主要原因。
> - 在真正的多用户生产环境中，安全管理员必须**制定详尽的安全计划**，结合 Windows 自身功能及外部工具来严格执行和实施。
