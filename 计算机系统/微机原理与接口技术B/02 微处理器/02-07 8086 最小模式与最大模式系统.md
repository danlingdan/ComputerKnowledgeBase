## 2.5 典型 CPU 应用系统

> [!info] 8086/8088 系统组成需求
> 8086/8088 CPU 不包括存储器和 I/O 端口，只能进行数据处理，不能记忆，更不能与外界交换信息。构成完整的微机系统必须加上：
> - 时钟电路
> - 地址锁存器
> - 总线驱动器
> - 存储器和 I/O 接口电路
> - 基本外围设备
>
> 为适用不同应用场合，可设计两种工作模式：
> - **单 CPU（最小）系统**
> - **多 CPU（最大）系统**

> [!tip] 系统总线概念
> 以 8086/8088 CPU 为核心，将微机各部件连接起来的一组线称为**系统总线**。本节以 8086/8088 CPU 为核心，说明如何通过系统总线来设计两种不同模式的系统。详见 [[02-06 系统总线操作与典型时序]] 与 [[02-05 微处理器引脚与总线信号]]。

### 2.5.1 8086/8088 支持芯片

#### 1. 8284 时钟发生器

> [!abstract] 8284 时钟发生器
> 8086/8088 CPU 内部没有时钟发生器，8284 是 Intel 公司专门为 8086/8088 系统配套设计的单片时钟发生器。
>
> **能提供的信号**：
> - 给 CPU 提供 **CLK 时钟、READY 准备就绪、RESET 复位** 信号。
> - 向外提供晶体振荡信号 **OSC**、外围芯片所需时钟 **PCLK** 等其他信号。
>
> **封装**：18 引脚（PIN），双列直插封装（DIP）。

![[计算机系统/微机原理与接口技术B/附件/第2章/Pasted image 20260719155804.png]]
*图 2-48　8284 时钟发生器的引脚及内部结构*

**8284 引脚功能**：

| 引脚 | 方向 | 功能 |
| :--- | :--- | :--- |
| CLK | 输出 | 系统时钟。直接接 CPU 及其他以此为时钟的设备，频率为晶体频率或 $EFI$ 信号频率的 $1/3$，占空比 $1/3$ |
| F/C | 输入 | 频率/晶振选择。$F/C=0$ 选 CLK 为晶体产生；$F/C=1$ 由外接输入频率产生 |
| $X_1$、$X_2$ | 输入 | 晶体输入。直接接晶体两引脚，晶体频率（14.318 MHz）为 CPU 时钟频率（4.77 MHz）的 3 倍 |
| PCLK | 输出 | 外部设备时钟。占空比 $1/2$，频率（2.5 MHz）约为 CLK 的 $1/2$ |
| $EFI$ | 输入 | 外接输入频率。$F/C=1$ 时 CLK 由该端输入信号产生，要求频率是 CLK 的 3 倍方波 |
| $\overline{AEN_1}$、$\overline{AEN_2}$ | 输入 | 地址允许信号。状态分别控制 $RDY_1$、$RDY_2$；为低时 $RDY_1$、$RDY_2$ 产生 READY 信号，使 CPU 产生等待周期 |
| $RDY_1$、$RDY_2$ | 输入 | 总线就绪信号。系统总线上某设备已收到数据或已准备好数据时，使 $RDY_1$ 或 $RDY_2$ 有效 |
| READY | 输出 | 就绪信号。由 $RDY_1$ 或 $RDY_2$ 形成。高电平已准备好；低电平使 CPU 产生等待周期 |
| RES | 输入 | 外部复位信号。经过施密特触发器及 D 触发器产生加到 CPU 的 RESET 信号 |
| CSYNC | 输入 | 实现多片 8284 相位相同的时钟信号 |

#### 2. 8282/8283 8 位三态输出锁存器

> [!abstract] 锁存器用途
> 8086/8088 的 $AD_{15} \sim AD_0$ / $AD_7 \sim AD_0$ 既可作为低 16 位/8 位地址线，又可作为 16 位/8 位数据线。为把地址信息分离出来保存，为外接存储器或外设提供地址信息，需外加三态锁存器，由 CPU 产生的地址锁存允许信号 ALE 的下跳变将地址信息锁存入 8282/8283 锁存器中。
>
> **封装**：20 个引脚，双列直插封装。

![[计算机系统/微机原理与接口技术B/附件/第2章/Pasted image 20260719155821.png]]

**8282/8283 控制引脚**：

| 引脚 | 有效电平 | 功能 |
| :--- | :--- | :--- |
| STB | 高电平 | 选通信号，输入。STB 高电平时 $DO_7 \sim DO_0$ 随 $DI_7 \sim DI_0$ 变化；STB 由高变低时将输入数据锁存 |
| OE | 低电平 | 输出允许。OE 低电平时将被锁存的信号输出；OE 高电平时 8282/8283 输出呈高阻状态。系统中 OE 一般接地，保证总是允许输出 |

#### 3. 8286/8287 并行双向总线驱动器

> [!abstract] 总线驱动器用途
> 8 位并行双向总线驱动器用来将数据总线上数据接收到 CPU 或把 CPU 的数据发送到数据总线上去，同时增加数据总线的带负载能力。
>
> **封装**：20 引脚，双列直插封装。

| :--- | :--- | :--- |
| T | — | 传输方向控制端。T 高电平时数据由 $A_i \to B_i$（发送）；T 低电平时数据由 $B_i \to A_i$（接收） |
| OE | 低电平 | 允许输出端。OE 低电平时允许输出；OE 高电平时输出呈高阻状态 |

#### 4. 8288 总线控制器

> [!abstract] 8288 总线控制器
> 专门为 8086/8088 CPU 构成多 CPU（最大模式）而设计，用以提供有关的总线命令和具有较强的驱动能力，使整个系统性能大大提高。

![[计算机系统/微机原理与接口技术B/附件/第2章/Pasted image 20260719155854.png]]
*图 2-54　8288 总线控制器的输入与输出信号*

**8288 输入信号**：

| 引脚 | 功能 |
| :--- | :--- |
| $\overline{S_2}$、$\overline{S_1}$、$\overline{S_0}$ | 状态译码信号，输入。8288 通过对这些状态译码产生相应总线命令和输出控制信号 |
| CLK | 时钟信号，输入。通常接 8284 CLK 端 |
| AEN | 地址允许信号，输入。低电平时允许 8288 各种命令输出；高电平时各命令处于高阻状态 |
| CEN | 命令允许信号，输入。低电平时 8288 所有命令信号及 DEN、PDEN 全无效；高电平时允许上述信号输出 |
| IOB | 总线方式输入控制信号，输入。低电平时 8288 工作于系统总线方式；高电平时工作于 I/O 总线方式 |

**8288 输出信号**：

| 引脚 | 功能 |
| :--- | :--- |
| $\overline{AIOWC}$ | 超前 I/O 写命令信号。总线周期中提早发出 I/O 写命令 |
| $\overline{AMWC}$ | 超前存储器写命令。功能与 $\overline{AIOWC}$ 相似 |
| $\overline{IOWC}$ | I/O 写命令。通知 I/O 外设取数据总线上的数据 |
| $\overline{IORC}$ | I/O 读命令。通知 I/O 外设把数据放在数据总线上 |
| $\overline{MWTC}$ | 存储器写命令。通知所选存储单元接收数据总线上的数据 |
| $\overline{MRDC}$ | 存储器读命令。通知所选中的存储单元将数据放在数据总线上 |
| MCE/PDEN | 双功能输出。IOB 低时 MCE 高电平有效（中断控制器级联时从主中断控制器读出级联地址）；IOB 高时 PDEN 低电平有效（控制 I/O 总线上数据收发器） |
| $\overline{INTA}$、DT/R、ALE、DEN | 与 8086/8088 CPU 相应引脚类似 |

### 2.5.2 8086/8088 单 CPU（最小模式）系统

#### 1. 8088 单 CPU 系统

> [!info] 8088 单 CPU 系统连接原则
> 1. 8088 CPU 的 **MN/MX 引脚接高电平 $V_{CC}$**。
> 2. 8088 CPU 的 **IO/M、RD、WR、INTA** 引脚直接接在存储器和 I/O 端口相应控制线上。
> 3. 地址线、地址/数据线接到地址锁存器上（$AD_7 \sim AD_0$ 是地址/数据线复用引脚）。一般用锁存器 8282 或 74LS373，将 CPU 的 ALE 接到 STB（或 G）上。
> 4. 数据线有两种接法：直接接到数据总线上，或者经过数据总线驱动器（8286/8287 或 74LS245）接到数据总线上（IBM PC/XT 采用后一种）。
> 5. 8088 CPU 的 CLK 时钟，由 8284 时钟发生器/驱动器提供。8284 还为 8088 提供定时和宽度符合要求的 RESET 复位信号及符合同步要求的 READY"就绪"信号。

> [!info] 数据总线驱动器连接细节
> 双向数据总线驱动器 $A_7 \sim A_0$ 引脚接 8088 CPU $AD_7 \sim AD_0$，其 $B_7 \sim B_0$ 引脚接数据总线的 $D_7 \sim D_0$ 引脚上：
> - 8088 CPU 的 **DT/R** 接 8286 的 **T** 引脚（74LS245 接 DIR 引脚）：DT/R 高电平时数据从 8088 CPU 发送到系统总线；DT/R 低电平时 CPU 从系统数据总线接收数据。
> - 8286 的 **OE** 引脚（74LS245 为 $\overline{G}$）与 8088 CPU 的 **DEN** 引脚相接：DEN 低电平期间才允许数据输入/输出。

#### 2. 8086 单 CPU 系统

![[计算机系统/微机原理与接口技术B/附件/第2章/Pasted image 20260719155912.png]]
*图 2-56　以 8086 为 CPU 的最小模式系统结构*

> [!info] 8086 与 8088 单 CPU 系统差异
> 8086 的数据总线是 **16 位**的，所以需用**两片 8286** 双向数据驱动器。此外，8086 CPU 能直接寻址 1 MB 存储空间，可把它分为两个 512 KB 的存储体。

> [!important] 8086 的奇偶地址存储体
> | 存储体 | 组成 | 数据线连接 | 选中信号 |
> | :--- | :--- | :--- | :--- |
> | 偶地址存储体 | 偶地址单元 | 与 CPU 16 位数据总线的 $D_7 \sim D_0$ 连接 | $A_0=0$ 选中 |
> | 奇地址存储体 | 奇地址单元 | 与 CPU 16 位数据总线的 $D_{15} \sim D_8$ 连接 | $\overline{BHE}=0$ 选中 |
>
> 地址总线 $A_{19} \sim A_1$ 提供存储器片选的高位地址，$A_0$ 和 BHE 区分奇偶地址存储体。

**$\overline{BHE}$ 与 $A_0$ 编码对应的操作**：

| $\overline{BHE}$ | $A_0$ | 操作 |
| :--- | :--- | :--- |
| 0 | 0 | 同时对两个存储体进行读/写操作（存储 16 位数据） |
| 0 | 1 | 只对奇地址存储体进行读/写操作 |
| 1 | 0 | 只对偶地址存储体进行读/写操作 |
| 1 | 1 | — |

> [!note] 8086 单 CPU 系统其他部分
> 除此之外，与 8088 CPU 系统相同。

### 2.5.3 8086/8088 多 CPU（最大模式）系统

> [!abstract] 多 CPU 系统
> 多 CPU 模式又称为多处理器系统或最大模式系统，是在一个系统中存在两个或两个以上的 CPU。如在 IBM PC/XT 扩充系统中，一般加入 8087 浮点运算协处理器和 8089 I/O 处理器。

> [!important] 多 CPU 系统需解决的 4 方面问题
> 1. **多 CPU 并行处理时各 CPU 之间的同步问题**。
> 2. **各 CPU 任务协调处理，并保证协调操作**。
> 3. **多 CPU 系统共用设备的共享和分配问题**。
> 4. **系统总线使用权的占有问题**。

> [!info] IBM PC/XT 系统的实际构成
> 在实际的 IBM PC/XT 系统中，增加 8087 和 8089 辅助处理器后，并不是平等的并行操作，而是在主处理器 8088 的协调控制下进行操作：
> - 在任何情况下，两个处理器不能在同一时刻同时访问系统总线。
> - 各辅助处理器只有在请求主处理器并获得允许之后才能占用总线。
> - 各辅助处理器与主处理器之间采用**异步通信方式**交换数据。
>
> 根据以上原则构成的多 CPU 系统，在系统结构、总线时序和控制信号的定义及主处理器的引脚方面，都要做出相应变动。

#### 1. 以 8088 为核心构成的多 CPU 系统

![[计算机系统/微机原理与接口技术B/附件/第2章/Pasted image 20260719155921.png]]
*图 2-57　以 8088 为主的多 CPU 系统结构*

> [!info] 8088 多 CPU 系统配置要点
> 8088 用于多 CPU 系统时，**MN/MX 引脚必须接地**。由于 8088 不能直接提供总线控制信号，而必须用总线控制器 8288 对 CPU 提供的状态信息 $\overline{S_2}$、$\overline{S_1}$、$\overline{S_0}$ 进行译码来产生总线控制信号。
>
> 将几个 CPU 接到同一组系统总线上，或为了给 8088 提供几组系统总线，必须使用 8288 总线控制器。

> [!abstract] 8288 总线控制器主要功能
> 1. **状态信息译码功能**（见表 2-13）。
> 2. 能产生系统总线的控制信号或访问 I/O 总线控制信号（INTA、IORC、IOWC 等）。
> 3. 可使总线控制信号浮空，允许 DMA 操作。
> 4. 提供专门为慢速存储器或 I/O 设备设计的预写控制信号 $\overline{AIOWC}$、$\overline{AMWC}$。
> 5. 可产生地址锁存器的锁存允许信号 ALE 和双向数据驱动器的控制信号 DEN、DT/R。
> 6. 可产生简单或级联中断逻辑的控制信号 MCE/PDEN。

**表 2-13　8288 状态输入/输出命令对应关系**

| 状态输入 $\overline{S_2}, \overline{S_1}, \overline{S_0}$ | CPU 总线操作周期 | 8288 产生命令 | 功 能 |
| :--- | :--- | :--- | :--- |
| 0 0 0 | 中断响应 | INTA | 8288 输出的中断响应 |
| 0 0 1 | I/O 读 | IORC | 8288 I/O 读控制信号 |
| 0 1 0 | I/O 写 | IOWC, $\overline{AIOWC}$ | 8288 I/O 写控制信号 |
| 0 1 1 | 停机 | / | / |
| 1 0 0 | 取指令周期 | MRDC | 8288 存储器读控制信号 |
| 1 0 1 | 读存储器 | MRDC | 8288 存储器读控制信号 |
| 1 1 0 | 写存储器 | MWTC, $\overline{AMWC}$ | 8288 存储器写，提前写 |
| 1 1 1 | 空操作 | / | / |

> [!info] MCE/PDEN 双功能说明
> - 当 **IOB 接低电平**时，MCE/PDEN 引脚输出 **MCE** 信号，在中断响应总线周期的 $T_1$ 状态有效，用于将中断控制器 8259A 的级联地址送至地址总线的同步信号。
> - 当 **IOB 接高电平**时，MCE/PDEN 引脚输出 **PDEN** 信号，低电平有效，与 DEN 信号的功能和时序相同但相位相反，作为响应 I/O 总线收发器的允许信号。

### 最小模式 vs 最大模式对比

> [!abstract] 最小模式与最大模式对比
>
> | 项目 | 最小模式（单 CPU） | 最大模式（多 CPU） |
> | :--- | :--- | :--- |
> | **MN/MX 引脚** | 接 $V_{CC}$（高电平） | 接 GND（低电平） |
> | **CPU 数量** | 单 CPU | 两个或两个以上 CPU |
> | **总线控制信号来源** | CPU 直接提供（IO/M、RD、WR、INTA 等） | 通过 8288 总线控制器对 $\overline{S_2}$、$\overline{S_1}$、$\overline{S_0}$ 译码产生 |
> | **24~31 引脚定义** | ALE、DEN、DT/R、IO/M、WR、RD、HOLD、HLDA、$\overline{SS_0}$ | $\overline{S_2}$、$\overline{S_1}$、$\overline{S_0}$、$\overline{RQ}/\overline{GT_0}$、$\overline{RQ}/\overline{GT_1}$、LOCK、$QS_1$、$QS_0$ |
> | **是否需 8288** | 否 | 是 |
> | **典型应用** | 简单系统 | IBM PC/XT（含 8087 协处理器） |
> | **多 CPU 同步** | 不需处理 | 需处理同步、任务协调、共享、总线使用权 |
