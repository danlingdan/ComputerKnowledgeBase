## 2.5 典型 CPU 应用系统
8086/8088 CPU 不包括存储器和 I/O 端口，也就是说，它们只能进行数据处理，不能记忆，更不能与外界交换信息。为此，8086/8088 CPU 芯片必须加上必要的支持芯片，如时钟电路、地址锁存器、总线驱动器以及存储器和 I/O 接口电路及基本外围设备，才能构成一台完整的微机系统。为了适用不同的应用场合，可以设计两种不同的工作模式：单 CPU（最小）系统和多 CPU（最大）系统模式。以 8086/8088 CPU 为核心，将微机各部件连接起来的一组线称为系统总线，如第 1 章的图 1-2 所示。本节以 8086/8088 CPU 为核心，说明如何通过系统总线来设计两种不同模式的系统。
### 2.5.1 8086/8088 支持芯片

1. 8284 时钟发生器

8086/8088 CPU 内部没有时钟发生器，而 8284 时钟发生器是 Intel 公司专门为 8086/8088 系统配套设计的单片时钟发生器。它能给 CPU 提供时钟、准备就绪（READY）、复位（RESET）信号，还可向外提供晶体振荡信号（OSC）、外围芯片所需时钟 PCLK 等其他信号。
8284 时钟发生器的引脚及内部结构如图 2-48 所示，共 18 个引脚（PIN），采用双列直插封装（DIP）。下面简要介绍各引脚的功能。

![[计算机系统/微机原理与接口技术B/附件/第2章/Pasted image 20260719155804.png]]
*图 2-48　8284 时钟发生器的引脚及内部结构*

CLK：系统时钟，输出信号。直接接到 CPU 及其他以此为时钟的设备上，其频率为晶体频率或外接频率输入端 $EFI$ 信号频率的 $1/3$。CLK 信号占空比为 $1/3$。
F/C：频率/晶振选择，输入信号。当 F/C=0（低电平）时，选择 CLK 为晶体所产生；当 F/C=1（高电平）时，CLK 由外接输入频率产生。
$X_1$、$X_2$：晶体输入，输入信号。$X_1$、$X_2$ 直接接晶体的两个引脚，晶体的频率（14.318 MHz）为 CPU 所需时钟频率的 3 倍（4.77 MHz）。
PCLK：外部设备时钟，输出信号。输出波形占空比为 $1/2$，其频率（2.5 MHz）约为 CLK 的 $1/2$。
$EFI$：外接输入频率，输入信号。当 F/C=1 时，选择 CLK 由该端输入信号产生。要求输入信号的频率是 CLK 的 3 倍方波。
$\overline{AEN_1}$、$\overline{AEN_2}$：地址允许信号，输入信号。其状态分别控制 $RDY_1$ 及 $RDY_2$。当 $\overline{AEN_1}$、$\overline{AEN_2}$ 为低时，$RDY_1$ 和 $RDY_2$ 产生 READY 信号，致使 CPU 产生等待周期。
$RDY_1$、$RDY_2$：总线就绪信号，输入信号。当系统总线上某个设备已收到数据或已准备好数据时，该设备可使 $RDY_1$ 或 $RDY_2$ 有效。
READY：就绪信号，输出信号。它由 $RDY_1$ 或 $RDY_2$ 形成。高电平表示已准备好；低电平时，使 CPU 产生等待周期。
RES：外部复位信号，输入信号。经过施密特触发器及 D 触发器产生加到 CPU 的复位信号 RESET。
CSYNC：实现多片 8284 相位相同的时钟信号。输入信号。

2. 8282/8283 8 位三态输出锁存器

8086/8088 的 $AD_{15} \sim AD_0$/$AD_7 \sim AD_0$ 既可作为低 16 位/8 位地址线，又可作为 16 位/8 位数据线，为了把地址信息分离出来保存，为外接存储器或外设提供 16 位/8 位地址信息，一般需要外加三态锁存器，并由 CPU 产生的地址锁存允许信号 ALE 的下跳变将地址信息锁存入 8282/8283 锁存器中。
8282/8283 引脚及真值表如图 2-50 所示，采用 20 个引脚，双列直插封装。下面简要介绍 STB 及 OE 引脚的功能。

![[计算机系统/微机原理与接口技术B/附件/第2章/Pasted image 20260719155821.png]]
*图 2-50　8282/8283 引脚及真值表*

STB：选通信号，输入信号，高电平有效。当 STB 为高电平时，输出 $DO_7 \sim DO_0$ 随输入 $DI_7 \sim DI_0$ 而变，即起传输作用；当 STB 由高电平变到低电平时，将输入数据锁存。
OE：输出允许，低电平有效。当 OE 为低电平时，将被锁存的信号输出；当 OE 为高电平时，8282/8283 输出呈高阻状态。在系统中，OE 一般接地，保证总是允许输出状态。

3. 8286/8287 并行双向总线驱动器

8 位并行双向总线驱动器用来将数据总线上数据接收到 CPU 或把 CPU 的数据发送到数据总线上去，另一方面用以增加数据总线的带负载能力。
8286/8287 引脚及一位内部结构如图 2-52 所示，采用 20 引脚，双列直插封装。下面简单介绍控制信号 OE、T 的作用。

T：传输方向控制端，输入信号。当 T 为高电平时，数据由 $A_i \to B_i$（发送）；当 T 为低电平时，数据由 $B_i \to A_i$（接收）。
OE：允许输出端，输入信号。当 OE 为低电平时，允许输出；当 OE 为高电平时，输出呈高阻状态。

4. 8288 总线控制器

8288 总线控制器专门为 8086/8088 CPU 构成多 CPU（最大模式）而设计，用以提供有关的总线命令和具有较强的驱动能力，使整个系统性能大大提高。

![[计算机系统/微机原理与接口技术B/附件/第2章/Pasted image 20260719155854.png]]
*图 2-54　8288 总线控制器的输入与输出信号*

图 2-54 是 8288 引脚与 CPU 连接的两组输入信号和输出信号。下面简要介绍这两组信号。

$\overline{S_2}$、$\overline{S_1}$、$\overline{S_0}$：状态译码信号，输入信号。8288 通过对这些状态译码，产生相应总线命令信号和输出控制信号。
CLK：时钟信号，输入信号。通常接 8284 CLK 端。
AEN：地址允许信号，输入信号。当 AEN 为低电平时，允许 8288 的各种命令输出；当 AEN 为高电平时，各命令处于高阻状态。
CEN：命令允许信号，输入信号。当 CEN 为低电平时，8288 所有命令信号及 DEN、PDEN 全无效；当 CEN 为高电平时，允许上述信号输出。
IOB：总线方式输入控制信号，输入信号。当 IOB 为低电平时，8288 工作于系统总线方式；当 IOB 为高电平时，它工作于 I/O 总线方式。
$\overline{AIOWC}$：超前 I/O 写命令信号，输出信号。在总线周期中提早由 $\overline{AIOWC}$ 发出一个 I/O 写命令，以便较早地通知 I/O 设备执行的是写命令。
$\overline{AMWC}$：超前存储器写命令，输出信号。其功能与 $\overline{AIOWC}$ 相似，只是由 $\overline{AMWC}$ 发出。
$\overline{IOWC}$：I/O 写命令，输出信号。通知 I/O 外设取数据总线上的数据。
$\overline{IORC}$：I/O 读命令，输出信号。通知 I/O 外设把数据放在数据总线上。
$\overline{MWTC}$：存储器写命令，输出信号。通知所选存储单元接收数据总线上的数据。
$\overline{MRDC}$：存储器读命令，输出信号。通知所选中的存储单元将数据放在数据总线上。
MCE/PDEN：具有双功能，输出信号。当 IOB 为低电平时，MCE 高电平有效。在中断控制器级联工作时，MCE 用于从主中断控制器读出级联地址送到级联总线上。当 IOB 为高电平时，用于控制外设数据，控制 I/O 总线上的数据收发器，此时 PDEN 低电平有效。
$\overline{INTA}$、DT/R、ALE 及 DEN 与 8086/8088 CPU 相应引脚类似，不再赘述。

### 2.5.2 8086/8088 单 CPU（最小模式）系统

1. 8088 单 CPU 系统

在这种系统中，8088 CPU 引脚与总线连接关系遵循以下原则：

1. 8088 CPU 的 MN/MX 引脚接高电平 $V_{CC}$。
2. 8088 CPU 的 IO/M、RD、WR、INTA 引脚直接接在存储器和 I/O 端口相应控制线上。
3. 地址线、地址/数据线接到地址锁存器上。因为 $AD_7 \sim AD_0$ 是地址/数据线复用引脚，在发数据前必须先将地址锁存起来。一般用锁存器 8282 或 74LS373。当 STB（或 G）由高电平变为低电平时，芯片将输入端的信号存入锁存器。OE 为输出控制信号，当它为低电平时，将锁存器内的信息送到输出端。为此只需将 CPU 的 ALE 接到 STB（或 G）上即可。
4. 数据线有两种接法：直接接到数据总线上，或者经过数据总线驱动器接到数据总线上。IBM PC/XT 采用后一种。这是因为数据总线上连接的设备比较多，要求它有足够的负载驱动能力。数据是双向传输的，需要采用双向总线驱动器。常用的 8 位双向总线驱动器有 8286/8287 或 74LS245。双向数据总线驱动器 $A_7 \sim A_0$ 引脚接 8088 CPU $AD_7 \sim AD_0$，其 $B_7 \sim B_0$ 引脚接数据总线的 $D_7 \sim D_0$ 引脚上。8088 CPU 的 DT/R 接 8286 的 T 引脚（74LS245 是接 DIR 引脚），当 DT/R 为高电平时，数据从 8088 CPU 发送到系统总线上去；当 DT/R 为低电平时，CPU 则从系统数据总线上接收数据。8286 的 OE 引脚（74LS245 则为 $\overline{G}$）与 8088 CPU 的 DEN 引脚相接。8088 CPU 在 DEN 端为低电平期间，才允许数据输入/输出。
5. 8088 CPU 的 CLK 时钟，是由 8284 时钟发生器/驱动器提供的。8284 还为 8088 提供定时和宽度符合要求的 RESET 复位信号及符合同步要求的 READY“就绪”信号。

2. 8086 单 CPU 系统

![[计算机系统/微机原理与接口技术B/附件/第2章/Pasted image 20260719155912.png]]
*图 2-56　以 8086 为 CPU 的最小模式系统结构*

图 2-56 是以 8086 为 CPU 的单 CPU（最小模式）系统结构。8086 的数据总线是 16 位的，所以需用两片 8286 双向数据驱动器。此外，8086 CPU 能直接寻址 1 MB 存储空间，可把它分为两个 512 KB 的存储体。其中一个奇存储体由奇地址单元组成，用于存储 16 位数据的高字节；另一个存储体由偶地址单元组成，用于存储 16 位数据的低字节。前者称为奇地址存储体，后者称为偶地址存储体。偶地址存储体的 8 位数据线，与 CPU 16 位数据总线的 $D_7 \sim D_0$ 连接；奇地址存储体的 8 位数据线与 CPU 16 位数据总线的 $D_{15} \sim D_8$ 连接。地址总线 $A_{19} \sim A_1$ 提供存储器片选的高位地址，$A_0$ 和 BHE 区分奇偶地址存储体。当 $A_0=0$ 时，选中偶地址存储体；当 $A_0=1$ 时，不选中偶地址存储器。当 $BHE=0$ 时，选中奇地址存储体；当 $A_0$ 和 BHE 两个信号都为低电平时，可同时对两个存储体进行读/写操作，即存储 16 位数据；而当 $BHE=0$，$A_0=1$ 时，只对奇地址存储体进行读/写操作；当 $A_0=0$， $BHE=1$ 时，只对偶地址存储体进行读/写操作。除此之外，与 8088 CPU 系统相同。

### 2.5.3 8086/8088 多 CPU（最大模式）系统
多 CPU 模式又称为多处理器系统或最大模式系统，是在一个系统中存在两个或两个以上的 CPU。如在 IBM PC/XT 扩充系统中，一般加入 8087 浮点运算协处理器和 8089 I/O 处理器。一个多 CPU 系统，需要解决好以下 4 方面的问题：
- 多 CPU 并行处理时，各 CPU 之间同步的问题。
- 各 CPU 任务协调处理，并保证协调操作。
- 多 CPU 系统共用设备的共享和分配问题。
- 系统总线使用权的占有问题。
在实际的 IBM PC/XT 系统中，增加 8087 和 8089 辅助处理器后，并不是平等的并行操作，而是在主处理器 8088 的协调控制下进行操作。在任何情况下，两个处理器不能在同一时刻同时访问系统总线，各辅助处理器只有在请求主处理器并获得允许之后才能占用总线。各辅助处理器与主处理器之间采用异步通信方式交换数据。
根据以上原则构成的多 CPU 系统，在系统结构、总线时序和控制信号的定义及主处理器的引脚方面，都要做出相应变动。

1. 以 8088 为核心构成的多 CPU 系统

8088 用于多 CPU 系统时，MN/MX 引脚必须接地。由于 8088 不能直接提供总线控制信号，而必须用总线控制器 8288 对 CPU 提供的状态信息 $\overline{S_2}$、$\overline{S_1}$、$\overline{S_0}$ 进行译码来产生总线控制信号。为此，将几个 CPU 接到同一组系统总线上，或为了给 8088 提供几组系统总线，必须使用 8288 总线控制器。其主要作用是把原来的状态信号 $\overline{S_2}$、$\overline{S_1}$、$\overline{S_0}$ 及输入控制信号 CLK、AEN、CEN、IOB 变成相应总线命令和控制信号。
8288 总线控制器主要功能如下。

1. 状态信息译码功能，如表 2-13 所示。

**表 2-13　8288 状态输入/输出命令对应关系**

| 状态输入 $\overline{S_2}, \overline{S_1}, \overline{S_0}$ | CPU 总线操作周期 | 8288 产生命令 | 功 能 |
| :--- | :--- | :--- | :--- |
| 0 0 0 | 中断响应 | INTA | 8288 输出的中断响应 |
| 0 0 1 | I/O 读 | IORC | 8288 I/O 读控制信号 |
| 0 1 0 | I/O 写 | IOWC, $\overline{AIOWC}$ | 8288 I/O 写控制信号 |
| 0 1 1 | 停机 | / | / |
| 1 0 0 | 取指令周期 | MRDC | 8288 存储器读控制信号 |
| 1 0 1 | 读存储器 | MRDC | 8288 存储器读控制信号 |
| 1 1 0 | 写存储器 | MWTC, $\overline{AMWC}$ | 8288 存储器写，提前写 |
| 1 1 1 | 空操作 | / | / |

2. 能产生系统总线的控制信号或访问 I/O 总线控制信号（INTA、IORC、IOWC 等）。
3. 可使总线控制信号浮空，允许 DMA 操作。
4. 提供专门为慢速存储器或 I/O 设备设计的预写控制信号 $\overline{AIOWC}$、$\overline{AMWC}$。
5. 可产生地址锁存器的锁存允许信号 ALE 和双向数据驱动器的控制信号 DEN、DT/R。
6. 可产生简单或级联中断逻辑的控制信号 MCE/PDEN。当 IOB 接低电平时，MCE/PDEN 引脚输出 MCE 信号，在中断响应总线周期的 $T_1$ 状态有效，用于将中断控制器 8259A 的级联地址送至地址总线的同步信号。当 IOB 接高电平时，MCE/PDEN 引脚输出 PDEN 信号，低电平有效，并与 DEN 信号的功能和时序相同，但相位相反，它作为响应 I/O 总线收发器的允许信号。
