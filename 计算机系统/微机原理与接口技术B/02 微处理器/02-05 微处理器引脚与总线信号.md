## 2.3 微处理器的主要引脚及功能

### 2.3.1 8086/8088 CPU 引脚功能

> [!info] 8086/8088 封装规格
> 8086/8088 均为 **40 条引脚**（PIN）、双列直插式（DIP）封装，引脚分配如图 2-38 所示。

![[计算机系统/微机原理与接口技术B/附件/第2章/Pasted image 20260719155556.png]]
*图 2-38　引脚分配*

> [!abstract] 8086/8088 引脚设计动机
> 8086/8088 比 8 位 CPU 功能增强：
> - 数据宽度从 8 位扩展到 16 位（8086）。
> - 直接寻址从 64 KB 扩展到 1 MB，采用段寄存器和存储器分段技术。
> - 利用某些引脚的双功能，可构成单 CPU 和多 CPU 系统。

> [!info] 多功能引脚的两种复用方式
> 为减少外部引脚数目，8086/8088 CPU 内部设置若干多路开关，使某些引脚具有多种功能：
> 1. **分时复用**：在总线周期的不同时钟周期内功能不同。
> 2. **按工作模式定义**：同一引脚在单 CPU（最小模式）和多 CPU（最大模式）下连接不同信号。

下面先说明 8088 引脚的功能，再从 8086 与 8088 的差别角度说明 8086 引脚的功能。

#### 公共引脚（最小/最大模式通用）

> [!info] 地址/数据复用引脚
> | 引脚 | 方向 | 类型 | 功能 |
> | :--- | :--- | :--- | :--- |
> | $AD_7 \sim AD_0$ | 双向（入/出） | 三态 | 8 根分时复用多功能引脚。$T_1$ 状态用作地址总线低 8 位 $A_7 \sim A_0$，输出访问存储器或 I/O 口地址；之后多路转换开关将其转换为数据总线 $D_7 \sim D_0$ 传送数据。DMA 方式时浮空 |
> | $A_{15} \sim A_8$ | 输出 | 三态 | 地址总线，整个总线周期内保持有效。DMA 方式时浮空 |
> | $A_{19}/S_6 \sim A_{16}/S_3$ | 输出 | 三态 | 4 根分时复用引脚。$T_1$ 状态用作地址总线高 4 位；I/O 访问时为低电平（I/O 口只用 16 位地址）；$T_2 \sim T_4$ 期间输出状态信息。DMA 方式时浮空 |

> [!info] 状态位 $S_6 \sim S_3$ 含义
> - $S_6$：总是低电平。
> - $S_5$：可屏蔽中断允许标志，可在每个时钟周期开始修改。
> - $S_4$ 和 $S_3$：表示当前访问存储器所用段寄存器，提供段地址。

**表 2-6　$S_4$、$S_3$ 编码与段寄存器的对应关系**

| $S_4$ | $S_3$ | 性能 | 对应段寄存器 |
| :--- | :--- | :--- | :--- |
| 0 | 0 | 数据交替 | 使用附加段寄存器 ES |
| 0 | 1 | 堆栈操作 | 使用堆栈段寄存器 SS |
| 1 | 0 | 代码 | 使用代码段寄存器 CS |
| 1 | 1 | 数据 | 使用数据段寄存器 DS |

#### 中断与控制引脚

| 引脚 | 方向 | 有效电平 | 功能 |
| :--- | :--- | :--- | :--- |
| NMI | 输入 | 上升沿 | 非屏蔽中断请求。不能通过软件屏蔽。出现低到高电脉冲使 CPU 现行指令结束，立刻进入中断响应 |
| INTR | 输入 | 高电平 | 可屏蔽中断请求，电平触发。CPU 在每条指令最后一个时钟周期采样；$IF=1$ 时进入中断响应周期，从数据总线读取中断类型码并查找中断向量表 |
| CLK | 输入 | — | 时钟信号，为 CPU 和总线控制器提供定时基准 |
| RESET | 输入 | 高电平 | 复位信号。停止操作；FR、DS、SS、ES、IP 置 0；指令队列清空；至少保持 4 个时钟周期（主频 4.77 MHz 时 $>50\mu s$）；恢复低电平后从 FFFF0H 单元启动 |
| READY | 输入 | 高电平 | 准备就绪信号。被访问部件无法在规定时间内完成数据传输时，READY 处于低电平，CPU 进入等待状态插入 $T_w$；READY 高电平时 CPU 继续运行。慢速设备 READY 信号需经 8284 时钟发生器同步 |
| $\overline{TEST}$ | 输入 | 低电平 | 测试信号。执行 WAIT 指令时，每隔 5 个时钟周期采样；高电平重复执行 WAIT 等待，低电平脱离等待。与 WAIT 指令结合使用，主要用于多处理器环境（如协处理器 8087 的 BUSY 引脚接 $\overline{TEST}$） |

> [!warning] INTR 屏蔽机制
> 若中断允许标志位 $IF=0$，即使 INTR 引脚信号为高电平，CPU 对外界送来的中断请求信号也不予理睬。可通过软件使 $IF=0$ 实现屏蔽中断请求 INTR。

#### 最小模式专属引脚（24~31）

| 引脚 | 方向 | 有效电平 | 功能 |
| :--- | :--- | :--- | :--- |
| INTA | 输出（三态） | 低电平 | 中断响应信号，CPU 响应外部中断请求后发给请求中断设备的回答信号。在每个中断响应周期 $T_2$、$T_3$、$T_w$ 期间有效低电平，通知中断源送出中断矢量码 |
| ALE | 输出 | 高电平 | 地址锁存允许信号。用作把地址信号锁存入 8282/8283 或 74LS373 锁存器的锁存控制信号。总线周期 $T_1$ 期间高电平，下跳沿锁存地址信息。DMA 方式中不能浮空 |
| DEN | 输出（三态） | 低电平 | 数据允许。在单 CPU 系统中作为 8286/8287 双向驱动器的选通信号。每个存储器或 I/O 访问周期及中断响应周期变为有效低电平。DMA 方式时浮空 |
| DT/R | 输出（三态） | — | 数据发送/接收控制。$DT/R=1$ CPU 发送数据；$DT/R=0$ CPU 接收数据。DMA 方式时浮空 |
| IO/M | 输出（三态） | 高/低电平 | 外设/内存访问控制。高电平表示 I/O 访问周期；低电平表示存储器访问周期。DMA 方式时浮空 |
| WR | 输出（三态） | 低电平 | 写信号。表示 CPU 正在执行写存储器或写 I/O 口操作，由 IO/M 决定写入目标。DMA 方式时浮空 |
| RD | 输出（三态） | 低电平 | 读信号。表示 CPU 在读存储器或 I/O 口数据。$IO/M$ 低电平时读存储器。DMA 方式时浮空 |
| HOLD | 输入 | 高电平 | 保持请求信号。DMA 操作或外部处理器要求通过总线传送数据时为高电平，表示外界请求现有主 CPU 让出总线控制权 |
| HLDA | 输出 | 高电平 | 保持响应信号。CPU 同意让出总线控制权时输出高电平，通知外界可以使用总线；同时 CPU 所有"三态"线进入浮空状态。HOLD 变低时 HLDA 也变低，CPU 重新获得总线控制权 |
| $\overline{SS_0}$ | 输出（三态） | — | 状态信号。逻辑上等同于多 CPU 系统中的 $\overline{S_0}$，只在单 CPU 系统中存在。多 CPU 系统下总是输出高电平 |

> [!important] MN/MX 模式选择引脚
> **MN/MX**（单 CPU/多 CPU 方式控制，输入）：决定系统是构成单 CPU（最小模式）还是多 CPU（最大模式）系统。
> - **$MN/MX = V_{CC}$**：单 CPU（最小模式）工作，8088 的 $24 \sim 31$ 引脚功能如上所述。
> - **$MN/MX = GND$**：多 CPU（最大模式）工作，8088 的 $24 \sim 31$ 引脚定义如图 2-38 括号内所示。
>
> IBM PC/XT 具有协处理器 8087，硬件扩展功能强，通常设计 8088 工作于多 CPU（最大模式）方式下。

#### 最大模式专属引脚（24~31）

| 引脚 | 方向 | 有效电平 | 功能 |
| :--- | :--- | :--- | :--- |
| $\overline{S_2}$、$\overline{S_1}$、$\overline{S_0}$ | 输出（三态） | 低电平 | 总线周期状态标志。组合表示 CPU 总线周期操作类型，送入 8288 总线控制器产生访问存储器或 I/O 的控制信号。$T_1$ 有效，$T_3$ 或 $T_w$ 后半周期且 READY 高电平时返回无效。DMA 方式时进入高阻状态 |
| $\overline{RQ}/\overline{GT_0}$、$\overline{RQ}/\overline{GT_1}$ | 双向（三态） | 低电平 | 请求/允许控制信号。供外部主控设备请求总线控制权。$\overline{RQ}/\overline{GT_0}$ 优先权高于 $\overline{RQ}/\overline{GT_1}$ |
| LOCK | 输出（三态） | 低电平 | 封锁信号。$LOCK=0$ 时外部主控设备不能占用总线。由指令前缀 LOCK 设置，保持到指令结束。DMA 方式时浮空 |
| $QS_1$、$QS_0$ | 输出 | 高电平 | 指令队列状态。反映 CPU 内部当前指令队列状态，以便外部主控设备跟踪 |

#### $\overline{RQ}/\overline{GT}$ 请求/允许工作时序

![[计算机系统/微机原理与接口技术B/附件/第2章/Pasted image 20260719155616.png]]
*图 2-39　请求/允许控制信号工作时序*

> [!example] $\overline{RQ}/\overline{GT}$ 三步工作流程
> 1. **请求**：其他主控制设备在 $\overline{RQ}/\overline{GT}$ 引脚向 8088 发出一个时钟周期宽的负脉冲，表示请求占用总线。
> 2. **允许**：在当前总线周期 $T_4$ 或下个总线周期 $T_1$ 期间，8088 输出一个宽度为一个时钟周期的负脉冲，通知主控设备同意让出总线，并从下一个时钟周期开始进入"同意占有"总线状态。在此期间 8088 所有三态线（除了 $\overline{RQ}/\overline{GT}$ 线）均置成浮空状态。
> 3. **释放**：主控设备申请总线控制权结束，送出一个时钟周期宽的负脉冲给 8088，表示请求占用总线周期已经结束，8088 可以在下一个时钟脉冲上升沿收回总线控制权。

#### 8086 与 8088 引脚差异

> [!warning] 8086 与 8088 引脚关键区别
> 1. **数据/地址复用引脚数**：8086 有 16 条 $AD_{15} \sim AD_0$；8088 只有 $AD_7 \sim AD_0$ 分时复用。
> 2. **34 引脚定义不同**：8086 为 $\overline{BHE}/S_7$（8088 为 $\overline{SS_0}$），是高 8 位数据总线的允许和状态信息复用引脚。
> 3. **28 引脚逻辑相反**：8086 为 $M/\overline{IO}$；8088 为 $IO/M$，逻辑定义正好相反。

> [!info] $\overline{BHE}$ 与 $A_0$ 的作用
> - $\overline{BHE}$ 可看作附加地址总线，用来访问存储器的高字节。
> - $A_0$ 用来访问存储器的低字节。
> - $\overline{BHE}$ 通常作为接在高 8 位数据总线上的设备片选信号。
> - $A_0$ 作为接在低 8 位数据总线上的设备片选信号。

**表 2-11　$\overline{BHE}$ 和 $A_0$ 编码与数据总线数据传送的关系**

| $\overline{BHE}$ | $A_0$ | 数据传送状态 |
| :--- | :--- | :--- |
| 0 | 0 | 传送 16 位 $D_{15} \sim D_0$ |
| 0 | 1 | 传送高 8 位 $D_{15} \sim D_8$ |
| 1 | 0 | 传送低 8 位 $D_7 \sim D_0$ |
| 1 | 1 | 无操作 |

> [!note] $\overline{BHE}/S_7$ 时序
> - 在总线周期 $T_1$ 期间 $\overline{BHE}$ 输出有效信息。
> - 在 $T_2$、$T_3$、$T_w$、$T_4$ 期间，这条引脚输出 $S_7$ 状态信息，$S_7$ 低电平有效。
> - 在 DMA 工作方式下为浮空状态。

### 2.3.2 80386 CPU 主要引脚功能

> [!info] 80386 型号与封装
> 80386 有两个型号：
> - **80386DX**：132 引脚 PGA（引脚栅格阵列）封装，32 位数据和地址总线。
> - **80386SX**：缩减总线型号，16 位数据总线和 24 位地址总线。
>
> 下面以 80386DX 为例介绍。

![[计算机系统/微机原理与接口技术B/附件/第2章/Pasted image 20260719155632.png]]
*图 2-40　80386DX 的引脚分配图*

> [!warning] 80386 起的低电平有效信号表示法变更
> 从 80386 开始，产品数据手册中把低电平或负脉冲有效的表示方法由习惯上的加上划线改成后跟"#"。如 $M/IO\#$ 表示 $\overline{M/IO}$，$\overline{BE_3}$ 等同于 $BE_3\#$。

#### 80386DX 引脚功能表

| 引脚 | 方向 | 功能 |
| :--- | :--- | :--- |
| $A_{31} \sim A_2$ | 输出 | 地址总线，访问 1G×32 位存储单元。$A_1$、$A_0$ 被编码为总线允许信号 $\overline{BE_3} \sim \overline{BE_0}$ |
| $D_{31} \sim D_0$ | 双向 | 数据总线，CPU 与存储器及 I/O 间传送数据 |
| $\overline{BE_3} \sim \overline{BE_0}$ | 输出 | 存储体使能信号，选择字节、字或双字数据。由 CPU 内部根据 $A_1$、$A_0$ 产生 |
| ADS | 输出 | 地址数据选通引脚。当 80386 发出有效存储器地址或 I/O 地址时产生该信号 |
| RESET | 输入 | 复位引脚。初始化 80386，使其从内存 FFFFFFF0H 处开始执行，复位为实模式 |
| $BS_{16}$ | 输入 | 16 位总线宽输入引脚。选择 32 位或 16 位数据总线 |
| M/IO | 输出 | 存储器及 I/O 选择。1 选存储器设备，0 选 I/O 设备。I/O 操作期间地址总线含 16 位地址 $A_{15} \sim A_2$ 及低 2 位 |
| W/R | 输出 | 写/读引脚。1 写周期，0 读周期 |
| CLK2 | 输入 | 两倍频时钟引脚，由 2 倍于 80386 工作频率的时钟信号驱动 |
| INTR | 输入 | 中断请求信号。外部电路用该信号请求中断 |
| NMI | 输入 | 非屏蔽中断输入引脚，请求非屏蔽中断 |
| READY | 输入 | 就绪引脚，控制为延长访问时间而插入的等待状态个数 |
| LOCK | 输出 | 总线锁定引脚。指令带 LOCK 前缀时为逻辑 0，常用于 DMA 访问期间 |
| D/C | 输出 | 数据/控制输出。1 数据总线含数据；0 CPU 执行中断响应或已停机 |
| NA | 输入 | 下一地址信号。使 80386 在当前总线周期输出下一条指令或数据地址，常用于流水线操作 |
| HOLD | 输入 | 总线保持引脚，请求 DMA 操作 |
| HLDA | 输出 | 总线保持响应引脚，表明 80386 当前处于总线保持状态 |
| PEREQ | 输入 | 协处理器请求信号，要求 80386 放弃总线控制并直接连到 80387 算术协处理器 |
| BUSY | 输入 | 忙输入信号。由 WAIT 和 FWAIT 指令使用，等待协处理器变为空闲 |

### 2.3.3 Pentium CPU 主要引脚功能

> [!info] Pentium 版本与封装
> Pentium CPU 有两个版本：
> - **全功能型 Pentium**：237 引脚 PGA 封装，64 位数据总线，32 位地址总线。
> - **Pentium OverDrive（P24T 型）**：32 位数据总线，可方便地插到有 P24T 插座的 80486 机器中。
>
> 早期型号需要 +5 V 电源，功耗大；后期型号（90 MHz 以上）采用 3.3 V 电源，配有良好通风效果的散热器。

#### Pentium 主要引脚功能表

**地址与数据**：

| 引脚 | 方向 | 功能 |
| :--- | :--- | :--- |
| $A_{31} \sim A_3$ | 输出 | 地址总线引脚，寻址任何 512K×64 位存储空间。$A_2$、$A_1$、$A_0$ 在 $\overline{BE_7} \sim \overline{BE_0}$ 中被编码 |
| $\overline{BE_7} \sim \overline{BE_0}$ | 输出 | 存储器体允许信号，选择单字节、字、双字、四字数据。由 $A_2$、$A_1$、$A_0$ 产生 |
| $D_{63} \sim D_0$ | 双向 | 数据总线信号，CPU、内存和 I/O 间传输字节、字、双字、四字数据 |
| A20 | 输入 | 地址 $A_{20}$ 屏蔽输入引脚，用于实模式中通知 Pentium CPU 进行地址扩展 |

**控制与状态**：

| 引脚 | 方向 | 有效电平 | 功能 |
| :--- | :--- | :--- | :--- |
| ADS | 输出 | — | 地址数据选通信号。Pentium CPU 发出有效存储器或 I/O 地址时变为有效 |
| RESET | 输入 | — | 复位信号。初始化 Pentium CPU，使其从 FFFFFFF0H 处开始执行，复位为实模式，最高位 12 条地址线保持逻辑 1（FFFFH），直至执行远程调用或远跳转 |
| BRDY | 输入 | — | 猝发就绪信号。通知 Pentium CPU 外部系统已从数据总线连接中取得数据。可插入等待状态 |
| BREQ | 输出 | — | 总线请求输出信号，指示 Pentium CPU 已产生一个总线请求 |
| CLK | 输入 | — | 时钟引脚，由具有当前 Pentium CPU 工作频率的时钟信号驱动 |
| D/C | 输出 | — | 数据/控制信号。1 表明数据总线含数据；0 表明 CPU 处于停机或正在执行中断请求 |
| FLUSH | 输入 | — | 清 Cache 输入信号。使 Cache 清除所有写回行并使内部 Cache 无效。复位时该信号为 0，Pentium CPU 进入 Cache 测试模式 |
| CACHE | 输出 | — | Cache 输出信号，指示当前 Pentium CPU 周期可对数据进行缓存 |

**Cache 一致性与命中**：

| 引脚 | 方向 | 功能 |
| :--- | :--- | :--- |
| HIT | 输出 | 命中信号。表明在查询方式中内部 Cache 包含了合法数据 |
| HITM | 输出 | 命中修改信号。表明在查询周期中发现了一个修改过的 Cache 行，用于在已修改过的 Cache 行写回到存储器之前禁止其他单元访问数据 |
| WB/WT | 输出 | 写回/写直达信号，为 Pentium CPU 数据 Cache 选择相应操作 |
| KEN | 输入 | Cache 使能输入信号，允许使用内部高速缓存 |

**总线仲裁**：

| 引脚 | 方向 | 功能 |
| :--- | :--- | :--- |
| HOLD | 输入 | 保持信号。请求一个 DMA 操作 |
| HLDA | 输出 | 保持响应信号。指示 Pentium CPU 当前处于保持状态 |
| LOCK | 输出 | 锁定信号。指令带 LOCK 前缀时为逻辑 0，常用于 DMA 访问中 |

**读写控制**：

| 引脚 | 方向 | 功能 |
| :--- | :--- | :--- |
| M/IO | 输出 | 存储器及 I/O 选择。1 选存储器，0 选 I/O。I/O 操作期间地址总线含 16 位地址 $A_{15} \sim A_3$ 及低 3 位 |
| W/R | 输出 | 写/读引脚。1 写周期，0 读周期 |
| NA | 输入 | 下一地址信号。表明外部存储器系统已准备好接收新的总线周期 |
| INTR | 输入 | 中断请求信号。外部电路用该信号进行中断请求 |
| NMI | 输入 | 非屏蔽中断信号。请求一个非屏蔽中断 |
| INV | 输入 | 无效输入信号。决定一个查询后的 Cache 行状态 |

**流水线与中断**：

| 引脚 | 方向 | 功能 |
| :--- | :--- | :--- |
| IU | 输出 | U 管道指令完成输出信号。表明 U 管道中的指令已完成 |
| IV | 输出 | V 管道指令完成输出信号。表明 V 管道中的指令已完成 |

**系统管理**：

| 引脚 | 方向 | 功能 |
| :--- | :--- | :--- |
| SMI | 输入 | 系统管理中断输入信号。使 Pentium CPU 进入系统管理运行模式 |
| SMIACT | 输出 | 系统管理中断激活输出信号。表明 Pentium CPU 正工作在系统管理模式 |

**IEEE 1149.1 边界扫描测试**：

| 引脚 | 方向 | 功能 |
| :--- | :--- | :--- |

> [!note] Pentium CPU 复位地址
> RESET 后 Pentium CPU 从内存 FFFFFFF0H 处开始执行，最高位的 12 条地址线保持逻辑 1（FFFFH），直至执行一个远程调用或远跳转，这样可使它与早期的 CPU 兼容。详见 [[02-08 x86 处理器工作模式]]。
