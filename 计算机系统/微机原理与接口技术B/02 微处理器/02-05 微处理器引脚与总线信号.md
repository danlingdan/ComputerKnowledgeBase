## 2.3 微处理器的主要引脚及功能
### 2.3.1 8086/8088 CPU 引脚功能
8086/8088 均为 40 条引脚（PIN）、双列直插式（DIP）封装，引脚分配如图 2-38 所示。

![[计算机系统/微机原理与接口技术B/附件/第2章/Pasted image 20260719155556.png]]
*图 2-38　引脚分配*

8086/8088 的功能比 8 位 CPU 功能增强很多，因为：数据宽度从 8 位扩展到 16 位（8086）；直接寻址从 64 KB 扩展到 1 MB，采用段寄存器和存储器分段技术，利用某些引脚的双功能，可构成单 CPU 和多 CPU 系统。
为减少外部引脚的数目，在 8086/8088 CPU 内部设置了若干多路开关，使某些引脚具有多种功能，这些多功能引脚的功能转换分两种情况：① 分时复用，在总线周期的不同时钟周期内其功能不同；② 按工作模式定义引脚的功能，同一引脚在单 CPU（最小模式）和多 CPU（最大模式）下，连接不同的信号。
下面先说明 8088 引脚的功能，再从 8086 和 8088 的差别角度说明 8086 引脚的功能。
$AD_7 \sim AD_0$：地址/数据总线，双向（入/出），三态。这是 8 根分时复用多功能引脚，担负着双重任务。在每个总线周期 $T_1$ 状态用作地址总线低 8 位 $A_7 \sim A_0$，输出访问存储器或 I/O 口地址，然后内部的多路转换开关将它们转换为数据总线 $D_7 \sim D_0$，用来传送数据，直到总线周期结束。在 DMA 方式时，这些引脚成浮空状态。
$A_{15} \sim A_8$：地址总线，输出，三态。这些地址在整个总线周期内保持有效（即输出稳定的 8 位地址）。在 DMA 方式时，这些引脚为浮空状态。
$A_{19}/S_6 \sim A_{16}/S_3$：地址/状态线，输出，三态。这是 4 根分时复用的多功能引脚，担负着双重任务，在每个总线周期 $T_1$ 状态用作地址总线高 4 位。但对 I/O 访问时，这些线为低电平，因为 I/O 口只用 16 位地址。在总线周期 $T_2 \sim T_4$ 期间，输出状态信息：$S_6$ 总是低电平；
$S_5$ 是可屏蔽中断允许标志，可在每个时钟周期开始修改；$S_4$ 和 $S_3$ 表示当前访问存储器所用段寄存器，用来提供段地址。$S_4$ 和 $S_3$ 编码与段寄存器对应关系如表 2-6 所示。在 DMA 方式时，这些引脚为浮空状态。
NMI：非屏蔽中断请求，输入，上升沿有效。NMI 是上升沿触发的非屏蔽中断请求输入信号，不能通过软件进行屏蔽。只要该引脚上出现一个从低到高的电脉冲，就能使 CPU 现行指令结束，立刻进入中断响应，转去执行 NMI 中断处理。

**表 2-6　$S_4$、$S_3$ 编码与段寄存器的对应关系**

| $S_4$ | $S_3$ | 性能 | 对应段寄存器 |
| :--- | :--- | :--- | :--- |
| 0 | 0 | 数据交替 | 使用附加段寄存器 ES |
| 0 | 1 | 堆栈操作 | 使用堆栈段寄存器 SS |
| 1 | 0 | 代码 | 使用代码段寄存器 CS |
| 1 | 1 | 数据 | 使用数据段寄存器 DS |

INTR：可屏蔽中断请求，输入，高电平有效。INTR 是电平触发，CPU 在每条指令的最后一个时钟周期对 INTR 信号线采样，若发现 INTR 引脚信号为高电平，同时 CPU 内部中断允许标志 IF=1 时，CPU 就进入中断响应周期，CPU 响应中断时，暂停正在执行的主程序，从数据总线读取中断类型码，根据中断类型码从存储器的中断向量表里查找相应的中断服务程序的入口地址，转到该地址去执行中断服务程序，在中断结束后，再返回主程序的断点处继续执行程序。
若中断允许标志位 IF=0，即使 INTR 引脚信号为高电平，CPU 对外界送来的此中断请求信号也不予理睬。这可以通过软件的方法使 IF=0，以实现屏蔽中断请求 INTR。
CLK：时钟信号，输入。CLK 为 CPU 和总线控制器提供定时基准。
RESET：复位信号，输入，高电平有效。当 RESET 为高电平时，CPU 将停止正在进行的操作，把内部标志寄存器 FR、段寄存器 DS、SS、ES 以及指令指针 IP 置 0，使指令队列清空，如表 2-7 所示。为保证对 CPU 进行可靠的初始化，RESET 高电平信号至少保持 4 个时钟周期，即当主频为 4.77MHz 时，复位时间必须大于 $50\mu s$。RESET 恢复低电平后，CPU 就从 FFFF0H 单元开始启动。

**表 2-7　初始化操作**

| 复位操作影响部分 | 内 容 |
| :--- | :--- |
| 标志寄存器 FR | 0000H |
| 指令指针 IP | 0000H |
| 代码段寄存器 CS | FFFFH |
| 数据段寄存器 DS | 0000H |
| 堆栈段寄存器 SS | 0000H |
| 附加段寄存器 ES | 0000H |
| 指令队列 | 空 |

READY：准备就绪信号，输入，高电平有效。用来使 CPU 和慢速存储器或 I/O 设备之间速度实现匹配，完成数据传送。当被访问部件无法在 CPU 规定的时间内完成数据传输时，应使 READY 信号处于低电平，这时 CPU 进入等待状态，插入一个或几个等待周期 $T_w$ 来延长总线周期。当被访问的部件可以完成数据传送时，READY 输入高电平，CPU 继续运行。慢速存储器或外部设备送出的 READY 信号必须先经过 8284 时钟发生器，与时钟脉冲同步后，再送入 CPU 的 READY 引脚，可满足定时要求，避免动作失误。
TEST：测试信号，输入，低电平有效。当执行 WAIT 指令时，每隔 5 个时钟周期，CPU 就对 TEST 信号进行采样，若 TEST 为高电平，就使 CPU 重复执行 WAIT 指令并处于等待状态，一直等到它变为低电平时，CPU 才脱离等待状态，继续执行下一条指令。通常，TEST 引脚和 WAIT 指令结合使用，主要用于多处理器环境。例如，在具有协处理器 8087 的系统中，将其 BUSY 引脚接 TEST。每当主 CPU 执行 WAIT 指令时，反复采样 TEST 信号，直到协处理器的 BUSY 信号使 TEST 变为低电平为止。
INTA：中断响应信号，输出，三态，低电平有效，是 CPU 响应外部中断请求后，发给请求中断的设备回答信号。在每个中断响应周期 $T_2$、$T_3$、$T_w$ 期间，它变为有效低电平，通知中断源送出中断矢量码。所以外部设备可以用此信号作为读取矢量码的选通信号。
ALE：地址锁存允许信号，输出，高电平有效。当地址/数据总线分时传输地址信息时，ALE 用作把地址信号锁存入 8282/8283 或 74LS373 锁存器的锁存控制信号。在总线周期 $T_1$ 期间，ALE 出现高电平，而 ALE 的下跳沿将出现在地址/数据总线上的地址信息锁存入锁存器。在 DMA 方式中，ALE 引脚不能浮空。
DEN：数据允许，输出，三态，低电平有效。在单 CPU 系统中，如果用 8286/8287 作为数据总线的双向驱动器，这时要用 DEN 作为驱动器的选通信号。在每个存储器或 I/O 访问周期以及中断响应周期，DEN 变为有效低电平。在 DMA 方式时，它处于浮空状态。
DT/R：数据发送/接收控制，输出，三态。在单 CPU 系统中，如果用 8286/8287 作为数据总线的双向驱动，这时要用 DT/R 来控制 8286/8287 的数据传送方向。DT/R=1 时，CPU 发送数据；DT/R=0 时，CPU 接收数据。在 DMA 方式时，它处于浮空状态。
IO/M：外设/内存访问控制，输出，三态。输出高电平时，表示总线周期为 I/O 访问周期；输出低电平时，表示总线周期为存储器访问周期。在 DMA 方式时，它为浮空状态。
WR：写信号，输出，三态，低电平有效。WR 信号有效时，表示 CPU 正在执行写存储器或写 I/O 口操作。由 IO/M 信号的状态决定数据是写入存储器还是写入 I/O 端口。在 DMA 方式时，它处于浮空状态。
RD：读信号，输出，三态，低电平有效。RD 信号有效时，表示 CPU 在读存储器或 I/O 口的数据。IO/M 为低电平时，CPU 读存储器。在 DMA 方式时，它处于浮空状态。
HOLD：保持请求信号，输入，高电平有效。当 DMA 操作或外部处理器要求通过总线传送数据时，HOLD 信号为高电平，表示外界请求现有主 CPU 让出对总线的控制权。
HLDA：保持响应信号，输出，高电平有效。当 CPU 同意让出总线控制权时，输出 HLDA 高电平信号，通知外界可以使用总线。同时现有主 CPU 所有“三态”线都进入浮空状态；当 HOLD 变为低电平时，现有主 CPU 把 HLDA 变为低电平，此时将重新获得总线控制权。
$\overline{SS_0}$：状态信号，输出，三态。它在逻辑上等同于多 CPU 系统中的 $\overline{S_0}$。$\overline{SS_0}$ 只在单 CPU 系统中。$\overline{SS_0}$、IO/M、DT/R 组合表示的单 CPU 系统总线周期状态对应关系如表 2-8 所示。在多 CPU 系统下，$\overline{SS_0}$ 总是输出高电平。

| :--- | :--- | :--- | :--- |
| 1 | 0 | 0 | 中断响应 |
| 1 | 0 | 1 | 读 I/O 口 |
| 1 | 1 | 0 | 写 I/O 口 |
| 1 | 1 | 1 | 暂停 |
| 0 | 0 | 0 | 取指令 |
| 0 | 0 | 1 | 读存储器 |
| 0 | 1 | 0 | 写存储器 |

MN/MX：单 CPU/多 CPU 方式控制，输入。该引脚的电平，决定了系统是构成单 CPU（最小模式）还是多 CPU（最大模式）系统。当 MN/MX=$V_{CC}$ 时，按单 CPU（最小模式）工作，这时 8088 的 $24 \sim 31$ 引脚功能如上所述；若 MN/MX=GND，系统按多 CPU（最大模式）工作，8088 的 $24 \sim 31$ 引脚定义如图 2-38 括号内所示。
下面的引脚在单 CPU 和多 CPU 系统下具有不同的功能。IBM PC/XT 具有协处理器 8087，硬件扩展功能强，所以通常设计 8088 工作于多 CPU（最大模式）方式下，下面主要介绍图 2-38 括号内引脚的功能。
$\overline{S_2}$、$\overline{S_1}$、$\overline{S_0}$：总线周期状态标志，输出、三态，低电平有效。这三条引脚状态信号的不同组合，表示 CPU 总线周期的操作类型。这些信号送入 8288 总线控制器对应输入端，8288 利用这些信号的不同组合，产生访问存储器或 I/O 的控制信号或中断响应信号。这些信号在每个总线周期 $T_1$ 有效，而在 $T_3$ 或 $T_w$ 后半周期，且 READY 为高电平时，则返回无效状态，表示总线周期结束。$\overline{S_2}$、$\overline{S_1}$、$\overline{S_0}$ 的编码和总线周期的关系如表 2-9 所示。在 DMA 方式时，它们进入高阻状态。

**表 2-9　总线周期状态标志**

| $\overline{S_2}$ | $\overline{S_1}$ | $\overline{S_0}$ | 操作类型 |
| :--- | :--- | :--- | :--- |
| 0 | 0 | 0 | 中断响应 |
| 0 | 0 | 1 | 读 I/O 口 |
| 0 | 1 | 0 | 写 I/O 口 |
| 0 | 1 | 1 | 暂停 |
| 1 | 0 | 0 | 取指令操作码 |
| 1 | 0 | 1 | 读存储器 |
| 1 | 1 | 0 | 写存储器 |
| 1 | 1 | 1 | 无效状态 |

$\overline{RQ}/\overline{GT_0}$、$\overline{RQ}/\overline{GT_1}$：请求/允许控制信号，双向，三态，低电平有效。这两个引脚供外部主控设备用来请求获得总线控制权。首先由外部主控设备向 8088 输入请求使用总线的信号（保持信号 HOLD），然后在同一条线上，8088 输出允许外部主控设备使用总线的回答信号（保持响应 HLDA）两个控制线可同时接两个外部主控设备，但 $\overline{RQ}/\overline{GT_0}$ 的优先权高于 $\overline{RQ}/\overline{GT_1}$。若两者同时向 8088 申请，先响应 $\overline{RQ}/\overline{GT_0}$ 上的申请。
请求/允许控制信号工作时序如图 2-39 所示。

![[计算机系统/微机原理与接口技术B/附件/第2章/Pasted image 20260719155616.png]]
*图 2-39　请求/允许控制信号工作时序*

1. 由其他主控制设备在 $\overline{RQ}/\overline{GT}$ 引脚向 8088 发出一个时钟周期宽的负脉冲，表示请求占用总线。
2. 在当前总线周期 $T_4$ 或下个总线周期 $T_1$ 期间，8088 输出一个宽度为一个时钟周期的负脉冲，通知主控设备，表示 8088 同意让出总线，并从下一个时钟周期开始进入“同意占有”总线状态。在此期间 8088 所有三态线（除了 $\overline{RQ}/\overline{GT}$ 线）均置成浮空状态。
3. 主控设备申请总线控制权结束，送出一个时钟周期宽的负脉冲给 8088，表示请求占用总线周期已经结束，8088 可以在下一个时钟脉冲上升沿收回总线控制权。

LOCK：封锁信号，输出，三态，低电平有效，用来封锁外部主控设备的请求。当 8088 的 LOCK 信号为低电平时，外部主控设备不能占用总线。该信号是用指令在程序中设置的。如果一条指令加上指令前缀 LOCK，则 8088 执行这条指令时，LOCK 引脚为低电平，并保持到指令结束，防止该指令执行过程中被打断。在 DMA 工作方式时，LOCK 为浮空状态。
$QS_1$、$QS_0$：指令队列状态，输出，高电平有效。$QS_1$、$QS_0$ 的不同编码状态反映了 CPU 内部当前的指令队列状态，以便外部主控设备对 8088 进行跟踪。$QS_1$、$QS_0$ 不同编码和指令队列的状态如表 2-10 所示。

**表 2-10　指令队列状态**

| $QS_1$ | $QS_0$ | 意 义 |
| :--- | :--- | :--- |
| 0 | 0 | 无操作 |
| 0 | 1 | 取指令队列中第一操作码 |
| 1 | 0 | 队列空 |
| 1 | 1 | 取指令队列中后续字节 |

8086 与 8088 引脚有以下区别：

1. 8086 有 16 条地址/数据引脚 $AD_{15} \sim AD_0$。这是因为 8086 的数据总线是 16 位，所以 16 条引脚地址与数据分时复用；而 8088 只有 $AD_7 \sim AD_0$ 分时复用。
2. 8086 的 34 引脚为 $\overline{BHE}/S_7$（8088 为 $\overline{SS_0}$），是高 8 位数据总线的允许和状态信息复用引脚。$\overline{BHE}$ 可看成一条附加的地址总线，用来访问存储器的高字节，而 $A_0$ 用来访问存储器的低字节。所以 $\overline{BHE}$ 通常作为接在高 8 位数据总线上的设备片选信号，而 $A_0$ 作为接在低 8 位数据总线上的设备片选信号。$\overline{BHE}$ 和 $A_0$ 的状态与数据总线传送的数据之间的关系如表 2-11 所示。

**表 2-11　$\overline{BHE}$ 和 $A_0$ 编码与数据总线数据传送的关系**

| $\overline{BHE}$ | $A_0$ | 数据传送状态 |
| :--- | :--- | :--- |
| 0 | 0 | 传送 16 位 $D_{15} \sim D_0$ |
| 0 | 1 | 传送高 8 位 $D_{15} \sim D_8$ |
| 1 | 0 | 传送低 8 位 $D_7 \sim D_0$ |
| 1 | 1 | 无操作 |

在总线周期 $T_1$ 期间 $\overline{BHE}$ 输出有效信息，而在 $T_2$、$T_3$、$T_w$、$T_4$ 期间，这条引脚输出 $S_7$ 状态信息，$S_7$ 低电平有效。在 DMA 工作方式，它为浮空状态。

3. 8086 的 28 引脚为 M/IO，存储器/输入输出信号，输出，三态。当 M/IO=1 时，表示访问存储器；当 M/IO=0 时，表示访问 I/O 端口，与 8088 第 28 引脚（IO/M）逻辑定义正好相反。

### 2.3.2 80386 CPU 主要引脚功能
80386 通常有两个型号：80386DX 和 80386SX。80386DX 采用 132 引脚的 PGA（引脚栅格阵列）封装，具有 32 位数据和地址总线；80386SX 是一个缩减总线型号的 80386，具有 16 位数据总线和 24 位地址总线。下面以 80386DX 为例介绍 80386 的主要引脚功能。

![[计算机系统/微机原理与接口技术B/附件/第2章/Pasted image 20260719155632.png]]
*图 2-40　80386DX 的引脚分配图*

80386DX 的引脚分配图如图 2-40 所示。需要说明的是，为便于书写，从 80386 开始，其产品数据手册中把低电平或负脉冲有效的表示方法由习惯上的加上划线改成后跟“#”，如通常的 M/IO 信号变成了 $M/IO\#$，$\overline{BE_3}$ 等同于 $BE_3\#$。
$A_{31} \sim A_2$：地址总线，用来访问 80386 存储系统中的 1G×32 位存储单元。$A_1$、$A_0$ 被编码为总线允许信号（$\overline{BE_3} \sim \overline{BE_0}$），以便选择 32 位存储单元中 4 字节的任何一个或几个。
$D_{31} \sim D_0$：数据总线，用于在 CPU 和存储器及 I/O 之间传送数据。
$\overline{BE_3} \sim \overline{BE_0}$：存储体使能信号，选择字节、字或双字数据。这些信号由 CPU 内部根据地址线 $A_1$ 和 $A_0$ 产生。
ADS：地址数据选通引脚。当 80386 发出有效的存储器地址或 I/O 地址时，产生该信号。
RESET：复位引脚。用来初始化 80386，使其从内存 FFFFFFF0H 处开始执行软件，80386 被复位为实模式。
$BS_{16}$：16 位总线宽输入引脚。用来选择 32 位或 16 位数据总线。
M/IO：存储器及 I/O 选择引脚，为 1 时选择存储器设备，为 0 时选择 I/O 设备。在 I/O 操作期间，地址总线包含了 16 位的地址，即 $A_{15} \sim A_2$ 及低 2 位。
W/R：写/读引脚。W/R 为 1，表明当前总线周期是写周期；为 0，则表明是读周期。
CLK2：两倍频时钟引脚，由 2 倍于 80386 工作频率的时钟信号驱动。
INTR：中断请求信号。外部电路使用该信号请求中断。
NMI：非屏蔽中断输入引脚。请求非屏蔽中断，与早期 CPU 的作用一样。
READY：就绪引脚，该引脚控制为延长存储器或 I/O 访问时间而插入的等待状态的个数。
LOCK：总线锁定引脚。当指令带有 LOCK 前缀时，该引脚变为逻辑 0，经常用于 DMA 访问期间。
D/C：数据/控制输出引脚。D/C 为 1 时，表示数据总线含有来自或送到存储器或 I/O 的数据；为 0，则表明 CPU 正在执行中断响应或已停机。
NA：下一地址信号。该信号使 80386 在当前总线周期输出下一条指令或数据的地址，它通常用在流水线操作中。
HOLD：总线保持引脚。用来请求 DMA 操作。
HLDA：总线保持响应引脚。用来表明 80386 当前处于总线保持状态。
PEREQ：协处理器请求信号。该信号要求 80386 放弃总线控制并直接连到 80387 算术协处理器上。
BUSY：忙输入信号。由 WAIT 和 FWAIT 指令使用，等待协处理器变为空闲。

### 2.3.3 Pentium CPU 主要引脚功能
Pentium CPU 有两个版本：全功能型 Pentium 和称为 Pentium OverDrive 的 P24T 型。全功能型 Pentium 为 237 引脚 PGA 封装，数据总线 64 位，地址总线 32 位。P24T 型具有 32 位数据总线，可以方便地插到有 P24T 插座的 80486 机器中。早期型号的 Pentium 需要 +5 V 电源，电流较大，导致功耗很大；后期型号的 Pentium，主频为 90 MHz 以上，采用 3.3 V 电源供电，以减少电流消耗，同时配有良好通风效果的散热器来保证芯片不致过热。Pentium 微处理器引脚示意如图 2-41 所示。

Pentium 主要引脚的功能说明如下。
$A_{31} \sim A_3$：地址总线引脚，用于寻址 Pentium 存储系统中任何 512K×64 位存储空间，$A_2$、$A_1$、$A_0$ 在总线允许（$\overline{BE_7} \sim \overline{BE_0}$）中被编码用于选择 64 位宽存储单元中的任意或全部 8 字节。
ADS：地址数据选通信号。当 Pentium CPU 发出一个有效的存储器地址或 I/O 地址时，该信号变为有效。
A20：地址 $A_{20}$ 屏蔽输入引脚，用于在实模式中通知 Pentium CPU 进行地址扩展。
$\overline{BE_7} \sim \overline{BE_0}$：存储器体允许信号，用于选择访问单字节、字、双字、四字数据。这些信号在 CPU 内由地址 $A_2$、$A_1$、$A_0$ 产生。
$D_{63} \sim D_0$：数据总线信号。在 CPU、内存和 I/O 系统间进行字节、字、双字、四字数据的传输。
RESET：复位信号。初始化 Pentium CPU，使其从内存 FFFFFFF0H 处开始执行，Pentium CPU 被复位为实模式，最高位的 12 条地址线保持逻辑 1（FFFFH），直至执行一个远程调用或远跳转，这样可使它与早期的 CPU 兼容。硬件复位后 Pentium CPU 的状态如表 2-12 所示。

BRDY：猝发就绪信号。该信号通知 Pentium CPU 外部系统已从数据总线连接中取得数据。可用于向 Pentium CPU 时序中插入等待状态。
BREQ：总线请求输出信号，指示 Pentium CPU 已产生一个总线请求。
CLK：时钟引脚。由具有当前 Pentium CPU 工作频率的时钟信号驱动。
D/C：数据/控制信号。该信号为逻辑 1 时表明数据总线上含有来自存储器或 I/O（或写往存储器或 I/O）的数据，为逻辑 0 时则表明 CPU 处于停机状态，或者正在执行一个中断请求。
FLUSH：清 Cache 输入信号。使 Cache 清除所有写回行并使内部 Cache 无效。如果在进行复位时该信号为 0，则 Pentium CPU 进入 Cache 测试模式。
CACHE：Cache 输出信号。指示当前 Pentium CPU 周期可对数据进行缓存。
HIT：命中信号。表明在查询方式中内部 Cache 包含了合法数据。
HITM：命中修改信号。表明在查询周期中发现了一个修改过的 Cache 行，此输出信号用于在已修改过的 Cache 行写回到存储器之前禁止其他单元访问数据。
WB/WT：写回/写直达信号，为 Pentium CPU 数据 Cache 选择相应操作。
KEN：Cache 使能输入信号。允许使用内部高速缓存。
HOLD：保持信号。请求一个 DMA 操作。
HLDA：保持响应信号。指示 Pentium CPU 当前处于保持状态。
INTR：中断请求信号。外部电路用该信号进行中断请求。
INV：无效输入信号。决定一个查询后的 Cache 行状态。
IU：U 管道指令完成输出信号。表明 U 管道中的指令已完成。
IV：V 管道指令完成输出信号。表明 V 管道中的指令已完成。
LOCK：锁定信号。指令带有 LOCK 前缀时该信号变为逻辑 0。该信号通常用在 DMA 访问中。
M/IO：存储器及 I/O 选择引脚。M/IO 为 1 时选择存储器设备，为 0 时选择 I/O 设备。在 I/O 操作期间，地址总线包含了 16 位的地址，即 $A_{15} \sim A_3$ 及低 3 位。
W/R：写/读引脚。W/R 为 1 时表明当前总线周期是写周期，为 0 时表明是读周期。
NA：下一地址信号。表明外部存储器系统已准备好接收新的总线周期。
NMI：非屏蔽中断信号。请求一个非屏蔽中断。
SMI：系统管理中断输入信号。使 Pentium CPU 进入系统管理运行模式。
SMIACT：系统管理中断激活输出信号。表明 Pentium CPU 正在工作在系统管理模式。
TCK：可测试性时钟输入信号。根据 IEEE 1149.1 边界扫描接口选择时钟操作。
TDI：测试数据输入信号。测试由 TCK 信号扫入 Pentium CPU 的数据。
TDO：测试数据输出信号。获得由 TCK 移出的 Pentium CPU 测试数据和指令。
TMS：测试方式选择输入信号。在测试模式中控制 Pentium CPU 操作。
TRST：测试复位输入信号。使测试模式被复位。
