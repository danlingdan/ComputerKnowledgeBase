## 2.2.4 Pentium CPU 内部结构

1. Pentium CPU 内部结构

Intel 公司于 1989 年 4 月推出新型 32 位处理器 80486，与 80386 相比，它的软件、硬件体系结构更先进，将浮点数学协处理器与指令、数据高速缓存集成在芯片上，这两个重要的改进使微机运行速度大大提高。80486 虽然在寻址方式、存储管理、数据类型等方面与 80386 相比的改变很少，但在相同主频下其处理速度比 80386 快 $2 \sim 4$ 倍。

![[计算机系统/微机原理与接口技术B/附件/第2章/Pasted image 20260719155506.png]]
*图 2-33　Pentium CPU 内部结构*

继 80486 后，1993 年 Intel 公司又推出新一代名为 Pentium 的微处理器（P5）。图 2-33 为 Pentium CPU 内部结构。Pentium 的中文名字为奔腾，奔腾是由希腊语 Pente（意思为 5）演变而来的，该处理器按原来代号顺序排列应取名为 80586，但为了取得商标注册，防止其他公司的兼容产品再以相同的名称命名，Intel 公司按照美国有关法律将这种新一代微处理器取名为 Pentium，并注册专有。
Pentium CPU 采用全新设计，具有 64 位数据线和 32 位地址线，与 80486 相比内部结构也有很大改进，但是依然保持了和 8086/8088、80286、80386、80486 的兼容性，在相同的工作模式下可以执行所有的 80x86 程序。片内存储管理单元（MMU）也与 386 和 486 兼容，可以在实地址方式引导下转入保护模式和虚拟 86 模式，其指令集包括了 80486 的所有指令，并增加了新的指令。
Pentium CPU 主要由执行单元、指令 Cache、数据 Cache、指令预取单元、指令译码单元、地址转换与管理单元、总线单元以及控制器等组成，其中核心是执行单元（又称为运算器），它的任务是高速完成各种算术和逻辑运算，其内部包括两个整数算术逻辑运算单元（ALU）和一个浮点运算器，分别用来执行整数和实数的各种运算。
除执行单元外，其他单元都可认为是控制单元。其中主要包括指令预取单元、指令译码单元、地址转换与管理单元以及总线单元。总线单元为 Pentium CPU 提供了与周围其他硬件环境的接口，如从指令预取单元接收预取指令的请求，从执行单元接收传送数据的请求等，当有多个单元同时发出请求时，总线单元将进行协调，提供及时服务。总线单元的基本任务是使 Pentium CPU 能有效使用计算机中的总线。

2. Pentium CPU 的新特点

除了具有与 80x86 系列 CPU 完全兼容的特点，在体系结构上，Pentium CPU 还有如下一些新特点。

1. Pentium CPU 片内高速缓存采用了分离式结构。Cache 分为两个成组相联的 8 KB 指令 Cache（用来存放程序代码）和 8 KB 的数据 Cache（用来存放数据），这种将片内高速缓存分开的做法使各自 Cache 能够加快速度，减少等待及搬移数据的次数和时间，从而提高了整体性能。
2. Pentium CPU 采用了 RISC 技术。Pentium CPU 虽然属于 CISC 处理器，但在执行单元的设计中采用了较多的 RISC 技术，如具有超标量指令流水线功能。
3. Pentium CPU 具有高性能的浮点运算部件。其浮点运算部件完全重新设计，根据测试，其 FPU 速度是 80486 的 2 倍，其浮点操作已高度流水线化，并与整数流水线相结合。
4. Pentium CPU 具有分支指令预测功能。
5. 数据总线位宽增加。Pentium CPU 内部数据总线与 80486 一样都是 32 位（内部供程序控制用的寄存器组有 16 个，其中 32 位通用寄存器 10 个，32 位状态和控制标志寄存器 2 个，16 位段寄存器 6 个，另外还有更多的寄存器不再用来控制程序，而提供给 CPU 在操作过程中使用，这类寄存器有几十个），但 CPU 与内存进行数据交换的外部数据总线为 64 位，在一个总线周期内将数据传输量增加了一倍。另外，Pentium CPU 还支持多种类型的总线周期，在突发方式下，可以在一个总线周期内读入 256 B 的数据，其 64 位数据总线与内存数据交换的速率高达 528 MB/s，为 80486DX-50 的 3 倍还多。
6. 常用指令固化。在 Pentium CPU 中，把一些常用指令（如 MOV、INC、DEC、PUSH 等）改用硬件实现，不再使用微码操作，使指令的运行速度进一步加快。
7. 采用系统管理模式（SMM）。其最显著的应用就是电源管理，可以使 CPU 和系统外部部件在暂不工作时，休眠一定时间，然后在按下一键时唤醒它们，使之继续工作，通过使用管脚信号，SMM 甚至能完全控制整个系统，包括输入/输出和 RAM。
3. Pentium CPU 的内部寄存器

Pentium CPU 的内部寄存器按功能分为 4 类：基本寄存器、系统级寄存器、调试和模型特定寄存器、浮点寄存器。
### 1. 基本寄存器
基本寄存器包括通用寄存器、指令指针寄存器、标志寄存器和段寄存器，这些寄存器均与 80386 寄存器基本相同，见图 2-10。唯一有区别的是标志寄存器，如图 2-34 所示，Pentium CPU 的标志寄存器比 80386 增加了 4 个标志位。

![[计算机系统/微机原理与接口技术B/附件/第2章/Pasted image 20260719155519.png]]
*图 2-10　基本寄存器示意图*

1. AC（Alignment Check）：对准检查标志。$AC=1$，且 $CR_0$ 中的 $AM=1$ 时，进行字、双字或四字的对准检查，如发现要访问的操作数未按边界对齐，会产生异常中断。
2. VIF（Virtual Interrupt Flag）：虚拟中断标志。保护模式下中断允许标志位的副本。
3. VIP（Virtual Interrupt Pending）：虚拟中断挂起标志。用于在保护模式下提供中断信息，在多任务环境下，为操作系统提供虚拟中断标志和中断挂起信息。
4. ID（Identification）：标识标志。用于指示 CPU 对 CPUID 指令的支持状态。CPUID 指令为系统提供 Pentium CPU 的信息，如型号及制造商。

### 2. 系统级寄存器
系统级寄存器包括 4 个系统地址寄存器（与 80386 完全相同，见图 2-15）和 5 个控制寄存器 $CR_0 \sim CR_4$。图 2-35 给出了 Pentium CPU 的控制寄存器。

![[计算机系统/微机原理与接口技术B/附件/第2章/Pasted image 20260719155528.png]]
*图 2-15　系统级寄存器示意图*

$CR_0$ 定义了 11 个控制位，比 80386 多定义了 NE、WP、AM、NW、CD 这 5 位，其含义如下。

1. NE（Numeric Error）：数值异常控制位。控制是由中断向量 16 还是由外部中断来处理浮点数值异常。$NE=1$，任何未屏蔽的浮点异常将产生软件中断 16；$NE=0$，由外部中断来处理该异常。
2. WP（Write Protect）：写保护位。$WP=1$，禁止特权级程序对只读页面的写操作；$WP=0$，允许只读页面由特权级写入。
3. AM（Alignment Mark）：对准屏蔽位。控制标志寄存器中对准检查位（AC）是否允许对准检查。$AM=1$，允许 AC 位对准检查；$AM=0$，则禁止 AC 位对准检查。
4. NW（Not Write through）：不通写控制位。选择片内 Cache 的操作模式。$NW=1$，禁止通写，写命中时不修改内存；$NW=0$，允许通写。
5. CD（Cache Disable）：Cache 禁止位。控制是否允许向未命中的 Cache 填充新数据。$CD=1$，当 Cache 未命中时，禁止填充 Cache；$CD=0$，当 Cache 未命中时，也允许填充 Cache。

$CR_1$：未定义的控制寄存器，供后期的微处理器使用。
$CR_2$：页故障线性地址寄存器，保存最后出现页故障的全 32 位线性地址。
$CR_3$：页目录基地址寄存器，保存页目录表的物理基地址。其中另外定义了 2 位，用于控制高速缓存，其含义如下。

1. PWT（Page Write Through）：页写直达位。控制外部 Cache 写操作下的页直达方式。
2. PCD（Page Cache Disable）：页高速缓存禁止位。$PCD=1$，表示禁止片内 Cache 存储当前页。

$CR_4$ 中定义了 6 位，各位的含义如下。

1. VME（Virtual Mode Extension）：虚拟方式扩展位。$VME=1$，表示允许虚拟 8086 方式的扩展。
2. PVI（Protected mode Virtual Interrupt）：保护模式虚拟中断位。$PVI=1$，表示在保护模式下允许对虚拟中断标志的支持。
3. TSD（Time Stamp Disable）：时间标记计数器禁止位。$TSD=1$，且当前特权级不为 0 时，禁止读时间标记计数器指令 RDTSC；否则 RDTSC 将在所有特权级上执行。
4. DE（Debugging Extension）：调试扩展位。$DE=1$，允许 I/O 断点调试扩展。
5. PSE（Page Size Extension）：页尺寸扩展位。$PSE=1$，使能页面大小扩充为 4 MB。
6. MCE（Machine Check Enable）：机器检查允许位。$MCE=1$，允许机器检查异常。

### 3. 调试和模型特定寄存器

![[计算机系统/微机原理与接口技术B/附件/第2章/Pasted image 20260719155536.png]]
*图 2-36　调试和模型特定寄存器示意图*

如图 2-36 所示，Pentium CPU 中设置了 8 个调试寄存器 $DR_0 \sim DR_7$，它们为程序调试提供了硬件支持。其中 $DR_0 \sim DR_3$ 为线性断点寄存器，可保存 4 个断点地址，从而方便程序设计人员按照调试意图组合指令的执行。$DR_4$、$DR_5$ 定义为 $DR_6$、$DR_7$ 的别名。$DR_6$ 用于保存断点状态，$DR_7$ 用于控制断点设置。
专用模式寄存器 MSR（Model Special Register）是 Pentium CPU 特有的，用于跟踪和检查性能、测试和检查机器错误，它替换了 80386、80486 中的测试寄存器。Pentium CPU 使用 RDMSR 和 WRMSR 指令读/写专用模式寄存器，这两条指令都使用 ECX 寄存器给 CPU 传递寄存器号，使用 EDX:EAX 寄存器为 64 位宽读/写服务。Pentium CPU 专用模式寄存器及其内容参看表 2-5 所示。

### 4. 浮点寄存器
Pentium CPU 内嵌了算术运算协处理器，其浮点寄存器同 80x87，参见 2.2.3 节。
注意，Pentium CPU 的指令系统增加了 57 条新的多媒体扩展 MMX（MultiMedia Extensions）指令，专为动态视频、图像处理、音频合成等多媒体应用而设计。MMX 寄存器是利用别名（Aliasing）技术共享使用的浮点数据寄存器，即 MMX 寄存器（$MM_0 \sim MM_7$）与浮点数据寄存器是相同的，如图 2-37 所示。

![[计算机系统/微机原理与接口技术B/附件/第2章/Pasted image 20260719155546.png]]
*图 2-37　浮点寄存器示意图*

注意，MMX 寄存器组为 64 位宽，并使用浮点数据寄存器组的最右边 64 位。MMX 体系结构引入新的紧缩数据类型是：8 个紧缩的、连续的 8 位的字节，4 个紧缩的、连续的 16 位的字，以及 2 个紧缩的、连续的 32 位的双字。MMX 寄存器与 64 位宽内存有相同的格式，它有两种数据存取方式，即 64 位存取方式和 32 位存取方式。

## 2.2.5 Pentium 系列其他微处理器

1. Pentium Pro

Pentium Pro 仍是 32 位微处理器，在 Pentium 的基础上增加的主要功能如下：
- 重新设计了微结构，是第一个采用 P6 结构的处理器。
- 集成了两级高速缓存。
- 采用了三路超标量结构和 14 级流水线深度，每个时钟周期可执行三条指令。
- 综合运用了 RISC 技术和 CISC 技术，采用了乱序执行和推测执行。

2. Pentium II

Pentium II 在 Pentium Pro 的基础上扩展了 MMX 技术，其增加的主要功能如下：
- 采用多媒体增强技术，增加了 57 条 MMX 指令。
- 采用动态预测技术（也称随机推测执行技术）。
- 增加了 70 条 SSE 数据流单指令多数据扩展技术。
- 双独立总线技术。
Pentium II 的入门级处理器是减少了 L2（甚至取消 L2）的 Celeron。而 Pentium II 的高阶处理器是针对服务器设计的 Pentium II Xeon。

3. Pentium III

Pentium III 在 Pentium II 的基础上增加了 128 位的浮点寄存器，能同时处理 4 个单精度浮点数。增加了 70 条附加浮点多媒体指令，克服了不能同时处理 MMX 数据和浮点数据的缺陷，有效地增强了静止图像、视频处理的能力。
与 Pentium II 相同，Pentium III 也有低阶的 Celeron 版本和高阶的 Xeon 版本。

4. Pentium 4

Pentium 4 是第一个基于 Intel NetBurst 结构的处理器芯片，其主要特点如下：
- 采用超流水技术，流水线从 14 级提高到 20 级，有效地减少了执行时间。
- 采用快速执行引擎，为执行单元提供执行指令，防止执行单元停顿。
- 使用了包含 144 条指令的 SSE2 指令集，与 SSE 相比，使可操作的数据量加倍，从而提高了 CPU 执行指令的效率。
Pentium 4 同样有低阶的 Celeron 版本和高阶的 Xeon 版本。当其主频升为 4 GHz 时，遇到了无法解决的高功耗问题，这促使了功耗优化微处理器 Pentium M 的产生。
