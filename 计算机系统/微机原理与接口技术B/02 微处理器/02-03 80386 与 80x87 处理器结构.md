## 2.2.2 80386 CPU 内部结构
从 8086 到 80x86/Pentium 微处理器，内部结构虽有不少变化，究其实质仍属 8086 处理器体系，即内部结构各单元均采用并行处理技术，也就是说，微处理器内部多个处理单元可分别进行同步、独立并行操作，以实现高效流水线工作，避免串行处理，最大限度地发挥了处理器性能。本节至 2.2.4 节将重点说明 80386、Pentium CPU 的结构特点，并简要介绍数学协处理器 80x87。
8086/8088 后是 16 位数据线、24 位地址线的 80286 CPU，其内部结构由 4 部分组成：执行单元 EU、地址单元 AU、总线单元 BU 和指令单元 IU，采用流水线工作方式，并行操作，速度比 8086 CPU 快 5 倍。80286 CPU 支持两种工作模式，即实地址实模式和虚地址保护模式。当 80286 CPU 工作在保护模式时，能够支持多任务，处理器提供虚拟内存管理和多任务的硬件控制。在保护方式下 80286 CPU 可产生 24 位物理地址，使用 16MB 内存，并产生 1024MB（1GB）虚拟内存。
80386 CPU 内部、外部的数据总线、地址总线均为 32 位，直接寻址能力达 4 GB，虚地址保护模式下的存储空间可达 64 TB，图 2-9 为 80386 CPU 内部结构。

*图 2-9　80386 CPU 内部结构*

80386 CPU 结构比 8086/8088、80286 复杂，有较多的并行处理单元，增强了 80386 CPU 的性能。依其职能共有 6 个处理单元：执行单元、分段单元、分页单元、总线单元、指令预取单元和译码单元。下面简要介绍各处理单元的功能。
### 1. 总线单元
总线单元是 80386 CPU 通往外部世界的接口。总线接口提供 32 位数据总线、32 位地址总线以及控制总线传输的一些信号，采取多路分解方法实现 8 位、16 位和 32 位数据传输。地址和数据的多路分解不但改进了性能，而且简化了硬件设计。
总线单元负责 CPU 内部各部件与存储器、输入/输出接口之间数据或指令的传输。CPU 内部的其他部件都能与总线单元直接通信，并将它们的总线请求传输给总线单元。在指令执行的不同阶段，指令、操作数以及存储器偏移地址都可从存储器取出送到 CPU 内部的相关部件。但当 CPU 内部多个部件同时请求使用总线时，为了使程序的执行不被延误，总线单元的请求优先控制器将优先响应数据的传输请求，只有不执行数据传输操作时，总线单元才可以满足预取指令的请求。从图 2-9 可以看到，在数据访问时，存储地址来源于分页单元；在代码访问时，存储地址由指令预取单元提供。
### 2. 指令预取单元
80386 指令代码的预取不再由 BIU 负责，而由一个独立的指令代码预取单元完成。指令预取单元由预取器和预取队列组成。预取器管理一个预取指令指针和段预取界限，从分段单元来的线性地址和分段界限分别送到这里。当总线单元不执行指令执行部分的总线周期时，若预取队列存在空单元或发生一次控制转移，预取器便通过分页单元向总线单元发出指令预取请求。分页单元将预取指令指针送出的线性地址变为物理地址，再由总线单元及系统总线从内存单元中预取出指令代码，放入预取队列中。80386 CPU 的预取队列可存放 16 字节的指令代码。进入预取队列的指令代码将被送至指令译码单元进行译码。预取器保持预取队列总是满的。
假如指令预取单元中队列是满的，而且执行单元也不要求存储器提供操作数，则总线接口单元不需要完成总线周期，处于空闲状态。
### 3. 指令译码单元
从图 2-9 可以看到，译码单元接受指令预取单元的指令队列的输出，读入的形式是机器代码指令，将其译码为微代码指令形式供执行单元使用。这些内部编码中包含了控制其他处理部件的各种控制信号，因而指令译码单元为指令的执行做好了准备。在 80386 CPU 的指令单元内，指令队列允许有三条指令被译码以供执行单元使用，进一步提升了 CPU 的性能。
### 4. 执行单元
执行单元由控制部件、数据处理部件和保护检测部件组成，其任务是将已译码指令队列中的内部编码变成按时间顺序排列的一系列控制信息，并发送给 CPU 的其他处理部件，以便完成一条指令的执行。具体地说，执行单元内包含算术/逻辑单元（ALU）、通用寄存器、专用乘法/除法器和移位器，还有控制 ROM（只读存储器）。控制 ROM 内包含微代码序列，它们是完成 80386 CPU 机器代码指令的微程序。执行单元从指令队列中读出已译码的指令并实现其操作。ALU 实现指令所要求的算术、逻辑和移位操作。当执行指令时，如果需要，段和页单元产生操作数据的地址并由总线接口单元完成读或写周期，从存储器或 I/O 设备中存取操作数。此外，ALU 采用特定硬件模块来完成乘法、除法、移位和循环移位等操作，这使指令完成上述操作时的性能得到很大改善。
执行部件中还设有一条附加的 32 位内部总线和专门的控制逻辑部件，并提供同时执行两条指令所需要的控制回路，可使每条访问存储器的指令与前一条指令并行执行。
### 5. 分段、分页单元
分段、分页单元为 80386 提供存储器管理和保护服务，负责地址产生、地址转换和对总线接口单元的段检查，因此进一步提高了 CPU 性能。80386 CPU 存储器管理的分段模型是用硬件完成高速地址计算、从逻辑到线性地址的转换和保护性检查。例如，在实模式下，执行单元要求分段单元将代码段（CS）寄存器的内容加上指令指针（IP）的数值，得到下一个要提取的指令地址，将 20 位的物理地址输出到地址总线，最后到达总线单元。
在保护模式下，分段单元完成逻辑地址到线性地址的转换，并在总线周期内完成多种保护性检查。分页单元实现 80386 存储器管理的分页模型，具有转换后备缓冲器，用于存放近期使用的页目录和页表。当分页工作被允许时，分段单元产生的线性地址被分页单元当作物理地址从存储器或 I/O 读取。分页单元将线性地址转换成存储器和 I/O 能接受的物理地址，并输入到总线接口单元。
80386 CPU 有 7 大类 32 个寄存器：通用寄存器、段寄存器、指令指针和标志寄存器、控制寄存器、系统地址寄存器、调试寄存器、测试寄存器，如图 2-10 所示。

### 1. 通用寄存器
80386 的 8 个 32 位通用寄存器 EAX、EBX、ECX、EDX、ESP、EBP、ESI、EDI 都是由 8086/8088 的相应 16 位寄存器 AX、BX、CX、DX、SI、DI、BP、SP 扩展而来。32 位通用寄存器的低 16 位可单独使用，而 AX、BX、CX、DX 的高、低 8 位可分别作为 8 位寄存器使用。
### 2. 段寄存器
80386 的段寄存器仍是 16 位，共有 6 个：CS、DS、ES、FS、GS、SS。其中 CS、DS、SS、ES 与 8086/8088 相同，而 FS、GS 是 80386 扩充的两个附加数据段寄存器。
在实模式下，80386 与 8086/8088 段寄存器的使用完全相同。
在保护模式下，存储单元的逻辑地址仍由两部分组成，即段基址和段内偏移地址。段内偏移地址为 32 位，段基址也是 32 位，但并不由段寄存器的值直接确定，而是以段寄存器（段选择符）为索引从描述表中找到所指向的描述符，再从描述符中找到段基址后与段内偏移地址一起确定物理地址。下面将分别介绍段描述符和段选择符。

![[计算机系统/微机原理与接口技术B/附件/第2章/Pasted image 20260719155108.png]]
*图 2-11　段描述符的格式*

段描述符的格式如图 2-11 所示。低 48 位是 80286 描述符，包括 16 位段界限（也称为段边界）、24 位段基地址和 8 位属性。80386 以上的 32 位 CPU 则在此基础上扩充了 8 位基地址、4 位界限和 4 位属性，即为 20 位段界限、32 位段基地址和 12 位属性。
段基地址为段起始地址，段界限给出段内最大偏移地址，其他位均为属性，具体含义如下。

1. G（Granularity）：粒度值（段界限所用单位）。$G=0$ 表示段界限给出的是段内最大偏移地址，以字节为单位，段界限最大为 FFFFFH，即段的最大长度为 1MB；$G=1$ 表示单位是页，每页大小为 4KB，段的最大长度为 4GB。
2. D：对于代码段、可扩展数据段和堆栈段，其含义有所不同。

- 可执行代码段，表示指令中默认操作数的长度。$D=1$ 表示操作数长度为 32 位或 8 位，$D=0$ 表示操作数长度为 16 位或 8 位。
- 可扩展数据段，表示段的上边界。$D=1$ 表示 4GB；$D=0$ 表示 64KB。
- 堆栈段，表示堆栈操作指令（PUSH 和 POP）隐含使用的寄存器是 ESP（$D=1$）还是 SP（$D=0$）。

3. P（Present）：存在位。$P=1$ 表示存在，即在实际内存中。
4. DPL（Descriptor Privilege Level）：描述符特权级（2 位），可以为 0（00）～3（11）级，0 级为最高级。
5. S（Segment）：段描述符类别。$S=1$ 表示代码段或数据段描述符；$S=0$ 表示系统描述符。
6. TYPE：段的类型（4 位）。说明访问、读/写控制及扩展特点等，其中类型值的最高位（可写为 E）表明描述的是数据段还是代码段。$E=0$ 表明段是数据段，相应描述符为数据段（包括堆栈段）描述符，数据段是不可执行的，但总是可读的；$E=1$ 表明是可执行段，即代码段，其总是不可写的。类型值的最低位（可写为 A）表明该段是否被访问过。$A=0$ 表示尚未被访问；$A=1$ 表示已访问过。段描述符具体类型说明如表 2-2 所示。

**表 2-2　段描述符类型说明**

| 类型值 | 含义 | 类型值 | 含义 |
| :--- | :--- | :--- | :--- |
| 0000 | 只读 | 1000 | 只执行 |
| 0001 | 只读，已访问 | 1001 | 只执行，已访问 |
| 0010 | 读/写 | 1010 | 执行/读 |
| 0011 | 读/写，已访问 | 1011 | 执行/读，已访问 |
| 0100 | 只读，向下扩展 | 1100 | 只执行，一致代码段 |
| 0101 | 只读，向下扩展，已访问 | 1101 | 只执行，一致代码段，已访问 |
| 0110 | 读/写，向下扩展 | 1110 | 执行/读，一致代码段 |
| 0111 | 读/写，向下扩展，已访问 | 1111 | 执行/读，一致代码段，已访问 |

例如，某可读/写的有效（存在的）数据段的描述符为 $0000\ \text{F}210\ 0000\ \text{FFFFH}$，对照图 2-11 格式，第 3 字节为 00H，即 $G=0$，$D=0$，表明该段为 16 位段，其段界限单位为字节，由所有段基地址字段可得其基地址为 $100000\text{H}$，段界限是 0FFFFH。
再如，某可执行的 32 位代码段的描述符为 $12\text{C}0\ 9834\ 5678\ 0010\text{H}$，同样对照图 2-11 格式，可得其基地址为 $12345678\text{H}$，以 4 KB 为单位的段界限值是 $10\text{H}$（$G=1$），描述符特权级 $DPL=0$（最高特权级）。
段描述符存放在两个系统表 GDT 和 LDT 中。GDT 是全局描述符表，用于给出系统使用的和各任务共用的段描述符，换言之，其可用来管理全局存储器地址空间，GDT 只有一个；LDT 是局部描述符表，存放某个任务专用的段描述符，用于管理任务用到的局部存储器地址空间，LDT 表可以有若干。

![[计算机系统/微机原理与接口技术B/附件/第2章/Pasted image 20260719155119.png]]
*图 2-12　格式*

段寄存器中存放的是段描述符的 16 位选择符，其格式如图 2-12 所示。其中 $b_1b_0$ 位为请求特权级 RPL（Requestor Privilege Level）字段，与 DPL 类似，为 0～3 级（2 位），在数据访问的特权级检查中将使用该字段。$b_2$ 位为表指示符字段 TI，指明本描述符是在 GDT（$TI=0$）还是在 LDT（$TI=1$）中，$b_{15} \sim b_3$ 共 13 位构成描述符索引字段 INDEX，用于指明段描述符在指定的描述符表中的序号。
当将一个选择符装入一个段寄存器时，处理器将自动从 GDT 或 LDT 中找到其对应的描述符并装入相应段寄存器的描述符高速缓存器中，该过程对用户是透明的。以后，每当访问存储器时，与所用段相关的段描述符高速缓存器就自动参与存储器访问操作。使用描述符高速缓存器的原因是，在同一段访问时，可直接使用该寄存器内的描述符信息，而无需重新从 GDT 或 LDT 表中去取，大大加快了访问时间。
在高速缓存器中获取的段基地址成为线性地址或物理地址计算中的一个分量，界限用于段限检查，属性则对照所要求的存储器访问类型进行检查，即
线性地址 = 段描述符高速缓存器中的段基址 + 偏移地址
不使用页部件时，线性地址即为物理地址；使用页部件时，线性地址需经页管理部件使用页目录表和页表转换成物理地址。
### 3. 指令指针和标志寄存器
80386 的指令指针寄存器 EIP 也是由 8086/8088 的 16 位指令指针寄存器 IP 扩展而得到的。EIP 中存放的是下一条要执行指令在代码段的偏移地址。80386 在实方式下采用 16 位的指令指针 IP。80386 的标志寄存器是 32 位，如图 2-13 所示。其中低 16 位中比 8086/8088 增加了 IOPL（第 12、13 位）和 NT（第 14 位），高 16 位中只使用了 2 个标志位：RF（第 16 位）和 VM（第 17 位）。

![[计算机系统/微机原理与接口技术B/附件/第2章/Pasted image 20260719155134.png]]
*图 2-13　指令指针和标志寄存器示意图*

这些新增标志位的含义如下。

1. IOPL（I/O Priority Level）：I/O 特权标志位，只适用保护模式，指明 I/O 操作的级别。
2. NT（Nested Task）：嵌套标志位。当前执行的任务正嵌套在另一任务中时，$NT=1$；否则，$NT=0$。NT 也仅适用保护模式。
3. VM（Virtual 8086 Mode）：虚拟 8086 模式标志位。处于保护模式的 80386 转为虚拟 8086 方式时 $VM=1$。
4. RF（Resume Flag）：恢复标志位，配合调试寄存器的断点或单步操作一起使用。在处理断点时，在两条指令之间检查到 $RF=1$ 时，则下一条指令执行时的调试故障被忽略。在成功地完成一条指令后，$RF=0$；而当接收到一个非调试故障的故障信号时，$RF=1$。

### 4. 控制寄存器
80386 有 4 个 32 位的控制寄存器 $CR_0$、$CR_1$、$CR_2$ 和 $CR_3$，如图 2-14 所示，它们的作用是保存全局性机器状态。

![[计算机系统/微机原理与接口技术B/附件/第2章/Pasted image 20260719155143.png]]
*图 2-14　控制寄存器示意图*

$CR_0$ 定义了如下 6 位。

1. PE（Protection Enable）：允许保护标志，用于控制微处理器是否进入保护模式。若 $PE=1$，则 80386 转换成保护模式。PE 复位后，微处理器回到实地址模式。系统加电后，微处理器总是初始化为实地址模式。
2. MP（Monitor Processor extension flag）：协处理器监控标志，监控数学协处理器是否存在。若系统中有一个数学协处理器存在，则 $MP=1$；否则，$MP=0$。
3. EM（Emulate processor extension flag）：仿真协处理器标志，表示是否用软件仿真数学协处理器。若 $EM=0$，所有协处理器操作在实际协处理器 80387 上执行；若 $EM=1$，将采用软件仿真实现协处理器的功能。
4. TS（Task Switched）：任务切换标志，由硬件置位而由软件复位。当一个任务转换完成后，TS 标志自动置 1。
5. ET（Extension Type）：扩展类型标志，表示系统中所使用的数学协处理器的类型。该标志设置的原因在于 80386DX 可以支持 80287 和 80387。若 $ET=1$，表示系统中使用的数学协处理器是 80387；若 $ET=0$，表明使用的是 80287 或没有使用数学协处理器。
6. PG（PaGing flag）：允许分页标志，表示处理器是否允许对存储器线性地址进行分页，并进行物理地址转换。若 $PG=1$，允许分页，并由分页单元将线性地址转换为物理地址；若 $PG=0$，禁止分页，线性地址直接作为物理地址使用。

$CR_1$：未定义的控制寄存器，供后期的微处理器使用。
$CR_2$：页故障线性地址寄存器，保存最后出现页故障的全 32 位线性地址。
$CR_3$：页目录基地址寄存器，保存页目录表的物理基地址。
### 5. 系统地址寄存器
80386 CPU 设置了 4 个系统地址寄存器，保存操作系统所需要的保护信息和地址转换表信息，如图 2-15 所示。

![[计算机系统/微机原理与接口技术B/附件/第2章/Pasted image 20260719155150.png]]
*图 2-15　系统地址寄存器示意图*

1. GDTR（Global Descriptor Table Register）：全局描述符表寄存器，48 位寄存器，保存全局描述符表的 32 位线性地址和 16 位段界限。
2. IDTR（Interrupt Descriptor Table Register）：中断描述符表寄存器，48 位寄存器，保存中断描述符表的 32 位线性地址和 16 位段界限。
3. LDTR（Local Descriptor Table Register）：局部描述符表寄存器，16 位寄存器，保存当前局部描述符表的 16 位选择符。
4. TR（Task state Register）：任务状态寄存器，16 位寄存器，保存当前任务的任务状态段的 16 位选择符。

前文已述，段描述符存放在全局描述符表 GDT 或局部描述符表 LDT 中。事实上，80386 有 3 种描述符表，除了全局和局部描述符表，还有中断描述符表 IDT（Interrupt Descriptor Table），给出它们存放位置的寄存器分别是 GDTR、LDTR 和 IDTR。

![[计算机系统/微机原理与接口技术B/附件/第2章/Pasted image 20260719155159.png]]
*图 2-16　GDTR、IDTR 与 LDTR 的组成*

如图 2-16 所示，GDTR 是 48 位寄存器。该寄存器最低 2 字节为限长（Limit）字节，它规定了 GDT 按字节计算的大小。由于限长为 16 位，GDT 最大可达 65535 字节。GDTR 的高 4 字节为基址（Base），指示物理存储器中 GDT 的起始位置。该 32 位基址允许该表定位在 80386 线性地址空间的任何地方。
同全局描述符表寄存器一样，中断描述符表寄存器 IDTR 在物理存储器中给出了中断描述符表 IDT 的所在位置，其工作机制与 GDTR 相似。IDTR 和 IDT 描述符表提供一种机制，将程序控制转给中断并执行中断服务程序。
局部描述符表寄存器 LDTR 用于定位当前局部描述符表 LDT。“当前” 是因为在保护模式下，系统支持多任务，每个任务除了可访问全局描述符表外还可访问它自己的描述符表。
局部描述符表 LDT 定义了任务用到的局部存储器地址空间。由于每项任务都有它自己的存储器段，因此保护模式的软件系统可能会包含多个局部描述符表。图 2-17(a)说明了 $LDT_0$ 到 $LDT_n$ 的关系。

图 2-17(b)说明 16 位的 LDTR 值并不直接定义一个局部描述符表，只是一个指向 GDT 中 LDT 描述符的选择符。如果 LDTR 中装入了选择符，相应的描述符就能够从全局存储器中读出来并装入局部描述符表高速缓存中，只有该描述符才能确定局部描述符表。32 位的“基址”值标识物理存储器中表的起始地址，16 位的“限长”值定义表的大小。将该描述符装入高速缓存就为当前任务创建了一个 LDT。也就是说，每次 LDTR 中装入选择符，则局部描述符表的描述符被缓存，激活一个新的 LDT。

### 6. 调试寄存器

![[计算机系统/微机原理与接口技术B/附件/第2章/Pasted image 20260719155249.png]]
*图 2-21　调试寄存器示意图*

如图 2-21 所示，80386 CPU 中设置了 8 个调试寄存器 $DR_0 \sim DR_7$，它们为程序调试提供了硬件支持。其中，$DR_0 \sim DR_3$ 为线性断点寄存器，可保存 4 个断点地址，从而方便程序设计人员按照调试意图组合指令的执行。$DR_4$ 和 $DR_5$ 保留。$DR_6$ 用于保存断点状态，$DR_7$ 用于控制断点的设置。
### 7. 测试寄存器
80386 CPU 中设置了 2 个 32 位的测试寄存器 $TR_6$ 和 $TR_7$，$TR_6$ 为测试命令寄存器，用于对 RAM 和相关寄存器进行测试；$TR_7$ 用于保留测试后的结果。
80386 CPU 提供 3 种工作模式：实地址模式、虚地址保护模式、虚拟 8086 模式（本质上属于保护模式）。其地址种类也有 3 种：逻辑地址、线性地址和物理地址。虚地址保护模式提供两种主要的保护功能：一是通过给每个任务分配不同的虚拟地址空间，使任务之间完全隔离，每个任务有不同的虚拟地址到物理地址的转换映射；二是任务内的保护机制，以保护系统存储段及特别的处理器寄存器，使其不被其他应用程序所破坏。为了实现这些功能，CPU 以 4 个特权级（0～3 级，特权级依次降低）来隔离或保护各用户及系统，每个存储段都与同一个特权级相联系，只有足够级别的程序段才可对相应的段进行访问。
虚地址保护模式下的存储器组织相当复杂。这里仅考虑存储器管理单元（MMU）是如何管理地址空间，以及如何将虚拟（逻辑）地址转换成物理地址，不讨论保护功能。
保护模式的存储器管理单元使用 48 位的存储器指针。该 48 位存储器指针称为虚拟地址，在程序中用以规定指令或数据的存储位置。选择符长 16 位（见图 2-12），偏移量长 32 位。其中选择符可放在 CPU 的段选择符寄存器中，比如要访问存储器中的指令代码，则活动段选择符一定放在 CS 中。指针的这一部分实际上选择的是 CPU 虚拟地址空间中一个特定的段。偏移量放在 CPU 用户可访问的寄存器中，或用于直接寻址。如进行指令代码的访问（取指），偏移量可放在 EIP 寄存器中，指针提供要访问的存储器段中的偏移量，即要执行的指令的第 1 字节的相对偏移地址。由于偏移量长 32 位，段空间可达 4 GB。
由图 2-12 可知，16 位选择符又分为 13 位索引、1 位表选择位和 2 位请求特权级。其中 2 位 RPL 并不用于存储器段选择，因此 16 位中只有 14 位用于寻址，这样虚拟地址空间可容纳 $2^{14}$（16384=16K）个存储器段，每个段最大可达 4 GB。这些段就是 CPU 的 MMU 管理虚拟地址空间的基本元素。
另一种查看虚拟地址空间大小的办法是将 14 位段选择符和 32 位偏移量结合起来，得到 46 位虚拟地址，从而虚拟地址空间可包含 $2^{46}\text{B}=64\text{TB}$。
80386 的存储器管理单元可以实现虚存的分段和分页机制。在分段机制中，64 TB 虚拟地址空间分成了 32 TB 的全局存储器地址空间和 32 TB 的局部存储器地址空间。

![[计算机系统/微机原理与接口技术B/附件/第2章/Pasted image 20260719155305.png]]
*图 2-22　80386 虚拟地址空间的分段组织*

其划分大致如图 2-22 所示，由选择符的 TI 位选择是全局还是局部描述符表，每个地址空间可允许最多 8192 个存储器段。这种情况假定了全局描述符表和局部描述符表中的每个描述符都在使用中且设为最大值，实际情况下段的总数和各段大小可选定。描述符定义了相应段的属性，但是在实际系统应用中并不是所有的描述符都在使用。
80386 支持分段和分页两种存储管理机制，可以将 48 位虚拟地址映射到硬件所需的 32 位物理地址。
分段机制下各个存储块大小不一，可以在 4 GB 范围内自行定义，每个段最多可以分配 4 GB 的物理地址空间，最少 1 B。物理空间多次交换后可能变得非常零碎，因此存储和管理的对象较复杂。
分页机制下的对象是固定大小的存储块，如 4 KB（80386 及以上）或 4 MB（Pentium 及以上）等，称为页（Page）。这时整个线性地址空间和整个物理地址空间都可看成由很多页组成，线性地址空间的任何一页都可以映射到物理地址空间的任何一页。

![[计算机系统/微机原理与接口技术B/附件/第2章/Pasted image 20260719155314.png]]
*图 2-23(a)　80386 CPU 地址转换过程*

80386 CPU 地址转换过程如图 2-23(a)所示。其过程是，先对虚拟（逻辑）地址进行段转换，分段机制可产生的线性地址直接作为物理地址进行处理，这时如禁止分页机制（$PG=0$），发生的地址转换过程如图 2-23(b)中阴影部分所示。如果允许采用分页机制（$PG=1$），线性地址再经过分页转换产生物理地址。
作为地址转换的一部分，MMU 还要检查目标段或页是否驻留在物理存储器中。若页不存在，处理器产生缺页异常（Page Fault），由操作系统决定是否从外部存储器调入所需页面，并按需换出其他页面。分页以固定大小的页管理虚拟存储器，降低了可变大小分段带来的分配复杂度。80386 的 4 GB 物理地址空间可划分为 $1\,048\,576$ 个 4 KB 页，即 $2^{32}/2^{12}=2^{20}$ 页。

*图 2-24　80386 物理地址空间的分页管理*

![[计算机系统/微机原理与接口技术B/附件/第2章/Pasted image 20260719155330.png]]
*图 2-25　页存储机制下的线性地址格式*

图 2-24 表示物理地址空间按这种方式的组织形式。分页存储器使用这种固定大小块有一个缺点，存储管理程序每次都最少分配 4 KB，即使它们并不全用，降低了空间利用率，但分页机制避免了大小不一的分段反复交换造成的存储区碎片，简化了存储管理程序。

从图 2-23(a)可看到，段转换过程产生的线性地址不再直接用作物理地址，还要经过称为页转换的第二次转换过程。图 2-25 是页存储机制下的 32 位线性地址格式，由三部分（域）组成，形成两级分页机制。

1. 页目录地址，第 $31 \sim 22$ 位（10 位）。说明页表在页目录表中的位置，即应选择第几号页表。该值乘以 4 即为页表基地址在页目录表中的相对地址。页目录表基地址由 $CR_3$ 中页目录基址寄存器（PDBR）确定。
2. 页表地址，第 $21 \sim 12$ 位（10 位）。说明页在页表中的位置，即应选择第几号页。该值乘以 4 即为页基地址在页表中的相对地址。页目录地址和页表地址确定了物理地址的高 20 位（$A_{31} \sim A_{12}$）。
3. 页内偏移地址，第 $11 \sim 0$ 位（12 位）。说明某寻址单元在页中的相对位置。

80386 在分页方式下，以页（4 KB）为单位管理内存。对于某一任务，可以将用到的存储页的基地址（32 位）存放到一个或几个页表中。每个页表的长度为 4 KB，可以存放 1024 个页的基地址。所有页表的基地址（32 位）存放到一个页目录表中，每个页目录表可以存放 1024 个页表的基地址。页目录表的基地址则由指令装载到控制寄存器 $CR_3$ 中。

![[计算机系统/微机原理与接口技术B/附件/第2章/Pasted image 20260719155340.png]]
*图 2-26　两级分页机制下的线性地址转换*

图 2-26 表示两级分页机制下的线性地址是怎样转换成对应的物理地址的。存储器页目录表地址由 $CR_3$ 中页目录基址寄存器（PDBR）确定。这 20 位上实际决定该目录基址的高 20 位。低 12 位为相对页首地址（低 12 位为 0）的偏移量。分页机制避免了大小不一分段交换造成的存储区碎片。

实际上，线性地址中 10 位目录域是相对于 PDBR 的偏移量，由此获得 32 位页目录项指针，并缓存在转换检测缓冲器（TLB）中，其值被用作存储器中页表的基址。再结合线性地址的 10 位页域得到页帧表基址，并也被缓存在转换检测缓冲器中。由图 2-26 可知，由此基址加上线性地址 12 位偏移量则可标识出操作数在活动页帧中的位置。
80386 转换检测缓冲器实际可以保存 32 个表项，因此有 128 KB 存储区可直接被访问。这部分存储器中的操作数不需再从页表中读出而可直接存取。如果要访问的操作数不在这些页中，则产生将页表项读进转换检测缓冲器的额外开销。
80386 虚拟地址分页转换中涉及了页表项，只有两种类型的页表项：页目录项和页表项，接下来将给出目录项和页表项的结构。

![[计算机系统/微机原理与接口技术B/附件/第2章/Pasted image 20260719155349.png]]

单一页目录或页表项的格式如图 2-27 所示。如果表项在页目录表中，则最左边的 20 位是页表的基址；如果表项在页表中，则这 20 位表示页帧的基址。注意到表项只提供基址的第 $12 \sim 31$ 位，最右边的 12 位假定为 0，这样页表和页帧总是位于 4 KB 地址边界上。从图 2-26 中看到，该表项被缓存到转换检测缓冲器中。
表项的低 12 位提供页表或页帧使用的保护属性或统计信息。例如，用户/管理员（U/S）和读/写（R/W）位实施两级页保护机制。将 U/S 位设为 1，则选择用户级保护，用户级是较低的特权级，相当于分段模型中的保护级 3。也就是说，用户级指定给那些可被应用程序访问的存储器页。另一方面，将 U/S 设为 0，则指定对表或帧进行管理员保护。管理员级相应于分段模型中的 0、1、2 级，它是赋给操作系统资源的级别。读/写（R/W）位使用户级的表或帧只读或可读/写。表 2-4 总结了 U/S 和 R/W 的访问属性。

页目录项指定的保护属性对页表中所有页帧都适用，但是页表指定的属性只适用于它定义的页帧。由于页帧存在两种保护属性，所以 32 位 CPU 能够保证较高等级（更加严格）的保护权限。图 2-27 中，目录和页表项中的其他一些位提供表或帧的使用统计信息。例如，P 位表示该项是否可用于页地址转换，$P=1$ 表示该项有效，可用于地址转换，$P=0$ 表示该项可能未定义，或者不在物理存储器中。如果试图访问 $P=0$ 的页表或页帧，则发生缺页错误。该缺页错误由操作系统存储器管理程序来处理。
80386 也记录页表或页帧是否已被访问的事实。如果对表或帧中的某个地址进行读/写访问，则先将表项的已访问位（A 位）设为 1，这表示该项已被访问过。对于页帧来讲，它还记录该访问是读操作还是写操作。D 位只用于页表项，如果相应的页帧中任一个地址执行了写操作则设置该位。在虚拟申请分页存储器系统中，操作系统可以检查这些位以确定当新页交换进物理存储器空间时，存储器中的页是否需要对硬盘进行更新。最后三位为 AVL，可供程序员使用。
## 2.2.3 80x87 数学协处理器

80x87 是配合 80x86 微处理器用于完成浮点运算的数学协处理器。1980 年，8087 发布，然后为 80287、80387DX/SX 和 487SX 所替代。Intel 80486DX、Pentium 及其之后的处理器都在 CPU 中集成了数学协处理器。图 2-28 为数学协处理器的内部结构，主要分为两部分：控制单元和数字执行单元。

![[计算机系统/微机原理与接口技术B/附件/第2章/Pasted image 20260719155402.png]]
*图 2-28　数学协处理器的内部结构*

### 1. 控制单元
控制单元负责协处理器与微处理器的总线连接，两者同时监视指令流，若是 ESC 扩展指令，则由协处理器执行，否则由微处理器执行。
### 2. 数字执行单元
数字执行单元负责执行所有协处理器的任务，其中 8 个数据寄存器组成头尾相接后进先出的堆栈，用于存储算术指令的操作数和结果，对其数据的访问可采用特定寻址方式，但更多是使用堆栈的入栈和出栈操作。此外，数字执行单元还包括标记寄存器、状态寄存器和控制寄存器。

![[计算机系统/微机原理与接口技术B/附件/第2章/Pasted image 20260719155412.png]]
*图 2-29　数学协处理器的数据寄存器组成*

1. 数据寄存器，有 8 个，将 80 位的扩展精度浮点数（编号 $0 \sim 7$）做成堆栈形式，如图 2-29 所示。数据驻留在内存时，可以是任何形式（带符号数、BCD 数、单精度或双精度数），但数据从内存移到浮点数据寄存器堆栈时，这些数据的格式将被转换为扩展精度浮点数。
2. 标记寄存器，表明浮点数据寄存器堆栈中每个单元的内容。图 2-30 给出了标记寄存器以及每个标记指示的状态。标记表明寄存器内容是否合法、是否为零、是否非法或为无穷大以及是否为空。

*图 2-30　80x87 标记寄存器及其状态编码*

3. 状态寄存器，如图 2-31 所示。其中各位的含义如下。

![[计算机系统/微机原理与接口技术B/附件/第2章/Pasted image 20260719155447.png]]
*图 2-31　80x87 状态寄存器*

B（Busy）：忙标志位，表明当前浮点运算单元正在执行一项任务。
$C_3 \sim C_0$（Condition Code）：条件码位，表明浮点运算单元的工作条件（由这些码位的组合对应，但对于不同指令有不同的含义）。
ST（Top of Stack）：栈顶，表示当前寻址为栈顶的寄存器，通常是寄存器 $R_0$。
ES（Error Summary）：错误汇总位，当任何一个非屏蔽错误位被置位时，ES 被置位。
PE（Precision Error）：精度错误位，表明结果或操作数超过了设定的精度范围。
UE（Underflow Error）：下溢错误位，表明一个非 0 的结果太小，以致不能用当前精度来表示。
OE（Overflow Error）：上溢错误位，表明结果太大而不能被表示出来。
ZE（Zero Error）：被零除错误，表明当被除数是非无穷大和非零时，除数是零。
DE（Denormalized Error）：非规格化操作数错误，表明至少有一个操作数是非规格化的。
IE（Invalid Error）：非法操作错误，表明运算结果是不确定的形式，如 $0 \div 0$、$+\infty$ 等，或者使用了 NAN（非数值量）作为操作数。

4. 控制寄存器，如图 2-32 所示，包括精度控制、舍入控制和无穷大控制，也可以屏蔽或开放与状态寄存器最右边 6 位对应的异常位。

![[计算机系统/微机原理与接口技术B/附件/第2章/Pasted image 20260719155456.png]]
*图 2-32　80x87 控制寄存器*

以下是控制寄存器中各位及各组合位的功能。
IC（Infinity Control）：无穷大控制，选择是仿射无穷大还是投射无穷大。仿射允许正/负无穷大，而投射则假定无穷大为无符号数。
RC（Rounding Control）：舍入控制，确定舍入的类型。
PC（Precision Control）：精度控制，设置结果的精度。
异常屏蔽：决定异常错误是否影响状态寄存器的错误位。如果其中一个异常控制位被置为 1，则相应的状态寄存器位被屏蔽。
