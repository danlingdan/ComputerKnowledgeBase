## 2.6 CPU 的工作模式
80x86 CPU 共有 4 种工作模式：实地址模式、保护模式、虚拟 8086 模式和系统管理模式。其中，8086/8088 CPU 只能工作于实地址模式，80286～80486 CPU 可以工作于实地址模式、保护模式和虚拟 8086 模式，Pentium 及其以上 CPU 可以工作在全部 4 种模式下。下面分别介绍 4 种模式的特点并给出其相互切换的控制方式。
### 2.6.1 实地址模式
实地址模式（Real Mode）是最基本的工作方式，它实际就是 16 位 8086/8088 CPU 的工作模式，其后的 CPU 为了与其兼容，都支持这种工作方式。在该模式下，原有 16 位 CPU 的程序不加任何修改就可以在 80286 及其以上的 CPU 上运行。
在实地址模式下，CPU 的地址线中只有低 20 位起作用，即能寻址的物理存储空间为 1 MB，其存储器的管理方式与 8086/8088 CPU 存储器的管理方式相同。系统复位时，$CR_0$ 的 PE 位自动清 0，进入实地址方式。MS-DOS 操作系统要求 CPU 工作于实地址模式。

### 2.6.2 保护模式
通常在程序运行过程中，需要防止以下情况的发生：① 应用程序破坏了系统程序；② 某一应用程序破坏了其他应用程序；③ 错误地把数据当成程序来运行。为了避免以上情况的发生，所采取的措施称为“保护”。
保护模式（Protected Mode）是一种建立在虚拟存储器和保护机制基础上的工作模式，可最大限度地发挥 CPU 所具有的存储管理功能及硬件支持的保护机制，为多任务、多用户操作系统的实现提供硬件支持。
在保护模式下，CPU 通过设立 0～3 级 4 个特权级，实现应用程序之间及应用程序与操作系统之间的有效隔离，其保护结构如图 2-59 所示。

0 级为最高特权级，分配给操作系统内核；操作系统的系统服务程序为 1 级；应用系统服务程序为 2 级；3 级为最低特权级，分配给应用程序。每个存储段都与同一个特权级关联，当前活跃的代码段的特权级为 CPL（Current Privilege Level）。CPU 的保护机制规定，对给定 CPL 的执行程序，只允许访问同一级或低特权级的数据段，否则将产生异常。
保护模式的保护机制主要指两方面：① 通过给每个任务分配不同的地址空间，使任务之间完全隔离；② 任务内部的保护机制，以保护操作系统存储空间及特别的 CPU 寄存器，使其不被其他应用程序所破坏。
保护模式下的虚拟地址空间被分为全局地址空间和局部地址空间。每个任务各自占有的虚拟地址空间为局部地址空间，其内部的代码和数据是任务私有的，需要与系统中的其他任务相隔离。各任务共用的一部分虚拟地址空间为全局地址空间，操作系统存储在全局地址空间中，可以被所有任务共享，而且任务对它的访问不会破坏它的内容。
保护模式分两步实现虚拟地址空间到物理地址空间的映射，也就是分两步实现地址转换，但第二步是可选的。通过描述符表和描述符、分段管理机制实现虚拟地址空间到线性地址空间的映射，实现把二维虚拟地址转换为一维线性地址，这一步总是存在的。
分页管理机制把线性地址空间和物理地址空间分别划分为大小相同的块，这样的块称为页（Page）。通过在线性地址空间的页与物理地址空间的页之间建立映射表，分页管理机制实现线性地址空间到物理地址空间的映射，实现线性地址到物理地址的转换。分页管理机制是可选的，在不采用分页管理机制时，线性地址空间就等于物理地址空间，线性地址就等于物理地址。

### 2.6.3 虚拟 8086 模式
虚拟 8086 模式（Virtual 8086 Mode）是一种特殊的保护模式，工作方式与保护模式相同，但其对逻辑地址的解释是按照 8086/8088 的方式进行的。在这种模式下，不仅可以执行 8086/8088 程序，而且支持多个 8086/8088 实模式程序的运行，其实质是为运行于 32 位 CPU 上的 8086 程序提供独立的虚拟机。

### 2.6.4 系统管理模式
系统管理模式（System Management Mode, SMM）是 Pentium CPU 新增加的工作模式，它不是作为应用程序或系统级特性，而是为实现高级的系统功能，如电源管理和安全性。进入系统管理模式，CPU 的状态被保存，并有其特定的存储空间，这时系统可以进入休眠状态，其后在有键按下或鼠标移动时能自动唤醒，并使系统从原先的休眠中断点开始继续工作。
