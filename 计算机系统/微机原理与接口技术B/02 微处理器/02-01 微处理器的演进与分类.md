## 2.1 微处理器概述

> [!important] 先区分三个层次
> **指令集架构（ISA）**规定软件可见的指令、寄存器和异常模型；**微体系结构**规定流水线、执行单元和缓存如何实现 ISA；具体**处理器产品**还包含核心数量、制程、封装和片上外设。三者不能混为同一个“架构”。

> [!definition] 微处理器（Microprocessor）
> 微处理器是把运算器（ALU）与控制器（CU）集成在单一芯片上的中央处理单元（CPU），是微型计算机的核心，负责取指、译码、执行指令并控制总线。

前文已述及，从指令集的结构来区分，微处理器可分为 RISC 和 CISC 处理器，而从应用领域来区分，微处理器可分为通用微处理器、嵌入式微处理器（数字信号处理器）和微控制器。通用微处理器主要用于运行通用软件，追求总体高性能，通常配备功能完备、复杂的操作系统，典型代表为 Intel 的 80x86 系列、AMD 的 K 系列等；嵌入式微处理器主要用于面向特定应用领域，强调处理特定问题的高性能，通常配备轻量级的操作系统，主要用于消费类电子产品，典型代表为 ARM 系列、MIPS、PowerPC、DSP 等；微控制器实现系统的控制功能，性能和价格相对较低，主要用于汽车、自动机械等的自控设备，典型代表为单片机 MCU。本章将会以 Intel 80x86 系列处理器为例详细说明微处理器的特性，下面将简要介绍其他几类常用的微处理器并给出 64 位微处理器的最新发展。

### 1. ARM（Advanced RISC Machine）
ARM 是嵌入式与移动系统中广泛采用的指令集体系，历史产品包括 ARM7、ARM9、ARM11、StrongARM 和 XScale，后续则形成 Cortex-A、Cortex-R、Cortex-M 等系列。具体内核在缓存、流水线和存储接口上的组织各不相同，很多实现采用统一地址空间并在一级缓存处分离指令与数据通路，可视为改进型哈佛结构，不能仅按系列名称一概而论。作为典型 RISC 指令集体系，其常见特征包括：

1. 31 个无区别使用的通用寄存器，均可存放数据或地址，可为所有的数据操作提供快速的局部存储访问。
2. 在符合 RISC 指令长度固定的特性基础上，优化调整特定指令的周期，并给出 32 位指令集的 16 位压缩形式 Thumb 指令集，提高了代码密度。
3. 在输入寄存器被某条指令使用前，内嵌桶型移位寄存器可处理该寄存器中的数据，扩展了指令功能，优化内核性能。
4. 支持用户、快中断、中断、管理、中止、系统和未定义等 7 种处理器模式，除了用户模式外，其余的均为特权模式。
5. 具有片上总线 AMBA（Advanced Micro-controller Bus Architecture）。AMBA 定义了 3 组总线：先进高性能总线 AHB（Advanced High performance Bus），先进系统总线 ASB（Advanced System Bus），先进外围总线 APB（Advanced Peripheral Bus）。AMBA 可以方便地扩充各种处理器及 I/O，可以把 DSP、其他处理器和 I/O（如 UART、定时器和接口等）都集成在一块芯片中。
6. 指令执行采用 3 级/5 级流水线技术。
7. 采用 Load-Store 结构，即 CPU 并不会对内存中的数据进行操作，所有的计算都要求在寄存器中完成，而寄存器和内存的通信则由单独的指令来完成。
8. ARM 处理器主要应用的领域包括：无线、消费电子和图像应用的开放平台；存储、自动化、工业和网络应用的嵌入式实时系统；智能卡和 SIM 卡的安全应用等。

### 2. DSP（Digital Signal Processor）
DSP 芯片是适合于进行数字信号处理的微处理器，主要应用是实时快速地实现各种数字信号处理算法，具有运算速度快、可程控、可预见性、精确度高及接口灵活等优点，也有功耗大、系统复杂、应用的频率范围受限等缺点，典型产品为 TI 公司的 TMS320 系列。
由于面向实时信号处理任务，DSP 作为一种微处理器，其设计的出发点和通用 CPU 以及 MCU 等处理器不同，其设计核心是算法的高效实现，因而在体系结构方面 DSP 具有如下的特性。
DSP 具有独立的硬件乘法器，乘法指令可以在单周期内完成，使得卷积、数字滤波、FFT、相关运算、矩阵运算等算法中的大量乘法运算速度加快。
DSP 通常包含专门的地址产生器，能生成信号处理算法需要的特殊寻址，如循环寻址和位翻转寻址，其中循环寻址对应于流水 FIR 滤波算法，位翻转寻址对应于 FFT 算法。
DSP 具有多总线结构，程序空间与数据存储空间分开，各自独立的地址总线和数据总线（哈佛结构），取指令和读数据可以同时进行。
采用流水线技术，每条指令的执行可分为取指令、译码、取数、执行等若干步骤，由片内多个功能单元完成，相当于多条指令并行执行（RISC 特征），显著提高了运算速度。
片内具有快速的 RAM，通常可通过独立的数据总线在两块 RAM 中同时访问。
快速的中断处理和硬件 I/O 支持。
VLIW（超长指令字）技术使用静态调度方法，使用宽的并行指令独立控制各个功能单元，可以进一步优化执行速度。

### 3. MCU（Microcontroller Unit）
单片机 MCU 是将 CPU、RAM、ROM、多个 I/O 接口和中断系统、定时器/计数器等功能集成到单个芯片的微型计算机系统，主要应用在工业控制领域。单片机的主要特点如下。

1. 片内寄存器分为工作寄存器组和特殊功能寄存器，可实现数据访问和多种 I/O 接口的数据管理。
2. 片内有全双工串行通信接口，可较为方便地组成分布式控制系统。
3. 片内有独立的位处理器，又称为布尔处理器，具有相应的累加器和可按位寻址的 RAM 区、特殊功能寄存器，并设有专门的位指令，方便实现位控功能。
4. 具有面向控制、结构简单、体积小、可靠性高的特点，由于自带较多的 I/O 接口，可以直接对 I/O 进行算术、逻辑和位操作，指令简单而丰富，控制能力强。而且 I/O 端口多采用分时复用的方式，整体芯片体积紧凑，且易于采取屏蔽措施，适合在恶劣环境下工作。
5. 具有低电压、低功耗、较高性价比的特点，通常可工作在 2.2 V 或 1.2 V 等低电压，由于 MCU 构成的硬件结构简单、开发周期短、控制能力强、可靠性高，在同等功能的要求下，基于 MCU 开发的控制系统比其他类型微处理器开发的相应系统具有价格上的优势。

上文给出了几类常见处理器。讨论“64 位处理器”时必须说明所指维度：通用寄存器和算术操作数宽度、地址形成能力、外部数据总线宽度并不必然相同。x86-64（AMD64，Intel 实现曾称 Intel 64/EM64T）扩展了 IA-32 并保留兼容执行环境；IA-64 则是不同的指令集架构，不能视为 x86-64 的一种实现。
由于 80x86 系列也包括 16 位处理器，现在 Intel 公司把 32 位的 x86-32 称为 IA-32，全名为 Intel Architecture 32-bit。x86-64，简称 x64，是 64 位 CPU 架构及其相关指令集的一种，也是 Intel x86 架构的延伸产品。x64 最先由 AMD 设计，推出时被称为 AMD64，其后也为 Intel 公司所采用，现在 Intel 称之为 Intel 64，也称为 EM64T。
Intel 公司之前已经在 Itanium 处理器上使用了 64 位 IA-64 技术，虽然 EM64T 也是 64 位，但两者并不相容，即 IA-64 的软件不能直接在 EM64T 上运行。EM64T 是 IA-32 指令集的延伸，而 IA-64 则是另一种独立的架构，没有任何 IA-32 的影子。虽然 IA-64 可通过模拟方式来执行 IA-32 的指令，但指令需要转换才能在 IA-64 上执行，导致速度变慢。
EM64T 是 Intel IA-32 架构的扩展，即 IA-32E（Intel Architecture-32 Extension）。IA-32 处理器通过附加 EM64T 技术，可在兼容 IA-32 软件的情况下允许软件利用更多的内存地址空间，并且允许软件进行 32 位线性地址写入。EM64T 特别强调对 32 位和 64 位的兼容性。
在 x86-64 的早期发展中，AMD64 首先用于 Opteron、Athlon 64 等处理器，Intel 随后在部分 Xeon 与 Pentium 4 型号中实现兼容扩展。讨论 64 位 x86 时更重要的是区分体系结构能力、操作系统模式和应用程序位宽，而不是记忆早期产品代号。
微处理器的整体性能提高大都来源于处理器频率的大幅增加（在 1983 至 2002 年间，从 5 MHz 增加到 3 GHz）和制造工艺技术的进步，还有一部分改进来源于每周期所执行指令数的增加（流水线技术的不断改进）。不过，从 2002 年开始，随着功耗和散热量的不断增加，这种通过提高处理器频率改进性能的方法，已经开始显露出局限性。解决的思路是使用多核处理器（Chip Multi-Processor, CMP），通过将多个内核整合在一起，每个内核能够在较低的频率下运行，从而各单内核的功耗下降，由此得到的多核处理器性能大幅超过单核，能够显著地增强计算机的处理能力和计算资源，提供更好的响应能力、更大的多线程吞吐量，并把并行计算的优势运用到分配了合理线程的主流应用软件中。
IBM POWER4 在 2001 年采用双核设计，随后多核技术从服务器逐步进入通用 x86 处理器。此后处理器继续向多核、片上互连、共享与私有缓存层次以及专用加速单元发展。Nehalem、Knights Corner 等适合作为历史节点理解，但它们不代表现代处理器的统一结构。
多核处理器是处理器发展的必然趋势，无论是移动与嵌入式应用、桌面应用还是服务器应用，都将采用多核架构，其未来应用前景十分广阔。在体系结构、软件、功耗和安全性设计等方面，多核处理器面临着巨大的挑战，当然同时也指明了其未来优化的方向。
