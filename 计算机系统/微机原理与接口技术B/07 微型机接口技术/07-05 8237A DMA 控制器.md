## 7.5 DMA 控制器接口

DMA（Direct Memory Access）是指外部设备直接与计算机存储器进行数据交换的 I/O 方式。这种方式下，数据的 I/O 操作不需要 CPU 执行指令，也不经过 CPU 的内部寄存器，而是由 DMA 控制器（DMAC）控制，利用系统的数据总线在外设和存储器之间进行顺序读出、写入操作。这种方式可以达到很高的传输速率，因而广泛用于高速 I/O 设备接口。
正常情况下，系统总线归 CPU 控制，DMA 控制器作为普通接口芯片接受 CPU 的管理，在需要 DMA 操作时向 CPU 提出 DMA 请求（DREQ）；在 CPU 响应请求后，DMAC 可从 CPU 处获得对总线的控制权。DMAC 根据数据传输的起始地址和长度，提供对存储器或 I/O 设备的读/写命令，每传输一个数据后自动修改地址和待传数据的长度。数据传输完毕，发出 DMA 传送过程结束信号 EOP，也可由外界输入 EOP 信号中止 DMA 过程。
在功能上，一方面可编程 DMAC 能以主控方式接管 CPU 总线，直接在 I/O 设备与存储器之间进行连续、顺序的数据读/写操作，因此它有别于一般的 I/O 控制器；另一方面，作为一个可编程 I/O 器件，在未获得总线控制权时，可由 CPU 对它进行初始化编程，按通道设置数据传送类型和方式、内存首地址、自动地址增/减选择、数据长度等参数，CPU 执行 I/O 指令对 DMAC 相关寄存器写入和读出时，DMAC 与其他 I/O 电路一样，是连接在 CPU 总线上的从属器件。本节主要介绍 PC 中常用的可编程 DMA 控制器 Intel 8237A。

### 7.5.1 8237A 芯片的基本功能和引脚特性

Intel 8237A 芯片具有 4 个可编程独立通道，每个通道支持 64 KB 的数据块传输，5 MHz 工作频率下传输速率可达 1.6 MB/s，是典型的可编程 DMA 控制器。8237A 芯片的内部结构如图 7-55 所示。

#### 1. 8237A 芯片的基本功能

1. 4 个独立的 DMA 通道。每个通道有 16 位基地址寄存器、16 位字节计数寄存器、16 位当前地址寄存器、16 位当前字节计数寄存器、6 位方式寄存器各 1 个。方式寄存器接收并保存来自 CPU 的方式控制字，使本通道工作于指定的方式。
2. 定时和控制逻辑电路。在 DMA 请求服务之前，对 CPU 编程给定的命令字和方式控制字进行译码，以确定 DMA 的工作方式，并产生所需的定时信号。
3. 优先权编码和旋转优先权逻辑。对通道进行优先权编码，在同时接收不同通道的 DMA 请求时，确定响应的先后次序。通道的优先权可以通过编程确定是固定的或旋转的。
4. 共用寄存器。除了每个通道的寄存器，整个芯片还有一组公用的寄存器：1 个 16 位临时地址寄存器、1 个 16 位临时字节计数寄存器、1 个 8 位状态寄存器、1 个 8 位命令寄存器、1 个 8 位临时寄存器、1 个 4 位屏蔽寄存器、1 个 4 位请求寄存器等。8237A 内部寄存器的类型和数量如表 7-6 所示，其中地址、字节数临时寄存器等数量为 1 的寄存器为各通道所公用，其余数量为 4 的寄存器每个通道各有 1 组。
5. 8237A 数据总线。数据总线带有三态缓冲器，供读/写使用。

#### 2. 8237A 芯片的引脚结构

![[计算机系统/微机原理与接口技术B/附件/第7章/Pasted image 20260719162915.png]]
*图 7-56 8237A 芯片的引脚*

如图 7-56 所示，8237A 主要引脚定义如下。
CLK：时钟信号输入。对于标准的 8237A，输入时钟频率为 3 MHz；对于 8237A-2，输入时钟频率可达 5 MHz。
$\overline{CS}$：芯片选择信号，低电平有效。
RESET：复位信号，用来清除 8237A 中的命令、状态请求和临时寄存器，且使字节指针触发器复位并置位屏蔽触发器的所有位（即使所有通道工作在屏蔽状态）。在复位之后，8237A 工作于空闲周期。
READY：外设向 8237A 提供的高电平有效的“准备就绪”信号。若 8237A 在 $S_3$ 状态以后的时钟下降沿检测到 READY 为低电平，则说明外设还未准备好下一次 DMA 操作，需要插入 $S_W$ 状态，直到 READY 引脚出现高电平为止。
$\text{DREQ}_0 \sim \text{DREQ}_3$：DMA 请求信号。对应于 4 个独立通道，DREQ 的有效电平可以通过编程来确定，优先级可以固定或旋转。
$\overline{\text{DACK}}_0 \sim \overline{\text{DACK}}_3$：对相应通道 DREQ 请求的应答信号。
$\text{DB}_0 \sim \text{DB}_7$：8 条双向三态数据总线。在 CPU 控制系统总线时，可以通过 $\text{DB}_0 \sim \text{DB}_7$ 对 8237A 编程或读出 8237A 的内部状态寄存器的内容；在 DMA 操作期间，由 $\text{DB}_0 \sim \text{DB}_7$ 输出高 8 位地址信号 $A_8 \sim A_{15}$，并利用 ADSTB 信号锁存该地址信号。在进行内存不同区域之间的 DMA 传送时，除了送出 $A_8 \sim A_{15}$ 地址信号外，还分时输入从存储器源区域读出的数据，送入 8237A 的临时寄存器中，等到存储器写周期时，再将这些数据通过这 8 个引脚，由 8237A 的临时寄存器送到系统数据总线上，然后写入规定的存储单元中。
$A_3 \sim A_0$：低 4 位地址信号，双向、三态。在空闲周期，接收来自 CPU 的 4 位地址信号，用以寻址 8237A 内部的不同寄存器（组）；在 DMA 传送时，输出要访问的存储单元或 I/O 端口地址的低 4 位。
$A_7 \sim A_4$：三态地址信号。在 DMA 传送时，输出要访问的存储单元或 I/O 端口地址的高 4 位。
$\overline{IOR}$：双向、三态 I/O 读信号，低电平有效。在空闲周期，它是一条输入控制信号，CPU 利用它读取 8237A 内部状态寄存器的内容；而在 DMA 传送时，它是读端口控制信号输出引脚，与 $\overline{\text{MEMW}}$ 配合，使数据由外设传送到内存。
$\overline{IOW}$：双向、三态 I/O 写信号，低电平有效，其功能与 $\overline{IOR}$ 相对应。
$\overline{\text{MEMR}}$：双向、三态存储器读信号，低电平有效，用于 DMA 传送。
$\overline{\text{MEMW}}$：双向、三态存储器写信号，低电平有效，用于 DMA 传送。
AEN：地址使能信号，高电平有效，由它把锁存在外部锁存器中的高 8 位地址送入系统的地址总线，同时禁止其他系统驱动器使用系统总线。
ADSTB：地址选通信号，高电平有效，把 $\text{DB}_7 \sim \text{DB}_0$ 上输出的高 8 位地址信号锁存到外部锁存器中。
EOP：过程结束信号，双向。当字节数计数器减为 0 时，在引脚上输出一个有效的低电平脉冲，表明 DMA 传送过程已经结束；也可接收外部的 EOP 信号，强行结束 8237A 的 DMA 操作或重新进行 8237A 的初始化。当不使用 EOP 端时，应通过上拉电阻接到高电平上，以避免干扰。
其他还有 Vcc（+5 V）、Vss（GND）及 N/C 引脚（引脚 5，建议上拉）等。

### 7.5.2 8237A 芯片内部寄存器与编程

8237 芯片有 4 个独立的 DMA 通道，多个内部寄存器如表 7-6 所示，其功能和作用如下。

1. 基地址寄存器：存放 16 位地址，只可写入，不能读出。在编程时，与当前地址寄存器被同时写入起始地址，可用作内存区域的首地址或末地址。在 8237A 进行 DMA 数据传送过程中，其内容不发生变化，只是在自动预置时，其内容可被重新写到当前地址寄存器中。
2. 基字节计数寄存器：存放相应通道需要传送数据的字节数，只可写入，不能读出。在编程时它与当前字节计数寄存器被同时写入要传送数据的字节数。在 8237A 进行 DMA 数据传送过程中，其内容保持不变，只是在自动预置时，其内容可以被重新写到当前字节计数寄存器中。
3. 当前地址寄存器：存放 DMA 传送期间的地址值。每次传送后自动加 1 或减 1。CPU 可以对其进行读/写操作。在选择自动预置时，每当字节计数值减为 0 或外部 EOP 有效时，就自动将基地址寄存器的内容写入当前地址寄存器，恢复其初始值。
4. 当前字节计数寄存器：存放当前字节数。每传送 1 字节，该寄存器的内容减 1。当设为自动预置时，若计数值减为 0 或接收到来自外部的 EOP 信号，会自动将基字节计数寄存器的内容写入该寄存器，作为初始计数值。
5. 临时地址寄存器和临时字节计数寄存器：中间过程使用，不需对其进行读/写操作。
6. 方式寄存器（地址偏移+0BH）：每个通道有 1 个 8 位方式寄存器，但是它们占用同一个端口地址，用来存放方式控制字，依靠方式控制字本身的特征位来区分写入不同的通道，规定通道的工作方式，各位的作用如图 7-57 所示。

![[计算机系统/微机原理与接口技术B/附件/第7章/Pasted image 20260719162924.png]]

自动预置就是当某一通道按要求将数据传送完后，又能自动预置初始地址和传送的字节数，以便重复前面已进行的过程。
校验传送指实际并不进行传送，只产生地址并响应 EOP 信号，不产生读/写控制信号，用以校验 8237A 的功能是否正常。

7. 命令寄存器（+08H，写）。存放编程的命令字，命令字如图 7-58 所示，位定义如下。

![[计算机系统/微机原理与接口技术B/附件/第7章/Pasted image 20260719162931.png]]
*图 7-58 8237A 命令寄存器*

（图中标注了 $D_0 \sim D_7$ 各个控制位含义）

$D_0$：规定是否允许采用存储器到存储器的传送方式。若允许，则利用通道 0 和通道 1 来实现。
$D_1$：规定通道 0 的地址是否保持不变。如前所述，在存储器到存储器传送中，源地址由通道 0 提供，读出数据到临时寄存器，而后由通道 1 送出目标地址，将数据写入目标区域；若命令字中 $D_1=1$，则在整个数据块传送中（块长由通道 1 决定）保持内存源区域地址不变，因此就会把同一个数据写入到整个目标存储器区域中。
$D_2$：允许或禁止 8237A 芯片工作控制位。
$D_3$：选择总线周期中写信号的定时。在 PC 中，动态存储器的写操作是由写信号的上升沿启动的，若在 DMA 周期中写信号来得太早，可能造成错误，所以 PC 选择 $D_3=0$。命令字的其他位容易理解，不再说明。
$D_5$：选择是否扩展写信号。在 $D_3=0$（正常时序）时，如果外设速度较慢，有些外设用 8237A 送出的 MEMW 和 IOW 信号的下降沿来产生 READY 信号。为提高传送速度，使 READY 信号早些到来，须将 MEMW 和 IOW 信号加宽，以使它们提前到来。因此可以通过令 $D_5=1$ 使 IOW 和 MEMW 信号扩展 2 个时钟周期，以便提前到来。

8. 请求寄存器（+09H）：在软件控制下产生一个 DMA 请求，等效外部 DREQ 请求。请求字格式如图 7-59 所示，$D_0D_1$ 的不同编码表示向不同通道发出 DMA 请求。在软件编程时，这些请求是不可屏蔽的，利用命令字可使 8237A 按照命令字的 $D_0D_1$ 所指的通道，完成 $D_2$ 所规定的操作，这种软件请求只用于通道工作在数据块传送方式下。

※ 单个通道屏蔽字（+0AH）：格式如图 7-60 所示。利用这个屏蔽字，每次只能选择一个通道。其中 $D_0D_1$ 的不同编码指示所选的通道， $D_2=1$ 表示禁止该通道接收 DREQ 请求，$D_2=0$ 表示允许 DREQ 请求。

10. 状态寄存器（+08H，读）：存放各通道的状态，CPU 读出其内容后，可得知 8237A 的工作状态。包括：通道传输结束，计数已达到计数终点，对应位为 1；通道的 DMA 请求尚未处理，对应位为 1。状态寄存器格式如图 7-62 所示。

![[计算机系统/微机原理与接口技术B/附件/第7章/Pasted image 20260719163005.png]]
*图 7-62 8237A 状态寄存器*

⑪ 临时寄存器（+0DH，读）：用于存储器到存储器传送过程中数据的暂时存放。
⑫ 字节指针触发器（+0CH）：一个特殊触发器，用于对前述各 16 位寄存器寻址。由于寄存器的读/写必须分两次进行，先低字节，后高字节，为此要利用该触发器，当此触发器状态为 0 时，进行低字节操作。低字节读/写操作完成后，触发器自动置 1，再次操作又清 0。利用这种机制实现双字节读/写操作，16 位寄存器仅占用一个外设端口地址。
⑬ 主清除（复位）寄存器（+0DH，写）：与硬件复位等效，进入空闲周期。

### 7.5.3 8237A 应用与编程

DMA 的典型应用包括：利用存储器通过数模转换器（DAC）产生连续波形、由 ADC 高速转换存储到存储器中实现高速连续采样、高速通信中网卡与存储器高速数据交换等。8237A 应用在早期的 PC/XT 中，用于软盘、硬盘快速交换和用户扩展（I/O 总线、网卡等）。通道 0 用于定时动态刷新 DRAM，刷新周期 $T=15.08\ \mu\text{s}$，优先级最高，这时仅允许单字节方式。

#### 1. 8237A 芯片的初始化

PC 系统的 BIOS 对 8237A 芯片进行初始化和测试，测试程序对 8237A 的 4 个通道的 8 个 16 位寄存器先后分别写入 0FFFFH 和 0000H，再读出进行比较，以检测 8237A 是否工作正常。BIOS 测试程序段（DMA 端口地址=00H）如下：
```asm
        MOV   AL, 04           ; 命令字禁止 8237A 工作
        OUT   DMA+08H, AL      ; 命令字送命令寄存器
        OUT   DMA+0DH, AL      ; 写总清命令，8237A 进入空闲周期
;
T1:     MOV   AL, 0FFH         ; 0FFH→AL，8 位作全 1 检测
        MOV   BL, AL           ; 保存到 BX 寄存器
        MOV   BH, AL
        MOV   CX, 08           ; 检测次数
        MOV   DX, DMA          ; 0FFH 写入 0#~3#通道的地址或字节计数寄存器
T2:     OUT   DX, AL           ; 写低 8 位
        OUT   DX, AL           ; 写高 8 位
        IN    AL, DX           ; 读出写入的低 8 位
        MOV   AH, AL           ; 保存到 AH
        IN    AL, DX           ; 读出高 8 位
        CMP   BX, AX           ; 读出内容是否与写入内容相同
        JZ    T3               ; 相同，转 T3 继续
        JMP   ERROR            ; 不同，8237A 有错停机
T3:     INC   DX               ; 寄存器地址加 1，检查下一寄存器
        LOOP  T2
        INC   AL               ; 使(AL)=0
        JZ    T1               ; 返回作全 0 检测
...
```

在 PC 中，DRAM 的刷新通过 8237A 的 0#通道来实现，其定时则通过 8253 芯片的 1#通道来提供，下面是 PC BIOS 对 DRAM 刷新的初始设置程序（TIMER1=40H）：
```asm
...
        MOV   AL, 0FFH         ; 0#通道传送字节数
        MOV   DX, DMA+1
        OUT   DX, AL           ; 写基值和现行字节计数寄存器低 8 位
        OUT   DX, AL           ; 写高 8 位
        MOV   AL, 58H          ; DMA 工作方式字
        OUT   DMA+0BH, AL      ; 写入方式寄存器
        MOV   AL, 0            ; DMA 命令字
        MOV   CH, AL           ; 计数次数高 8 位为 0
        OUT   DMA+8, AL        ; 写命令字到命令寄存器
        OUT   DMA+10, AL       ; 写屏蔽寄存器，允许 0#通道请求
        MOV   AL, 54H          ; 系统 8253 定时/计数器控制字
        OUT   TIMER1+3, AL     ; 写入 8253 控制字
        MOV   AL, 12H          ; 设置 8253 的 1#通道计数初值
        OUT   TIMER1+1, AL     ; 写入 1#通道计数初值
...
```

#### 2. 8237A 的应用

> [!example] 例 7-4
> 把存储器从 80000H 开始的 300H 字节通过 8237A 通道 1 送往 I/O 端口。
```asm
        XOR   AL, AL           ; DMAC 地址 00H ~ 0FH
        OUT   0DH, AL          ; 主复位寄存器 MR 复位
        OUT   02, AL           ; A0~A7 当前地址→CH1 寄存器
        OUT   02, AL           ; A8~A15
        MOV   AL, 8            ; A16~A19
        OUT   83H, AL          ; 存储器页面寄存器，这是微机的规定
        MOV   AX, 300H-1       ; 计数值
        OUT   03H, AL
        MOV   AL, AH
        OUT   03H, AL
        MOV   AL, 49H          ; 单字节读，+1 计数
        OUT   0BH, AL
        MOV   AL, 40H          ; 命令字
        OUT   08H, AL          ; 通道 1，低电平有效，正常时序
        MOV   AL, 01           ; 允许 DMA#1
        OUT   0AH, AL          ; 写屏蔽字
WAITF:  IN    AL, 08H
        TEST  AL, 02
        JZ    WAITF
        MOV   AL, 05           ; 结束，屏蔽 DMA#1
        OUT   0AH, AL
```
