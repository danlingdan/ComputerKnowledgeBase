## 7.4.4 通用串行总线 USB

通用串行总线 USB（Universal Serial Bus）是主机调度的串行外设总线。USB 2.0 的高速模式标称信令速率为 480 Mb/s；后续 SuperSpeed 代际进一步提升速率。USB OTG 允许具备双角色能力的设备协商谁承担主机角色，但通信开始后仍保持主机—设备关系，并不是取消了 USB 的主从控制模型。

## 1. 主要特点

1. 集线器（Hub）使用树形连接，外设安装十分方便，具有真正的“即插即用”（Plug and Play，PnP）特性，用户可以方便地进行外设的连接、安装和拆卸，主机可按外设的增减情况自动配置系统资源，用户可在不关机情况下进行外设的更换，外设驱动程序的安装、删除实现了自动化。

USB 连接外围设备时可使用 Hub 进行树形连接，连接于 USB 上的装置不限定是终端点，也可利用 Hub 连接其他装置的分叉点。此外，它连接的装置之间不是平等关系，上、下游关系明确。
用 USB 连接的外部设备数目最多可达 127 个，节点间连接距离为 5 m，这对一般的计算机系统应用是足够的。

2. 低成本。一方面，USB 外设的设计制造过程比较简单，属于一种开放的、没有专利版权限制的理想工业标准。另一方面，USB 从 1996 年 4 月起并入 Intel 芯片组，从而使设备制造成本降低。
3. USB 2.0 定义低速 1.5 Mb/s、全速 12 Mb/s 和高速 480 Mb/s；这些是信令速率，不等于应用层持续吞吐量。
4. USB 定义**控制、中断、批量和等时**四类传输。控制传输负责枚举与配置；中断传输服务低延迟、小数据量事件；批量传输强调可靠吞吐；等时传输为连续音视频等实时数据预留带宽，但不提供重传保证。
5. 低速、全速和高速链路使用相应的线路编码、同步字段与位填充规则恢复时钟。协议版本与速率代际不同，不能把所有 USB 物理层都概括为同一种 NRZI 编码。
6. 装置和 Hub 可从总线直接获得电源。
7. 作为 PC 标准接口，通过专用芯片实现各种串/并行连接适配功能，市售产品包括 RS-232、I²C/SPI、CAN 总线、有线/无线以太网转换器等，大大方便了微机应用。

## 2. USB 系统的结构框架

一个经典 USB 系统可分为 4 部分：USB 主机、USB 设备、USB Hub 和 USB 互连。USB 系统中的设备与主机的连接方式采用星形连接，其拓扑结构如图 7-52 所示。

Hub 是配线连接器，连接点被称为端口，每个 Hub 把单个连接点转换成多个连接点，这种结构支持多个 Hub 的连接。Hub 只有一个上游端口，其余为下游端口，通过上游端口连到主机，每个下游端口允许连接一个 Hub 或功能设备，直到所有外设加起来达到 127 个为止。多个不同功能的应用设备可作为复合设备放在一起，在主机看来，这个复合设备与一个带着若干设备的单独 Hub 是一样的。主机上有一组插口，主要作为根集线器（Root Hub）。根 Hub 是一个 USB 系统的总控制端口，既可以直接连接外设，也可通过 Hub 扩展控制更多的外设。

## 3. USB 物理接口

标准 USB 连接器分为两类：A 类和 B 类，其外形如图 7-53 所示。A 类连接器适合不经常插拔的设备，如键盘、鼠标、Hub 等；B 类连接器适合经常需要插拔的设备，如打印机、扫描仪、调制解调器等。A 类和 B 类连接器不能互换。每类连接器都具有坚实的屏蔽外壳和易于插拔的特性。所有设备都有一个上行（Up-stream）连接点，Hub 则具有上行和下行（Down-stream）连接点，上游连接器和下游连接器在机械上不可互换，因此消除了 Hub 的非法环路连接。

## 4. USB 通信流与传输方式

USB 在客户软件及其 USB 设备之间提供通信服务，根据客户与 USB 设备之间不同的操作情况，USB 设备可以有不同的通信流要求。通过向一个 USB 设备分配不同的通信流，USB 可以充分利用总线。每个通信流使用总线访问来完成客户与 USB 设备之间的通信。
USB 标准规定了 4 种传输方式：控制传输、等时传输（Isochronous Transfer）、中断传输和数据块（Bulk）传输。每种传输类型确定通信流的多种特性，包括 USB 所用的数据格式、通信流的方向、包规限制、总线访问限制、延迟时间限制、请求的数据序列、错误处理等，以满足不同类型的传输需要。
USB 属于协议总线，通过软件实现各种功能，更多内容可参考 USB 规范（www.usb.org）。

## 5. USB 接口与设备开发

USB 控制器通常需要专门的接口芯片，各类新型微处理器都集成了相应的模块。Intel CPU 系列从 PII 开始即将 USB 控制器集成到芯片组中，可直接驱动应用，并成为 PC 的标准串行接口总线。与一般串行接口类似，CPU 与 USB 控制器进行通信时，可使用输入/输出寄存器。如控制器作为总线设备挂在 I/O 总线（如 PCI）上，还需编程配置相应的寄存器，即 USB 主控制器模块包含两组软件可以访问的寄存器组，即输入/输出寄存器组和可选用的 PCI（总线）配置寄存器。
主要的输入/输出寄存器有命令寄存器和状态寄存器。USB 命令寄存器是一个 16 位寄存器，给出 USB 主控制器要执行的命令，对该寄存器进行一次写入操作将使 USB 主控制器执行一次写入命令。USB 状态寄存器也是 16 位，反映 USB 主控制器在被读取时未处理的中断和所处的状态。
另外，USB 主控制器模块还带有中断请求寄存器（USB INTR）、反映当前帧编号的配置寄存器（FRNUM）、指定帧列表基地址的帧列表基地址配置寄存器（FLBASEADD）、用于启动起始帧（SOF）修正的配置寄存器、反映端口状态（挂起、线路状态、连接状态等）和进行端口控制（如端口复位、传输速率选择、恢复检测等）的配置寄存器（PORTSC）等。

## 6. USB OTG 接口

OTG（On-The-Go）技术 2001 年公布，用于在没有 Host 的情况下，实现不同设备或移动设备间的连接和数据交换，进而使 USB 应用从 PC 外设跨越到了消费电子产品和通信电子产品的领域。根据 USB 接口所属协议，OTG 可分为 3 类。
※ USB2.0 OTG：一种是 Micro 5PIN OTG，常见安卓手机 OTG 接口；另一种是 Mini 5PIN OTG，常见安卓手机和平板 OTG 接口。
※ Micro USB3.0 OTG：部分 2016 年以前的安卓手机 OTG 接口。
※ Type C OTG：部分在 2016 年以后的安卓手机 OTG 接口。
USB OTG 在完全兼容 USB 2.0 的基础上增加了电源管理（节省功耗）功能，允许设备既可作为主机，也可作为外设操作（两用 OTG）。OTG 两用设备完全符合 USB 2.0 标准，并提供一定的主机检测能力，支持主机通令协议（HNP）和对话请求协议（SRP）。在 OTG 中，初始主机设备称为 A 设备，外设称为 B 设备。可用电缆的连接方式来决定初始角色。

## 7.4.5 I²C 与 SPI 串行总线

近年来，多种嵌入式处理器/微控制器（单片机）集成了 SPI 或 I²C 接口，使之成为重要的串行总线，并有专门的转换芯片实现到 UART 或 USB 的转换。

### 1. I²C 总线

I²C（Inter-Integrated Circuit）是由 Philips（后来的 NXP）提出的板级两线串行总线，常用于连接微控制器、传感器、存储器、时钟和电源管理器件。它使用串行数据线 SDA 与串行时钟线 SCL；两条信号线通常采用开漏/开集电极输出并连接上拉电阻，因此可由多个节点共享。

控制器通过 START 条件发起传输并发送目标地址，被控器在地址匹配后应答。节点在一次传输中分别承担发送器或接收器角色；多控制器系统还要处理仲裁与时钟同步。标准模式速率为 100 kbit/s，快速模式为 400 kbit/s，此外还定义了更高速模式。I²C 并没有脱离电气条件的固定“最大距离”或“最大器件数”：可用速率和规模受总线电容、上拉电阻、布线和器件能力共同限制。
在传送数据过程中，I²C 总线共有 3 种信号。

1. 开始信号：SCL 为高电平时，SDA 由高电平向低电平跳变，开始传送数据。
2. 结束信号：SCL 为高电平时，SDA 由低电平向高电平跳变，形成 STOP 条件并结束传输。
3. 应答信号：接收数据的 IC 在接收到 8 位数据后，向发送数据的 IC 发出特定的低电平脉冲，表示已收到数据。CPU 向受控单元发出一个信号后，等待受控单元发出一个应答信号；CPU 接收到应答信号后，根据实际情况做出是否继续传递信号的判断。

送到 SDA 线上的字节必须为 8 位长度，每次传送的字节数不受限制，每字节后必须跟一个应答位。数据传送时，先传送最高位，如果接收设备不能接收下一字节，如在处理一个外部中断，可以把 SCL 线拉成低电平，迫使主机处于等待状态。当从机准备好接收下一字节时再释放时钟线 SCL，使数据传送继续进行。

### 2. SPI 总线

串行外围设备接口 SPI（Serial Peripheral Interface）总线是 Motorola 公司推出的一种同步串行接口。许多微控制器（如 MC68xx）、DSP 及 DSP 控制器（如 TI 公司的 TMS 系列）、ARM 架构处理器等也都配有 SPI 硬件接口，可以方便地实现一个主器件与一个或多个从器件相连，一些采用串行接口的存储器和转换芯片（如 Microchip 公司的 25Cxx E²PROM、Maxim 公司的 DAC MAX538、ADC MAX187 等）也使用该总线，部分应用参见 7.6 节。
SPI 使用两根数据线，一根串行时钟线和一片选信号线。
※ 串行数据输入：主入从出，简称 MISO。
※ 串行数据输出：主出从入，简称 MOSI。
※ 串行时钟：记为 SCLK 或 SCK、SCL。
※ 从器件使能信号：记为 $\overline{SS}$ 或 SS#、NSS。SPI 协议是一种主-从设置，SPI 主控把 $\overline{SS}$ 拉低以开始传输，然后驱动串行时钟 SCLK 把数据同步输入/输出到从设备。SPI 主控通过把 $\overline{SS}$ 拉回到高电平，以终止传输。
SPI 有两种工作模式：主模式和从模式，无论哪种模式，都支持 3 Mb/s 的传输速率，并且提供为传输完成标志和写冲突保护标志置位的功能。在从模式下，可使用简化的三线 SPI，即由 $\overline{SS}$、SCLK、DATA 构成，与 I²C 类似，但多了 $\overline{SS}$。SPI 接口有 4 种时钟模式，反映了 2 个信号 CPOL（时钟极性）和 CPHA（时钟相位）的 2 个状态，以 CPOL、CPHA 的形式表示。
