## 4.3.2 80x86/Pentium CPU 扩展伪指令

宏汇编 MASM 5.0 及以上版本针对 80x86/Pentium CPU 的发展增加了很多伪指令，这里仅介绍与选择 CPU 指令集、存储器工作模式及编程模式等相关的部分伪指令，其余结合汇编语言程序设计方法再讨论。

### 1. 处理器选择伪指令

80x86/Pentium 系列 CPU 的指令集在不断地扩充，编程时，需要在程序的开头处用选择伪指令来说明 CPU 所使用的指令系统，如表 4-7 所示，通知汇编程序当前 CPU 的工作方式和使用的指令集。如果省略，则系统默认为 8086/8088 及 8087 指令集。若想使用 32 位寄存器，则必须加上 .386 及以上处理器的选择伪指令。

**表 4-7 处理器选择伪指令**

| 伪指令 | 功 能 | 伪指令 | 功 能 |
| :--- | :--- | :--- | :--- |
| .286/.286C | 选择 80286 非特权（实地址模式）指令和 80287 协处理器指令 | .286P | 选择全部 80286 指令和 80287 协处理器指令 |
| .386/.386C | 选择 80386 非特权指令和 80387 协处理器指令 | .386P | 选择全部 80386 指令和 80387 协处理器指令 |
| .486/.486C | 选择 80486 非特权指令 | .486P | 选择全部 80486 指令 |
| .586/.586C | 选择 Pentium 非特权指令 | .586P | 选择全部 Pentium 指令 |
| .686/.686C | 选择 Pentium Pro～Pentium 4 非特权指令 | .686P | 选择全部 Pentium Pro～Pentium 4 指令 |
| .287 | 选择 80287 协处理器指令 | .387 | 选择 80387 协处理器指令 |
| .MMX | 选择 MMX 指令 | .XMM | 选择 SSE、SSE2、SSE3、SSE4 指令 |

其中，增加了针对 32 位 CPU 的"字长选择"项，该项有两种选择。

> [!info] 字长选择项 USE16 / USE32
> - **USE16**：16 位段模式（对应实地址模式），字长为 16 位，按 16 位方式寻址（段地址 16 位，偏移地址 16 位），最大段长 64 KB。
> - **USE32**：32 位段模式（对应保护模式），字长为 32 位，按 32 位方式寻址（段选择子 16 位，偏移地址 32 位），最大段长 4 GB。

> [!warning] USE16/USE32 注意事项
> ① **寄存器的位数和段的位数是两个不同的概念**，80386 及以上 CPU 无论选择 USE16 还是 USE32，均可使用 32 位寄存器，即 32 位寄存器的使用是由 `.386`、`.386p` 等伪指令决定的，与 16 位或 32 位段的选择无关。
> ② 字长选择项省略时默认的段属性为 USE32。因此，在实地址模式编程时，一定要先使用 USE16 说明为 16 位段。
> ③ 选择了 USE16，也可以使用 32 位寄存器，按 16 位方式寻址，这时它们的高 16 位为 0。

#### 2. 80x86/Pentium 汇编语言源程序结构

> [!info] 80x86/Pentium 与 8086/8088 源程序框架差异
> 80x86/Pentium 汇编语言源程序较 8086/8088 源程序在框架结构上的主要差异为：
> ① 开始处增加了**处理器选择**；
> ② 段定义中有**字长选择**可选项以确定段模式；
> ③ 允许同时使用代码段 CS、数据段 DS、堆栈段 SS 及附加段 ES、FS 和 GS 共 **6 个段**。

> [!example] 例 4-3-1　两个 32 位带符号数的乘法
> 设在内存的数据段连续存放了两个 32 位（4 字节）的带符号数 DATA1 和 DATA2，编程将它们的 32 位带符号乘积置于 RESULT 的 4 个内存单元中（设结果不超过 32 位）。

```asm
        .586                     ; 定义处理器
DATA    SEGMENT USE16            ; 数据段按 16 位寻址
DATA1   DD    0FFFFFBFBH         ; 被乘数
DATA2   DD    0FFFFF000H         ; 乘数
RESULT  DD    ?                  ; 乘积
DATA    ENDS
STACK   SEGMENT PARA STACK USE16 'STACK'  ; 堆栈段按 16 位寻址
        DB    100 DUP (?)
STACK   ENDS
CODE    SEGMENT USE16            ; 代码段按 16 位寻址
        ASSUME  CS:CODE, DS:DATA
START:  MOV     AX, DATA
        MOV     DS, AX
        MOV     EAX, DATA1
        MOV     EBX, DATA2
        IMUL    EAX, EBX          ; 两个 32 位数相乘→EAX
        JO      ERR               ; 若结果超过 32 位，OF=1，转 ERR
        MOV     RESULT, EAX       ; 否则，结果送存
EXIT1:  MOV     AH, 4CH
        INT     21H
ERR:    MOV     DWORD PTR RESULT, 0  ; 结果单元送 0
        JMP     EXIT1
CODE    ENDS
        END     START
```

### 3. 存储模式定义伪指令

存储模式定义伪指令用来定义程序中数据和代码的存放格式以及它们占用内存的大小。

> [!info] .MODEL 伪指令
> **格式**：`.MODEL  存储模式[, 高级语言]`
> - MASM 有 7 种存储模式可以选择：TINY、SMALL、MEDIUM、COMPACT、LARGE、HUGE、FLAT，含义如表 4-8 所示。
> - `[ , 高级语言 ]` 是可选项，可使用 C、BASIC、FORTRAN、PASCAL 等关键字来指定与哪种高级程序设计语言接口。

**表 4-8 内存模式类型**

| 内存模式 | | 说 明 |
| :--- | :--- | :--- |
| TINY | 微型模式 | 数据和代码放在同一个 64 KB 段内，产生 .com 程序。MASM 6.x 支持 |
| SMALL | 小模式 | 程序包含两个段：1 个 64 KB 代码段和 1 个 64 KB 数据段（一般应用程序常用） |
| MEDIUM | 中模式 | 可以有多个代码段（1 MB 内），但是只有 1 个 64 KB 数据段 |
| COMPACT | 紧凑模式 | 可以有多个数据段（1 MB 内），但是只有 1 个 64 KB 代码段 |
| LARGE | 大模式 | 多个数据段（1 MB 内）与多个代码段（1 MB 内），但数据指针不能跨越 64K 段边界 |
| HUGE | 巨型模式 | 多个数据段（1 MB 内）与多个代码段（1 MB 内），数据指针可以跨越段边界 |
| FLAT | 平展模式 | 允许有 1 个 512 KB 的段，用于存储代码和数据。MASM 6.0 支持 |

> [!info] 存储模式选择要点
> - MS-DOS 环境下的程序只能选用前 6 种模式，第 7 种模式定义的程序只能在 Windows 9x 等 32 位系统环境中运行。
> - 当独立的汇编语言源程序与高级语言混合编程时，存储模式的选择应该与高级语言的一致。
> - 对于**简化段定义**，存储模式定义伪指令不能缺少。

### 4. 80x86/Pentium 简化的段定义伪指令

简化的段定义伪指令用来表示一个段的开始，同时说明前一个段的结束。若这个段是程序中的最后一个段，则该段以 END 伪指令结束。具体的简化段定义语句见表 4-9。

**表 4-9 简化段定义语句**

| 格 式 | 功 能 | 段默认名 |
| :--- | :--- | :--- |
| `.CODE [段名]` | 定义代码段。如果有多个代码段，要用段名区别 | 在 TINY、SMALL、COMPACT 及 FLAT 内存模式下，为 `_TEXT` |
| `.STACK [长度]` | 定义堆栈段，并形成 SS、SP 初值。SP=长度，若省略长度，SP=1024 | 为 `_STACK` |
| `.DATA` | 定义数据段，要赋初值 | 为 `_DATA` |

使用简化段定义的程序框架如下。此程序将变量名为 ABCD 所指的两个双字进行按位"与"运算，结果存入 EFGP 双字单元。

```asm
        .386                      ; 定义处理器方式
        .STACK  64                ; 定义堆栈段长度为 64 字节，并给 SS、SP 赋初值
        .DATA                     ; 定义数据段
ABCD    DD      11112222H, 0000FFFFH ; 定义双字数据
EFGP    DD      0                 ; 运算结果缓冲区
        .CODE                     ; 定义代码段
MAIN:   MOV     AX, @DATA         ; @DATA 对应数据段的段地址
        MOV     DS, AX            ; 建立数据段寄存器 DS 的初值
        MOV     EAX, ABCD         ; 取出第 1 个双字给 EAX
        MOV     EBX, ABCD+4       ; 取出第 2 个双字给 EBX
        AND     EAX, EBX          ; 与运算
        MOV     EFGP, EAX         ; 结果存入 EFGP 中
        MOV     AH, 4CH
        INT     21H               ; 返回 DOS
        END     MAIN              ; 整个程序结束
```

> [!info] SMALL 存储模式下的装入顺序
> SMALL 存储模式下，代码段长度可达 64 KB，数据段和堆栈段为一个组，其总长度可达 64 KB，数据段名 `@DATA` 代表组所对应物理段的段地址。在汇编链接后，程序按代码段（默认 PARA）、数据段（默认 WORD）、堆栈段（默认 PARA）的顺序装入存储区。

> [!warning] CPU 选择伪指令的位置含义与简化段扩展
> ① 80386 及以上 CPU 选择伪指令在源程序中的位置不一样可有不同的含义。如果 CPU 选择伪指令放在 `.MODEL` 伪指令前面，段将被定义成 **32 位的段**；如果希望 CPU 使用 **16 位的段**，则应在 `.MODEL` 伪指令的后面使用 CPU 选择伪指令。
> ② `.STARTUP` 与 `.EXIT 0` 也是简化段定义的伪指令。前者表示程序开始，并初始化 DS、SS、SP，程序结束处的 END 伪指令不必再指定程序入口处的标号；后者相当于 `MOV AH,4CH` 和 `INT 21H` 两条指令。据此，上例的 CODE 部分可以改写成：
> ```asm
>         .CODE                     ; 定义代码段
>         .STARTUP                  ; 程序开始
>         …
>         .EXIT 0                   ; 程序结束，返回 DOS
>         END                       ; 整个程序结束
> ```
