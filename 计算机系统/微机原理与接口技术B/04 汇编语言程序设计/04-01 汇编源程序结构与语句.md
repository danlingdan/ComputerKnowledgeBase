## 4.1 程序设计语言概述

> [!note] 示例环境
> 本章示例主要面向 MASM 5.x/6.x、16 位 DOS/BIOS 与部分 32 位 x86 教学环境。伪指令、目标文件格式、调用约定和系统服务均依赖工具链；迁移到 x86-64 或现代操作系统时不能只替换寄存器名称。

用于计算机程序设计的语言可分为机器语言、汇编语言和高级语言三种。

> [!abstract] 三种程序设计语言对比
> - **机器语言**：指令与数据均用二进制编码表示，CPU 可直接识别与执行；面向机器，编写、阅读困难，随 CPU 型号而异，通用性差。任何语言最终都需转换成机器语言才能由 CPU 执行。
> - **高级语言**：面向过程，采用接近自然语言和数学语言的语法，不依赖具体机器；可移植性好。但其源程序须经编译程序翻译成目标程序（机器语言程序）才能运行，编译后目标程序较大、执行时间较长，难以充分利用 CPU 特性（如寄存器、标志、中断），限制了它在实时数据采集、检测与控制等场合的运用。
> - **汇编语言**：采用助记符表示的符号化程序设计语言，面向机器；一个助记符通常表示一条机器指令（如 `MOV`、`ADD`），方便书写、阅读、记忆与检查。源程序与目标程序之间有一一对应关系。

> [!info] 汇编语言适用场合与代价
> - **优势**：能利用计算机硬件特性（寄存器、标志、中断等），直接处理位、字节、字、寄存器、存储单元或 I/O 端口，直接使用 CPU 指令系统的寻址方式；占用内存少、执行速度快。
> - **典型应用**：系统软件、实时通信、实时控制、I/O 接口驱动程序；或与高级语言混合使用。
> - **代价**：与硬件结构强相关，与数学模型关系不直观，设计难度大、时间长、出错率高，对程序员要求高。

汇编语言源程序在交付计算机执行前需要先翻译成目标程序，这个编译过程称为**汇编**，完成汇编任务的工具被称为**汇编程序**（Assembler）。汇编程序是计算机系统软件之一。

> [!info] 汇编程序的功能
> 除将汇编语言源程序翻译成机器代码外，还能够：
> - 根据用户要求自动分配存储区域（程序区、数据区等）；
> - 自动将各种进制数转换成二进制数、将字符转换成 ASCII 码、计算表达式的值；
> - 自动对源程序进行检查，给出错误信息（非法格式、未定义的助记符、标号、漏掉操作数等）。
>
> 在使用汇编语言编程之前应首先熟悉相应的汇编程序。

在 C 语言源程序变成可执行目标代码过程中，需要经过编译和汇编两个阶段，常用的汇编程序有以下几种。

> [!info] 常用汇编程序对比
> | 名称 | 全称/厂商 | 主要特点 | 版本支持 |
> | :--- | :--- | :--- | :--- |
> | **GAS** | GNU 汇编器（GCC 编译器套件） | 支持多处理器（x86、ARM、MIPS 等），多数类 UNIX 系统标准汇编器 | 2.31；2.10 起支持 `intel_syntax` 伪指令 |
> | **MASM** | 微软宏汇编语言 | 支持宏操作、条件汇编、结构、记录等高级功能 | 4.0→80286，5.0→80386，6.11→Pentium，6.13→MMX，6.14/6.15→SSE/SSE2/SSE3，7.0 集成于 VC++.NET 2002，8.0 支持 64 位 |

## 4.2 汇编语言的程序结构与语句格式

一个源程序必须用其编译器能够理解的语法进行编写，这些语法是指语句和程序结构必须遵循的规则，汇编语言源程序的编写也是如此。从本节开始介绍 Microsoft MASM 汇编程序的基本语法规则。

### 4.2.1 汇编语言源程序的框架结构

首先，通过以下可在 MS-DOS 环境下运行的 8086/8088 汇编语言源程序实例，初步认识标准的具有完整段定义格式的汇编语言源程序的框架结构。

> [!example] 例 4-1
> 打印输出字符串 “THIS IS A SAMPLE PROGRAM。”

```asm
; SAMPLE PROGRAM DISPLAY MESSAGE      ; 注释行
DATA      SEGMENT                      ; 定义数据段
MESSAGE   DB    'THIS IS A SAMPLE PROGRAM. '
          DB    0DH, 0AH, '$'          ; 在存储器中存放供显示的数据
DATA      ENDS

STACK     SEGMENT PARA STACK 'STACK'   ; 定义堆栈段
          DB    1024 DUP (0)           ; 在存储器的某个区域建立一个堆栈区
STACK     ENDS

CODE      SEGMENT                      ; 定义代码段
          ASSUME  CS:CODE, DS:DATA, SS:STACK ; 段寄存器说明
MAIN      PROC    FAR                  ; 将程序定义为远过程
START:    PUSH    DS
          MOV     AX, 0
          PUSH    AX                   ; 标准序，以便返回操作系统
          MOV     AX, DATA
          MOV     DS, AX               ; 取数据段地址
                                       ; 建立数据段的可寻址性
LOOP1:    LEA     BX, MESSAGE          ; MESSAGE 地址偏移量→BX
          CMP     BYTE PTR [BX], '$'
          JE      LOOP2
          MOV     AH, 5
          MOV     DL, [BX]             ; 5 号 DOS 系统功能调用打印输出字符
          INT     21H
          INC     BX
          JMP     LOOP1
LOOP2:    RET                          ; 返回 DOS 操作系统
MAIN      ENDP                         ; 过程结束
CODE      ENDS                         ; 代码段结束
          END     START                ; 程序从 START 开始，汇编结束
```

鉴于 80x86/Pentium CPU 采用存储器分段管理方式，其汇编语言源程序都以逻辑段为基础，按段来组织代码和数据。源程序一般具有代码段、堆栈段、数据段和附加数据段，通常只有代码段是必不可少的。

```mermaid
flowchart TB
    SRC["汇编源程序 .asm"] --> SEG["若干逻辑段"]
    SEG --> CODE["代码段 CODE（必不可少）"]
    SEG --> DATA2["数据段 DATA（存放常数/变量）"]
    SEG --> EXTRA["附加段 ES（数据工作区）"]
    SEG --> STACK2["堆栈段 STACK（中断/子程序调用/参数传递）"]
    CODE --> ASSUME["ASSUME 伪指令：说明段寄存器与逻辑段的关系"]
    CODE --> INIT["对 DS、ES 等段寄存器赋初值"]
    CODE --> DOS["返回 DOS 的指令序列"]
```

> [!info] 80x86/Pentium 源程序结构特点
> 1. 源程序由若干逻辑段组成，各逻辑段都有段名，由段定义伪指令定义与说明。
> 2. 在代码段起始处，用 `ASSUME` 伪指令说明各段寄存器与逻辑段的关系；用户自己设置各段寄存器（CS 除外）初值，建立逻辑段的可寻址性。
> 3. 每个逻辑段由若干行汇编语言语句组成，每行只有一条语句且不能超过 128 个字符，一条语句允许有续行；整个源程序必须以 `END` 语句结束，其后的标号 `START` 表示执行起始地址。
> 4. 每一条汇编语言语句最多由 4 个字段组成，各字段间用空格或 Tab 隔开。
> 5. 每个源程序在代码段中往往需要含有返回 MS-DOS 操作系统的指令语句，以保证程序执行完后能自动回到 DOS 状态。

> [!tip] 数据段定义在代码段之前的原因
> 通常将数据段定义在代码段前面，以产生正确的机器代码：① 程序中所使用的变量应先定义；② 汇编程序在汇编过程遇到变量时要知道变量的属性。

> [!info] 各 CPU 同时可用的段数与段长
> | CPU | 同时可用段数 | 段数构成 | 实模式段最大长度 | 保护模式段最大长度 |
> | :--- | :--- | :--- | :--- | :--- |
> | 8086/8088/80286 | 4 | CS、SS、DS、ES | 64 KB | — |
> | 80386～Pentium | 6 | CS、SS、DS、ES、FS、GS | 64 KB | 80286: 16 MB；80386+: 4 GB |

#### 返回 DOS 的常用方法

终止当前程序、使其正确返回到 DOS 状态的常用方法有如下两种。

> [!example] 方法一：标准序法
> 采用标准源程序框架结构：先将主程序定义为 FAR 过程，其最后一条指令为 RET；再在主程序开始处使用以下三条指令：
> ```asm
> PUSH  DS          ; 保护程序段前缀 PSP 的段地址
> MOV   AX, 0
> PUSH  AX          ; 保护 PSP 的 0 偏移地址
> ```
> 程序执行完毕后自动返回 DOS。上例例 4-1 使用的就是这种"标准序"法。

> [!example] 方法二：DOS 4CH 号功能调用
> 不需定义主程序为 FAR 过程并去掉上述三条指令，只在代码段结束（`CODE ENDS`）前增加调用语句：
> ```asm
> MOV   AH, 4CH     ; 功能号 4CH→AH
> INT   21H         ; 中断调用
> ```
> 程序执行完毕后自动返回 DOS，这是因为执行了功能号为 4CH 的 DOS 系统功能调用。

> [!warning] MASM 5.0+ 简化段定义
> MASM 汇编程序从 5.0 版开始还支持简化段定义，通过伪指令 `EXIT` 返回 DOS。

### 4.2.2 汇编语言的语句

语句是汇编语言源程序的基本组成单位，源程序是一个语句序列。语句序列中的每条语句规定了一个基本操作要求，语句序列则完成某个特定的操作任务。

#### 1. 语句的种类与格式

##### 1. 语句的种类

> [!abstract] 80x86/Pentium 汇编语言的三种基本语句
> | 语句种类 | 出现位置 | 是否产生机器代码 | 是否可执行 | 实例 |
> | :--- | :--- | :--- | :--- | :--- |
> | **指令语句** | 代码段 | 是（一一对应机器代码） | 是 | `MOV DS, AX`（机器代码 8EH 和 D8H） |
> | **伪指令语句** | 任一段 | 否 | 否（CPU 不执行） | `SEGMENT/ENDS`、`MESSAGE DB '...'` |
> | **宏指令语句** | 任一段（宏扩展处） | 是（宏扩展后插入宏体） | 是 | 一般性指令语句的扩展 |

##### 2. 语句的格式

指令语句与伪指令语句的格式类似，由 4 部分组成（宏指令语句格式在稍后专门介绍）。

> [!info] 语句的一般格式
> - **指令语句**：`[标号:] [前缀] 指令助记符 [操作数] [;注释]`
> - **伪指令语句**：`[名字] 伪指令定义符 [操作数] [;注释]`
>
> 其中“`[ ]`”中的内容为可选部分；操作数部分或是 0、1 个操作数，或是由“`,`”隔开的多个操作数。

**1. 标号和名字**

标号和名字分别是指令单元和伪指令单元所起的符号名称，是用户自定义的标识符。

> [!important] 标号与名字的区别
> - **指令语句的标号**后面必须有“`:`”；代表指令所在存储单元的符号地址，可作为转移（JMP）、循环（LOOP）等指令的转移目标，与具体的指令地址相联系。
> - **伪指令语句的名字**后面不能有“`:`”；一般用于定义常量、变量、过程、段名等，指示所定义变量、过程以及段的起始地址。

> [!warning] 标识符规则
> 标号和名字必须符合目标 MASM 版本的标识符规则，通常可由字母、数字以及 `?`、`@`、`$`、`_` 等字符组成，并且不能与寄存器名、指令助记符或其他保留字冲突。允许的首字符、有效长度和大小写规则可能随汇编器版本变化，应以所用 MASM 版本为准。

**2. 助记符和定义符**

助记符和定义符分别规定指令语句的操作性质和伪指令语句的伪操作功能，是语句中唯一不可省略的。指令助记符前面可根据需要加前缀，80x86/Pentium 指令系统中允许与助记符同时使用的前缀是**重复前缀** `REP`、`REPE` 及**段超越前缀**等。

**3. 操作数**

指令语句中的操作数提供该指令的操作对象，并说明要处理的数据存放在什么位置以及如何访问它。伪指令语句中操作数的格式和含义随伪操作命令的不同而不同。

**4. 注释**

注释由“`;`”开始，对程序的功能加以说明，以构成源程序的编程文档，不被汇编。

#### 2. 语句中的操作数

根据寻址方式等因素的不同，语句中操作数的表现形式有**常量**、**寄存器**、**存储器**、**表达式**。

##### 1. 常量操作数

在汇编时，已经确定其值且程序运行期间不变的量称为常量，如语句中的立即数或端口地址等。

> [!info] 4 种整型数值常量
> 4 种整型数值常量只需在数的后面加上字母 `B`、`D`、`Q` 和 `H` 即可定义。例如，十进制数 9 可表示为：$1001B$、$9D$、$9H$、$11Q$。
> - **十进制**后缀 `D` 可省略；
> - **十六进制**的第一个数字必须在 0～9 之间，故 $9D$ 可写作 $9$；$ABH$ 必须写作 $0ABH$。
> - 在汇编语言源程序中常用十六进制表示数据和地址。

> [!info] 其他常量
> - **字符串常量**：用单引号括起来的一个或多个字符，其值为每个字符对应的 ASCII 值。如 `'2'` 的值为 $32H$，`'AB'` 的值为 $4142H$；可与整型数值常量交替使用。
> - **符号常量**：在程序中用标识符形式表示的常量，可提高程序的通用性。
> - 其余常量操作数还有十进制浮点数和十六进制实数。

##### 2. 寄存器操作数

指操作数部分是寄存器名，如 `AX`、`SI`、`DS` 等。

##### 3. 存储器操作数

存储器操作数是地址型操作数，有**标号**和**变量**两种。标号是指令所存放单元的符号地址，一定在代码段内。变量一般在数据段或堆栈段中，是可以用存储器寻址方式访问的操作数。

> [!important] 标号和变量的三种属性
> 1. **段属性 SEGMENT**：标号和变量对应存储单元的段地址，是段基地址的高 16 位。
> 2. **偏移量属性 OFFSET**：标号和变量对应存储单元的偏移地址。
> 3. **类型属性 TYPE**：
>    - 标号的类型是指其距离属性：**NEAR**（近距离属性，用于段内转移或调用）和 **FAR**（远距离属性，用于段间转移或调用）；
>    - 变量的类型是指变量占用存储单元的字节数：**BYTE**（字节）、**WORD**（字）、**DWORD**（双字）等。

##### 4. 表达式操作数

表达式由各种操作数、运算符和操作符组成，根据其表示内容的不同又分为数值表达式与地址表达式两类。汇编时，汇编程序对表达式进行运算，得到一个确定的值。

**1. 数值表达式和地址表达式**

> [!info] 两类表达式
> - **数值表达式**：由数值常量、字符串常量或符号常量等与算术、逻辑或关系运算符等连接而成；运算结果是一个数值（仅有大小而无其他属性），可作为指令中的立即操作数或数据区中的初值使用。
> - **地址表达式**：由常量、变量、标号、寄存器（如 BX、BP、SI、DI）的内容以及一些运算符组成；运算结果表示存储器地址（一般是段内的偏移地址），具有段属性、偏移量属性和类型属性。如 $ES:[SI+4]$、$PLACE+2*3$。

算术运算符实现加、减、乘、除和求余的算术运算。它们都可用于数值表达式，只有 $+$、$-$ 运算符可用于地址表达式。

> [!tip] "标号或变量±常量"的类型属性
> 当标号或变量加上或减去某个常量时，结果仍为标号或变量，其三种属性中的**类型与段地址属性不变**，偏移量属性被修改。例如，$PLACE+2*3$ 是指 $PLACE$ 字节单元后的第 6 个存储单元的地址。

逻辑运算符实现按位相与、相或、异或、求反的逻辑运算，只适用于数值表达式。

> [!warning] 逻辑运算符与逻辑运算指令的区别
> 两者在符号形式上相同，但含义有本质差异：
> - 作为**逻辑运算符**时，在程序汇编时由汇编程序计算，结果构成指令操作数的一部分；操作对象只能是整型常量。
> - 作为**指令助记符**时，在程序运行时被执行，操作对象还可以是寄存器或存储器操作数。

**2. 运算符和操作符**

MASM 宏汇编中有三类运算符（算术、逻辑和关系运算符）和两类操作符（分析和合成操作符），如表 4-1 所示。运算符实现对操作数的相关运算，操作符则完成对操作数属性的定义、调用和修改。

**表 4-1 MASM 支持的运算符和操作符**

| 类型 | 符号 | 名称 | 运算结果 | 实例 |
| :--- | :--- | :--- | :--- | :--- |
| 算术 | $+$ | 加法 | 和 | $5+3=8$ |
| | $-$ | 减法 | 差 | $8-3=5$ |
| | $*$ | 乘法 | 乘积 | $3*5=15$ |
| | $/$ | 除法 | 商 | $22/5=4$ |
| | $MOD$ | 模除 | 余数 | $12 MOD 3=0$ |
| | $SHL$ | 左移 | 左移后的二进制数 | $0010B SHL 2=1000B$ |
| | $SHR$ | 右移 | 右移后的二进制数 | $1100B SHR 1=0110B$ |
| 逻辑 | $NOT$ | 非运算 | 逻辑非结果 | $NOT 1010B=0101B$ |
| | $AND$ | 与运算 | 逻辑与结果 | $1011B AND 1100B=1000B$ |
| | $OR$ | 或运算 | 逻辑或结果 | $1011B OR 1100B=1111B$ |
| | $XOR$ | 异或运算 | 逻辑异或结果 | $1010B XOR 1100B=0110B$ |
| 关系 | $EQ$ | 相等 | 结果为真——输出全 1；结果为假——输出全 0 | $5 EQ 11B=全 0$ |
| | $NE$ | 不等 | | $5 NE 11B=全 1$ |
| | $LT$ | 小于 | | $5 LT 3=全 0$ |
| | $LE$ | 不大于 | | $5 LE 101B=全 1$ |
| | $GT$ | 大于 | | $5 GT 100B=全 1$ |
| | $GE$ | 不小于 | | $5 GE 11B=全 0$ |
| 分析 | $SEG$ | 返回段地址 | 段地址 | $SEG N1=N1 所在段段地址$ |
| | $OFFSET$ | 返回偏移地址 | 段偏移地址 | $OFFSET N1=N1 的偏移地址$ |
| | $LENGTH$ | 返回变量单元数 | 单元数 | $LEGTH N2=N2 单元数$ |
| | $TYPE$ | 返回元素字节数 | 字节数 | $TYPE N2=N2 中元素字节数$ |
| | $SIZE$ | 返回变量总字节数 | 总字节数 | $SIZE N2=N2 总字节数$ |
| 合成 | $PTR$ | 修改类型属性 | 修改后类型 | $BYTE PTR [BX]$ |
| | $THIS$ | 指定类型/距离属性 | 指定后类型 | $ALPHA EQU THIS BYTE$ |
| | $:$ | 段寄存器名 | 修改段 | $ES:[BX]$ |
| | $HIGH$ | 分离高字节 | 高字节 | $HIGH 3355H=33H$ |
| | $LOW$ | 分离低字节 | 低字节 | $LOW 3355H=55H$ |
| | $SHORT$ | 短转移说明 | 转移距离在 $-128 \sim +127$ | $JMP SHORT LABEL$ |
| 其他 | $( )$ | 圆括号 | 改变运算符优先级 | $(8-3)*3=15$ |
| | $[ ]$ | 方括号 | 下标或间接寻址 | $MOV AX, [BX]$ |
| | $< >$ | 点运算符 | 连接结构与成员变量 | $TAB.T1$ |
| | $.$ | 尖括号 | 修改变量 | $< 8, 5>$ |
| | $MASK$ | 返回字段屏蔽码 | 字段屏蔽码 | $MASK C$ |
| | $WIDTH$ | 返回记录宽度 | 记录/字段位数 | $WIDTH W$ |

关系运算符用于比较和测试符号数值。MASM 用 FFH 或 FFFFH 表示关系成立(为真)，用 00H 或 0000H 表示关系不成立（为假）。结果值在汇编时获得。例如：

```asm
MOV  BX, PORT LT 5   ; 若 PORT 值小于 5，汇编结果为：MOV BX, 0FFFFH
                     ; 否则，汇编结果为：MOV BX, 0000H
```

分析操作符与合成操作符。分析操作符又称为数值返回运算符，它的运算对象是存储器操作数，返回标号或变量的属性值。合成操作符作用于存储器操作数时可以改变它们的属性，又称为修改属性操作符。这两种操作符在后面会有详细介绍。

**3. 运算符和操作符的优先级**

运算符和操作符的优先级如表 4-2 所示。优先级相同时顺序为先左后右。

**表 4-2 运算符和操作符的优先级**

| 优先级 | 运算符和操作符 |
| :--- | :--- |
| 1 (高) | $LENGTH, SIZE, WIDTH, MASK, ( ), [ ], < >, .$ |
| 2 | $PTR, OFFSET, SEG, TYPE, THIS, 段寄存器名, (加段前缀)$ |
| 3 | $HIGH, LOW$ (操作数高、低字节) |
| 4 | $+, -$ (单目) |
| 5 | $*, /, MOD$ (求模), $SHL, SHR$ |
| 6 | $+, -$ (双目) |
| 7 | $EQ, NE, LT, LE, GT, GE$ |
| 8 | $NOT$ |
| 9 | $AND$ |
| 10 | $OR, XOR$ |
| 11 (低) | $SHORT$ |
