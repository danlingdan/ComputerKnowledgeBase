## 4.6 综合应用程序设计举例

80x86/Pentium CPU 有多种工作模式。8086/8088 的工作模式是实模式，也是 80286～Pentium 兼容 8086/8088 并复位后默认的工作模式；通过设置相应的寄存器，80286～Pentium 可进入保护模式，充分发挥 CPU 的全部性能；如果希望在保护模式下运行一个或多个 16 位的程序，可将系统切换到虚拟 8086 模式。本节进一步说明实模式与保护模式下的编程方法。

### 4.6.1 16 位实模式程序设计

实模式的默认条件下，MASM 只接受 8086/8088 的 16 位指令集。其编程的主要特点是，只能使用 8 位或 16 位寄存器，对内存寻址只能使用 SI、DI 及 BX、BP 寄存器。

> [!example] 例 4-18
> 多位非压缩的 BCD 数算术运算。

内存区从 FIRST 和 SECOND 开始的单元中分别存放两个 4 位非压缩 BCD 数，数据存放的规则是，低位在低地址，高位在高地址。求这两个数的和并存放在 THIRD 开始的内存单元。两个 4 位的 BCD 数相加，最高位可能产生进位。为了避免丢失这个进位，给加数、被加数及结果存放单元都开辟 5 个字节单元。程序代码如下：
```assembly
DATA    SEGMENT
FIRST   DB    4, 7, 9, 2, 0             ; 十进制数 02974
SECOND  DB    6, 2, 3, 7, 0             ; 十进制数 07326
THIRD   DB    5 DUP(0)                  ; 存放结果
DATA    ENDS
CODE    SEGMENT
        ASSUME CS:CODE, DS:DATA, ES:DATA
UNPACK: MOV    AX, DATA
        MOV    DS, AX                   ; DS 指向数据段
        MOV    ES, AX                   ; ES 指向数据段
        LEA    SI, FIRST
        LEA    BX, SECOND
        LEA    DI, THIRD
        MOV    CX, 4+1                  ; 进行 5 次加法
        CLD
RETRY:  LODSB                           ; 对应的 1 位进行加法运算
        ADD    AL, [BX]
        AAA
        STOSB                           ; 保存当前值
        INC    BX
        ADC    BYTE PTR[BX], 0          ; 进位值加到加数的下一位
        LOOP   RETRY
        MOV    AH, 4CH
        INT    21H
CODE    ENDS
        END    UNPACK
```

> [!example] 例 4-19
> BCD 码转换为二进制数。

将 AX 中的 4 位 BCD 码转换为二进制数，结果仍存于 AX 中。
BCD 码用 4 位二进制数表示 1 位十进制数的编码，所以 AX=9827H 表示值为 9827D。4 位 BCD 码 $D_3 D_2 D_1 D_0$（0～9999）对应的二进制数应为：
$$
(((D_4 \times 10 + D_3) \times 10 + D_2) \times 10 + D_1) \times 10 + D_0 \quad (\text{设 } D_4=0)
$$
显然，完成转换需操作 4 次“×10”后再取数累加，因此设计如下含有 4 次循环的程序。
```assembly
W10     DW    10                       ; 十进制数权值
; 子程序名：BTO2。入口参数：AX=BCD 码。出口参数：AX=二进制数。
BTO2    PROC  NEAR
        …
        MOV    BX, AX                  ; 保护现场
        MOV    AX, 0                   ; 保存 AX 中的 BCD 码到 BX
        MOV    CX, 4                   ; 结果单元清 0
RETRY:  PUSH   CX                      ; 共处理 4 位 BCD 码
        MOV    CL, 4
        ROL    BX, CL                  ; 1 位 BCD 码左移到 BX 中的低半字节
        POP    CX
        MUL    W10                     ; AX×10→DX:AX
        PUSH   BX                      ; 保存 BX 当前值
        AND    BX, 000FH               ; 取出 BX 中的 1 位 BCD 码
        ADD    AX, BX                  ; 累加到 AX 中
        POP    BX                      ; 恢复 BX
        LOOP   RETRY                   ; 4 次循环未做完，继续
        …                             ; 恢复现场
        RET
BTO2    ENDP
```

> [!example] 例 4-20
> 二进制数转换为 BCD 码。

将 AX 中的二进制数转换成 4 位 BCD 码，转换的结果仍存于 AX 中（设 AX 中的数值小于十进制数 10000）。
算法：将 AX 中的二进制数除以 1000 得到的商是千位上的 BCD 码，所得余数除以 100 得到的商是百位上的 BCD 码，所得余数再除以 10 得到的商是十位上的 BCD 码，最后所得的余数是个位上的 BCD 码。显然，完成转换需要操作 4 次“被除数÷权值后再整合”。
```assembly
W1000   DW    1000, 100, 10, 1          ; 十进制数千、百、十、个位的权值
; 子程序名：AX2TOB。入口参数：AX=二进制数。出口参数：AX=压缩 BCD 码。
AX2TOB  PROC  NEAR
        …                             ; 保护现场
        XOR    BX, BX                  ; 存放 BCD 码的临时单元清 0
        MOV    SI, OFFSET W1000        ; 权值所在首地址送 SI
        MOV    CX, 4                   ; 循环次数 4→CX
RETRY:  PUSH   CX
        MOV    CL, 4
        SHL    BX, CL                  ; 先转换高位 BCD 码
        MOV    DX, 0                   ; DX:AX 组成被除数
        DIV    WORD PTR [SI]           ; 除以权值，商→AX,余数→DX
        OR     BX, AX                  ; BX 中为压缩 BCD 码
        MOV    AX, DX                  ; 余数送 AX
        POP    CX
        ADD    SI, 2                   ; 地址加 2，指向下一个权值
        LOOP   RETRY                   ; 连续转换 4 次
        MOV    AX, BX                  ; BCD 码由 BX→AX
        …                             ; 恢复现场
        RET
AX2TOB  ENDP
```
二进制数转换成 BCD 码的另一种算法是“连续除以 10 取余”：首先将 AX 中的数除以 10，所得余数为个位上的 BCD 数；所得的商再作为被除数来除以 10，得到的余数为十位上的 BCD 数……直到被除数小于 10 时，得到最高位的 BCD 数。可据此另编制程序实现这个转换。

> [!example] 例 4-21
> 从键盘输入字符。若是回车键，则退出程序；若是小写字母，则显示对应的大写字母，否则直接显示。

A～Z 字符对应的 ASCII 值为 41H～5AH，a～z 字符对应的 ASCII 值为 61H～7AH。所以，要将小写字母转换为大写字母，只需将其 ASCII 值减去 20H 即可。程序设计如下：
```assembly
.MODEL  SMALL
.STACK
.CODE
.STARTUP
START:  MOV    AH, 01H                 ; 等待从键盘上输入一个字符
        INT    21H                     ; 按下键的 ASCII 值存放在 AL 中
        CMP    AL, 0DH                 ; 判断按下的键是否为回车键
        JZ     LP2                     ; 是，转移到 LP2，退出程序
        CMP    AL, 61H                 ; 不是，与 'a' 比较
        JC     LP1                     ; 若 AL<'a'，则转去直接送显
        CMP    AL, 7BH                 ; 与 'z' 比较
        JNC    LP1                     ; 若 AL>'z'，则转去直接送显
        SUB    AL, 20H                 ; 否则在 'a'～'z' 之间，减去 20H，变为大写字母
LP1:    MOV    DL, AL                  ; DL=欲显示字符的 ASCII 值
        MOV    AH, 02H                 ; 设置 02H 号功能，显示字符
        INT    21H
        JMP    START                   ; 重新读取新的键值，判断…
LP2:    .EXIT  0
END
```

> [!example] 例 4-22
> 在字节数组中找出第一个非零元素，并显示它的下标。
```assembly
.MODEL  SMALL
.STACK
.DATA
ARRAY   DB    0, 0, 0, 25H, 0, 0, 34H, 36H, 45H
COUNT   EQU   9
.CODE
.STARTUP
        MOV    CX, COUNT
        MOV    DI, -1                  ; DI 为元素下标
NEXT:   INC    DI                      ; 从 0 号元素起，逐个与 0 比较
        CMP    ARRAY[DI], 0
        LOOPZ  NEXT
        JNE    OK                      ; ZF=0，即有非零元素，转 OK
        MOV    DL, '0'                 ; 没有找到非零元素，显示一个 '0'
        JMP    SHOW
OK:     MOV    DX, DI
        OR     DL, 30H                 ; 将下标转换成相应的 ASCII 值
SHOW:   MOV    AH, 02H                 ; 显示下标
        INT    21H
        .EXIT  0
END
```

> [!example] 例 4-23
> 冒泡法给无序表中的元素排序。

排序的算法有很多，冒泡排序是最常用的一种。其方法是：对无序表从前至后，两两元素比较，逆序则交换。设从地址 ARRAY 开始的缓冲区中有一个字数组，要使该数据表中的 $N$ 个带符号数按照从小到大的次序排列，用冒泡算法实现的过程如下。
从第 1 个数开始依次进行相邻两个数的比较，即第 1 个数与第 2 个数比，第 2 个与第 3 个比……若两个数的次序是顺序（符合排序要求），则不做任何操作；若是逆序，就交换这两个数的位置。经过一遍全表扫描的 $N-1$ 次比较后，最大的数排到了表的最后。用同样的方法再进行第二遍扫描，这时只需考虑前 $N-1$ 个数之间的 $N-2$ 次比较，扫描完毕，次大的数排到了后面……以此类推，在进行了 $N-1$ 遍的扫描比较后则完成了排序。
冒泡法最大可能的扫描遍数为 $N-1$。但是，往往有的数据表在第 $I$ 遍（$I < N-1$）扫描后可能已经成序。为了避免后面不必要的扫描比较，可在程序中引入一个交换标志。若在某一遍扫描比较中一次交换也未发生，则表示数据已按序排列，在这遍扫描结束时就停止程序循环，结束排序过程。用双重循环实现的冒泡算法程序清单如下：
```assembly
.MODEL  SMALL
.STACK
.DATA
ARRAY   DW    d1, d2, d3, …, dn        ; 定义带符号数的字数组
COUNT   EQU   ($-ARRAY)/2              ; 表中的数据个数
FLAG    DB    -1                       ; 交换标志，初值为 -1
.CODE
.STARTUP
SORT:   MOV    BX, COUNT               ; 元素个数→BX
LP1:    CMP    FLAG, 0                 ; 数组已有序？
        JE     EXIT1                   ; 是，排序结束
        DEC    BX                      ; 否，置本遍扫描比较（内循环）次数
        MOV    CX, BX
        MOV    SI, 0                   ; 置数组的偏移地址
        MOV    FLAG, 0                 ; 预置交换标志 FLAG 为 0
LP2:    MOV    AX, ARRAY[SI]           ; 取一个数据→AX
        CMP    AX, ARRAY[SI+2]         ; 与下一个数比较
        JLE    NEXT                    ; 前一个数<=后一个数，顺序，转 NEXT
        XCHG   AX, ARRAY[SI+2]         ; 逆序，交换两个数
        MOV    ARRAY[SI], AX
        MOV    FLAG, -1                ; 置交换标志 FLAG 为 -1
NEXT:   ADD    SI, 2                   ; 修改地址指针
        LOOP   LP2                     ; 内循环进行两两数据的比较
        JMP    LP1                     ; 内循环结束，继续下一遍排序
EXIT1:  .EXIT  0                       ; 排序完成，返回 DOS
END
```

### 4.6.2 基于 32 位指令的实模式程序设计

如果针对 80386 及以上 CPU 来编写基于 32 位指令的实模式程序，先要用 CPU 选择伪指令来说明所使用的 32 位 CPU，并以选择 USE16 段模式（完整段定义）或在 CPU 选择伪指令之前加 `.MODEL` 伪指令（简化段定义）来明确是在实模式下编写程序。这时可以使用所有的 8、16 和 32 位寄存器，32 位操作数，且所有 32 位寄存器都可用于内存寻址（按 16 位方式在 1MB 内存空间中寻址，分段最大 64 KB）。

> [!example] 例 4-24
> 将一个 64 位数据逻辑左移 8 位。
```assembly
.386                             ; 选择处理器
DATA    SEGMENT USE16            ; 数据段以 16 位寻址
QVAR    DQ    1234567887654321H  ; 定义 8 字节变量
DATA    ENDS
CODE    SEGMENT USE16            ; 代码段以 16 位寻址
        ASSUME CS:CODE, DS:DATA
SHIFT:  MOV    AX, DATA
        MOV    DS, AX
        MOV    ESI, OFFSET QVAR
        MOV    EAX, DWORD PTR [ESI]    ; 取低 32 位
        MOV    EDX, DWORD PTR [ESI+4]  ; 取高 32 位
        MOV    CX, 8                   ; 设置循环次数
AGAIN:  SHL    EAX, 1                  ; 低 32 位逻辑左移
        RCL    EDX, 1                  ; 高 32 位带进位循环左移
        LOOP   AGAIN                   ; 不足 8 次继续移位
        MOV    DWORD PTR QVAR, EAX     ; 保存结果低 32 位
        MOV    DWORD PTR QVAR[4], EDX  ; 保存结果高 32 位
        MOV    AH, 4CH
        INT    21H
CODE    ENDS
        END    SHIFT
```

> [!example] 例 4-25
> 采用冒泡法，将 ESI 指定的缓冲区中 ECX 个 32 位带符号数按从大到小排序。
```assembly
.MODEL  SMALL
.386                             ; 16 位段
.STACK
.DATA
COUNT   EQU   10
DARRAY  DD    20, 4500H, 3F40H, -1, 7F000080H, 81000000H, 0FFFFFFFFH, -45000011, 12345678, 87654321
.CODE
.STARTUP                         ; 程序开始
        XOR    ESI, ESI
        MOV    SI, OFFSET DARRAY
        MOV    ECX, COUNT-1
OUTLP:  MOV    EDX, 0                  ; 外循环
INLP:   CMP    EDX, ECX                ; 内循环
        JAE    BOTM
        MOV    EAX, [ESI+EDX*4+4]
        CMP    [ESI+EDX*4], EAX        ; 比较前后两个数据的大小
        JGE    NSWAP
        XCHG   [ESI+EDX*4], EAX
        MOV    [ESI+EDX*4+4], EAX
NSWAP:  INC    EDX
        JMP    INLP
BOTM:   LOOP   OUTLP
        .EXIT  0                       ; 程序结束，返回
END
```

### 4.6.3 基于多媒体指令的实模式程序设计

只有 Pentium MMX 及以上的 CPU 才支持 MMX、SSE～SSE4 等多媒体指令。其中，MMX 指令集只能处理紧缩整型数据，而 SSE～SSE4 指令集则提供了紧缩浮点型数据的并行处理能力。这里仅介绍 MMX 指令的编程应用。

> [!example] 例 4-26
> 8 个元素（每个元素 16 位，1 个字）的向量点积计算程序设计，即求
$$
a \times b = a_1 \times b_1 + a_2 \times b_2 + a_3 \times b_3 + \dots + a_8 \times b_8
$$
```assembly
.MODEL  SMALL                              ; 本例需使用 MASM 6.11 以上版本，在 Pentium 及以上的机器上运行
.586
.MMX
.STACK
.DATA
A_VER   DQ    0001000200030004H, 0005000600070008H ; 给出 a[i]元素（共 8 个字）
B_VER   DQ    0001000200030004H, 0005000600070008H ; 给出 b[i]元素（共 8 个字）
AB_VER  DD    ?                           ; 存放结果（32 位）
NOMSG   DB    'MMX NOT FOUND', 0DH, 0AH, '$'
YESMSG  DB    'MMX FOUND', 0DH, 0AH, '$'
.CODE
.STARTUP
        MOV    EAX, 1                      ; 判断 MMX CPU 是否存在
        CPUID
        TEST   EDX, 00800000H              ; 取 CPU 的 ID 标识码
        JNZ    MMX_F                       ; D23=1 表示有 MMX 功能
        MOV    AH, 9                       ; 有 MMX 功能，转去 MMX_F
        MOV    DX, OFFSET NOMSG            ; 提示 CPU 不支持 MMX 功能
        INT    21H
; 以下利用整数指令实现元素的积之和运算
        MOV    SI, OFFSET A_VER
        MOV    DI, OFFSET B_VER
        MOV    CX, 8                       ; 8 次乘法(不支持 MMX 时使用 16 位乘)
START1: MOV    AX, [SI]                    ; 取元素 a[i]
        IMUL   WORD PTR [DI]               ; 与 b[i]相乘，乘积在 DX、AX 中
        PUSH   DX
        PUSH   AX
        ADD    SI, 2
        ADD    DI, 2
        LOOP   START1
        MOV    CX, 7                       ; 7 次加法
        POP    EAX
START2: POP    EDX
        ADD    EAX, EDX                    ; 取乘积
        LOOP   START2                      ; 求和
        MOV    AB_VER, EAX                 ; 存结果
        JMP    START4
MMX_F:  MOV    AH, 9
        MOV    DX, OFFSET YESMSG
        INT    21H
; 以下采用 MMX 指令实现元素的积之和运算，MM0 中为：a4×b4+a3×b3 及 a2×b2+a1×b1
; MM1 中为：a8×b8+a7×b7 及 a6×b6+a5×b5
        MOVQ   MM0, A_VER                  ; 取元素 a[i]前 4 个（64 位）到 MM0
        PMADDWD MM0, B_VER                 ; 与元素 b[i]前 4 个相乘，再加
        MOVQ   MM1, A_VER+8                ; 取元素 a[i]后 4 个（64 位）到 MM1
        PMADDWD MM1, B_VER+8               ; 与元素 b[i]后 4 个相乘，再加
; 组合结果
        PADDD  MM0, MM1                    ; MM0+MM1→MM0（64 位）
        MOVQ   MM1, MM0                    ; 复制 MM0 到 MM1（64 位 t）
        PSRLQ  MM1, 32                     ; 将 MM1 右移 32 位
        PADDD  MM0, MM1                    ; MM0+MM1→MM0（32 位）
        MOVD   AB_VER, MM0                 ; 存 32 位结果值
        EMMS                                 ; 退出 MMX 状态
START4: .EXIT  0
END
```
> [!warning] 注意
> MMX 寄存器是使用浮点数据寄存器实现的，编程时需将 MMX 程序与浮点处理程序置于各自独立的代码段中，不要在指令级混用 MMX 指令和浮点指令。而 SIMD 浮点寄存器 XMM7～XMM0 是独立设计的，程序设计时可以混用 SSE 指令与 80x87 浮点指令。

### 4.6.4 保护模式程序设计

保护模式支持多任务。基于保护模式的保护机制，在设计保护模式程序时需要掌握两点：① 对系统中的多个不同任务，将每个任务放在不同的虚拟地址空间；② 对同一任务，依其代码和数据的信任度与重要性的不同为之分配不同的特权级。
保护模式和实模式以同样的方法使用和执行指令。开机或复位时 CPU 总是进入实模式，实模式具有特权级 0。从实模式切换到保护模式的一般步骤如下，反向操作则可回到实模式。

1. 利用伪指令构造系统进入保护模式工作的各种表，如全局/中断描述符表 GDT/IDT、任务状态段 TSS 和各任务所涉及的堆栈等。
2. 初始化全局描述符表寄存器 GDTR 和中断描述符表寄存器 IDTR 等。
3. 打开 A20 地址线。
4. 通过指令使 CR0 的 D0（PE）置 1，切换到保护模式。
5. 根据保护模式，设置各段的选择子。

实模式和保护模式的相互切换是保护模式编程的要点，以下举例说明。

> [!example] 例 4-27
> 以十六进制数的形式显示从内存地址 110000H 开始的 256 字节的值。

本例指定 110000H 内存区域，说明有必要切换到保护模式，因为在实模式下不能访问该内存区域。按照上述步骤本例应该：① 定义相关结构与宏，做好切换到保护模式的准备（在保护模式下需通过描述符表数据结构实现对内存的访问）；② 切换到保护模式；③ 将指定内存区内容传送到常规内存区；④ 切换回实模式；⑤ 使用 INT 21H 显示缓冲区内容。具体如下：
```assembly
; 定义段描述符的结构
DESC    STRUC                           ; 每个段描述符占 8 字节
LIMITL  DW    0                         ; 段界限（BIT0～BIT15）
BASEL   DW    0                         ; 段基地址的低 16 位（BIT0～BIT15）
BASEM   DB    0                         ; 段基地址的 16～23 位（BIT16～BIT23）
ATTRIBUTES DB  0                        ; 段属性（取值 98H 或 92H）
LIMITH  DB    0                         ; 段界限 BIT16～19（含段属性的高 4 位）
BASEH   DB    0                         ; 段基地址的 24～31 位（BIT24～BIT31）
DESC    ENDS
; 定义伪描述符（用于装入全局或中断描述符表寄存器）
PDESC   STRUC                           ; 装入 48 位 GDTR 时要用到
LIMIT   DW    0                         ; 16 位段界限
BASE    DD    0                         ; 32 位段基地址
PDESC   ENDS
; 定义打开 A20 地址线的宏（保护模式下访问的地址空间超过 1MB，要用到 A20 地址线）
ENABLE_A20 MACRO                        ; 通过 92H 端口的输出，使地址线 A20 打开
        PUSH    AX
        IN      AL, 92H
        OR      AL, 00000010B            ; 在 92H 端口的 D1 位输出 1，打开 A20 地址线
        OUT     92H, AL
        POP     AX
ENDM
; 定义将 CR0 寄存器中的 D0 置 1 的宏，以便切换到保护模式
TO_PROTECT MACRO
        MOV     EAX, CR0
        OR      EAX, 1
        MOV     CR0, EAX
ENDM
; 定义一个段间跳转的宏指令，以保证在进入保护模式时将代码段的选择子装入 CS 中
JUMP16  MACRO   SELECTOR, OFFSETS
        DB      0EAH                    ; 0EAH 为 JUMP 指令的机器码
        DW      OFFSETS                 ; 16 位段偏移量
        DW      SELECTOR                ; 段选择子
ENDM
```
从保护模式返回到实模式时，只要做和以上相反的动作即可。
```assembly
; 定义一个回到实模式的宏（将 CR0 寄存器中的 D0 清 0）
TO_REAL MACRO
        MOV    EAX, CR0
        AND    EAX, 0FFFFFFFEH
        MOV    CR0, EAX
ENDM
; 用一个段间跳转的宏，清指令预取队列，使 CS 内容为代码段地址
; 关闭 A20 地址线
DISABLE_A20 MACRO
        PUSH    AX
        IN      AL, 92H
        AND     AL, 0FDH                 ; 在 92H 端口的 D1 位输出 0，关闭 A20 地址线
        OUT     92H, AL
        POP     AX
ENDM
```
基于以上定义，设计保护模式下的 0 级代码段 PCODE 来实现数据传送，完整程序如下：
```assembly
.386P                                   ; 使用 386 全部指令集
DATA    SEGMENT USE16                   ; 定义 16 位数据段
GDT     LABEL   BYTE                    ; 定义全局描述符表 GDT
DUMMY   DESC<>                          ; 空描述符
PSEG    DESC<0FFFFH, PCODE, , 98H, , >  ; 保护模式下的代码段描述符
PCODE_SEL =PSEG - GDT                   ; 保护模式下代码段描述符的选择子
DSEGS   DESC<0FFFFH, 0, 11H, 92H, , >   ; 源数据段描述符，段基地址 110000H
DSEGS_SEL = DSEGS - GDT                 ; 源数据段描述符的选择子
DSEGD   DESC<0FFFFH, DATA, , 92H, , >   ; 目标数据段描述符
DSEGD_SEL = DSEGD - GDT                 ; 目标数据段描述符的选择子
GDTLEN  = $-GDT                         ; 全局描述符表长度
VGDTR   PDESC<GDTLEN-1, >               ; 伪描述符，界限为 GDT 的实际长度
BUFFER  DB    256 DUP (0)               ; 256 字节的目标缓冲区
DATAE   NDS
CODE    SEGMENT USE16                   ; 定义 16 位代码段
        ASSUME CS:CODE, DS:DATA
START:  MOV    AX, DATA
        MOV    DS, AX
        MOV    BX, 16                   ; 准备要加载到 GDTR 的伪描述符
        MUL    BX                       ; 现在是实模式，所以段地址要左移 4 位
        ADD    AX, OFFSET GDT
        ADC    DX, 0                    ; 得到 GDT 的 32 位段基地址 DX,AX
        MOV    WORD PTR VGDTR.BASE, AX  ; 设置 VGDTR 的 BASE 内容
        MOV    WORD PTR VGDTR.BASE+2, DX
        MOV    AX, PSEG.BASEL           ; 初始化代码段描述符
        MUL    BX
        MOV    PSEG.BASEL, AX           ; 代码段开始偏移为 0
        MOV    PSEG.BASEM, DL           ; 代码段界限已在定义时设置好
        MOV    PSEG.BASEH, DH
        MOV    AX, DSEGD.BASEL          ; 初始化目标数据段描述符
        MUL    BX
        ADD    AX, OFFSET BUFFER        ; 加上 BUFFER 的偏移地址
        ADC    DX, 0
        MOV    DSEGD.BASEL, AX
        MOV    DSEGD.BASEM, DL
        MOV    DSEGD.BASEH, DH
        LGDT   QWORD PTR VGDTR          ; 加载 GDTR
        CLI                              ; 关中断（未建立 IDT，程序必须运行在关中断下）
        ENABLE_A20                       ; 打开 A20 地址线
        TO_PROTECT                       ; 切换到保护模式
        JUMP16 <PCODE_Sel>, <OFFSET PROTECT> ; 清指令预取队列，进入保护模式
REAL:   DISABLE_A20                      ; 关闭 A20 地址线
        MOV    AX, DATA
        MOV    DS, AX
        STI                               ; 开中断
        …                               ; 根据具体任务要求实现“显示字符”程序段
CODE    ENDS

PCODE   SEGMENT USE16                   ; 定义 16 位代码段
        ASSUME CS:PCODE
PROTECT:                                ; 现在开始在保护模式下运行
        MOV    AX, DSEGS_SEL            ; 加载源数据段描述符
        MOV    DS, AX
        MOV    AX, DSEGD_SEL            ; 加载目标数据段描述符
        MOV    ES, AX
        CLD
        XOR    SI, SI                   ; 设置指针初值
        XOR    DI, DI
        MOV    CX, 256/4                ; 设置以 4 字节为单位的数据长度
        REPZ   MOVSD                    ; 传送
        TO_REAL                          ; 将 CR0 寄存器中的 D0 清 0，切换回实模式
        JUMP16 <SEG REAL>, <OFFSET REAL> ; 清指令预取队列，进入实模式
PCODE   ENDS
        END    START
```
说明：

1. 在从实模式切换到保护模式前，基本的准备工作是建立合适的全局描述符表 GDT，并使 GDTR 指向 GDT。因为在切换到保护模式时，至少把保护模式下代码段的选择子装载到 CS，所以 GDT 中至少含有保护模式下代码段的描述符。

本例全局描述符表 GDT 有 4 个描述符：第 1 个是空描述符，可以保证 GDT 中的第 1 个描述符永远不会被访问；第 2 个是代码段描述符……各描述符中的段界限是在定义时设置的，除了伪描述符 VGDTR 中的界限按 GDT 的实际长度设置，其余各段描述符的界限都规定为 0FFFFH。另外，描述符中的段属性也根据所描述段的类型被预置，92H 表示可读/可写数据段，98H 表示只可执行代码段，各段的特权级都设为 0。

2. 由于在切换到保护模式后就要引用 GDT，因此在切换到保护模式前使用指令 LGDT 装载 GDTR，其功能是将 48 位伪描述符 VGDTR 装入全局描述符表寄存器 GDTR 中。低字是以字节为单位的全局描述符表段的界限，高双字为描述符表段的线性基地址（本例不启用分页机制，所以线性地址等同于物理地址）。
3. 在做好准备后执行 TO_PROTECT 宏指令，CPU 转入保护模式。安排一条段间转移宏指令：JUMP16 <PCode_Sel>, <OFFSET PROTECT>，在实模式下被预取并在保护模式下被执行。利用这条段间转移指令可把保护模式下代码段的选择子装入 CS，同时刷新指令预取队列。从此真正进入保护模式。
4. 在保护模式下实现数据传送时，需将源数据段和目标数据段的选择子装入 DS 和 ES，这两个描述符已在实模式下设置好，将选择子装入段寄存器意味着把包括基地址在内的段信息装入到段描述符高速缓冲寄存器。然后设置 SI、DI、CX 的初值，利用串操作指令实施传送。回到实模式下的显示是在常规内存中操作，代码很容易理解，故省略未写。

实际上，保护模式编程还可能涉及 16 位段和 32 位段的切换、多任务之间的切换以及中断描述符表 IDT、局部描述符表 LDT、任务状态段 TSS 的使用等。在此不再详述。

### 4.6.5 浮点指令程序设计

采用浮点指令的汇编语言程序格式，与整数指令源程序格式类似，仅需注意：① 编程时需使用 FPU 选择伪指令 .387 等；② 使用伪指令 DD、DQ、DT（MASM 6.11 建议采用 REAL4、REAL8、REAL10）定义 32/64/80 位的浮点数据；③ 程序的第一条指令必须是初始化 FPU 的指令 FINIT，这表示开始执行一个新的浮点指令程序。

> [!example] 例 4-28
> 用浮点运算指令构建一个 $0^\circ \sim 90^\circ$ 的高精度正弦表。
```assembly
.MODEL  SMALL
.386
.387                                 ; 汇编 80387 指令
.STACK
.DATA
FCONST  DQ    180.0
TEMP    DW    0
FSINE   DQ    91 DUP(?)              ; 共 91 个正弦函数值
.CODE
.STARTUP
        MOV    SI, OFFSET FSINE
        MOV    CX, 91
START1: FINIT                        ; 初始化 FPU
        FLDPI                        ; 压入 π
        FLD    FCONST                ; 压入 180
        FDIV                         ; 计算 π÷180
START2: FILD   WORD PTR TEMP         ; 压入 0～90
        FMUL   ST(0), ST(1)          ; 度转换成弧度
        FSIN                         ; 求正弦函数值
        FSTP   QWORD PTR [SI]        ; 保存
        INC    WORD PTR TEMP         ; 计算下一度
        ADD    SI, 8
        LOOP   START2
START3: .EXIT  0
END
```
