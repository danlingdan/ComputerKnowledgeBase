## 4.3 汇编语言的伪指令

MASM 宏汇编提供以下功能的伪指令：选择处理器方式、存储模式定义、符号定义、数据定义、程序分段、过程和宏定义、模块定义、条件汇编、结构定义以及列表控制等。除了数据定义伪指令，其余伪指令都不分配存储空间，仅起说明作用。本节分别介绍基本伪指令（支持 8086/8088）和扩展伪指令（MASM 5.0 及以上版本支持 80386/Pentium）的功能和使用方法。

### 4.3.1 基本伪指令语句

#### 1. 符号定义伪指令

符号定义伪指令用于给程序中的常量或表达式赋予一个符号名，方便它们在程序中以此符号名的形式出现，提高程序的可读性与灵活性。符号定义伪指令有 EQU 和 $=$ 两种。

##### 1. 等值语句 EQU
格式：`符号名  EQU  表达式`
例如：
```assembly
PORT1   EQU   312              ; 给常量赋予符号名
PORT2   EQU   PORT1+1          ; 给数值表达式赋予符号名 PORT2
ADDR    EQU   ES:[SI+4]        ; 给地址表达式定义一个名字
CHAR    EQU   'COMPUTER'       ; 为字符串定义新的名字
COUNT   EQU   CX               ; 为寄存器 CX 定义新的符号名 COUNT
LD      EQU   MOV              ; 为 MOV 定义新的符号名 LD
```
> [!warning] 注意
> 在同一源程序中，同一符号不能用 EQU 重复定义。

##### 2. 等号语句 $=$
格式：`符号名 = 表达式`
此语句的功能与 EQU 类似，唯一区别是能对同一符号重复定义。
例如：
```assembly
EMP=60             ; 定义 EMP 等于 60
EMP=EMP+1          ; 重新定义 EMP 等于 61
EMP=123*4          ; 重新定义 EMP 等于 492
```

#### 2. 数据定义伪指令

数据定义伪指令用于定义变量，指定变量的类型和名称，为变量分配存储单元并赋初值。

##### 1. 数据定义伪指令语句的格式
格式：`[变量名] 数据定义符 操作数 [, 操作数]…[, 操作数]`
其中，变量名是可选的；操作数是赋给变量的初值，多个相同类型的变量可在一条语句中定义；常用的“数据定义符”是下列三种伪指令之一。

1. DB：定义字节变量，每个变量分配 1 字节的存储单元。
2. DW：定义字变量，每个变量分配 2 字节存储单元，且遵循“数据的低位部分在低地址，高位部分在高地址”的存放规则。
3. DD：定义双字变量，每个变量分配 4 字节存储单元，存放时同样按“低位字在低地址，高位字在高地址”的存放规则。

还有 DQ 和 DT 两种整型数据定义符，DQ 定义 4 字变量，DT 定义 10 字节变量。MASM 6.x 还支持实数定义，如数据定义符 REAL4、REAL8 分别用于定义 4、8 字节的实型变量。

经过定义的变量名有三种属性：段地址（可用操作符 SEG 获得）、偏移量（可用操作符 OFFSET 获得）和数据类型（字节、字或双字等）。若某个变量所表示的是一个数组，则其数据类型属性为变量的单个元素所分配的字节数。

##### 2. 数据定义伪指令语句的具体形式和功能

1. 为数据项分配存储单元，变量名作为该存储单元的名称。例如：

```assembly
X       DB    25H                  ; 定义 X 为字节变量
Y       DW    4142H                ; 定义 Y 为字变量
Z       DD    12345678H            ; 定义 Z 为双字变量
```
汇编时汇编程序将 $25H$、$4142H$、$12345678H$ 这些初值分别放入名为 X、Y、Z 的存储单元中，如图 4-2 所示。

![[计算机系统/微机原理与接口技术B/附件/第4章/Pasted image 20260719160401.png]]
*图 4-2　数据定义伪指令语句的具体形式和功能示意图*

初始值也可以是一个数值表达式，因为表达式的值是在汇编时计算的。例如：
```assembly
IN_PORT  DB     PORT_VAL           ; PORT_VAL 是符号常量，已由之前的 EQU 语句赋值
OUT_PORT DB     PORT_VAL+1
```

2. 预留若干字节（或字、双字等）存储单元但并不赋具体的初值，用于存放程序的运行结果。这时操作数部分可使用 “?”。例如：

```assembly
A        DB     ?                  ; 为变量 A 分配 1 字节单元
B        DW     ?                  ; 为变量 B 分配 1 个字单元
```

3. 引入若干用“,”分隔的操作数来定义一个表（数组）。

例如：
```assembly
W_TABLE  DW     1122H, 3344H       ; 定义一个字表
B_TABLE  DB     1, 2, 4, 8, 16     ; 定义一个字节表
```
其中，字变量 W_TABLE 的内容为 $1122H$，下一个字单元（起始地址 $W_TABLE+2$）的内容为 $3344H$。字节变量 B_TABLE 的内容为 1，B_TABLE+1 单元的内容为 2，其余以此类推。
当表中的操作数相同时，可以用重复操作符 DUP 来缩写。
例如：`ALL_ZERO  DB  0, 0, 0, 0, 0, 0` 可以写成 `ALL_ZERO  DB  6 DUP(0)`。
DUP 是重复数据定义操作符，利用给出的一个初值（或一组初值）以及这些值应该重复的次数（由 DUP 前面的常数决定）来初始化存储器。例如：
```assembly
W_TABLE   DW    10H DUP(?), 20H DUP(7)   ; 定义 30H 个字单元
B_TABLE   DB    10H DUP(12H, 34H)        ; 重复存放 10H 字节 12H、34H，共占用 20H 字节单元
```
DUP 操作符还可以嵌套使用。例如：
`; 定义了 0、1、2、1、2、1、2、3 的 2 个副本，共占用 16 字节单元`
```assembly
ARRAY     DB    2  DUP(0, 3 DUP(1, 2), 3)
```

4. 用 DB 伪指令在内存中定义一个字符串。字符串中的每个字符用它的 ASCII 值来表示，为 1 字节，所以字符串的定义必须用 DB 命令。例如：

```assembly
MESSAGE   DB    'HELLO'                 ; MESSAGE 所指的单元中存放字符 H 的 ASCII 值
                                        ; MESSAGE+1 单元中存放字符 E 的 ASCII 值……
                                        ; MESSAGE+4 单元中存放字符 O 的 ASCII 值
```
当字符串的长度不超过两个字符时，也可用 DW 来定义。例如：
```assembly
STRING1   DB    'AB'
STRING2   DW    'AB'
STRING3   DW    'AB', 'CD'
```
这三条语句在汇编后，存储器初始化的情况如图 4-3 所示。

![[计算机系统/微机原理与接口技术B/附件/第4章/Pasted image 20260719160410.png]]
*图 4-3　存储器初始化的情况*

5. 当操作数是标号或变量时，可用 DW 或 DD 伪指令将标号或变量的偏移地址或全地址来初始化存储器。例如：

```assembly
XX        DW    CYCLE                ; 变量 XX 的初值为 CYCLE 的偏移地址
YY        DD    CYCLE                ; 变量 YY 的初值为标号 CYCLE 的段地址和偏移地址
CYCLE:    MOV   BX, AX               ; CYCLE 是程序中一条指令语句的标号
```
第 1 条伪指令将标号 CYCLE 的 16 位偏移地址存放在与变量 XX 对应的 2 字节存储单元内；第 2 条伪指令在与变量 YY 对应的 4 字节存储单元内先存放 CYCLE 的 16 位偏移地址，再存放 CYCLE 的 16 位段地址。

6. 数据定义语句定义了变量的类型，能使汇编程序对访问存储器的指令产生正确的目标代码。例如：

```assembly
SUM    DW    ?      ; 定义 SUM 为字变量
       ……
INC    SUM          ; 将变量 SUM 的字内容加 1
```
前已述及，当“标号或变量±常量”时，形成的新操作数与原操作数有相同的类型属性。如对上述举例有：SUM+2 是字型变量，CYCLE+1 也是一条指令语句的标号。

##### 3. 分析操作符与合成操作符
作为存储器操作数的标号或变量都有段、偏移量和类型三种属性。在程序设计中，有时需要随时查询、调用或修改这些属性，以满足不同的访问要求。这些操作由分析与合成操作符来完成。

1. 分析操作符：作用于存储器操作数时返回它们的属性值，这些操作符是 SEG、OFFSET、TYPE、SIZE、LENGTH 等。

使用格式： `操作符  标号或变量`
SEG 和 OFFSET 操作符：分别用来返回标号或变量的段地址和偏移地址。例如：
```assembly
ARRAY   DB    100 DUP(0)      ; 定义字节变量数组
        …
MOV     AX, SEG ARRAY         ; 取变量 ARRAY 的段地址→AX
MOV     DS, AX                ; AX→DS
MOV     BX, OFFSET ARRAY      ; 取 ARRAY 的偏移地址→BX
MOV     AL, [BX]              ; 取字节
```
设数据段 DATA 的起始地址为 $06000H$，字节变量 ARRAY 在 DATA 段内，则上述指令被汇编时，SEG ARRAY 回送 ARRAY 所在段的段地址 $0600H$，OFFSET ARRAY 回送 ARRAY 在段内的偏移量 $0000H$。CPU 执行指令时，立即数 $0600H$ 被送入 AX，$0000H$ 被送入 BX。
TYPE 操作符返回一个数值，表示存储器操作数的类型。各种存储器操作数的类型值如表 4-3 所示。LENGTH 操作符返回变量用 DUP 重复定义的数据项总数，其他情况返回 1。SIZE 操作符返回 TYPE 和 LENGTH 的乘积，表示为变量所分配的字节单元总数。

**表 4-3 存储器操作数的类型值**

| 存储器操作数 | 类型属性 | 类型值 | 说明 |
| :--- | :--- | :--- | :--- |
| 字节变量 | BYTE | 1 | 变量类型的每个数据占用的字节数 |
| 字变量 | WORD | 2 | |
| 双字变量 | DWORD | 4 | |
| 标号 | NEAR | $-1$ | 距离属性值（没有实际物理意义） |
| 标号 | FAR | $-2$ | |

例如：
```assembly
MOV  AX, TYPE ARRAY    ; ARRAY 是字节变量，该语句汇编结果为 MOV  AX, 1
```
例如：

| 数据定义 | LENGTH 操作符 | SIZE 操作符 |
| :--- | :--- | :--- |
| `BUFFER1 DB 01H, 02H` | `LENGTH BUFFER1=1` | `SIZE BUFFER1=1*1=1` |
| `BUFFER2 DW 1234H` | `LENGTH BUFFER2=1` | `SIZE BUFFER2=2*1=2` |
| `BUFFER3 DD 100 DUP(20H)` | `LENGTH BUFFER3=100` | `SIZE BUFFER3=4*100=400` |
| `BUFFER4 DB 'ABCD……Z'` | `LENGTH BUFFER4=1` | `SIZE BUFFER4=1*1=1` |

2. 合成操作符：又称为修改属性操作符，作用于存储器操作数时，能对其类型进行设置，建立一些新的存储器操作数以满足不同的访问要求，常用的有 PTR、LABEL、THIS 等。

PTR 操作符，用来临时指定或修改存储器操作数的类型属性，而保持其原有的段属性和偏移地址属性。如果这些变量或标号已有定义，则原定义的类型属性不变。
使用格式：`类型  PTR  标号或变量`
其中，类型可以是 BYTE、WORD、DWORD、NEAR、FAR 等，它们仅在当前所在的指令中有效。例如：
```assembly
MOV     BX, WORD PTR ARRAY1[3]    ; 使 ARRAY1 为字变量，0403H→BX
MOV     CL, BYTE PTR ARRAY2[6]    ; 使 ARRAY2 为字节变量，03H→CL
MOV     WORD PTR [SI], 4          ; 将 0004H 放入 SI 开始的字单元中
```
配合 EQU 伪指令，还可以为变量 ARRAY1 和 ARRAY2 定义新的变量名。例如：
```assembly
WBYTE   EQU WORD PTR ARRAY1
BWORD   EQU BYTE PTR ARRAY2
```
新变量 WBYTE、BWORD 的段地址和偏移地址分别与 ARRAY1、ARRAY2 的相同，只是类型已改变，在之后的程序中可以直接使用新变量。例如：
```assembly
MOV     BX, WBYTE[3]            ; 0403H→BX
MOV     CL, BWORD[6]            ; 03H→CL
```
PTR 用来指明标号或变量的类型属性时，可确定指令是段内转移还是段间转移。例如：
```assembly
W_VAR   DD    INCHES
        …
INCHES: CMP   SUM, 100
        …
JMP     WORD PTR W_VAR          ; 段内转移
或
JMP     DWORD PTR W_VAR         ; 段间转移
或
JMP     NEAR PTR INCHES         ; 段内转移（NEAR PTR 可以略去）
或
JMP     FAR PTR INCHES          ; 段间转移
```
LABEL 操作符为当前存储单元定义一个指定类型的标号或变量，使它们具有多重名字和属性。
例如：
使用格式： `标号或变量名  LABEL  类型`
例如：
```assembly
WBYTE   LABEL   WORD            ; 为变量 WBYTE 定义一个字类型的数据区
ARRAY   DB      1, 2, 3, 4      ; 为变量 ARRAY 定义一个字节类型的数据区
        MOV     AL, ARRAY       ; 01H→AL
        MOV     AX, WBYTE       ; 0201H→AX
```
WBYTE 和 ARRAY 两个变量指向同一数据区，具有同样的段属性和偏移量属性，但是类型不同。这样可根据需要按不同类型去操作数据区中的数据。
THIS 操作符与 EQU 配合使用，具有 LABEL 操作符同样的作用。
使用格式：`标号或变量名  EQU  THIS  类型`
例如：
```assembly
BWORD   EQU     THIS BYTE
ARRAY   DW      100 DUP(?)
```
此例表明，可以将 100 个字的存储区 ARRAY 按 200 字节的存储区 BWORD 来使用。

#### 3. 程序分段伪指令

程序分段伪指令用于指示汇编程序和链接程序如何按逻辑段来组织程序和利用存储器，规定了源程序中段的起始和结束，并指定属性，如段的起始地址边界、同名段的组合方式等。
段定义有两类：完整段定义和简化段定义。本节仅介绍完整段定义伪指令，主要包括段定义、指定段寄存器等语句，如 SEGMENT/ENDS、ASSUME、ORG 等。

##### 1. 段定义伪指令 SEGMENT/ENDS
SEGMENT/ENDS 这对段定义语句将程序中的指令或语句分成若干逻辑段，按性质一般分为数据段、代码段、堆栈段和附加段。
段定义格式：`段名  SEGMENT  [定位类型][组合类型][‘类别名’]`
                       …
                       (段体)
                       …
`段名  ENDS`
功能：指出段名及段的各种属性，并表示段的开始和结束位置。
段名是用户定义的段的标识符，是识别段的标志，表示汇编程序为该段分配的存储区的起始位置。段体是段内的指令和伪指令语句序列。三个可选参数项代表段的三个属性，为源程序的汇编与链接提供必要的信息，顺序不可交换，其意义如下。

1. 定位类型。汇编程序对源程序汇编后生成的是地址浮动的目标代码文件 `.obj`，其中的段地址和偏移量都未最后确定，还需要用链接程序（LINK）把各模块链接起来。定位类型参数通知 LINK 程序，链接时本段起始地址的边界定位方式。定位类型有 4 种，如表 4-4 所示，默认定位类型是 PARA。

**表 4-4 各种定位类型所表示的段起始地址**

| 定位类型 | 含 义 |
| :--- | :--- |
| PARA (节，默认状态) | 段的起始地址必须为 16 的倍数 (xxxx0H) |
| PAGE (页) | 段的起始地址必须为 256 的倍数 (xxx00H)，就是页的起点上 |
| WORD (字) | 段的起始地址必须为偶数，最适合安排类型为字的数据段 |
| BYTE (字节) | 段可以从任意地址开始 |

2. 组合类型。大型的汇编语言源程序可以由许多模块组成，每个模块常常有自己的数据段、代码段和堆栈段。组合类型用于通知 LINK 程序，在对多个模块进行链接形成一个可执行文件时，如何将本段与其他模块中的同名段进行合理组合。组合类型有 6 种，如表 4-5 所示。

**表 4-5 各种组合类型表示的链接关系**

| 组合类型 | 含 义 |
| :--- | :--- |
| NONE (默认状态) | 本段与其他模块中的同名段无逻辑关系，不组合，各自有自己的段起始地址。 |
| PUBLIC | 在满足定位类型的前提下，LINK 程序将本段与其他模块中说明为 PUBLIC 的同名段邻接在一起，共用一个段地址，即合成一个物理段。 |
| STACK | 默认堆栈段需要的组合类型。LINK 程序将所有堆栈段链接成一个连续段（其长度是各堆栈段的长度的之和），链接后自动初始化 SS、SP。如果在定义堆栈段时没有将其说明为 STACK 组合类型，则需要在程序中用指令设置 SS 和 SP 的值，此时 LINK 程序会给出一个警告信息。 |
| COMMON | 各模块中由 COMMON 方式说明的同名段重叠覆盖，有着相同的起始地址。段的长度取决于最长 COMMON 段的长度，段的内容为所链接的最后一个模块中 COMMON 段的内容以及没有覆盖到的前面 COMMON 段的内容。 |
| MEMORY | 本段定位在所有链接在一起的其他段的最后（存储器高地址区域）。若有多个 MEMORY 段，汇编程序则认为所遇到的第一个为 MEMORY，其余为 COMMON。 |
| AT 表达式 | 本段定位在表达式的值所指定的段地址上。如 `AT 1930H` 表示本段的起始地址为 `19300H`。 |

3. 类别名：用单引号括起来的字符串，以表示该段的类别。链接时类别名相同的段在内存中相邻存放，但各段仍然是独立的。类别名可任意选择，典型的可选 STACK、CODE 和 DATA 等。若省略“类别名”，则表明该段类别为空。

##### 2. 指定段寄存器伪指令 ASSUME
ASSUME 一般出现在代码段中 SEGMENT 伪指令的后面，用于建立段与段寄存器之间的联系，通知 MASM 用指定的段寄存器去寻址对应的逻辑段。但是，ASSUME 语句并未真正将段地址装入相应的段寄存器，段寄存器（CS 除外）的初值设定还需在程序中用指令来完成。
格式：`ASSUME  段寄存器:段名[, 段寄存器:段名, …]`
下面一段程序表示如何使用 SEGMENT、ENDS、ASSUME 伪指令。
```assembly
DATA    SEGMENT                ; 定义 DATA 段
X       DB      ?
DATA    ENDS

ESTRA   SEGMENT                ; 定义 ESTRA 段
ALPHA   DB      ?
ESTRA   ENDS

STACK   SEGMENT PARA STACK 'STACK'  ; 定义 STACK 段
STAPN   DB      100 DUP(?)     ; 100 字节堆栈空间
TOP     EQU     LENGTH STAPN
STACK   ENDS

CODE    SEGMENT                ; 定义 CODE 段
        ASSUME  CS:CODE, ES:ESTRA, DS:DATA, SS:STACK ; 定义了 CODE、ESTRA、DATA、STACK 分别为代码段、附加段、数据段和堆栈段
MAIN    PROC    FAR
        PUSH    DS
        SUB     AX, AX
        PUSH    AX             } ; 标准序
        MOV     AX, DATA
        MOV     DS, AX
        MOV     AX, ESTRA
        MOV     ES, AX         } ; 给段寄存器（CS 除外）赋值
        MOV     AX, STACK
        MOV     SS, AX
        MOV     AX, TOP
        MOV     SP, AX         ; 给 SP 赋值
        MOV     BL, X          ; 把字节 X 单元的内容→字节 ALPHA 单元
        MOV     ALPHA, BL
        …
        RET
MAIN    ENDP
CODE    ENDS
        END     MAIN
```
说明：

1. 因为堆栈段定义时选用了组合类型 STACK，系统链接时就会自动初始化 SS、SP，这时应该去掉 ASSUME 语句中的 `SS: STACK` 以及代码段中初始化 SS、SP 的 4 条语句。
2. CS 和 IP 寄存器的初值都是由伪指令语句 `END MAIN` 装入的，无须在程序中设置。`END [标号]`语句是程序结束伪指令，通知 MASM，主程序或模块程序在哪儿结束，其中的标号是可选项。只有主程序的结束语句允许带标号，它是程序运行时第一条要执行的指令语句的标号。如上例中的 MAIN，它的段地址和偏移地址就是 CS 和 IP 的内容。

基于①和②，通常在程序中仅需对 DS、ES 段寄存器作初始化赋值。

3. 程序中把字节 X 的内容传送至字节 ALPHA 时使用的两条指令，因为 ASSUME 伪指令的作用，MASM 把它们汇编成以下两条等价指令：

```assembly
MOV   BL, DS:X
MOV   ES:ALPHA, BL
```
有时为了更明确地指明段寄存器或替代 ASSUME 语句的作用，可在指令中直接增加段超越前缀，用以通知 MASM，该指令执行时应使用哪一个段寄存器。

4. 可以用 `ASSUME 段寄存器:NOTHING` 语句取消先前建立的段与段寄存器的联系，需要时可再重新建立段寄存器与新的段之间的关联。

##### 3. 组定义伪指令 GROUP
伪指令 GROUP 用于将程序中若干不同名的段集合成一个组，使它们都装在一个 64 KB 的物理段中。
格式：`组名  GROUP  段名[, 段名…]`
组名是识别组的标志，表示组的起始地址。段名可以是用 SEGMENT 语句定义的或者由 SEG 操作符得到的段名。如果组名在 ASSUME 语句中已说明，且相应段寄存器（DS、ES 等）有初始化语句，则系统把组内各段中的标号或变量的偏移地址调整为相对于组名的起始地址。可用 `OFFSET 组名:变量` 得到变量相对于组名地址的偏移量，组名在这里相当于段前缀。例如：
```assembly
DGROUP   GROUP   DATA1, DATA3          ; 定义段 DATA1、DATA3 为一个组
DATA1    SEGMENT WORD PUBLIC 'DATA'
X        DW      ?
DATA1    ENDS
DATA3    SEGMENT WORD PUBLIC 'DATA'
Z        DW      ?
DATA3    ENDS
CODE     SEGMENT WORD PUBLIC 'CODE'
         ASSUME  DS:DGROUP, CS:CODE
START:   MOV     AX, DGROUP            ; DGROUP 为已定义的组名
         MOV     DS, AX                ; 给 DS 赋初值
         MOV     X, OFFSET DGROUP:Z    ; 变量 Z 相对于 DGROUP 的偏移量→BX
         …
CODE     ENDS
         END     START
```

##### 4. 指定地址伪指令 ORG
ORG 用来强行指定地址计数器（记载正在汇编的指令目标代码或数据存放在当前段内的偏移量的部件）的当前值，以改变段内在它之后的代码或数据块存放的偏移地址。
格式 1：`ORG  表达式`
格式 2：`ORG  $+表达式`        ; 符号 `$` 表示地址计数器的当前值
格式 1 直接将表达式的值（0～65535）送入地址计数器，格式 2 将地址计数器的当前值加上表达式的值后送入地址计数器。随后以“表达式”或 `$+表达式` 的值为起点放置 ORG 语句后的代码或数据，直到遇到新的 ORG 语句。例如：
```assembly
DATA   SEGMENT
       ORG     10H                  ; 在数据段 10H 偏移地址处开始存放 20H、30H
X      DB      20H, 30H
       ORG     $+5                  ; 使偏移地址+5
Y      DB      40H, 50H             ; 在数据段 17H 偏移地址处开始存放 40H、50H
```
又如：
```assembly
ARRAY  DB      12, 34, 56
LEN    EQU     $-ARRAY              ; LEN 的值就是 ARRAY 变量所占的字节数
```

#### 4. 过程和宏定义伪指令

过程（又称为子程序）和宏都是程序段的一种形式，汇编语言中常用定义过程或宏的方法来简化源程序，使程序结构简洁清晰，并便于建立和使用相应的库以减少编程工作量等。

##### 1. 过程定义伪指令
过程是一段可由 CALL 指令调用的程序，执行完后用 RET 指令从过程返回到原调用处。若整个源程序由主程序和若干子程序组成，则主程序和这些子程序都应包含在代码段内，且都可以用过程定义语句来定义。
过程定义格式： `过程名  PROC  [NEAR/FAR]`
                        …
                       [RET]
                        …
                       RET                ; 过程体
                 `过程名  ENDP`
功能：定义一个过程，并指出过程名及过程的属性。
伪指令 PROC/ENDP 标识过程的起始与结束。过程名是给所定义的过程取的名字，代表该过程存放的起始地址，是调用指令 CALL 的目标操作数。过程属性有 NEAR 或 FAR 两种。NEAR 属性（近过程）允许过程在段内调用，FAR 属性（远过程）允许过程在段间调用。省略时默认属性为 NEAR。
RET 为过程的返回指令，是过程的出口，但不一定是过程的最后一条指令。一个过程可以有多条 RET 指令，正常情况下至少要执行到一条 RET 指令。
汇编程序根据过程定义语句，当汇编到调用该过程的 CALL 语句时，会自动识别出是段内调用还是段间调用；当汇编到过程中的 RET 时，也会自动识别出是段内返回还是段间返回，从而给出适合的指令代码。
过程定义与调用程序结构举例如下。
```assembly
CODE   SEGMENT
       ASSUME  CS:CODE
MAIN   PROC    FAR                 ; 主程序
       ……
       CALL    P1                  ; 汇编成段内调用
       ……
       RET
MAIN   ENDP
P1     PROC    NEAR                ; 定义过程 P1 是 NEAR 过程
       ADD     CX, 1
       RET
P1     ENDP
CODE   ENDS
       END     MAIN
```
一个段可以有多个过程。过程调用还允许嵌套或递归，嵌套或递归的深度只受堆栈的限制。

##### 2. 宏定义伪指令
宏的概念与过程很相似。若源程序中经常要用到一个程序段（宏体），可将其定义成一条宏指令（宏定义），于是在源程序中可用这条宏指令语句替代所定义的程序段（宏调用）。当 MASM 汇编程序处理到这条宏指令时，会自动用宏体替换它而扩展成原来的程序段（宏扩展）。

1. 宏的定义、调用与扩展。

宏定义格式： `宏指令名  MACRO [形参, 形参, …]`
                           …
                            ; 宏体
                 `ENDM`
宏调用格式： `宏指令名  [实参, 实参, …]`
MACRO/ENDM 语句限定所定义的宏。宏指令名代表所定义的宏体的内容，宏一经定义，就像为指令系统增加了一条新的指令一样，在程序中可通过宏指令名对它进行任意的调用。宏定义中的形参为可选项，若选用了形参，所定义的宏称为带参数的宏。当调用这样的宏时，需用对应的实参去取代，以实现向宏中传递信息。
例如，为了实现 ASCII 与 BCD 码之间的相互转换，往往需要把 AL 中的内容左移 4 位或右移 4 位。若左移 4 位，则可用下列宏定义和调用来实现。
```assembly
SHIFT   MACRO                   ; 宏定义
        MOV     CL, 4
        SAL     AL, CL
        ENDM
        …
        SHIFT                   ; 宏调用
        …
```
当汇编程序遇到 SHIFT 这样的调用时，将对应的宏体插入到源程序宏指令语句所在的位置上，来替换这条宏指令并产生目标代码，这就是宏扩展。汇编程序在每一条由宏扩展而产生的指令前冠以“+”号。例如：
```assembly
+ MOV   CL, 4          ; 对应宏指令 SHIFT
+ SAL   AL, CL
```
为了使宏定义适用于不同的情况，可以在宏定义中引入参数。如上例中希望每次用不同的寄存器移位不同的次数，可用下列宏定义和调用来实现。
```assembly
SHIFT   MACRO   X, Y            ; 带参数的宏定义
        MOV     CL, X
        SAL     Y, CL
        ENDM
        …
SHIFT   4, AL                   ; 宏调用，AL 左移 4 位。实参与形参按序一一对应
```

2. 取消宏指令名伪指令 PURGE。

格式：`PURGE  宏指令名 [, …]`
用 MACRO 定义过的宏指令名可用 PURGE 来注销。其他一些宏指令这里不详述。

##### 3. 宏与过程的区别
对程序中需重复使用的一些程序段，既可用宏也可用过程来定义，然后在程序中对它们进行调用。那么，宏与过程又有些什么区别，应如何选用它们呢？总结起来有以下 4 点：

1. 宏指令语句由汇编程序识别，在程序汇编时完成宏扩展的处理。而调用过程的 CALL 指令语句是在程序执行时完成过程调用的。
2. 宏操作可以直接传递参数，而过程不能直接带有参数。当过程之间需要传递参数时，必须通过寄存器、存储器或堆栈来进行。
3. 宏调用只能简化源程序的书写，缩短源程序的长度，并没有缩短其目标代码的长度。汇编程序处理宏指令时，是把宏体插入到宏调用处，对一个宏定义多次调用就要多次插入，所以目标程序占用内存空间并不因宏操作而减少；过程调用却能缩短源程序目标代码的长度，有效地节省内存空间。因为过程在源程序的目标代码中只有一段，无论过程被调用多少次，除了增加 CALL 指令代码，并不增加过程段的代码。
4. 引入宏操作不会在执行目标代码时增加额外的时间开销。相反，过程调用时的 CALL 和 RET 指令涉及保护和恢复现场及断点，这些额外的时间开销会延长目标程序的执行时间。

#### 5. 模块定义与通信伪指令

一个任务的汇编语言源程序可由一个或多个模块组成。在多个模块的情况下，每个模块实现一个特定的功能，并且有自己的代码段和数据段等，对应一个 `*.asm` 源文件，是一个独立的汇编单位。当所有模块单独汇编完毕后，再通过 LINK 程序链接成一个完整的可执行程序。MASM 汇编程序提供了模块化编程伪指令，用以实现模块的定义与通信。

##### 1. 模块定义伪指令 NAME/END
它们定义一个可以独立编写及汇编的程序模块。
模块定义格式： `NAME  模块名         ; 为模块命名`
                          …
                          ; 若干语句
              `END  [标号] ; 结束模块`
通常，模块命名语句 NAME 可以省略。省略时，模块所在的源程序文件名为模块名。
模块结束语句 END 告诉汇编程序本模块到此结束。如果该模块是程序的主模块或一个任务的独立模块，END 语句中必须有表征程序启动地址的标号。

##### 2. 模块间通信伪指令 PUBLIC/EXTERN
程序的各模块之间必须解决过程或数据的互访和共享问题，使用全局符号说明伪指令 PUBLIC/EXTERN 可以实现这个连接任务。
格式：`PUBLIC  名字 [, 名字, …]`
`EXTERN  名字:类型 [, 名字:类型, …]`
其中，名字可以是变量名、标号、过程名或由 EQU（或=）伪指令定义的符号名。类型可以是 BYTE/WORD/DWORD（变量）或 NEAR/FAR（过程）和 ABS（由 EQU 定义的常量的属性）。
PUBLIC 说明一个模块内哪些名字为全局的，可以被其他模块引用。EXTERN 说明本模块中使用的哪些名字为外部的（已在其他模块中定义）并已被说明为 PUBLIC 的。各模块间的 PUBLIC/EXTERN 要配对使用，并且指明的类型要一致。

#### 6. 条件汇编伪指令

条件汇编伪指令可使汇编程序根据某种条件有选择地对源程序中的某部分语句进行汇编处理。各种条件汇编语句的一般格式如下：
```assembly
IFxx     条件
         …                   ; 语句体 1
[ELSE
         …                   ; 语句体 2
ENDIF
```
根据条件的不同，条件汇编伪指令有 5 种，每种含两条相反条件的伪指令，如表 4-6 所示。

**表 4-6 条件汇编伪指令**

| | 格 式 | 功 能 |
| :--- | :--- | :--- |
| 1 | IF 表达式 | 表达式值非 0，则条件为真，汇编语句体 1 |
|   | IFE 表达式 | 表达式值为 0，则条件为真，汇编语句体 1 |
| 2 | IFDEF 符号 | 符号已被定义，则条件为真，汇编语句体 1 |
|   | IFNDEF 符号 | 符号未被定义，则条件为真，汇编语句体 1 |
| 3 | IFB <形参> | 用在宏定义体。如果宏调用没有用实参替代该形参，则条件为真 |
|   | IFNB <形参> | 用在宏定义体。如果宏调用有用实参替代该形参，则条件为真 |
| 4 | IFIDN <字符串1>, <字符串2> | 两字符串相同，则条件为真，区别大小写 |
|   | IFIDNI <字符串1>, <字符串2> | 两字符串相同，则条件为真，不区别大小写 |
| 5 | IFDIF <字符串1>, <字符串2> | 两字符串不同，则条件为真，区别大小写 |
|   | IFDIFI <字符串1>, <字符串2> | 两字符串不同，则条件为真，不区别大小写 |

每条 IFxx 伪指令都必须与 ENDIF 配对使用，汇编程序根据要求对条件进行检测：若条件为真，汇编语句体 1 中的语句部分；条件为假，语句体 1 不被汇编。若存在可选的 ELSE 伪指令，语句体 2 中的语句将在条件为假时被汇编。

#### 7. 结构定义与使用伪指令

MASM 中还允许用结构（STRUC/ENDS）或记录（RECORD）等伪指令来自定义一些复杂的数据结构，如学籍管理中的学生数据项，包括学生姓名、性别、年龄、成绩等多个不同数据类型的相关字段，可通过这些数据结构对若干相关的单个变量进行整体数据定义。结构的定义与使用需要三个步骤：结构类型说明→结构变量定义→结构引用。

##### 1. 结构类型说明伪指令 STRUC/ENDS
格式： `结构名  STRUC`          ; MASM 6.1x 后用 STRUCT
                   …
                   ; 结构体，由数据定义语句序列组成
        `结构名  ENDS`
例如，将一个学生的姓名、性别、学号、数学成绩、电路成绩等定义为一个结构。
```assembly
STUDENT STRUC              ; 结构由 5 个字段共 9 字节构成
SID     DB   ?             ; 学号
NAME    DB   5 DUP("")     ; 姓名预留 5 个字符空间
SEX     DB   'M'           ; 性别
MATH    DB   0             ; 数学成绩
CIRCUIT DB   0             ; 电路成绩
STUDENT ENDS
```
结构中用数据定义语句 DB、DW 等定义的变量称为字段或结构域，在某些情况下变量本身还可以是另一个结构。

##### 2. 结构变量的定义
结构类型说明只是通知汇编程序增加了一个新的数据类型，但并未给结构分配内存空间，需要通过定义结构变量为它分配内存空间。
格式： `[结构变量名] 结构名 <[域值表]>`
其中，结构变量名是用户为定义的结构变量起的名字，可以默认。结构名与结构定义时的结构名相同，不可省略。域值表用于给结构变量的各域赋初值，初值间用逗号分开。初值可以是数值，可以用 DUP 重复操作符，也可以省略，省略时由汇编程序自动给出。
例如：对上例说明的结构 STUDENT，定义若干结构变量并赋初值。
```assembly
ONE     STUDENT < >           ; 初值为结构说明时的值
TWO     STUDENT <02, 'ZHANG', , 87, 90> ; SEX 字段是结构说明时的值
THREE   STUDENT <03, 'WANG', 'F', 78, 92>
FOUR    STUDENT 5 DUP(<>)     ; 定义有 5 个结构变量的结构数组，初值为结构说明时的值
```
一旦定义结构变量，即为该变量分配内存空间，确定了其在内存中存放的起始地址。结构中各字段具有各自的局部偏移量，是指各字段第一字节与结构起始地址间的字节距离。

##### 3. 结构变量及其字段的引用
引用结构变量，只需直接书写结构变量名；要引用其中的某个字段（也称为成员变量），需要使用专门的结构字段运算符“.”。
格式： `结构变量名.结构字段名`
例如，对上例定义的各结构变量进行引用，实现从 ONE 到 FOUR[3]、FOUR[4]共 8 个结构变量的 SID 字段分别赋值 1～8。
```assembly
        MOV      ONE.SID, 1
        MOV      SI, OFFSET FOUR       ; 取结构数组首地址→SI
        MOV      AL, 4
        MOV      CX, LENGTH FOUR       ; 取结构数组元素个数 5→CX
INIT:   MOV      [SI].SID, AL          ; 依次给结构数组的 SID 字段赋值
        INC      AL
        ADD      SI, TYPE FOUR         ; SI+9，修改地址指针
        LOOP     INIT
```
