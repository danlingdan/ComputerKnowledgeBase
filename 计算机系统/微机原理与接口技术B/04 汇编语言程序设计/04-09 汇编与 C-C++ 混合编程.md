## 4.7 汇编语言与 C/C++ 语言混合编程

将汇编语言与 C/C++ 高级语言结合起来混合编程，可以充分利用系统与硬件技术的支持，取长补短，是编写高质量程序，特别是编写实时性要求比较高的程序的有效方法。
使用汇编语言与 C/C++ 语言混合编程的方法通常有两种：内嵌模块或多模块编程。

### 4.7.1 内嵌模块方法

内嵌模块方法是在 C/C++ 程序中直接嵌入汇编语言指令的语句。这种方法简单，只需在 C/C++ 程序中的 `_asm{}` 模块中嵌入汇编语言程序段即可。对于 Visual C++，除了在 `_asm{}` 模块中嵌入汇编语言程序段，还可以在汇编指令的前面用保留字 `_asm` 进行标记。

> [!example] 例 4-29
> 用汇编语言与 C/C++ 语言混合编程，实现在屏幕上显示字符串。

分析：可用汇编语言指令通过 EDX 寄存器传递待输出字符串的地址，然后调用 C 语言的标准输出函数 `printf`，将字符串输出显示在屏幕上，这里使用了嵌入汇编语言的两种格式。

```c
#include<stdio.h>
char const *MESSAGE ="The planet Earth.\n$";
char *output ;
void main(void)
{
    _asm  MOV   EDX, MESSAGE
    _asm  MOV   output, EDX
    printf("MESSAGE IS:%s", output);   //在屏幕上显示字符串
}
```

或

```c
_asm { MOV  EDX, MESSAGE
       MOV  output, EDX }
```

注意，混合编程时，能够使用的伪指令较少，如 OFFSET 这样的伪指令就不能用。

### 4.7.2 多模块混合编程

混合编程时，还可以采用之前介绍的模块化编程方法。将不同语言的程序分别编写并作为一个独立过程保存（如汇编语言程序），用 PUBLIC 伪指令将过程声明为公共的，提供给其他程序（如 C/C++ 语言程序）调用。C/C++ 语言程序在调用时，使用 EXTERN 伪指令声明所调用的过程（或函数）为外部的，同时指明该过程（或函数）的类型。
为了保证能够正确链接，各模块必须遵守相同的约定规则，这里以 C/C++ 语言程序对汇编语言程序的调用为例进行说明。

#### 1. 存储模式与标识符命名规则

通常，C/C++ 语言程序与汇编语言程序需具有相同的存储模式，以便自动产生相互兼容的调用与返回类型。
汇编语言程序中与 C/C++ 语言程序共享的变量名、过程名等标识符名称，都必须在前边加上下画线。如果在存储模式选择语句 `.model` 中，增加了参数 "c"，如 `.model MEDIUM, c`，则表明此汇编语言模块是按 C/C++ 语言规范编写的，这是 C/C++ 语言程序默认的调用约定，其入口参数和返回值均为 32 位。此时，不必在共享名称前加下画线。

#### 2. 寄存器使用规则

汇编语言程序可以任意使用通用寄存器 (EAX)、(EBX)、(ECX)、(EDX) 和 ES，其中的 (EAX)、(EDX) 承担了传递返回值的任务。对于段寄存器 DS、CS、SS 和 (EBP)、(ESP)，汇编语言程序在使用时必须先保护（入栈），退出前再恢复。对于 (ESI)、(EDI)，仅当 C/C++ 语言程序未启用寄存器变量时，汇编语言程序可直接使用，否则也必须先保护再使用，退出前恢复。

#### 3. 过程调用的参数传递规则

在 C/C++ 语言程序中，被调用的过程（函数）可以带参数，对它的说明形式如下：
```c
EXTERN 返回值类型 函数名称(参数列表)
```

其中，返回值类型是 C/C++ 语言中函数允许的任意类型，当返回值类型省略时默认为 int 型，参数在传递过程中要求个数、类型和顺序一一对应。
所有高级语言在调用过程（函数）时，都是通过堆栈来传递参数的。C/C++ 语言程序从汇编语言程序中获取参数时，实模式程序规定用 AX、DX，保护模式程序规定用 EAX、EDX。而将参数从 C/C++ 语言程序传递给所调用的汇编语言程序时，必须遵守如下规则：

1. 将 C/C++ 语言函数中的参数从右向左压入堆栈。例如，在 C/C++ 语言中函数 `sum(x, y, z)` 的数据传输顺序是 $z$、$y$、$x$，即先将 $z$ 压入栈，然后是 $y$、$x$。C/C++ 语言传递普通参数时，传递的是参数本身；传递数组时，传递的是数组的指针。

2. 参数压栈后，将当前的 CS、IP 压栈保护。如果 C/C++ 语言程序是在 TINY、SMALL 或 COMPACT 存储模式下编译的，则汇编语言程序中应将过程设置成 NEAR 型，将只保存 IP。如果 C/C++ 语言程序是在 MEDIUM、LARGE 或 HUGE 模式下编译的（过程被设置成 FAR 型），则先将 CS 压栈再将 IP 压栈。

注意使用的 C/C++ 语言程序开发环境。如果是 Visual C++ 6.0 环境，则只能编写 32 位应用程序，汇编子模块程序中必须用如 `.386P` 这样的伪指令来说明采用的指令集，并选用 FLAT 存储模式。32 位 Visual C++ 中没有近、远调用之分，所有调用都是 32 位的偏移地址。同时不能使用 DOS 和 BIOS 功能调用指令（INT n），因为这些功能调用只适用于 16 位应用程序。

3. 汇编语言程序的最后一条指令应该是不带参数的 RET。因为汇编语言程序返回后，断点自动弹出，然后 C/C++ 语言程序重新获得控制权。

> [!example] 例 4-30
> 在 Visual C++ 6.0 环境下编写一个函数，将三个数值传递给汇编过程，汇编过程对三个数求和后再将其传回 C++ 程序，由 C++ 程序将这个和在屏幕上显示。

设计汇编过程是 NEAR 类型，在调用汇编过程时，C++ 语言程序通过堆栈将三个数值传递，同时将 EIP 压栈保护（32 位编程环境的寄存器是 32 位的）。汇编程序用 EBP 寻址堆栈，得到参数的偏移地址应为 EBP+8、EBP+12、EBP+16，堆栈的变化如图 4-12 所示。

因为数据类型被定义成 int 型，汇编过程通过 EAX 将计算出的和传递给 C++ 程序。参考程序如下（在 VC 6.0 和 MASM 32 V10.0 版本编译调试通过）：

```cpp
#include<stdio.h>        // following is main.cpp
extern int sum (int, int, int);
main()
{
    printf("begin count\n");
    printf("The sum is equal to %d", sum(500, 6500, 200));
}
```

```asm
.386p                   // following is sum.asm
.MODEL  FLAT, C
.CODE
PUBLIC  SUM
SUM     PROC    NEAR
        PUSH    EBP
        MOV     EBP, ESP
        SUB     EAX, EAX
        MOV     EDX, EAX
        ADD     EAX, [EBP+08H]
        ADC     EDX, 0
        ADD     EAX, [EBP+0CH]
        ADC     EDX, 0
        ADD     EAX, [EBP+10H]
        ADC     EDX, 0
        POP     EBP
        RET
SUM     ENDP
END
```
