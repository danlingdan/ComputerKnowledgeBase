## 5.4 存储器连接与扩充应用

> [!check] 接口设计检查表
> 依次确认容量与数据宽度、地址范围与片选译码、电平兼容、读写时序、总线负载以及未对齐访问行为。位扩展解决“每个地址有多少位”，字扩展解决“共有多少个可寻址单元”。

在微机设计和应用中往往需要用一组存储器芯片构成一个存储系统，或根据需要扩充存储器容量。存储器提供指令代码和数据的信息存储功能，CPU 在取指令和执行操作数为存储器单元的指令的过程中访问相应的存储单元，需要在硬件电路上通过地址、数据及控制总线把存储器芯片与 CPU 连接起来，并使之协调工作。CPU 与存储器交换信息时一般是先输出地址，再送出读/写控制信号，并通过数据总线进行信息交换。如前所述，每个存储器都有特定的读、写及擦除时序，读/写系统如 CPU 必须在时序上支持相应的操作过程。

半导体存储器中信息数据的正确读/写或擦除，每种芯片都有严格的性能规范。由于 CPU 和存储器往往根据各自的技术标准，由不同厂商生产，要保证正确的访问，除了要求连接电平标准相同，还必须考虑信号连接、时序配合等问题。

CPU 的负载驱动能力也是需要考虑的因素，因为在微机系统中，CPU 通过总线与存储器芯片、I/O 接口芯片连接，而 CPU 的总线驱动能力是有限的。但存储器芯片多为 MOS 电路，直流负载很小，主要负载为电容负载。在小型系统中，CPU 可直接与存储器芯片连接，但当与大容量的存储器连接时就应考虑总线的驱动问题。

当 CPU 数据总线在 16 位以上，组合使用 8 位数据总线等存储器时，需要考虑读/写效率和逻辑允许等问题。有些 CPU 要求低位地址必须对齐（如 Intel 8096），有些在低位地址不对齐时需要多个读/写周期完成 16 位或 32 位操作，如 80x86 CPU。CPU 按字访问时，存储单元地址可以是偶数（$A_0=0$）或奇数（$A_0=1$），对于 16 位字长的 CPU 如 8086/80286，对偶数地址进行字访问时，地址是对齐时（$A_0=0$），访问效率高；对于 32 位数据总线的 CPU 如 80386/Pentium，则当地址对齐（$A_1A_0=00$）时，能够高效实现双字（32 位）数据访问。

### 5.4.1 存储器芯片选择

#### 1. 存储器类型选择

如前所述，半导体存储器分成 RAM 和 ROM 两类。RAM 的特点是其存储的信息可以在程序中用读/写指令随机访问，但断电后信息丢失。所以 RAM 一般用于存储用户的调试程序（或程序存储器中的用户区）、程序运算的中间结果及断电后不需保护（存）的信息。静态 RAM 在与 CPU 接口时，一般不需要外围电路，连接比较简单，故在智能仪器仪表、小型系统中，一般采用静态 RAM。

动态 RAM 集成度高，但需要专门的刷新电路，因此与 CPU 的接口设计较为复杂，但是由于成本相对较低，在需要较大存储器容量的计算机产品中得到了广泛应用。

ROM 中的内容断电后不丢失，但不能随机写入，故一般用于存放系统程序（监控程序）和不需在线修改的参数等。掩膜 ROM 和 PROM 用于大批量生产的微电子产品或计算机产品中。在产品研制和小批量生产时，宜选用 EPROM 或 Flash ROM 等芯片，以便多次修改程序或用户自行编程。

E²PROM 多用于保存这样一些数据或参数，它们在系统工作过程中被写入而又需要掉电保护。近年来串行接口的 E²PROM 因其接口简单、工作可靠被大量用于系统设计，而 Flash ROM 被大量用于程序存储，尤其是支持软件的在线升级。

利用后备电源（一般是可充电电池）配合掉电保护电路，也可以使静态 RAM 在正常电源掉电时数据不丢失，这时需要选择有双（高、低电平）片选的低功耗 RAM 芯片。

存储器系统设计先确定整机存储容量，再确定选用存储芯片的类型和数量，之后划分 RAM、ROM 区，画出地址分配图。存储器空间的划分和地址编码靠地址线来实现。多片存储芯片构成的存储器的地址编码原则是：低位地址总线作为片内寻址，高位地址总线将用来产生存储芯片的片选信号。需要设计专门的片选译码电路，详见 5.5 节。

#### 2. 连接接口电平

通常，CPU 总线与存储器连接引脚的电平标准应相同，同为 TTL、CMOS、LVTTL、SSTL2、SSTL18 等标准电平，必要时需增加电平转换电路。

#### 3. 存储器芯片与 CPU 的时序配合

存储器的存取时间是反映其工作速度的重要指标，选用存储芯片时，需要分别从 CPU 和存储器芯片两个方面进行分析，保证存储器芯片读/写存取时间等工作时序和参数与 CPU 的时序和参数相匹配。

在 CPU 和存储器的数据手册中已设计规定了各自读/写操作时的地址、数据和控制总线信号的时序，因此连接时，必须注意它们的配合，如当 CPU 发出读数据信号时，存储器能输出数据并稳定在数据总线上，保证读操作能顺利进行。

典型的 8088 CPU 存储器读周期时序重画在图 5-21 中。对照表 5-6 并由图 5-21 可以看出，CPU 工作在 4.77 MHz 时钟频率时，从 $T_1$ 状态开始到地址信号有效其最长时间 $TCLAV_{max}=110$ ns。由 5.2 节中的图 5-8 得出 HM6264BL 读取时间 $t_{AA\max}=70$ ns。于是从 $T_1$ 状态开始到 HM6264BL 中指定单元读出信息到数据总线上的时间为 $TCLAV_{max}+t_{AA\max}=110\ \text{ns}+70\ \text{ns}=180\ \text{ns}$。

8088 CPU 在 $T_3$ 状态下跳沿采样数据总线得到数据。为确保采样数据是可靠的，要求 RAM 输出到总线上的数据比 $T_3$ 后沿提前 TDVCL 时间达到稳定。从 $T_1$ 开始到数据必须稳定的时间为 $3T-TDVCL=3\times200\ \text{ns}-30\ \text{ns}=570\ \text{ns}$。

对于 HM6264BL，从 $T_1$ 开始约 180 ns 后数据即可有效，早于 8088 要求的 570 ns 截止时间，具有约 390 ns 的时序裕量。因此在上述条件下，二者的读周期时序能够配合。

存储器写周期时序配合与读周期类似，不再重复叙述。如果存储器芯片读或写周期的工作速度不能满足 CPU 的要求，可在 CPU 的相应周期内插入一个或数个 $T_w$ 周期，人为地延长 CPU 读/写时间，使两者相配。为简化外围电路及充分发挥 CPU 的工作速度，应尽可能选择与 CPU 时序相配的芯片。

### 5.4.2 存储器容量扩充

> [!abstract] 位扩充与字扩充
> - **位扩充**：芯片数 = 总容量 ÷ 单片容量（字数不变、位数增加）；各芯片地址线并联、数据线分别接不同位。
> - **字扩充**：芯片数 = 总字数 ÷ 单片字数（位数不变、字数增加）；需片选译码把各芯片映射到不同地址区间。

当一片存储器芯片的容量不能满足系统要求时，需多片组合以扩充字长（位扩充）或字数（字扩充）。下面以 SRAM 为例说明容量扩充的方法，ROM 的处理方法与此类似。

#### 1. 存储器位扩充

用 $8K \times 8$ b 的 SRAM 芯片 HM6264 扩充形成 $8K \times 16$ b 的芯片组，所需芯片为 $16\ \text{b}/8\ \text{b}=2$（片）。这两个芯片（0# 和 1#）的地址线 $A_0 \sim A_{12}$ 分别连在一起，各芯片相应的片选信号及读/写控制信号也都分别连在一起，两个芯片只有数据线各自独立，一片连低 8 位（$D_0 \sim D_7$），另一片连高 8 位（$D_8 \sim D_{15}$），如图 5-22 所示。也就是说，每个 16 位数据的高、低字节分别存储于两个芯片单元中，一次读/写操作同时访问两个芯片的相同地址单元。

*图 5-22　存储器位扩充示意图*

#### 2. 存储单元数扩充

当存储器位数满足要求而需要扩充单元数时，也需要用若干芯片（组）组成新的芯片组，如用 $8K \times 8$ b 芯片 6264 构成 $32K \times 8$ b 的存储区时，所需要的 $8K \times 8$ b 芯片数为 $32K/8K=4$（片）。

![[计算机系统/微机原理与接口技术B/附件/第5章/Pasted image 20260719161124.png]]
*图 5-23　存储单元数扩充示意图*

这 4 个 8 KB 芯片（0#～3#）连成 $32K \times 8$ b 芯片组，如图 5-23 所示，即各 6264 芯片的地址线 $A_0 \sim A_{12}$、数据线 $D_0 \sim D_7$ 及读/写信号都是同名信号连在一起。字数的扩充使 $32K \times 8$ b 芯片组较 $8K \times 8$ b 芯片增加了两位地址信号 $A_{13}$、$A_{14}$，它们译码后产生 4 个片选信号，分别选中 4 个 $8K \times 8$ b 芯片中的哪一个，这 32K 的地址范围分配如表 5-7 所示。这里地址线 $A_0 \sim A_{12}$ 实现片内寻址，$A_{13} \sim A_{14}$ 实现片间寻址。

| $8K \times 8$ b 芯片号 | $A_{14}$ | $A_{13}$ | $A_{12} \sim A_0$ | 地址范围（空间） |
| :---: | :---: | :---: | :---: | :--- |
| 0# | 0 | 0 | $00 \cdots 0 \sim 11 \cdots 1$ | 00000H～01FFFH |
| 1# | 0 | 1 | $00 \cdots 0 \sim 11 \cdots 1$ | 02000H～03FFFH |
| 2# | 1 | 0 | $00 \cdots 0 \sim 11 \cdots 1$ | 04000H～05FFFH |
| 3# | 1 | 1 | $00 \cdots 0 \sim 11 \cdots 1$ | 06000H～07FFFH |

存储器的单元数和位数都需要扩充时，如用 $8K \times 8$ b 芯片构成 $32K \times 16$ b 存储区，需要 $4 \times 2$ 个芯片。可以先扩充位数，每 2 个芯片一组，构成 4 个 $8K \times 16$ b 芯片组；再扩充单元数，将这 4 个芯片组组合成 $32K \times 16$ b 存储块。

### 5.4.3 RAM 存储模块

高集成度 RAM 模块就是习惯上所说的内存条（Memory Module），其产生与发展与 CPU 性能的提高密切相关，利用 DRAM 基本存储电路存储信息，把动态刷新电路集成在片内，克服了 DRAM 需要加外部刷新电路的缺点，从而兼有静态和动态 RAM 的优点。

高集成度 RAM 将多片高容量 DRAM 芯片装配在条状印制线路板上，线路板配有标准单边或双边沿连接插脚，可直接插入微机主板的存储器插座，加上相应的控制电路，构成具有规定字长宽度及奇偶校验特性的存储器模块。

PC 系统常用的模块中，按其不同标准和数据字长，可分为 30 线（8+1 位）SIMM（Single In-line Memory Module）、72 线（32+4 位）SIMM 和 168 线、184 线、240 线（64+8 位）DIMM（Dual In-line Memory Module）等，并满足不同的总线速度。其中，30 线 SIMM 多用于 80386 以下系统，72 线 SIMM 多用于 80486 系统，168 线 DIMM 主要用于 Pentium 以上机型（PC66、PC100、PC133 等），而 PC 标准内存条 DDR SDRAM 184 线 DIMM、DDR2 和 DDR3 为 240 线（如图 5-24 所示），DDR4 为 288 线，主要应用参数包括前端总线频率、传输速率、存储容量等。

![[计算机系统/微机原理与接口技术B/附件/第5章/Pasted image 20260719161134.png]]
*图 5-24　典型 DIMM 内存模块及其组织结构*

事实上，内存条是根据 CPU 的特性组合扩充得到的具有标准接口和读/写特性的存储模块。例如，30 线 LGS 产品 GMM79100 将二片 $1M \times 4$ b 和一片 $1M \times 1$ b（奇偶校验位）构成 $1M \times 9$ b 的 SIMM 模块，刷新频率为 1 kHz，5 V 供电，FPM 模式，存取速度为 $60/70$ ns；72 线 GMM7324210，以 8 片 $4M \times 4$ b 为基片，构成 $4M \times 32$ b DRAM SIMM，EDO 模式，5 V，刷新频率为 4 kHz，存取速度为 $50/60$ ns。

Pentium Ⅱ 和 Pentium Ⅲ CPU 系统主要采用 168 线 SDRAM 模块，典型芯片如 Samsung 公司的 KMM375S1620BT，是一种容量 $16M \times 72$ b 的 SDRAM DIMM（128 MB 带奇偶校验），上面集成了 18 片 $16M \times 4$ b 的 SDRAM 芯片，2K SPD（Serial Presence Detect）串行 E²PROM，以及锁相环 PLL 等辅助电路，工作电压为 3.3±0.3 V，LVTTL 输入/输出兼容，支持突发模式，自动及自刷新频率 44 kHz/64 ms，符合 PC100 标准，最高频率可达 125 MHz；支持 SPD 的母板可以从 E²PROM 中读取 SDRAM 的工作参数，以避免由于参数设置不当而引起系统的不稳定。

Pentium 4 CPU 系统主要采用 DDR SDRAM，由一组 DDR SDRAM 存储器芯片组合集成，模块数据字长 72（64+8 位 ECC 校验纠错位），如 Micron 公司的 184 线 DDR 模块 MT18VDDT3272D 为 256 MB（$32M \times 72$ b）、MT18VDDT6472D 为 512 MB（$64M \times 72$ b）、MT18VDDT12872D 为 1 GB（$128M \times 72$ b）等，传输速率 PC1600/PC2400。240 线 DDR2 模块 MT9HTF3272 为 256 MB、MT9HTF6472 为 512 MB、MT9HTF12872 为 1 GB（$128M \times 72$ b），前端总线频率 400 MT（传输次）/s 和 533 MT/s，传输速率 PC2-3200 或 PC2-4200，其原理框图见图 5-24。

DDR SDRAM 已经历多代演进，不同平台可能采用 DDR3、DDR4、DDR5 或面向特定场景的其他存储技术。教材中的 DDR3/DDR4 参数用于说明代际变化；实际选型必须同时匹配处理器内存控制器、主板布线、模块类型、工作电压、速率等级、容量与纠错需求。
