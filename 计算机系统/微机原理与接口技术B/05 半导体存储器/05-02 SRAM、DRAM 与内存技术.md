## 5.2 随机存取存储器（RAM）

### 5.2.1 静态 RAM（SRAM）

#### 1. SRAM 原理

##### 1. 基本存储电路

基本存储电路主要由 R-S 触发器构成，其两个稳态分别表示存储内容为 0 或 1。容易理解，SRAM 在正常电源供电时，存入的数据可以保存和读出；只要不写入新的数据，触发器状态就保持不变，读（触发器的输出）操作不会改变它；但 RAM 芯片在断电后，原存储的信息将全部丢失，这就是所谓的“易失性”。

##### 2. 外围电路

与一般存储器类似，外围电路包括如下。

1. 地址译码器：对输入的外部地址信号译码，用以选择要访问的单元。在目前所用的 SRAM 存储器中，通常采用双译码结构，如图 5-6 所示。

2. I/O 控制电路：处于数据总线和被选中的单元之间，如用 $\text{R}/\overline{\text{W}}$（读/写控制信号），也可用独立的存储器读（$\overline{\text{RD}}$）和存储器写（$\overline{\text{WR}}$），负脉冲有效）来控制被选中单元的读出或写入。$\overline{\text{CE}}$（或 $\overline{\text{CS}}$）为片选信号，当 $\overline{\text{CE}}$ 为低电平时，该芯片被选中使能，并能正常读/写信息；而当 $\overline{\text{CE}}$ 为高电平时，表示该芯片未被选中。

### 5.2.2 动态 RAM（DRAM）

#### 1. 单管基本动态存储电路

![[计算机系统/微机原理与接口技术B/附件/第5章/Pasted image 20260719160900.png]]
*图 5-10　单管基本动态存储电路*

基本动态存储电路是利用 MOS 管栅极和源极之间的电容 $C$ 来存储信息的，图 5-10 为单管基本动态存储电路。以电容上“有”“无”电荷两种状态来区分其存储的二进制信息是“1”还是“0”。由于工艺问题，一般 $C$ 的电容值小于数据线的分布电容 $C_D$，每个数据读出后，$C$ 上的电荷经 $C_D$ 释放，信息易被破坏，所以每个数据读出后，要重新恢复 $C$ 上的电荷量。即使无“读操作”，电容随着时间变化放电，电荷泄漏也会造成信息丢失。为保持 $C$ 中信息（电荷），需周期性地充电，这一过程称为（动态）刷新。刷新周期通常为 $2 \sim 8$ ms。

与 SRAM 相比，DRAM 的基本存储电路较简单，故集成度较高、功耗小，除了正常读/写，额外增加了刷新操作，需要相应的外部电路支持，设计使用较复杂。另外，DRAM 的存取速度一般比 SRAM 慢。

DRAM 一般用于组成大容量 RAM 存储器，而容量不大时（如常规小型系统或高速缓存）选用 SRAM 芯片更为简单实用。

#### 2. 典型芯片 $\mu$PD424256

##### 1. 内部结构

$\mu$PD424256 芯片的容量是 $256K \times 4$ b，片内译码需 $\log_2 256K=18$ 条地址线。为了减少封装引线、缩小体积，地址线被分成两部分：行地址和列地址，这样芯片的外部地址引线只有 9 条，行、列地址分时送入，利用内部多路开关和锁存器，由行地址选通信号 $\overline{\text{RAS}}$ 先把出现在地址线上的 9 位地址信号锁存至行地址锁存器，然后列地址选通信号 $\overline{\text{CAS}}$ 把后出现在地址线上的 9 位地址锁入列地址锁存器，如图 5-11 所示。

![[计算机系统/微机原理与接口技术B/附件/第5章/Pasted image 20260719160907.png]]
*图 5-11　μPD424256 内部结构*

##### 2. 读/写控制

行列地址译码后选中 256K 个单元中的一个单元进行访问，读/写由 $\overline{\text{OE}}$ 和 $\overline{\text{WE}}$ 控制。当 $\overline{\text{WE}}$ 有效（低电平）时，表示写入，I/O 线上的信号经输入三态缓冲器写入被选中单元；当 $\overline{\text{WE}}$ 无效（高电平）时，表示读出，$\overline{\text{OE}}$ 有效时被选中单元的内容经输出三态门出现在 I/O 线上。$\mu$PD424256 无片选信号 $\overline{CS}$。$\overline{\text{RAS}}$ 和 $\overline{\text{CAS}}$ 均有效即表示芯片被选中。

##### 3. $\mu$PD424256 工作时序

![[计算机系统/微机原理与接口技术B/附件/第5章/Pasted image 20260719160918.png]]
*图 5-12　μPD424256 读写周期时序*

1. 读周期（时序如图 5-12(a)所示）。为了使芯片能正确地接收行、列地址并实现读操作，各信号间的关系应满足如下条件：

- 行地址必须在 $\overline{\text{RAS}}$ 信号有效之前送到芯片的地址输入端，以确保被选中的单元信息正确读出。
- $\overline{\text{CAS}}$ 信号应迟后 $\overline{\text{RAS}}$ 一段时间，并迟后于列地址送到芯片地址输入端的时间。
- $\overline{\text{RAS}}$、$\overline{\text{CAS}}$ 应有一定宽度，才能保证被选中单元信息的正确读出。
- 读有效（$\overline{\text{WE}}=1$）应在 $\overline{\text{CAS}}$ 有效之前建立。

2. 写周期（时序如图 5-12(b)所示）。写周期中 $\overline{\text{RAS}}$ 与 $\overline{\text{CAS}}$ 之间的关系，以及它们与地址信号之间的关系和读周期相同。不同的是：

- 写命令（$\overline{\text{WE}}=0$）应在 $\overline{\text{CAS}}$ 信号有效之前建立，且在 $\overline{\text{RAS}}$（或 $\overline{\text{CAS}}$）变高前结束。
- 写入的数据必须在 $\overline{\text{CAS}}$ 有效之前出现在 I/O 引脚，才能保证数据可靠地写入。

3. “读-修改-写”时序。在指令中，常常需要对某个单元的内容读出后进行修改，再写回该单元。为提高操作速度，存储器设计了专门的“读-修改-写”周期，基本上是读/写周期的组合。

4. 页模式周期。为提高数据块操作时数据传送的速度，存储器还设计有页模式操作。在该模式下，$\overline{\text{RAS}}$ 不变，由连接的 $\overline{\text{CAS}}$ 信号对不同的列地址进行操作。注意，这时 $\overline{\text{RAS}}$ 脉冲宽度有一个最大限度，即每次能连续扫描的列数是有限的。在页模式操作时，可实现读/写及“读-修改-写”操作。

##### 4. 刷新周期

刷新操作是通过执行只有 $\overline{\text{RAS}}$ 信号的 DRAM 访问周期来实现的。刷新时，在 $\overline{\text{RAS}}$ 信号期间行地址有效，选中存储矩阵中的一行单元，将其内容分别读至相应的内部读出放大器后又重新写回单元。对由行地址决定的 512 个组合逐一施加 $\overline{\text{RAS}}$ 信号，$\mu$PD424256 芯片内部的所有单元都进行一次刷新，刷新周期为 8 ms。注意，因在刷新期间 $\overline{\text{CAS}}=1$，刷新数据不会出现在 I/O 线上。

### 5.2.3 随机存取存储器 RAM 的应用

在实际应用中，随着 CPU 存取速度的不断提高，要求存储器的读/写速度随之提高。在技术上，一方面要提高存储器容量和速度，主要通过 DRAM 改进，缩短延迟、提高带宽并结合应用采取专门措施帮助提高系统性能；另一方面需要引入高速缓存技术，提高系统级存储器读/写速度。

#### 1. 基于预测技术的 DRAM

要加快普通 DRAM 的访问速度，简单实用的方法是在芯片上附加一些逻辑电路。以 DRAM 为例，这些附加电路包括地址多路转换电路、地址选通、刷新逻辑以及读/写控制逻辑等。通过增加少量的附加逻辑电路，可以提高单位时间内的数据流量，即通过预测增加带宽的技术。较为成功的技术包括快速页模式 FPM（Fast Page Mode）和扩展数据输出 EDO（Extended Data Output），相应的存储器称为 FPM-DRAM 和 EDO-DRAM。

1. 快速页模式 FPM。通常在 DRAM 阵列中读取一个单元时，首先提供一个行地址，并置 $\overline{\text{RAS}}$ 为低电平，然后通过周期性的列信号 $\overline{\text{CAS}}$ 实现多个列存取。$\overline{\text{RAS}}$/$\overline{\text{CAS}}$ 选择时充电线路在稳定前会有一定的延迟，制约了存储器的读/写速度。在大多数情况下，要存取的数据在 RAM 中是连续存放的，即下一个要存取的单元常常位于当前单元的下一地址，FPM 采用这一预测技术（这里的页指 DRAM 芯片一行存储单元中的一个 2048 位区间），增加了快速页读或写操作来缩短页模式周期。

2. EDO 模式。在 FPM 的基础上，增加了超页（Hyper Page）读或写以及超页“读-修改-写”等操作，即利用地址预测，可在当前读/写周期中启动下一个存取单元的读/写周期，进而在宏观上缩短地址选择的时间。采用这一技术，可比 FPM 提高效率 10%～15%。此外，在 EDO 技术上引入突发模式，即假定对若干后续地址进行预取操作。

#### 2. 同步 DRAM（SDRAM）

上述 FPM-RAM 和 EDO-RAM 都属于异步内存，其输入/输出过程不与 CPU 同步，CPU 在存取数据时必须保持和监测各种信号，等待 DRAM 完成所有的内部操作。典型的等待时间为 $50 \sim 70$ ns，这个延迟时间也就代表了异步 DRAM 的存取速度。

相比之下，SDRAM（Synchronized DRAM）则具有同步接口。它的基本原理是，将 CPU 和 RAM 通过一个相同的时钟锁在一起，使它们共享一个时钟周期，以相同的速度同步工作。即在系统时钟的控制下，SDRAM 能够把地址、控制和数据信号同步起来，经过指定的时钟数后，CPU 可以直接从数据总线上获得所需数据或将数据写到总线存入相应的存储单元。由于不再需要监测数据的存取过程，一些具有乱序执行功能的高级 CPU 能够在等待存取数据时完成其他工作，这也是 SDRAM 的优点之一。SDRAM 芯片的速度通常用其最高输出频率来表示，如 66 MHz、100 MHz、133 MHz，转换成时间分别为 15 ns、10 ns、7.5 ns。但是当多片 SDRAM 被集成为模块（内存条）使用时，实现各种信号之间的同步则会引入额外的延迟，因此模块的稳定工作频率要低于芯片最高输出频率。

SDRAM 基于多存储体结构，内含 2 或 4 个交错的存储阵列体（bank），当 CPU 从一个阵列体访问数据的同时，另一个阵列体已准备好读/写数据。通过存储阵列体的紧密切换，读取效率可得到成倍的提高。

SDRAM 芯片还支持突发传输模式，在这种工作模式下，当第一个列地址输入之后，芯片内部自动产生后续若干（2、4、8 个或 FP 全页）连续的列地址，从而可以快速输出后续地址数据（预取）。突发数据长度可通过修改突发计数器对应的寄存器来设定。SDRAM 与支持突发模式的高速缓存相配合，可以有效地提高系统的性能。“全页”方式时，只在时钟脉冲的上升沿对行地址选通信号 $\overline{\text{RAS}}$ 及其有关的控制信号进行采样，之后不再需要这些信号，也就是说，$\overline{\text{RAS}}$ 信号仅需持续一个时钟周期，而呈现为一个窄脉冲。这不同于传统的 DRAM，后者通常要求 $\overline{\text{RAS}}$ 信号在整个操作过程中始终保持有效。

SDRAM 通常支持两种刷新方式：自动刷新（Auto-refresh）和自刷新（Self-refresh）。前者为标准方式，当 CKE 信号有效且时钟允许时进行刷新；后者当芯片处于低功耗情况（CKE 及时钟禁止）时芯片自行刷新。

LGS 公司的 GM72V28441 是典型的 SDRAM 芯片，其容量为 $8M \times 4$ b$\times$4 bank，符合 PC66/100/133 标准，LVTTL 接口，3.3 V 电源，每 64 ms 包含 4096 个刷新周期，具有双刷新操作，可编程突发访问模式，顺序或间隔，长度 1、2、4、8 或 FP（全页）；可编程 CAS 延迟 2 或 3 个时钟；4 个存储阵列块可独立或同时操作；可突发读/突发写或者突发读/单独写。图 5-13 为 GM72V28441 芯片引脚图及功能框图。

DDR SDRAM（Double Data Rate SDRAM，双倍数据速率同步动态随机存储器）在时钟上升沿和下降沿都传输数据，因此在相同 I/O 时钟频率下，其理论数据传输次数是单倍数据速率 SDRAM 的两倍。双沿传输提高了带宽，但系统功耗仍取决于工作电压、频率、负载、电路活动率和终端方式，不能据此断言“不会增加能耗”。第一代 DDR 常称 DDR 或 DDR1，典型工作电压为 2.5 V，并采用 SSTL_2 接口电平。

后续 DDR2、DDR3、DDR4 和 DDR5 通过预取、Bank 组织、信号完整性、供电与命令协议等多方面改进提高带宽和容量。预取深度描述一次核心阵列访问向 I/O 缓冲区传送的数据量，并不等同于“一个时钟周期执行若干次完整存储器访问”。各代 DDR 的电气规范、引脚和控制协议不同，通常不能直接互换；实际选型必须与处理器的内存控制器和主板设计匹配。表 5-4 保留到 DDR4，用于展示早期代际演进。

#### 3. 基于协议/内部串行接口的 RDRAM

RDRAM（Rambus DRAM）是 DDR SDRAM 相对昂贵的替换品，由美国 Rambus 公司开发。与 DDR 和 SDRAM 不同，RDRAM 引入了新一代高速简单内存架构，采用串行数据传输模式，使整个系统性能得到提高。从本质上讲，RDRAM 是一种基于协议的 DRAM。由于种种原因，如无法保证与原有的制造工艺兼容、专利费用、制造成本及价格高昂等而被淘汰。
