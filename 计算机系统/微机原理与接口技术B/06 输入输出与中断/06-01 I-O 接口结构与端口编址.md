## 6.1 输入/输出及接口

输入和输出设备（称为 I/O 设备或外设）是计算机系统的重要组成部分，计算机通过它们与外界进行数据交换。如程序、原始数据及各种现场采集得到的信息，都必须通过输入装置输入到计算机；计算机也必须把计算的结果或各种控制信号送到各输出装置，以便显示、打印和实现各种控制动作。计算机常用的 I/O 设备有键盘、鼠标、显示器和打印机等。

### 6.1.1 I/O 信息的组成

> [!abstract] 三类信息
> 在计算机控制系统中，外设与 CPU 之间需频繁交换的信息有以下三种：
> - **数据信息**：CPU 与外设间交换的基本内容，分为数字量、模拟量和开关量。
> - **状态信息**：CPU 与外设间交换数据时的联络信息。
> - **控制信息**：CPU 发给外设的命令信息。

#### 1. 数据信息

CPU 与外设间交换的基本信息就是数据信息，分为数字量、模拟量和开关量三种形式。

| 形式 | 含义 | 与 CPU 的接口关系 |
| :--- | :--- | :--- |
| **数字量** | 按编码标准（二进制或 ASCII）由若干位组合表示的数或字符 | 直接进入 CPU，如键盘、打印机 |
| **模拟量** | 现场非电量经传感器、放大后得到的电压或电流 |须经 A/D 转换输入、D/A 转换输出 |
| **开关量** | 开关的断开与闭合等二值状态 | 经电平转换后连接，1 位二进制即可表示；8 位机一次可输入/输出 8 位开关量 |

数据的传送可采用并行传送和串行传送两种形式，本章只讨论并行传送。

#### 2. 状态信息

> [!info] 状态信息的作用
> 状态信息指在 CPU 与外设之间交换数据时的**联络信息**。CPU 通过对外设状态信息的读取，可得知其工作状态。如了解输入设备的数据是否准备好、输出设备是否空闲等。了解状态信息是 CPU 与 I/O 设备之间进行正确数据交换的重要条件。

#### 3. 控制信息

> [!info] 控制信息
> 控制信息指 CPU 发给外设的**命令信息**，如设置 I/O 设备工作模式的信息等。CPU 通过发送控制信息控制外设的工作。

### 6.1.2 I/O 接口概述

> [!abstract] 接口的定义
> 接口是连接计算机和外设的部件，通常指**接口电路**，广义的接口还包括管理接口电路的驱动程序。

#### 1. I/O 接口的功能

理论上，CPU 与 I/O 设备之间的连接及信息处理和 CPU 与存储器之间的连接及信息处理类似。实际上，I/O 设备类型繁多，工作速度高低不一，传输数据的字长多样，CPU 与其交换信息相应复杂。因此，I/O 设备通常并不直接与 CPU 交换信息，而是通过相应的输入/输出接口（称为 I/O 接口）来进行。I/O 接口是 CPU 和 I/O 设备之间进行信息交换的**中间环节**。

> [!info] I/O 接口的主要功能
> 在实际应用中，I/O 接口电路种类繁多，功能各异，可分为专用接口和通用接口，或分成可编程接口和不可编程接口，也可分成并行接口和串行接口，其主要功能有：
> 1. 对输入/输出数据进行**缓冲、隔离和锁存**；
> 2. 完成信息格式的**变换**，如串-并转换等；
> 3. 实现电气特性的**匹配**；
> 4. 选择设备或**地址译码**等。

#### 2. I/O 接口的构成

当 CPU 通过 I/O 接口与外设交换数据、状态和控制信息时，一般是通过接口电路中的寄存器来实现的，这些寄存器分别称为数据、状态和控制端口。每个 I/O 接口电路通常包含若干这样的寄存器端口，CPU 通过对这些端口的读/写实现与外设的信息交换。

*图 6-1　I/O 接口的构成示意图*

> [!info] 接口电路的三大组成模块
> - **总线驱动**：实现对 CPU 数据总线速度和驱动能力的匹配。
> - **地址译码**：实现对外设中各寄存器端口的寻址操作。
> - **控制逻辑**：实现 CPU 对接口电路中各寄存器端口的读/写操作和时序控制。

> [!important] 状态/控制信息也走数据总线
> 状态信息和控制信息通常也是通过数据总线传送的，由于它们的性质不同于数据信息，在传送时被赋予不同的端口。因此一个外设往往占用几个端口——CPU 对外设的控制或 CPU 与外设间的信息交换，实际上转换成了 CPU 通过 I/O 指令进行端口的操作。只是对不同的端口，读/写的数据性质不同罢了。

#### 3. I/O 接口的类型

早期的计算机外设种类非常有限，相应的接口标准也很简单。通常使用 PS/2 接口连接键盘和鼠标，串行接口（RS-232）连接外置 Modem，并行接口接入针式打印机或扫描仪等。

接口标准会随设备类型和平台代际演进。USB 通过统一的主机—设备模型支持键盘、鼠标、存储设备和打印机等多类外设；IEEE 1394 曾较多用于数字视频设备；VGA 是模拟显示接口；常见的 8P8C 模块化连接器（口语中常称"RJ-45"）通常承载以太网物理链路。它们在电气特性、拓扑、协议和带宽上各不相同，不能只依据连接器外形判断接口功能。

> [!note] 本章的接口层次
> 8255A、8253/8254、8259A 等芯片用于讲解端口、握手、定时和中断等基础机制。现代 USB、PCIe 等接口封装层次更高，但底层仍要解决地址译码、数据缓冲、时序同步、状态报告和错误处理等同类问题。

### 6.1.3 I/O 端口的编址

> [!abstract] 两种 I/O 端口编址方式
> - **独立编址**：I/O 端口与内存地址空间相互独立，用 `IN` / `OUT` 指令访问，地址线少、译码简单。
> - **统一编址（存储器映射 I/O）**：I/O 端口占用内存地址空间，可用任意访存指令，但会占用部分内存地址。

由于 CPU 与外设间的信息交换是通过接口中的 I/O 端口完成的，因此 CPU 必须对 I/O 端口进行寻址。CPU 寻址外设端口一般有两种方式。

#### 1. I/O 端口独立编址与统一编址的对比

| 对比项 | 独立编址（直接 I/O 映射） | 统一编址（存储器映射 I/O） |
| :--- | :--- | :--- |
| 地址空间 | 存储器与 I/O 端口为**两个独立**地址空间 | 存储单元与外设端口**统一编址** |
| 访问指令 | 专门的 `IN`/`OUT` 指令 | 任意访存指令均可 |
| 区分依据 | 通过不同指令和专门控制信号区分 | 通过地址区域不同加以区分 |
| 译码电路 | I/O 端口空间小，译码电路相对简单 | 端口寻址手段丰富 |
| 内存空间 | 不占用内存地址空间 | 占用部分内存地址空间，使可用内存减小 |
| 程序易读性 | 易分清是访存还是访外设 | 不易分清是访存还是访外设，易读性较差 |

#### 2. 80x86/Pentium 的独立编址实现

> [!important] 80x86/Pentium 采用独立编址
> 80x86/Pentium CPU 采用 I/O 端口独立编址方式，端口地址 **16 位**。与存储器地址空间相似，I/O 地址空间在物理上也由 1、2、4、8 个 8 位 I/O 体构成，以支持 8、16、32、64 位 I/O 端口操作。因而 16 位地址线可寻址：
> - **64K** 个 8 位端口
> - **32K** 个 16 位端口
> - **16K** 个 32 位端口
> - **8K** 个 64 位端口

> [!info] PC/XT 的端口寻址
> 在 PC/XT 中，实际参与端口寻址的只有低 10 位地址线 $A_9 \sim A_0$。因此 PC/XT 的 I/O 端口空间的大小为 **1 KB**，以 $A_9$ 来区别端口所在位置：
> - $A_9=0$：寻址**主板**上的 512 个端口
> - $A_9=1$：寻址 **I/O 扩展卡**上的 512 个端口
>
> 译码条件：AEN 为低电平（表示 CPU 掌控总线；AEN 高电平时表示由 DMA 控制器送出的地址有效）。

| CPU 系列 | 寻址地址线 | I/O 体选择信号 | 备注 |
| :--- | :--- | :--- | :--- |
| 8086/8088 | $A_{15} \sim A_0$（理论 16 位） | — | PC/XT 实际只用 $A_9 \sim A_0$ |
| 80386 / 80486 | $A_{15} \sim A_2$ | $\overline{BE_3} \sim \overline{BE_0}$ | 选择 8/16/32 位 I/O 数据 |
| Pentium 系列 | $A_{15} \sim A_3$ | $\overline{BE_7} \sim \overline{BE_0}$ | 选择 8/16/32/64 位 I/O 数据 |

> [!tip] 用户扩展接口的地址选择
> 用户在设计扩展接口时，需注意不要使用系统已占用的地址空间。表中**未指明**的端口地址是最可靠的用户可用地址。

### 6.1.4 简单的 I/O 接口

> [!abstract] 简单接口的基本元件
> 不同的 I/O 设备所需采用的 I/O 接口电路的复杂程度相差甚远，但分解到最基本的功能，接口中应用最多的是**三态缓冲器**和**数据锁存器**。

#### 1. 三态缓冲器

> [!important] 三态缓冲的必要性
> 在微机系统中，每个输入设备都需通过数据总线向 CPU 传送数据。若不经过三态门电路进行缓冲隔离而直接和数据总线相连，就会造成总线上数据的混乱。因此，每个接入设备必须经过缓冲隔离——当 CPU 选通时，才允许选定的输入设备将数据送到系统总线，其他的输入设备此时与数据总线隔离。

74LS244 是一种常用的三态缓冲器，其引脚及内部结构如图 6-3 所示，可以构成简单的输入接口，如图 6-4 所示。8 个开关 $K_1 \sim K_8$ 接 74LS244 的输入，其输出接到系统的数据总线上。当对此端口进行输入操作时，地址译码和 $\overline{IOR}$ 同时有效，于是 $\overline{1G}$、$\overline{2G}$ 有效，8 个开关的开合状态所表示的二进制数被输入到计算机中。

#### 2. 数据锁存器

> [!important] 数据锁存的必要性
> 数据总线是 CPU 和外部交换数据的公用通道。当 CPU 把数据送给输出设备时，只有执行总线周期的部分阶段总线会送出有效数据，因而必须利用数据锁存器及时把数据锁存起来，以便较慢的外设有足够的时间进行处理，使 CPU 和总线能够脱身去做其他工作。

74LS373 和 Intel 8282 都是常用的 8 位锁存器。在微机系统接口中，通常采用 8D 触发器 74LS273 进行数据锁存，其引脚及内部结构如图 6-5 所示，可以构成简单的输出接口，如图 6-6 所示。

> [!info] 锁存时序
> 当 CPU 对此端口进行输出操作时，地址译码和 $\overline{IOW}$ 同时有效。在 $\overline{IOW}$ 的后沿，将数据总线上的数据锁存到 74LS273 的输出端，再通过驱动电路控制发光二极管。此处取 $\overline{IOW}$ 的上升沿锁存是为了**等待数据总线稳定**。
