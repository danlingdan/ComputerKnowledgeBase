## 6.6 中断程序设计

### 6.6.1 设计方法

80x86 系列微机可以处理 256 个中断，分为内部与外部中断两种类型。对不同类型中断的处理过程有些差异，但中断服务程序的编程原则大致相同。由于可屏蔽中断涉及对中断控制器的操作，所以其编程情况较内部中断和非屏蔽中断复杂些。在此以可屏蔽中断为例，介绍中断程序设计的一般过程（假设采用的中断控制器是 8259A）。

#### 1. 设置中断向量表

当外设申请中断得到响应后，CPU 首先在中断向量表中查找中断服务程序的入口地址，以便转入中断服务程序。所以在主程序中需设置中断向量，使其指向相应的中断服务程序。
如果利用 PC 已有的内部 8259A 处理中断，必须在设置中断向量表之前，首先保存系统原中断向量的内容，待用户程序执行完毕退出之前，取出原中断向量写入到中断向量表中，以恢复系统原始状态。
设置中断向量表可以利用传送指令直接访问中断向量表的相应存储单元，写入中断向量；PC 也可利用 DOS 系统功能调用 INT 21H 的 25H 和 35H 子功能修改中断向量。假如借用 PC/XT 中的 $\text{IRQ}_3$（中断码 0BH）响应外部中断，中断后须执行的中断服务程序名为 INTPROC，其中断向量表的设置可以用下面的程序段实现：

```assembly
; 保存原中断向量的内容
INTSEG  DW  ?               ; 定义存放原中断向量的单元
INTOFF  DW  ?
...
MOV     AH, 35H             ; 读取原中断向量在 ES:BX 中
MOV     AL, 0BH
INT     21H
MOV     INTSEG, ES          ; 保存原中断向量的段基址
MOV     INTOFF, BX          ; 保存原中断向量的偏移量
; 重新修改中断向量的内容
CLI                         ; 关中断，设置中断向量新内容
PUSHDS
MOV     AX, SEG INTPROC
MOV     DS, AX
MOV     DX, OFFSET INTPROC
MOV     AH, 25H             ; 将 DS:DX 置于中断向量表中
MOV     AL, 0BH
INT     21H
POP     DS
```

#### 2. 设置中断控制器

主程序中还需对中断控制器 8259A 进行初始化设置。若利用 PC 内部的 8259A 处理中断，由于操作系统已对 8259A 初始化，所以只需对 IMR 进行相应处理，开放那些允许的中断请求即可。同样，为了使应用程序返回 DOS 后系统能恢复原状态，应在修改 IMR 之前保存其原有内容，并于程序退出前予以恢复。假设允许 $\text{IRQ}_3$ 响应外部中断，编程修改 IMR 的程序段如下：

```assembly
INTIMR  DB  ?               ; 定义存放原 IMR 的单元
...
IN      AL, 21H             ; 读出 IMR
MOV     INTIMR, AL          ; 保存原 IMR 内容
AND     AL, 0F7H            ; 允许 $\text{IRQ}_3$，其他不变
OUT     21H, AL             ; 设置新 IMR 内容
...
; 恢复原先的 IMR
MOV     AL, INTIMR          ; 取出保留的 IMR 原内容
OUT     21H, AL             ; 重写 OCW1
```

#### 3. 设置 CPU 的中断允许标志 IF

硬件中断是随机的，当外设提出可屏蔽中断请求，系统是否允许响应这个中断可以通过设置 CPU 中的中断允许标志位 IF 来控制，设置 IF=0 或 IF=1 的关中断和开中断指令分别是 CLI 和 STI。通常，在系统刚上电需要初始化时不能产生中断，这段时间必须关中断；初始化完毕则应开中断，以便及时响应中断请求为外设提供服务。另外，进入中断服务程序后，也可马上开中断，以允许较高级的中断能够嵌套。

#### 4. 设计中断服务程序

无论中断服务程序完成什么服务，其中须含有：保护现场、中断服务、向 8259A 发送中断结束命令、恢复现场、中断返回的程序段。程序段的一般结构如下：

```assembly
INTPROC PROC               ; 中断服务程序
        PUSH    AX          ; 保护现场
        PUSH    BX
        ...
        STI                 ; 开中断，以允许中断嵌套
        ...
        }                   ; 中断处理
        CLI                 ; 关中断
        MOV     AL, 20H     ; 向 8259A 发送 EOI 命令
        OUT     20H, AL
        POP     BX          ; 恢复现场
        POP     AX
        IRET                ; 中断返回
INTPROC ENDP
```

外部中断服务程序是处理较紧急的事物的，因此服务时间应尽量短，能放在主程序完成的任务就不要由中断服务程序完成。另外，外部中断服务程序不要使用 DOS 系统功能调用 INT 21H，因为 DOS 版本的内核是不可重入的。例如，当主程序执行一个 DOS 系统功能调用时产生了外部中断，中断服务程序又调用这个 DOS 系统功能，就出现了 DOS 重入，这是不允许的。在中断服务程序中若要控制 I/O 设备，最好调用 ROM BIOS 功能子程序或对 I/O 端口直接编程。
上述中断程序设计过程，除了对中断控制器 8259A 的操作外，其他均适合内部中断和非屏蔽中断。但非屏蔽中断在 PC 中有特殊作用，一般不要改写其中断服务程序。

### 6.6.2 中断程序设计举例

#### 1. 软中断程序设计举例

> [!example] 例 6-5
> 自己定义一个软中断，中断类型码为 80H，在中断服务程序中完成 ASCII 码加偶校验位（第 7 位）的工作。ASCII 码首地址为 ASCBUF，字节数为 COUNT，加偶校验位后仍放回原处。程序如下：

```assembly
DATA    SEGMENT                 ; 数据段
ASCBUF  DB    'ABCDEFGHIJ1234567890'
COUNT   EQU   $ - ASCBUF
DATA    ENDS
STACK   SEGMENT STACK    'STACK' ; 堆栈段
        DB    100 DUP (?)
STACK   ENDS
CODE    SEGMENT                 ; 代码段
        ASSUME  CS:CODE, SS:STACK, DS:DATA
START:  MOV     AX, DATA
        MOV     DS, AX
        CLI                     ; 关中断，设置中断向量表
        SUB     AX, AX
        MOV     ES, AX          ; 中断向量表的段基址为 0000H
        MOV     DI, 4 * 80H     ; 在中断向量表中的偏移量
        MOV     AX, OFFSET INTSV
        CLD
        STOSW                   ; 写中断向量的偏移量
        MOV     AX, SEG INTSV
        STOSW                   ; 写中断向量的段基址
        STI
        ...
        INT     80H             ; 软中断指令
        ...
        MOV     AH, 4CH         ; 返回 DOS
        INT     21H
INTSV   PROC                    ; 中断服务程序
        PUSH    AX              ; 保护现场
        PUSH    BX
        PUSH    CX
        MOV     CX, COUNT
        MOV     BX, OFFSET ASCBUF
L2:     MOV     AL, [BX]
        AND     AL, AL          ; 建立标志 PF
        JP      L1              ; PF=1，偶数个 1
        OR      AL, 80H         ; PF=0，奇数个 1
        MOV     [BX], AL        ; 加入偶校验后写回
L1:     INC     BX
        LOOP    L2
        POP     CX              ; 恢复现场
        POP     BX
        POP     AX
        IRET                    ; 中断返回
INTSV   ENDP
CODE    ENDS
        END     START
```

#### 2. 硬中断程序设计举例

> [!example] 例 6-6
> 8259A 的 $\text{IRQ}_0$ 中断请求（对应中断类型码 08H）来自定时器 8253 的通道 0。系统中设置它每隔 55 ms 产生一次中断，并提供完成日时钟计时功能的中断服务程序。本示例中将替换系统计时程序，使得每次中断显示一串信息，显示 10 次后中止，程序返回 DOS。

```assembly
STACK   SEGMENT STACK            ; 堆栈段
        DB    256 DUP (0)
STACK   ENDS
DATA    SEGMENT                  ; 数据段
MSG     DB    'A 8259A INTERRUPT!', 0DH, 0AH   ; 显示信息，共 20 个字符
COUNTER DB    0                  ; 中断次数记录单元
INTSEG  DW    ?                  ; 保存原中断服务程序的段基址
INTOFF  DW    ?                  ; 保存原中断服务程序的偏移量
INTIMR  DB    ?                  ; 保存原 IMR 内容
DATA    ENDS
CODE    SEGMENT                  ; 代码段
        ASSUME  CS:CODE, DS:DATA, SS:STACK
START:  MOV     AX, DATA
        MOV     DS, AX
        MOV     AX, 3508H        ; 保存原中断向量内容
        INT     21H
        MOV     INTSEG, ES
        MOV     INTOFF, BX
        CLI                      ; 关中断
        PUSH    DS               ; 设置中断向量表新内容
        MOV     AX, SEG INTPROC
        MOV     DS, AX
        MOV     DX, OFFSET INTPROC
        MOV     AX, 2508H
        INT     21H
        POP     DS
        IN      AL, 21H          ; 读出 IMR
        MOV     INTIMR, AL       ; 保存原 IMR 内容
        AND     AL, 0FEH         ; 允许 $\text{IRQ}_0$，其他不变
        OUT     21H, AL          ; 设置新 IMR 内容
        MOV     COUNTER, 0       ; 设置中断次数初值
        STI                      ; 开中断
L1:     CMP     COUNTER, 10      ; 循环等待中断
        JB      L1               ; 中断 10 次退出
        CLI                      ; 关中断
        MOV     AL, INTIMR       ; 恢复 IMR
        OUT     21H, AL
        MOV     DX, INTOFF       ; 恢复中断向量内容
        MOV     AX, INTSEG
        MOV     DS, AX
        MOV     AX, 2508H
        INT     21H
        STI                      ; 开中断
        MOV     AX, 4C00H        ; 返回 DOS
        INT     21H              ; 主程序结束

INTPROC PROC                     ; 中断服务程序
        PUSH    AX               ; 保护现场
        PUSH    BX
        PUSH    CX
        PUSH    DS
        STI                      ; 开中断
        MOV     AX, DATA         ; 在真正由外部随机信号引起的中断中，中断时数据段
        MOV     DS, AX           ; 地址 DS 是不确定的，所以程序中必须设置 DS
        INC     COUNTER          ; 中断次数加 1
        MOV     BX, OFFSET MSG   ; 显示信息
        MOV     CX, 20           ; 20 个字符
        CALL    DISPLAY
        CLI                      ; 关中断
        MOV     AL, 20H          ; 发送 EOI 命令
        OUT     20H, AL
        POP     DS               ; 恢复现场
        POP     CX
        POP     BX
        POP     AX
        IRET                     ; 中断返回
INTPROC ENDP

DISPLAY PROC                     ; 显示字符串程序
        PUSH    AX
DISPL:  MOV     AL, [BX]         ; 入口参数：BX=字符串首址
        CALL    DISPCHAR         ; CX=显示字符串个数
        INC     BX
        LOOP    DISPL
        POP     AX
        RET
DISPLAY ENDP

DISPCHAR PROC                    ; 显示 AL 中的字符
        PUSH    BX               ; 入口参数：BL=字符颜色，BH=页号
        MOV     BX, 0
        MOV     AH, 0EH          ; 调用 ROM BIOS 功能显示 AL 中的字符
        INT     10H
        POP     BX
        RET
DISPCHAR ENDP

CODE    ENDS                     ; 代码段结束
        END     START            ; 从 START 标号处开始执行程序
```

> [!example] 例 6-7
> 采用时间片轮转方法实现实模式下的多任务调度。

假设有 A、B、C 三个独立任务：A 任务 4ms 执行一次，B 任务 4ms 执行一次，C 任务 2ms 执行一次，任务内容可自行设计，单个任务的程序段应在 1ms 内执行完毕。时间片长度为 1ms，由外部定时器 8253 每隔 1ms 产生中断请求信号，连接至 8259 的 $\text{IRQ}_2$。
定时中断调度任务在中断服务程序中实现，在中断服务程序中进行任务的调度与切换，首先以分支结构判断执行某个任务的条件是否满足，若满足则调用相应任务的子程序；否则继续判断执行下一个任务的条件是否满足。任务执行完毕后返回到中断服务程序部分，通知中断控制器中断处理结束，最后返回到主程序，等待下一次中断的来临。主程序中判断中断次数，若中断次数超过 20，则中止程序，返回 DOS。具体的主程序流程图和中断服务程序流程图如图 6-39 所示。
