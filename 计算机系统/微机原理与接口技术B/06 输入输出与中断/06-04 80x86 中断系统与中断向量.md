## 6.4 80x86/Pentium 中断系统

### 6.4.1 中断结构

Intel 80x86 系列微机的中断系统可以处理 256 个中断源，每个中断源都有对应的中断类型码（0～255）供 CPU 识别。在 8086/8088 系统中，中断源分为外部（硬件）中断与内部（软件）中断两类，如图 6-22 所示。

### 1. 外部中断

外部中断是由外部硬件产生的突发事件，包括非屏蔽中断和可屏蔽中断两种，通过 CPU 的 NMI 引脚或 INTR 引脚向 CPU 申请中断。外部中断的发生具有随机性，可能由 I/O 引起，或由时钟或硬件故障引起。

#### 1. 非屏蔽中断
由 NMI 引脚引起的中断称为非屏蔽中断，不受中断允许标志位 IF 的限制，不能在 CPU 的内部被屏蔽掉，中断类型码固定为 2。NMI 信号是上升沿触发，CPU 内部会把该信号锁存起来，但要求该信号的有效高电平持续 2 个时钟周期以上。CPU 接收到非屏蔽中断请求信号后，不论当前正在做什么，都会在执行当前指令后立即响应中断请求而进入相应的中断处理。在实际系统中，非屏蔽中断通常用来处理系统中出现的重大故障或紧急情况，如系统掉电处理、紧急停机处理等。与一般硬件中断不同，对于 NMI，CPU 不执行中断响应周期。

#### 2. 可屏蔽中断
一般外部设备提出的中断请求是从 CPU 的 INTR 端引入的，所产生的中断称为可屏蔽中断。可屏蔽中断受中断标志位 IF 的影响，只有在 IF=1 时，CPU 才会在执行当前指令后响应可屏蔽中断请求，所以 INTR 信号有效也必须保持到当前指令执行结束。如果 IF=0，中断被屏蔽，即使有中断请求 CPU 也不会响应。在 IBM PC 系列微机中，通常外部设备提出的中断请求首先由中断控制器 8259A 预处理，再决定是否向 CPU 的 INTR 引脚提出中断请求。CPU 响应可屏蔽中断请求信号后，会通过 $\overline{INTA}$ 引脚送出两个负脉冲，并获得中断源对应的中断类型码。

### 2. 内部中断

内部中断是 CPU 执行某些指令或指令执行的结果使标志寄存器中某个标志位置 1 而引发的，与外部硬件电路完全无关，中断类型码或由系统预先设定，或包含在指令中。

1. 除法出错中断。当 CPU 执行除法指令时，若发现除数为 0 或商超过了结果寄存器所能表示的数的范围，则立即产生一个中断类型码为 0 的除法出错中断。该中断的服务处理一般由操作系统安排。
2. 单步调试中断。若标志寄存器中 TF=1，CPU 便处于单步工作方式，即每执行完一条指令就自动产生一个中断类型码为 1 的中断。单步方式为系统提供了一种方便的调试手段，成为能够逐条指令地观察系统操作的一个 "窗口"。如 DEBUG 中的跟踪命令 T，就是将 TF 置 1，便进入单步工作方式。单步方式为系统提供了一种方便的调试手段，成为能够逐条指令地观察系统操作的一个“窗口”。如 DEBUG 中的跟踪命令 T，就是将 TF 置 1，进而去执行一个单步中断服务程序，以跟踪程序的具体执行过程，找出程序的问题所在。

说明，所有类型的中断在其处理过程中，CPU 会自动地把标志压入堆栈，然后清除 TF 和 IF 位。因此当 CPU 进入单步处理程序时，就不再处于单步工作方式，而以正常方式工作。只有在单步处理结束，从堆栈中弹出原来的标志后，才使 CPU 又返回到单步方式。注意，80x86 指令系统中没有设置或清除 TF 标志的专门指令，可利用 PUSHF 和 POPF 来实现对 TF 的置位和复位，程序段如下：
```assembly
PUSHF             ; 标志寄存器 FLAGS 入栈
POP   AX          ; (FLAGS) → AX
OR    AX, 0100H   ; TF 标志置 1，若要使 TF 清 0，应改为 AND AX，0FEFFH
PUSH  AX
POPF              ; 改变标志寄存器的值
```

3. INTO 指令，为溢出中断指令。若算法操作结果产生溢出（OF=1）时，执行 INTO 指令后立即产生一个中断类型码为 4 的中断。INTO 为程序员提供了一种处理算术运算出现溢出的手段，通常与算术指令配合使用。
4. INT *n* 指令。INT *n* 是双字节代码指令，第一字节为操作码，第二字节为中断类型码 *n*（$n=0 \sim 255$）。CPU 执行一条这种指令便发生一次中断。

其中，INT 3 是 INT *n* 的特例，是单字节指令，因而能被方便地插入到程序的任何地方，专供调试程序时在程序中设置断点之用。插入 INT 3 指令之处便是断点，在其中断服务程序中可显示有关的寄存器和存储单元等内容，以分析断点为止程序运行是否正确。
80386 及以上 CPU 扩展了一些内部中断，把许多执行指令过程中产生的错误也纳入中断处理的范围，并将它们和内部中断一起统称为异常中断，简称异常；将通常意义上的外部中断简称为中断。因此，对于 80386 及以上 CPU 来说，也可将中断源分为中断和异常两类。
异常包括由指令引起的异常（如 INT *n*、INTO、BOUND 等）和处理器检测异常（如除法出错、无效操作码、堆栈故障、段/页不存在、浮点协处理器、单步调试异常等）。所有的异常按其发生异常的报告方式和异常服务程序返回方式的不同，又可归为 3 类：失效、陷阱和终止。

1. 失效一般在引起失效的指令启动之后、执行之前被检测到，且在处理异常的中断服务完成后返回该条指令，重新启动并执行完毕。如在读虚拟存储器时，当访问的页或段不在物理存储器中，便产生一个失效字，引起异常中断；其中断服务程序立即从磁盘上读取这个页或段的内容到物理内存中，再返回主程序中重新执行这条指令。
2. 陷阱是在产生陷阱的指令执行完后才被报告，且其中断服务程序结束后返回到主程序中该条指令的下一条指令。如中断指令 INT *n* 属于此类异常。
3. 终止则是一种对引起异常指令的确切位置无法确定的异常，如硬件错误或系统表中的非法数值等造成的异常。当出现此类异常时，原来的程序已无法继续执行，只好终止，而由中断服务程序重新启动操作系统并重建系统表格。

## 6.4.2 中断向量表

> [!info] 8086 中断向量表
> 在内存 **00000H – 003FFH** 存放 **256 个**中断向量，每个占 **4 字节**（低字 IP、高字 CS），按类型号 N 索引：入口地址 = N × 4。

在 80x86/Pentium 中断系统中，每个中断源对应一个确定的 8 位中断类型码。CPU 在响应中断后，会根据中断类型码查询中断向量表转入相应的中断服务程序。
中断向量表是指内存最低 1 KB（00000H～003FFH）的存储区域，其中存放对应 256 个中断源的中断服务程序的入口地址（中断向量）。每个中断向量占 4 字节，前 2 字节为中断服务程序入口地址的偏移地址（IP），后 2 字节为段地址（CS）。中断向量按照中断类型码的序号，在中断向量表中有规则地顺序存放，如表 6-3 所示。
中断类型码与中断向量在向量表中所处位置（中断向量地址指针）之间的对应关系为：
$$
\text{中断向量地址指针} = \text{中断类型码} \times 4
$$

## 表 6-3 中断向量表

| 分类 | 中断源 | 入口地址 |
| :--- | :--- | :--- |
| **专用中断** | 00H | 类型 0 中断入口地址（除法出错） |
| | 01H | 类型 1 中断入口地址（单步中断） |
| | 02H | 类型 2 中断入口地址（NMI） |
| | 03H | 类型 3 中断入口地址（断点中断） |
| | 04H | 类型 4 中断入口地址（溢出中断） |
| **系统备用** | 05H | 类型 5 中断入口地址 |
| | $\vdots$ | $\vdots$ |
| | 07CH | 类型 31 中断入口地址 |
| **供用户使用** | 080H | 类型 32 中断入口地址 |
| | $\vdots$ | $\vdots$ |
| | 3FCH | 类型 255 中断入口地址 |

在表 6-3 中，前 5 个中断类型是 80x86 规定的专用中断，有固定的含义和处理功能；类型码 5～31 是 Intel 公司为系统功能或将来扩展特殊用途而保留的；用户可以自行使用类型码 32～255。在一种具体的微机系统中，对用户可使用的中断类型码又有特殊的规定。如在 PC/AT 机中，类型码 70H～77H 已有安排，所以在系统开发和应用时应加以注意。
由于 ROM BIOS 和 MS-DOS 的版本不同以及多种 PC 主板，中断类型码的分配在不同的计算机上会有些差异。

## 6.4.3 中断响应过程

80x86/Pentium 系统中各种中断的响应和处理过程略有不同，主要在于如何获取相应的中断类型码。

### 1. 内部中断响应过程

内部中断包括系统使用的专用中断与用户使用的 INT *n* 指令。内部中断没有随机性，不受中断允许标志位 IF 的影响，而且其中除单步中断外都无法用软件禁止。当内部中断发生时，CPU 无须执行中断响应周期去获得中断类型码。对于专用中断，中断类型码是自动形成的；而对于 INT *n* 指令，类型码即为指令中给定的 *n* 值。
CPU 取得中断类型码后的处理过程如下：

1. 将中断类型码×4，作为中断向量表的地址指针；
2. 将 CPU 的标志寄存器入栈保护，此操作类似于 PUSHF 指令；
3. 清除 IF 和 TF 标志位，屏蔽新的 INTR 中断和单步中断；
4. 保存断点，即把断点处的 IP 和 CS 值压入堆栈，先压入 CS 值，再压入 IP 值；
5. 从中断向量表中取中断服务程序的入口地址，分别送至 CS 和 IP 中；
6. 按新的 CS:IP 执行中断服务程序。

在中断服务程序中，通常要保护现场，进行相应的中断处理，恢复现场，最后执行中断返回指令 IRET。IRET 的执行将使 CPU 依次恢复断点处的 IP 和 CS 值，恢复标志寄存器（相当于 POPF），于是程序就恢复到断点处继续执行。

### 2. 外部中断响应过程

1. 非屏蔽中断响应。CPU 采样到非屏蔽中断请求时，自动提供中断类型码 2，其后的处理过程和内部中断一样。
2. 可屏蔽中断。当 INTR 信号有效时，如果中断允许标志 IF=1，则 CPU 就会在当前指令执行完毕后，响应外部中断请求，转入中断响应周期。中断响应周期有 2 个，每个响应周期都由 4 个 T 状态组成。在第一个中断响应周期，CPU 输出 $\overline{INTA}$ 负脉冲，表示 CPU 响应外设中断请求，并通知外设现在应该准备好中断类型码，同时用以撤销外设提出的 INTR 信号；在第二个中断响应周期，CPU 又输出 $\overline{INTA}$ 负脉冲，要求请求中断的外设在收到第二个负脉冲以后（第二个中断响应周期的 $T_3$ 状态前），把中断类型码通过数据总线（通常通过 8259A 传送），CPU 在 $T_4$ 状态的前沿采样数据总线，获取中断类型码，如图 6-23 所示。

![[计算机系统/微机原理与接口技术B/附件/第6章/Pasted image 20260719161804.png]]
*图 6-23 中断响应总线周期*

CPU 获取中断类型码后的处理过程和内部中断一样，自动从中断向量表中取出中断服务程序的入口地址并转入中断服务。但使用过程中应注意两点：

1. CPU 复位后，中断允许位处于关闭状态。因此要使中断得到响应，必须使用指令开中断。
2. CPU 响应中断后，自动关中断。因此，要允许中断嵌套，必须在进入中断服务程序后使用指令重新开中断。

一个可屏蔽中断被响应时 CPU 实际执行的总线周期有 7 个：

1. 执行两个中断响应总线周期（8086 在两个周期之间用 2～3 个空闲状态隔开），CPU 获得相应的中断类型码，将它左移两位形成中断向量表指针，存入暂存器。
2. 执行一个写总线周期，把标志寄存器的内容压入堆栈。同时置 IF 和 TF 为 0，以禁止中断响应过程中其他可屏蔽中断的进入，同时也禁止中断处理过程中出现单步中断。
3. 执行两个写总线周期，把断点地址（当前的 CS:IP）的内容压入堆栈。
4. 执行两个读总线周期，从中断向量表中分别取出中断服务程序入口地址的偏移量与段基值送到 IP 和 CS 寄存器中。

### 3. 中断响应及处理过程举例

> [!example] 例 6-4
> 设某输入设备在数据准备就绪后向 CPU 申请可屏蔽中断，中断类型码为 80H，中断响应后执行的中断服务程序名为 INTPROC。
在允许中断前，首先要写中断向量到中断向量表，即先进行中断初始化，程序段如下：
```assembly
CLI
PUSHD
MOV    AX, 0000H          ; 中断向量指针的段基址
MOV    DS, AX
MOV    BX, 0200H          ; 中断向量指针的偏移量
MOV    AX, OFFSET INTPROC
MOV    [BX], AX           ; 中断向量的偏移量部分 → 中断向量表
MOV    AX, SEG INTPROC
MOV    [BX+2], AX         ; 中断向量的段基址部分 → 中断向量表
POP    DS
STI
```

开中断后，执行任何指令期间都有可能接收到中断申请。假设执行下列程序中的 ADD 指令时收到中断申请，且 IF=1，则 CPU 响应中断，响应过程如图 6-24 所示。

```assembly
MOV AX, 2000H
ADD AX, AX
1234: 5678  MOV [SI], AX
...
INTPROC PROC
        MOV DX, DPORT
        IN AL, DX
        IRET
INTPROC ENDP
```

### 4. 各类中断的优先权

80x86/Pentium 系统中所有的中断都是在当前指令结束后处理的。在当前指令执行完毕，CPU 总是按照中断优先级别，从高到低自动查询中断类型，先查询到的中断先被响应。80x86/Pentium 系统的中断优先级别从高到低排列顺序为：内部中断（除法出错、INTO、INT *n*）、NMI、INTR、单步中断。

## 6.4.4 80386/80486/Pentium CPU 中断系统

80386/80486/Pentium CPU 在中断处理机制上有实模式和保护模式两种不同情况。

### 1. 实模式下的中断与异常

80386 及以上 CPU 工作于实模式时，中断处理机制与 8086/8088 的基本一致，都可定义 256 种中断类型，并将中断向量表存放在系统物理存储器的最低 1 KB 区域。中断类型及各类中断处理流程也基本相同，都是通过中断类型码查找中断服务程序入口地址。表 6-4 给出了 8086～Pentium CPU 在部分微机中的中断分配表。
Intel 公司保留前 32 个中断类型为 Intel 各种 CPU 系列成员专用，最后 224 个可作为用户中断类型。表 6-4 中前 5 个中断类型在 8086～Pentium 的所有 Intel 系列 CPU 中是相同的。其他中断类型存在于 80286 及向上兼容的 386、486、Pentium CPU 中，但不向下兼容 8086 或 8088。所以对于 80386～Pentium 微机系统，类型码 08H～0FH 既被内部中断使用，又供外部可屏蔽中断 $\text{IRQ}_0 \sim \text{IRQ}_7$ 使用，类型码 70H～77H 同样可同时供内部中断及外部可屏蔽中断 $\text{IRQ}_8 \sim \text{IRQ}_{15}$ 使用。为区分某个类型码是内部中断，还是由 INTR 引脚引起的外部中断，在处理之前，要判断出错代码中的 EXT 位，如果为 “1”，则执行外部中断处理过程，否则执行内部异常处理过程。

### 2. 保护模式下的中断与异常

保护模式下的中断处理过程与实模式相比有三点不同：① CPU 根据中断类型码从中断描述符表 IDT 而不是中断向量表获取中断服务程序的入口信息。② 中断过程中要对被中断的程序代码进行保护，即要进行特权级检查。③ 若有出错码，还要将出错码压入堆栈。
#### 1. 中断描述符表
IDT 的起始位置通过中断描述符表寄存器 IDTR 设置，可置于内存的任意区域。在整个系统中，IDT 只有一个。它的作用同样是通过类型码引导程序转移到中断或异常处理程序中。但是其表项内容与实模式完全不同，IDT 包含的是门描述符而不是中断向量。门描述符可以是中断门、陷阱门或任务门。IDT 最多允许有 256 个门描述符，标识为门 $0 \sim 255$，对应 256 种中断类型。
#### 2. 获得中断服务程序地址
使用中断门/陷阱门描述符进入中断服务程序入口的过程如图 6-25 所示。

![[计算机系统/微机原理与接口技术B/附件/第6章/Pasted image 20260719161824.png]]
*图 6-25 使用中断门/陷阱门进入中断程序入口的过程*

具体描述如下：

1. 将中断类型码 *n* 乘以 8 得到相应的中断门或陷阱门描述符首字节在 IDT 中的地址。
2. 从 IDT 读出门描述符（8 字节），将其中的 16 位选择符装入代码段寄存器 CS。
3. 用 CS 寄存器从 GDT 或 LDT 中选择相应的段描述符。
4. 将段描述符中的基址（32 位）与门描述符中的偏移量（32 位）相加得到中断服务程序的入口地址（线性地址）。

中断门和陷阱门只能设置在 IDT 内。它们的不同之处在于，通过中断门访问中断服务程序将清除 EFLAGS 中的 IF 标志，而陷阱门则不改变 IF 内容。一般来讲，外部硬件中断由中断门描述符来配置。这样，在执行中断服务程序时，除非软件允许再次中断，否则可以防止另外的中断影响当前的中断处理过程。软件中断及内部中断一般用陷阱门。

3. 中断响应及异常处理。保护模式下的中断响应过程基本与实地址方式相似。响应中断时，要将标志寄存器 EFLAGS、段寄存器 CS、指令指针 EIP 的内容压入堆栈；清除 NT、TF、IF 标志（陷阱门不清除 IF）；如果异常导致错误的代码要保存，还需将错误码压入堆栈。与实模式下中断处理不同之处有以下两点：

1. 需要通过类型码判断所使用的门描述符是否超出 IDT 的界限。若超出，则引起一般保护异常错误。
2. 从 IDT 中取得对应的门描述符后要进行相应的检查。

门描述符只能是任务门、286 中断门、286 陷阱门、386 中断门和 386 陷阱门，否则同样会引起一般保护异常错误。
门描述符中的 P 位必须是 1，表示该描述符可用，否则就引起段不存在故障。
中断时，中断服务程序的代码段必须比被中断程序的特权级高或相同（$\text{DPL} \leqslant \text{CPL}$），从而对被中断的程序代码进行保护，否则将产生一般保护异常错误。如果 $\text{DPL} = \text{CPL}$，中断服务程序使用当前的堆栈；如果 $\text{DPL} < \text{CPL}$，将出现堆栈转换，主程序的堆栈指针（由 SS:ESP 提供）也将压入堆栈，而中断服务程序使用的选择符以及堆栈指针从当前所执行任务的 TSS 获得。CPU 自动将主程序堆栈中的 SS、ESP、FLAGS、CS、EIP 和出错码信息复制到中断服务程序的堆栈中。堆栈的变化过程如图 6-26 所示。

3. 在保护模式下，除了可通过中断门或陷阱门实现由当前任务进行中断处理，也允许通过任务门实现由另一个任务完成中断处理。使用任务门的优点是，被中断程序的环境资源入栈和中断任务环境资源的加载全部自动进行，而且中断任务与其他任务隔离。缺点是，由于要保存大量的机器状态，处理程序较复杂，处理时间比中断门要长。

从异常或中断服务程序返回时，必须使用 IRET/IRETD 指令，两种指令分别应用于 16 位和 32 位操作。
