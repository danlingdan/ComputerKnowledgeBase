## 3.3 8086/8088 CPU 指令系统

Intel 8086/8088 CPU 指令系统是 80x86/Pentium 系列 CPU 的基本指令集。指令的操作数宽度是 8 位或 16 位，偏移地址宽度是 16 位。可以分成 6 个功能组：数据传送类、算术运算类、逻辑运算与移位类、串操作类、控制转移类和处理器控制类。

> [!info] 指令操作数符号说明（表 3-4）
> 为方便理解指令，定义如下符号：

| 符 号 | 意 义 |
| :--- | :--- |
| OPRD, OPRDN | 操作数，第 N 个操作数，如 OPRD1、OPRD2、OPRD3 等 |
| DEST, SRC | 目的操作数，源操作数 |
| reg | 通用寄存器，长度可以是 8 位或 16 位 |
| reg8 | 8 位通用寄存器 AH/AL/BH/BL/CH/CL/DH/DL |
| reg16 | 16 位通用寄存器 AX/BX/CX/DX/SI/DI/BP/SP |
| Sreg | 段寄存器 DS/CS/SS/ES (FS/GS) |
| mem | 存储器操作数，长度可以是 8 位或 16 位 |
| mem8 | 8 位存储器操作数 |
| mem16 | 16 位存储器操作数 |
| imm | 立即数，长度可以是 8 位或 16 位 |
| imm8 | 8 位立即数 |
| imm16 | 16 位立即数 |

### 3.3.1 数据传送类指令

> [!abstract] 数据传送类指令概览
> 数据传送是计算机中最基本最重要的一类操作，在程序中使用的比例最高。它们用于实现：
> - 寄存器与寄存器、寄存器与存储器、累加器（AL/AX）与 I/O 端口之间的字节或字传送；
> - 将立即数传送到寄存器或存储器单元中。
>
> 数据传送类指令有 **14 条**，分为 4 组（见表 3-5）。除了 `SAHF` 和 `POPF` 指令，其余指令执行后**对标志位都没有影响**。

| 分组 | 指令类别 | 主要指令 |
| :--- | :--- | :--- |
| 通用数据传送 | 传送、堆栈、交换、查表 | MOV / PUSH / POP / XCHG / XLAT |
| 地址传送 | 有效地址、地址指针 | LEA / LDS / LES |
| 标志位传送 | 标志读写、入栈/出栈 | LAHF / SAHF / PUSHF / POPF |
| 输入/输出 | I/O 端口数据传送 | IN / OUT |

#### 1. 通用数据传送指令

> [!info] 通用数据传送指令
> 共 5 条，除了 `XCHG`，是**唯一允许以段寄存器作为操作数的指令**。

##### 1. MOV 指令

> [!abstract] MOV 指令
> - **指令格式**：`MOV  OPRD1, OPRD2`
> - **功能**：将源操作数传送给目标操作数，即 $\text{OPRD2}\rightarrow\text{OPRD1}$，源操作数保持不变。OPRD1 和 OPRD2 可以是字节或字，但两者必须等长（同为字节传送或字传送）。

MOV 指令的几种形式：

```asm
MOV  reg/Sreg/mem, reg     ; reg/Sreg/mem ← reg
MOV  reg/Sreg, mem         ; reg/Sreg ← mem
MOV  reg/mem, imm          ; reg/mem ← imm
```

源操作数可以是通用寄存器、段寄存器、存储器以及立即操作数；目标操作数可以是通用寄存器、段寄存器（CS 除外）或存储器。各种数据传送关系如图 3-4 所示。

![[计算机系统/微机原理与接口技术B/附件/第3章/Pasted image 20260719160213.png]]
*图 3-4　各种数据传送关系*

```asm
MOV  AL, CH           ; 通用寄存器之间传送字节数据
MOV  DS, AX           ; 通用寄存器与段寄存器（CS 不能是目标）之间传送数据
MOV  AX, 0FF3BH       ; 立即数 FF3BH 传送到通用寄存器
MOV  AL, BUFFER       ; 存储器与通用寄存器之间传送数据，BUFFER 为字节存储单元名
MOV  DAT[BP+DI], ES   ; 段寄存器与存储器之间传送数据，DAT 为常量或字存储单元名
```

> [!warning] MOV 指令使用注意
> 1. 立即数和段寄存器 CS 不能作为目标操作数；
> 2. 立即数不能直接传送到段寄存器；
> 3. 不允许在两个存储单元之间或在两个段寄存器之间直接传送数据；
> 4. 将立即数传送到存储单元时，必须显式说明存储器操作数的宽度类型。

##### 2. 堆栈操作指令 PUSH/POP

> [!abstract] 堆栈基本概念
> 堆栈是内存的一个数据区，按照**"后进先出"原则**组织的一段内存区域。80x86/Pentium CPU 规定：
> - 堆栈设置在堆栈段（SS）内，**向下生长**；
> - 堆栈指针 SP 始终指向堆栈的顶部；
> - 在子程序（过程）调用或处理中断时，堆栈用于保存当前的**断点地址**（8086/8088 中为 CS 和 IP）和现场数据。

> [!important] PUSH / POP 操作语义
> - **指令格式**：`PUSH  OPRD2`、`POP  OPRD1`
> - **PUSH**：$\text{SP}-2\rightarrow\text{SP}$，然后将 16 位源操作数压入堆栈。入栈时，高字节存放在高地址，低字节存放在低地址。
> - **POP**：从栈顶弹出 16 位操作数到目标操作数。低地址字节送低字节，高地址字节送高字节，同时修改 $\text{SP}+2\rightarrow\text{SP}$。
> - 目标操作数可以是通用寄存器、段寄存器（CS 除外）或存储器。
>
> 8086/8088 堆栈操作总是按**字**（不是字节）存取。

```asm
PUSH reg16   ; SP=SP-2, [SP] ← reg16    POP reg16   ; reg16 ← [SP], SP=SP+2
PUSH Sreg    ; SP=SP-2, [SP] ← Sreg     POP Sreg    ; Sreg ← [SP], SP=SP+2
PUSH mem16   ; SP=SP-2, [SP] ← mem16    POP mem16   ; mem16 ← [SP], SP=SP+2
```

##### 3. 交换指令 XCHG

> [!abstract] XCHG 指令
> - **指令格式**：`XCHG  OPRD1, OPRD2`
> - **功能**：将长度相同（同为字节或字）的源操作数与目标操作数进行交换。

```asm
XCHG  reg/mem, reg    ; reg/mem ↔ reg
XCHG  reg, mem        ; reg ↔ mem
```

```asm
MOV  AX, 2244H        ; AX=2244H
MOV  BX, 3366H        ; BX=3366H
XCHG AX, BX           ; 交换后，AX=3366H, BX=2244H
```

> [!warning] XCHG 注意
> 1. 段寄存器和立即数不能作为操作数；
> 2. 不能在两个存储单元之间直接交换数据。

##### 4. 查表转换指令 XLAT（或称换码指令）

> [!abstract] XLAT 指令
> - **指令格式**：`XLAT  或  XLAT  OPRD`
> - **功能**：完成 1 字节的查表转换，将数据段中偏移地址为 $(\text{BX+AL})$ 的存储单元的内容送入 AL 寄存器，即 $\text{DS:}[\text{BX+AL}]\rightarrow\text{AL}$。

> [!info] XLAT 使用要点
> 1. 先在数据段建立一个长度小于 256 字节的表格；
> 2. 表的首地址置于 BX 中；
> 3. AL 中存放查找对象在表中的下标；
> 4. 指令执行后，所查找的对象存于 AL 中，BX 内容保持不变。

> [!example] 例：BCD 码转 7 段 LED 显示代码
> 数字 0～9 的 BCD 码对应的 7 段 LED 显示代码为 40H，79H，24H，30H，19H，12H，02H，78H，00H，18H，依次存放在数据段中偏移地址为 0800H 开始的区域。若待转换的 BCD 码为 0100B，则：
> ```asm
> MOV  BX, 0800H
> MOV  AL, 4
> XLAT             ; AL 中为 0100B 对应的 7 段代码 19H
> ```

> [!note] XLAT 操作数与段超越
> `XLAT` 指令的操作数是隐含的，汇编语言程序中可指定表的首地址以方便阅读。表的首地址名称仅供汇编程序使用，对指令的执行没有任何影响。该指令还允许段超越，如 `ES:XLAT`。

#### 2. 地址传送指令

> [!info] 地址传送指令
> 用于传送存储器的逻辑地址信息（见表 3-5），指令中的源操作数只能是**存储器操作数**。

##### 1. 有效地址传送指令 LEA

> [!abstract] LEA 指令
> - **指令格式**：`LEA  OPRD1, OPRD2`
> - **功能**：将源操作数在当前段内的有效地址（即偏移地址）传送至目标操作数。

```asm
LEA  reg16, mem    ; reg16 ← Addr(mem)
```

> [!warning] LEA 与 MOV 的区别
> `MOV` 指令传送操作数的内容，而 `LEA` 传送的是操作数的存储地址。例如：
> ```asm
> MOV  DI, TABLE         ; 将变量 TABLE 的内容传送至 DI
> LEA  DI, TABLE         ; 将变量 TABLE 的偏移地址传送至 DI
> LEA  DX, [1000H]       ; 将 1000H 单元的偏移地址 →DX, DX=1000H
> ```

##### 2. 地址指针传送指令 LDS 和 LES

> [!abstract] LDS / LES 指令
> - **指令格式**：`LDS  OPRD1, OPRD2`、`LES  OPRD1, OPRD2`
> - **功能**：将源操作数有效地址决定的双字存储单元中的**第一个字**的内容送入指令指定的 16 位通用寄存器，**第二个字**的内容传送给段寄存器 DS 或 ES。通常，双字存储单元中是一个 32 位的地址指针。

```asm
LDS ( LES )  reg16, mem  ; reg16 ← [EA], DS(ES) ← [EA+2]
```

```asm
TABLE  DD  12345678H
LDS    BX, TABLE         ; BX=5678H, DS=1234H
```

#### 3. 标志位传送指令

CPU 通过这类指令对[[02-02 8086 与 8088 的内部结构#4. 状态标志寄存器 FR|标志寄存器 FR]]进行保护与更新。指令的操作数用隐含方式规定，指令共 4 条。

##### 1. 标志寄存器读/写指令 LAHF/SAHF

> [!info] LAHF / SAHF
> - **LAHF**：将标志寄存器的低字节（含 SF、ZF、AF、PF、CF）读出后传送给 AH 寄存器。这条指令本身不影响标志位。
> - **SAHF**：把寄存器 AH 中的内容写入标志寄存器的低字节，取代某些标志位（SF、ZF、AF、PF、CF）的原来状态。

##### 2. 标志寄存器入栈/出栈指令 PUSHF/POPF

> [!info] PUSHF / POPF
> - **PUSHF**：执行过程同 `PUSH` 指令，只是将标志寄存器 FR 的内容压入堆栈。
> - **POPF**：将当前栈顶的一个字传送给标志寄存器 FR，同时修改栈顶指针 SP。

> [!tip] PUSHF/POPF 的应用
> 在子程序调用和中断服务中，常用这两条指令保护和恢复需要的标志位。利用它们还可以方便地改变标志寄存器中任一位的状态——8086/8088 指令系统中没有直接修改 TF 标志位的指令，可用以下程序段来实现 TF 的改变：
> ```asm
> PUSHF
> POP    AX           ; 标志寄存器的内容送 AX
> OR     AH, 01H      ; 将 TF 位置 1（TF 在标志寄存器的 8 位）
> PUSH   AX
> POPF               ; AX 的内容送标志寄存器
> ```

#### 4. 输入/输出数据传送指令 IN/OUT

> [!abstract] IN / OUT 指令
> - **指令格式**：`IN  OPRD1, OPRD2`、`OUT  OPRD1, OPRD2`
> - **功能**：专门用于在 AL 或 AX 寄存器与 I/O 端口之间传送数据。

> [!info] I/O 端口寻址规则
> 一台计算机可以配接许多外部设备，每个外部设备与 CPU 之间要交换数据、状态信息和控制命令，每一种这样的信息交换都要通过一个端口来进行。计算机系统中的各端口也像存储器那样用地址来区分。
>
> 80x86/Pentium CPU 中有 16 条 I/O 地址线，最多可提供 64K 个传送 8 位数据的端口地址或 32K 个传送 16 位数据的端口地址。
>
> - 当**端口地址 < 256** 时，采用**直接寻址方式**，在指令中直接指定端口地址；
> - 当**端口地址 ≥ 256** 时，采用**间接寻址方式**，端口地址放在 DX 中。

```asm
IN   AL, imm8    ; AL ← [端口 imm8](其他同)
IN   AX, imm8    ; AX ← [imm8+1], [imm8]
IN   AL, DX      ; AL ← [DX]
IN   AX, DX      ; AX ← [DX+1], [DX]

OUT  imm8, AL    ; [imm8] ← AL
OUT  imm8, AX    ; [imm8+1], [imm8] ← AX
OUT  DX, AL      ; [DX] ← AL
OUT  DX, AX      ; [DX+1], [DX] ← AX
```

> [!example] I/O 指令示例
> ```asm
> IN   AX, 20H          ; 从端口 20H 和 21H 输入 16 位数据到 AL 和 AH(AX)
> MOV  DX, 3F0H
> IN   AL, DX           ; 从端口 03F0H 输入 8 位数据到 AL
> OUT  27H, AL          ; 将 8 位数据从 AL 输出到端口 27H
> OUT  DX, AX           ; 将 16 位数据从 AX 输出到 DX 和 DX+1 指定的端口
> ```
