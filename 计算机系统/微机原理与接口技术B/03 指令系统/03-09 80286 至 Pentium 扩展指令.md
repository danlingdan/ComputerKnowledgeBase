## 3.4 80x86/Pentium CPU 指令系统

x86 指令集长期保持较强的向后兼容性，但具体指令是否可用仍取决于处理器代际、工作模式与功能检测结果。80286 以后逐步加入保护模式控制、32 位整数与寻址、原子操作、MMX 和 SSE 等 SIMD 扩展；程序不应仅凭“x86 兼容”假定所有扩展都存在，而应通过 CPUID 等机制确认功能。

### 3.4.1 80286 CPU 的增强与增加指令

80286 是超级 16 位 CPU，从它开始的 CPU 都将 8086/8088 的工作模式作为实模式，在此基础上增加了保护等其他工作模式。80286 CPU 增强与增加的指令如表 3-14 所示。

#### 1. 80286 CPU 的增强指令

在 8086/8088 指令系统中，许多指令不能用立即数作为源操作数，80286 增强了此功能，允许立即数作为源操作数，如：① 堆栈操作指令 PUSH 允许将 16 位立即数直接压栈；② 允许移位指令中移位计数值从原来规定的常数 1 扩充至在 1～31 之间的立即数。例如：

```asm
SAR  DX, 3
ROL  BYTE PTR [BX], 5
```

特别地，有符号数乘法指令不仅允许立即数作为一个乘数操作数，而且允许有两个或三个操作数。例如：

```asm
IMUL  BX, 50                ; BX←BX×50
IMUL  AX, [BX+DI], 0342H    ; AX←0342H×DS:[BX+DI]
IMUL  AX, BX, 20            ; AX←20×BX
```

#### 2. 80286 CPU 的增加指令

80286 CPU 增加的一些新功能指令如下。

1. 16 位通用寄存器全部进栈/出栈指令 PUSHA/POPA。
2. 从外设端口进行串输入的指令 INS/INSB/INSW 和串输出指令 OUTS/OUTSB/OUTSW，它们都可在其前面加重复前缀 REP，以连续实现整个串的输入/输出操作，这时 CX 寄存器中为重复操作的次数。例如，以下程序段从端口 PORT 输入 100 字节存到附加段中以 TABLE 为首地址的内存区。

```asm
CLD                ; DF=0，按地址增量传送
LEA  DI, TABLE
MOV  CX, 100       ; CX 中为计数值
MOV  DX, PORT
REP  INSB          ; [ES:DI]←[DX], DI←DI+1, CX←CX-1，直至 CX=0
```

3. 高级语言类指令 BOUND、ENTER、LEAVE。BOUND 指令用于验证在指定寄存器 reg16 中的操作数是否在 mem32 所指向的两个界限内。若不在，则产生一个 5 号中断。指令中假定上、下界值（即数组的起始和结束地址）依次存放在相邻存储单元中。例如：

```asm
ARRAY  Dw  0000H, 0063H     ; 定义数组的最小下标 0 及最大下标 99（63H）
NUMB   DW  0019H
       ...
MOV    BX, NUMB             ; BX 中为被测下标值 25（19H）
BOUND  BX, ARRAY            ; 检查被测下标值是否在规定的下标边界范围之内
       ...
```

程序中的每个子程序都有自己的局部变量，这些局部变量只有当它们所在的子程序执行时才有意义。为了保存这些局部变量，当执行到这些子程序时，应为它们建立相应的堆栈框架，退出子程序时再撤除。80286 用 ENTER、LEAVE 两条指令来完成这些功能。

ENTER 指令为局部变量建立一个堆栈区，指令中的目标操作数 imm16 指出子程序所要使用的堆栈字节数，源操作数 imm8 指出子程序的嵌套层数：0～31。LEAVE 指令用于撤销前面 ENTER 指令的动作。例如：

```asm
TASK   PROC NEAR
       ENTER  60, 4     ; 建立堆栈区并保存 60 字节长的局部变量
       ...
       LEAVE            ; 撤销建立的栈空间，恢复栈指针
       RET
TASK   ENDP
```

#### 3. 新增的保护模式编程指令

保护模式与实模式的不同，主要表现在它对存储器的管理和使用上。在开机或复位后，80x86/Pentium CPU 先进入实地址模式。如果进入保护模式，需要对各系统表和寄存器进行初始化，包括建立全局/中断描述符表 GDT/IDT、任务状态段 TSS 以及设置 GDTR、CR0 等。切换进入保护模式后，系统需要重新对 CS、SS、DS、ES 及任务状态段寄存器 TR 等进行设置。80286 为保护模式增加了一套如表 3-15 所示的编程指令，又称为系统控制类指令，包括描述符表操作和任务切换两大类。

### 3.4.2 80386 CPU 的增强与增加指令

80386 是 32 位 CPU，兼容全部的 80286 指令，且直接扩展为 32 位，表现在：① 所有指令都扩展支持 32 位操作数，包括 32 位立即数；② 所有涉及存储器寻址的指令都可以使用 32 位寻址方式。MASM 5.0 开始支持 80386 指令系统。

80386 增强与增加的指令如表 3-16 所示。增强的指令都是允许 32 位操作后出现的新指令助记符；增加的指令中，控制、调试与测试寄存器传送指令属于系统控制类指令。

**表 3-16 80386 CPU 增强与增加的指令**

| 类别 | 指令书写格式（助记符） | 指令功能 |
| :--- | :--- | :--- |
| **增强** | | |
| 传送类 | PUSHAD/POPAD | 将通用寄存器 EAX、ECX、EBX、EDX、ESP、EBP、ESI、EDI 顺序压入堆栈，弹出顺序与压入顺序相反 |
| | PUSHFD/POPFD | 将 32 位标志寄存器 EFLAGS 进栈/出栈 |
| | LFS/LGS reg, mem | 将 mem 中存放的偏移地址送 16 或 32 位 reg，段地址送 FS/GS |
| 算术运算类 | CWDE | 将 AX 中 16 位有符号数扩展为 32 位，存放在 EAX 中 |
| | CDQ | 将 EAX 中 32 位有符号数扩展为 64 位，存放在 EDX:EAX 中 |
| 串操作类 | 所有串操作指令后面扩展 D，如：MOVSD、OUTSD… | 将所有串操作直接扩充至 32 位 |
| 传送类 | MOVSX/MOVZX reg,reg/mem | 若两个有（无）符号数等长，则进行传送操作；否则，扩展补齐后传送 |
| 逻辑运算与移位类 | SHLD/SHRD reg/mem,reg,CL | reg/mem 和 reg 一起左（右）移 CL 位或 imm8（1～31）位，且 reg 不变，移出位→CF |

| | MOV reg32, CRn | |

对 80386 新增指令的使用举例如下。

> [!example] 例 3-13
> 传送指令 MOVSX/MOVZX 可以对有符号数/无符号数进行扩展传送。

```asm
MOVSX  ECX, AL           ; 将 AL 的内容带符号扩展为 32 位送入 ECX
MOVZX  EAX, CX           ; 将 CX 中 16 位数加 0 扩展为 32 位送入 ECX
```

> [!example] 例 3-14
> 双精度左移/右移指令 SHLD/SHRD 是三操作数指令，将 OPRD1 和 OPRD2 两个 16 位或 32 位操作数连接成双精度值（32 位或 64 位），然后向左或向右移位 OPRD3 次。移位时，OPRD2 的内容移入 OPRD1，而 OPRD2 本身不变。CF 中为 OPRD1 移出的最后一位。

```asm
MOV  AX, 2A80H
MOV  BX, 9A78H
SHLD  AX, BX, 8           ; AX=809AH, BX=9A78H, CF=0。
```

> [!example] 例 3-15
> 位测试指令 BT/BTR/BTS/BTC 测试目标操作数 OPRD1 中由源操作数 OPRD2 指定的那一位（最低位为 $\text{b}_0$），并将测试位送入 CF。测试完成后，BT 指令不改变目标操作数，其他指令则分别将测试位清 0、置 1 或取反。

```asm
MOV  CX, 4
BT  [BX], CX              ; 检查 [BX] 的 $\text{b}_4$ 位，且将 $\text{b}_4$ 位放入 CF 中
JC  NEXT                  ; $\text{b}_4$ 位=1 转移至 NEXT
```

> [!example] 例 3-16
> 位扫描指令 BSF/BSR 分别实现前向位扫描和后向位扫描，当对 16 位或 32 位源操作数 OPRD2 从低位 $\text{b}_0$ 到高位（$\text{b}_{15}$ 或 $\text{b}_{31}$）扫描或反向扫描时，将扫描到的第一个 1 的位号送入 OPRD1，并将 ZF 清 0 或置 1。

```asm
MOV  BX, 40A0H
BSF  AX, BX                ; 指令执行后，AX=5
BSR  AX, BX                ; 指令执行后，AX=14
```

> [!example] 例 3-17
> 条件设置指令 SETcc OPRD 有 16 条，根据指令中给出的条件“cc”是否满足来设置 OPRD 指定的 8 位寄存器或存储器操作数，常用于需要判断转移的场合，但无须转移指令，如表 3-17 所示。

**表 3-17 条件设置指令表**

| :--- | :--- | :--- |
| SETO reg8/mem8 | OF=1 | 溢出 |
| SETNO reg8/mem8 | OF=0 | 无溢出 |
| SETC/SETB/SETNAE reg8/mem8 | CF=1 | 有进位/低于/不高于或等于 |
| SETNC/SETNB/SETAE reg8/mem8 | CF=0 | 无进位/不低于/高于或等于 |
| SETZ/SETE reg8/mem8 | ZF=1 | 为零/等于 |

在 32 位 CPU 中增加了一些系统控制用的寄存器，包括：控制寄存器 CRn、调试寄存器 DRn 和测试寄存器 TRn，所以增加了相关传送指令对它们进行操作，如表 3-16 中的控制、调试与测试寄存器传送指令。这些指令一般只在系统程序中使用，且在 Pentium CPU 以后才支持 CR4。

### 3.4.3 80486 CPU 增加的指令

80486 CPU 在 80386 体系结构上进行了一些扩展，因而只有新增指令，如表 3-18 所示。其中，三条管理 Cache 的指令属于系统控制类指令。MASM 6.0 开始支持 80486 指令系统。

**表 3-18 80486 CPU 增加的指令**

### 3.4.4 Pentium 系列 CPU 增加的指令

从早期 Pentium、Pentium MMX 到 Pentium II、Pentium III 和 Pentium 4，处理器在保持基础 x86 兼容性的同时，逐步加入 MMX、SSE 和 SSE2 等扩展，以提高整数并行、浮点、多媒体和数据流处理能力。本节按历史代际介绍这些经典扩展，不把它们视为当前处理器的完整指令集。

#### 1. Pentium CPU 的指令系统

Pentium CPU 的指令系统主要是在 80486 指令集上增加了三条整数指令和三条系统控制指令，如表 3-19 所示。MASM 6.11 开始支持 Pentium CPU 指令系统。

**表 3-19 Pentium CPU 增加的指令**

| 类别 | 操作 | 指令书写格式（助记符） | 指令功能 |
| :--- | :--- | :--- | :--- |
| **新增整数指令** | 比较和交换 | CMPXCHG8B mem64 | 若 EDX:EAX=mem64，则 ZF←1，mem64←ECX:EBX；否则，ZF=0，EDX:EAX←mem64 |
| | 读 CPU 标识 | CPUID | 若 EAX=0，EAX:EBX:ECX:EDX←厂商信息；若 EAX=1，EAX:EBX:ECX:EDX←CPU 信息 |
| | 读时间标记计数器 | RDTSC | 将 Pentium 处理器中 64 位时间标记计数器 TSC（每个时钟周期自动加 1）读到 EDX:EAX 中 |
| **新增系统控制指令** | 读专用模式寄存器 | RDMSR | 将 ECX 中（00H～13H）指明的专用模式寄存器 MSR（Model-Specific Registers）中的 64 位数据读到 EDX:EAX 中 |
| | 写专用模式寄存器 | WRMSR | 将 EDX:EAX 中的 64 位数据写入 ECX 中（00H～13H）指明的专用模式寄存器 MSR 中 |
| | 由系统管理模式返回 | RSM | 使 Pentium CPU 从系统管理模式（SMM）返回到原来的实模式或保护模式 |

简单说明：从 Pentium CPU 开始，可以通过处理器识别指令 CPUID 获得 CPU 的 ID，以便程序在合适的 CPU 上运行。系统上电时 64 位的时间标记计数器 TSC 清 0，通过读 TSC 可检测程序运行速度。专用模式寄存器 MSR 一般由系统开发人员用于跟踪程序、查错等。

多媒体数据的每个样本点由多个数据组成，而且呈现出某种对称性，如音频信号中的左、右两个声道，视频信号中的 R、G、B 三基色信号，3D（三维）图形处理中的 X、Y、Z 坐标数据等，Pentium MMX 之前的“单指令单数据”指令系统处理这样的数据效率是非常有限的。

MMX 技术的主要特点是，提出了“单指令多数据（Single Instruction Multiple Data，SIMD）”的概念，使得一条指令能够同时处理多个数据，即一条指令的操作数中包含有多个经过组合的紧缩数据。MMX 的这种思想将 CPU 处理数据的速度从一次处理一个变成了一次处理一组，这极大地提高了 Pentium/Pentium Pro CPU 处理多媒体数据的性能。

紧缩整型数据分为 4 种数据类型：紧缩字节、紧缩字、紧缩双字、紧缩四字。
- **紧缩字节（Packed Byte）：** 8 字节组合成为一个 64 位数据。
- **紧缩字（Packed Word）：** 4 个字组合成为一个 64 位数据。
- **紧缩双字（Packed Doubleword）：** 2 个双字组合成为一个 64 位数据。
- **紧缩四字（Packed Quadword）：** 一个 64 位数据。

MMX 技术使用 8 个 64 位的 MMX 寄存器来适应多个数据的并行处理，它们以 MM7～MM0 来标识，与浮点处理单元 FPU 的 8 个 80 位寄存器共享，使用了其中的 64 位尾数部分。因此，MMX 指令集与 x87 浮点运算指令不能同时执行，两者之间只能分时切换，这在需密集切换时无疑会造成整个系统运行质量的下降。

#### 3. Pentium III CPU 的指令系统

MMX 技术只适合整型数据处理，不能用于浮点处理。为此，Intel 公司使用 MMX 指令的 SIMD 关键技术，在 Pentium III 中增加了适应网络发展的数据流扩展（Streaming SIMD Extension，SSE）技术，新增 SSE 指令 70 条，使得浮点数处理具备了单指令多数据的能力。

为了解决与浮点处理单元 FPU 的冲突问题，SSE 技术新增了 8 个 128 位的 SIMD 浮点寄存器 XMM7～XMM0，每个寄存器可存放 4 个 32 位的单精度浮点数，同时处理的数据比 64 位的 MMX 翻了一番。应该说，MMX 指令集开创了 Intel CPU 引入 SIMD 指令集之先河，MMX 可以同时对 2 个 32 位的整数进行操作，而 SSE 可以同时对 4 个 32 位的浮点数操作，并避开了 MMX 中使用浮点寄存器的冲突。SSE 对于 3D 几何运算，动画处理，图像处理，音频/视频信号的编辑、合成、识别以及解压缩处理的性能都有极为明显的改善。

Pentium III 中 SSE 指令集的 70 条指令分为如下 3 类。
##### 1. 提高 3D 图形运算效率的 50 条单指令多数据浮点运算指令
计算机表示的三维图形是用大量的表示图形顶点的浮点数组成的，通过操作这些顶点数据就可以改变三维图形的外观。通过使用 SSE 指令集及 128 位的浮点寄存器，CPU 可以在一个时钟周期内处理更多的数据，可以显著改善一些在三维操作中经常用到的计算，如矩阵变换、矩阵的加、减、乘运算以及光照计算等，大大加快了三维图形的顶点计算速度，结合复杂的算法可以创造出更为生动的三维图像效果。
##### 2. 12 条 MMX 整数运算增强指令
Pentium III 采用了比 Pentium II 更先进的算法，进一步增强了 CPU 在视频和图形处理方面的能力。
##### 3. 8 条优化内存中连续数据块传输的指令
Pentium III 采用新的数据预存取技术，减少了 CPU 处理连续数据流的中间环节，提高了连续数据流的处理效率，使视频、音频处理的速度更快、质量更高。

#### 4. Pentium 4 CPU 的指令系统

MMX 与 SSE 指令集主要提供了整型数据及单精度浮点数据的并行处理能力。
2000 年年底，Intel 公司开始推出 Pentium 4 CPU，从 Willamette 核心的 Pentium 4 开始采用 SIMD 技术，基于 SSE 开发了第 2 代浮点多媒体处理器指令集 SSE2，扩展了双精度浮点数并行处理能力。

SSE2 指令共 144 条，通过支持多种数据类型（如双字和四字）的算术运算，使最初的 SSE 指令得到增强，支持灵活且动态范围更广的计算功能。同时，随 MMX 技术引进的 SIMD 整数指令从 64 位扩展到了 128 位，使 SIMD 整数类型操作的有效执行率成倍提高。最具体的特点是，SSE2 中增加了 64 位双精度浮点 SIMD 指令，允许以 SIMD 格式同时执行两个浮点操作，提供双倍精度操作支持；加上对 Cache 的控制指令，使得 SSE2 非常适合 3D 渲染、图形驱动、游戏和多媒体编码等应用。

2004 年，Intel 公司发布了 Prescott 核心的 Pentium 4 处理器，SSE 家族又添新丁 SSE3。SSE3 只有 13 条指令，其中 10 条用于完善 MMX、SSE、SSE2 指令，1 条用于 x87 浮点处理单元编程中浮点数转换为整数的加速，另外 2 条用于加速线程的同步。这些新增指令强化了 CPU 在浮点数到整数的转换（不需要进行运算模式切换）、复杂算法（DSP 及 3D）、视频编码、SIMD 浮点寄存器操作以及线程同步等 5 方面的表现，达到提升多媒体和游戏性能的目的。

后续 SIMD 扩展需要区分 SSE3 与 SSSE3：SSE3 最早随 Pentium 4 Prescott 引入；SSSE3（Supplemental Streaming SIMD Extensions 3）随后随 Core 微架构处理器普及，它不是“SSE3 的另一名称”。SSE4 又分为 SSE4.1 与 SSE4.2：前者随 Penryn 引入 47 条指令，后者随 Nehalem 增加 7 条指令。不同扩展并不由处理器名称自动保证，软件仍应通过 CPUID 检测后再选择相应代码路径。
