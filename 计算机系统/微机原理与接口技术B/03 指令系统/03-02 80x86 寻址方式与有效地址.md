## 3.2 寻址方式

### 3.2.1 寻址方式与有效地址 EA 的概念

> [!note] 有效地址 EA 与物理地址 PA
> 有效地址 **EA**（Effective Address）是段内的偏移地址；8086 实模式下物理地址 **PA = 段基址 × 16 + EA**。绝大多数寻址方式解决的是"如何计算 EA"。

寻址方式是指令中用来说明操作数怎样存放以及如何寻找操作数的方式。指令中的操作数有 3 种可能的存放位置：

| 存放位置 | 操作数类型 | 对应寻址方式 |
| :--- | :--- | :--- |
| 操作数包含在指令中 | 立即数 | 立即寻址 |
| 操作数存放在 CPU 内部寄存器中 | 寄存器操作数 | 寄存器寻址 |
| 操作数在内存的数据区中 | 存储器操作数 | 存储器寻址 |

> [!info] 逻辑地址与 MMU
> CPU 指令中给出的存储器地址都是**逻辑地址**，逻辑地址与实际物理地址之间的映射由内存管理单元 MMU 完成。以 80x86/Pentium 为例，若内存中某单元的逻辑地址用 `DS:TABLE` 表示：
> - **实地址模式**：DS 中是段基地址的高 16 位（段地址），TABLE 为 16 位段内偏移量；
> - **保护模式**：DS 中存放指向段描述符的 16 位段选择符，通过它得到 32 位段基地址；TABLE 为 32 位段内偏移量。

为了适应各种数据结构的需要，80x86/Pentium 规定段内偏移量可由几个基本部分组合而成，称为**有效地址 EA**。

> [!important] 有效地址四分量与计算公式
> 组成有效地址 EA 的 4 个基本部分（称为有效地址四分量）：
> - **基址寄存器内容**：通常为某局部存储区的起点；
> - **变址寄存器内容**：通常为某局部存储区的起点；
> - **位移量**：指令中的 `disp` 字段，是具体数值，也可以是符号常量或变量；
> - **比例因子**：32 位寻址方式中特有的部分。
>
> 计算公式：
> $$\text{EA}=(\text{基址寄存器})+(\text{变址寄存器} \times \text{比例因子})+\text{位移量}$$

**表 3-1　有效地址 4 种分量的使用规则**

| 有效地址分量 | 16 位寻址 | 32 位寻址 |
| :--- | :--- | :--- |
| 基址寄存器 | BX, BP | 任何 32 位通用寄存器（EAX/EBX/ECX/EDX/ESI/EDI/EBP/ESP） |
| 变址寄存器 | SI, DI | 除 ESP 外的任何 32 位通用寄存器（EAX/EBX/ECX/EDX/ESI/EDI/EBP） |
| 比例因子 | 1 | 1, 2, 4, 8 |
| 位移量 | 0, 8, 16（位） | 0, 8, 32（位） |

### 3.2.2 80x86/Pentium 各种寻址方式

> [!info] 寻址方式数量
> 80x86/Pentium CPU 指令系统的寻址方式有 **11 种**；对 8086/8088/80286（16 位寻址）只有 **8 种**；80386/80486 及 Pentium 系列除 16 位寻址外还支持 32 位寻址方式。以下介绍均以指令的**源操作数**为对象进行分析。

#### 1. 立即寻址

> [!abstract] 立即寻址
> 操作数作为立即数直接包含在指令中，紧跟在操作码之后并一起存放在代码段区域。立即数总是和操作码一起被存入 CPU 的指令队列，**指令执行时不再需访问存储器**。

> [!info] 立即数位数与存放规则
> 立即操作数可以是 8 位、16 位或 32 位：
> - 16 位时，低位字节存放在相邻 2 字节存储单元的低地址单元中；
> - 32 位时，低位字存放在相邻 2 字存储单元的低地址单元中。
>
> 立即寻址方式**仅用于源操作数**，常用来给寄存器赋初值。

```asm
MOV  BL, 12H        ; 将立即数 12H 送入 BL
MOV  AX, 1234H      ; 将立即数 1234H 送入 AX
MOV  EDX, 12345678H ; 将立即数 12345678H 送入 EDX
```

#### 2. 寄存器寻址方式

> [!abstract] 寄存器寻址
> 操作数存放在指令规定的 8 位、16 位或 32 位 CPU 内部寄存器中。这是 CPU 内部操作，不需要使用访问总线周期，**指令执行速度较快**。

```asm
INC  CL        ; CL ← CL+1
MOV  DS, AX    ; DS ← AX
MOV  ECX, EAX  ; ECX ← EAX
```

#### 3. 存储器寻址

存储器寻址时，操作数在存储区中，指令的操作数部分指出此操作数的有效地址 EA，而段地址在默认的或段超越前缀指定的段寄存器中。根据 EA 的不同组合方式，存储器寻址方式可分为以下 9 种。

##### 1. 直接寻址

> [!abstract] 直接寻址
> 指令中操作数部分直接给出操作数的有效地址 EA，它是 16 位或 32 位的位移量数据，与操作码一起放在代码段中。操作数一般在数据段（DS）中（默认）；若要访问其他段（CS、ES、SS、FS、GS）中的数据，应在指令中增加**段超越前缀**指出段寄存器名。

```asm
MOV  AX, [2000H]    ; 将 DS 段中 2000H 和 2001H 单元内容分别送 AL 和 AH
MOV  AX, ES:[2000H] ; 将 ES 段中 2000H 和 2001H 单元内容分别送 AL 和 AH
```

汇编语言程序中，直接寻址中的有效地址 EA 也可用变量名形式给出：

```asm
VALUE  DB  12H           ; 定义变量 VALUE 并赋初值
MOV    AL, VALUE         ; 将变量 VALUE 所指的字节单元内容 12H 存入 AL
```

##### 2. 寄存器间接寻址

> [!abstract] 寄存器间接寻址
> 操作数在存储区内，操作数的有效地址 EA 在指令指定的寄存器中，即 $\text{EA}=(\text{寄存器})$。寄存器的使用规定在 16 位寻址和 32 位寻址时不一样。

**16 位寻址时**：EA 放在 SI、DI、BP、BX 中，有两种段默认情况：

| 寄存器 | 默认段 | 示例 |
| :--- | :--- | :--- |
| SI、DI、BX | DS | `MOV AX, [SI]` |
| BP | SS | `MOV AX, [BP]` |

```asm
MOV  CX, DS:[BP]  ; "DS:"是该指令的段超越前缀
```

**32 位寻址时**：8 个 32 位通用寄存器均可作为寄存器间接寻址：

| 寄存器 | 默认段 |
| :--- | :--- |
| EBP、ESP | SS |
| 其余 6 个通用寄存器 | DS |

```asm
MOV  CH, [EAX]
MOV  DX, [EBP]
```

> [!tip] 直接寻址 vs 寄存器间接寻址
> - **直接寻址**：有效地址 EA 来自指令自身，相当于一个**常量**；
> - **寄存器间接寻址**：有效地址 EA 来自寄存器，寄存器的内容由前面的指令确定，相当于一个**变量**。

##### 3. 基址寻址

> [!important] 基址寻址 EA 公式
> $$\text{EA}=(\text{基址寄存器})+\text{位移量}$$
> 位移量在指令中给出并与操作码一起存放在代码段中。

| 寻址模式 | 基址寄存器 | 默认段寄存器 | 位移量位数 |
| :--- | :--- | :--- | :--- |
| 16 位寻址 | BP / BX | BP→SS，BX→DS（可段超越） | 8 或 16 位 |
| 32 位寻址 | 8 个 32 位通用寄存器 | ESP、EBP→SS，其余→DS（可段超越） | 8 或 32 位 |

```asm
MOV  AX, [BX+24]          ; 也可写成 MOV  AX, 24[BX]
MOV  ECX, [EBP+50]        ; 也可写成 MOV  ECX, 50[EBP]
MOV  DX, [EAX+TABLE]      ; 或写成 MOV  DX, TABLE[EAX], TABLE 是符号常量或变量
```

##### 4. 变址寻址

> [!important] 变址寻址 EA 公式
> $$\text{EA}=(\text{变址寄存器})+\text{位移量}$$
> 指令格式及寻址过程与基址寻址相同，区别仅在于将基址寄存器改成变址寄存器。

| 寻址模式 | 变址寄存器 | 默认段寄存器 |
| :--- | :--- | :--- |
| 16 位寻址 | SI、DI | DS |
| 32 位寻址 | 除 ESP 外的任何 32 位通用寄存器 | EBP→SS，其余→DS |

```asm
MOV  AX, COUNT[SI]      ; 16 位寻址示例
MOV  EAX, 5[EBP]        ; 32 位寻址示例
MOV  ECX, DATA[EAX]
```

> [!tip] 基址/变址寻址的应用
> 基址或变址寻址**非常适合对一维数组元素进行检索操作**：常用位移量表示数组起始地址，基址或变址寄存器表示数组元素的可变下标，通过修改寄存器内容可方便地访问数组中的任意元素。

##### 5. 比例变址寻址

> [!important] 比例变址寻址 EA 公式
> $$\text{EA}=(\text{变址寄存器})\times\text{比例因子}+\text{位移量}$$
> 乘以比例因子的操作由 CPU 内部硬件完成。**只适合 32 位寻址**。

```asm
MOV  EAX, TABLE[ESI*4]
```

> [!tip] 适用场景
> 比例变址寻址更适合对**多字节一维数组**进行检索：当数组元素大小为 2、4、8 字节时，可取比例因子 2、4、8，访问更方便、高效。

##### 6. 基址加变址寻址

> [!important] 基址加变址寻址 EA 公式
> $$\text{EA}=(\text{基址寄存器})+(\text{变址寄存器})$$
> 当基址、变址寄存器默认的段寄存器不同时，**一般由基址寄存器决定默认段寄存器**作为段地址。

```asm
MOV  AX, [BX+SI]        ; 或写成 MOV  AX, [BX][SI], BX 决定默认 DS 为段寄存器
MOV  EAX, [EDX][EBP]    ; 由 EBP 决定默认 SS 为段寄存器
```

> [!tip] 适用场景
> 基址加变址寻址适合**检索二维数组元素**和**双重循环**等操作。

##### 7. 基址加比例变址寻址

> [!important] 基址加比例变址寻址 EA 公式
> $$\text{EA}=(\text{基址寄存器})+(\text{变址寄存器}\times\text{比例因子})$$
> **只适合 32 位寻址**。

```asm
MOV  ECX, [EDX*4][EAX]      ; 或  MOV  ECX, [EDX*4+EAX]
MOV  AX,  [EBX*8][ESI]      ; 或  MOV  AX,  [EBX*8+ESI]
```

> [!tip] 适用场景
> 适合检索**多字节二维数组元素**等操作。

##### 8. 带位移量的基址加变址寻址

> [!important] 带位移量的基址加变址寻址 EA 公式
> $$\text{EA}=(\text{基址寄存器})+(\text{变址寄存器})+\text{位移量}$$
> 也分 16 位寻址和 32 位寻址两种情况。

```asm
MOV  AX, [BX+SI+MASK]      ; 或  MOV  AX,  MASK[BX][SI]
```

> [!tip] 适用场景
> 适合对**二维数组**操作，位移量即为数组起始地址。

##### 9. 带位移量的基址加比例变址寻址

> [!important] 带位移量的基址加比例变址寻址 EA 公式
> $$\text{EA}=(\text{基址寄存器})+(\text{变址寄存器}\times\text{比例因子})+\text{位移量}$$
> 用到有效地址的全部 4 个分量，**只有 32 位寻址一种情况**。变址寄存器内容乘以比例因子的操作在 CPU 内部由硬件完成。

```asm
MOV  AX, [EDI*8+ECX+40]      ; 或  MOV  AX,  [EDI*8][ECX+40]
```

> [!tip] 适用场景
> 适合**起始地址不为零**的多字节二维数组元素的检索操作。

### 3.2.3 80x86/Pentium 存储器寻址的段约定

进行存储器操作数访问时，指令中一般不特别指出段寄存器，这是因为对各种类型的存储器寻址，80x86/Pentium 中关于段和偏移地址寄存器有基本默认约定。

**表 3-2　存储器操作数访问中关于段和偏移地址寄存器的约定**

| 访问存储器类型 | 默认段寄存器 | 允许超越的段寄存器 | 偏移地址寄存器 |
| :--- | :--- | :--- | :--- |
| 取指令 | CS | 无 | (E)IP |
| 堆栈操作 | SS | 无 | (E)SP |
| 源串数据访问 | DS | CS、ES、SS (FS、GS) | (E)SI |
| 目的串数据访问 | ES | 无 | (E)DI |
| 通用数据访问 | DS | CS、ES、SS (FS、GS) | 偏移地址 EA |
| 以(E)BP、(E)SP 间接寻址的指令 | SS | CS、DS、ES (FS、GS) | 偏移地址 EA |

> [!warning] 段超越的限制
> 表 3-2 说明，除了**程序只能在代码段**、**堆栈操作数只能在堆栈段**、**目的串操作数只能在附加数据段**，其他约定都是允许段超越的。

### 3.2.4 几种处理器寻址方式比较

> [!note] RISC 与 CISC 的寻址差异
> MIPS 或 ARM 等 RISC 处理器的指令条数和寻址方式少，因而指令的译码速度快，控制逻辑规整简单。MIPS 及 ARM 与 80x86/Pentium 指令在寻址方式上的重要不同就是存储器寻址——由于前者存储器操作数的地址分量少，所以它们的寻址方式较为简洁。

**表 3-3　各种处理器寻址方式比较**

| | | 80x86/Pentium | MIPS | ARM |
| :--- | :--- | :---: | :---: | :---: |
| **立即寻址** | | √ | √ | √ |
| **寄存器寻址** | | √ | √ | √ |
| **存储器寻址方式** | 直接寻址 | √ | √ | √ |
| | 寄存器间接寻址 | √ | √ | √ |
| | 基址寻址 | √ | √ | √ |
| | 变址寻址 | √ | | |
| | 比例变址寻址 | √ | | |
| | 基址加变址寻址 | √ | | |
| | 基址加比例变址寻址 | √ | | |
| | 带位移量的基址加变址寻址 | √ | | |
| | 带位移量的基址加比例变址寻址 | √ | | |
| | 多寄存器寻址 | | | √ |
| | 寄存器移位寻址 | | | √ |

在 ARM 中，定义了不同工作模式下的多组通用寄存器 R0-R15，它们都可作为基址寄存器。除了与 80x86/Pentium 的一些相同寻址方式，ARM 增加了专有的多寄存器和寄存器移位寻址方式。MIPS 的所有通用寄存器都可以定义为基址寄存器或变址寄存器。

> [!example] ARM 多寄存器寻址
> ```asm
> STMIA  R0!, {R1-R4}  ; [R0]←R1, [R0+4]←R2, [R0+8]←R3, [R0+12]←R4
>                      ; 将 R1-R4 的内容送到以 R0 为首地址的连续内存区域并更新 R0
> ```

> [!example] ARM 寄存器移位寻址
> ```asm
> ADD  R0, R1, R2, LSL #1  ; R0←R1+(R2<<1), 将 R2 左移一位后与 R1 相加，结果存 R0
> ```

> [!example] MIPS 基址加变址寻址
> ```asm
> ADD  $1, ($2+$3)    ; R1←R1+[R2+R3], 将内存地址 R2+R3 中的值与寄存器 R1 相加，结果保存到 R1
> ```
