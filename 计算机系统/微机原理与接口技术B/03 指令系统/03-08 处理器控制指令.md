## 3.3.6 处理器控制类指令

处理器控制类指令可分为三组，如表 3-13 所示。它们都是无操作数指令，除了对指定的标志位进行操作，对别的标志位没有影响。

**表 3-13　处理器控制类指令**

| 类 别 | 指令功能 | 指令书写格式(助记符) |
| :--- | :--- | :--- |
| **标志位操作** | 进位标志置 1 | STC |
| | 进位标志复位 | CLC |
| | 进位标志取反 | CMC |
| | 方向标志置 1 | STD |
| | 方向标志复位 | CLD |
| | 中断标志置 1 | STI |
| | 中断标志复位 | CLI |
| **外同步** | 停机 | HLT |
| | 等待 | WAIT |
| | 封锁总线 | LOCK |
| | 交权 | ESC |
| **空操作** | 空操作 | NOP |

### 1. 标志位操作指令

`CLC/STC/CMC`: CLC 和 STC 指令使进位标志 CF 置为 0 或 1，CMC 使标志 CF 取反。
`CLD/STD`: 将方向标志 DF 置为 0 或 1，以控制串操作指令执行时 SI 和 DI 是增量还是减量。
`CLI/STI`: 将中断标志 IF 置为 0 或 1，以控制 CPU 是否响应可屏蔽中断 INTR 引线上出现的外部中断请求。CLI 禁止 CPU 响应（$\text{IF}=0$），STI 允许 CPU 响应（$\text{IF}=1$）。

### 2. 外同步类指令

外同步类指令的作用是实现 CPU 与外部事件的同步，共 4 条。

#### 1. 处理器暂停指令 HLT

HLT 指令使 CPU 进入暂停状态，不做任何操作，程序停止执行。只有当 CPU 复位或外部中断发生时，CPU 才能退出暂停状态。

对含有外部中断的系统进行程序调试时，常用 HLT 指令模拟主程序。当中断到来使 CPU 退出暂停而去执行中断服务程序时，当前的断点地址 CS:IP 指向 HLT 的下一条指令，在中断服务完成返回时，返回到此断点处继续程序的执行。

#### 2. 处理器等待指令 WAIT

WAIT 指令在 CPU 的 TEST 引脚无效（高电平）时，使 CPU 进入等待状态。在等待状态，CPU 没有任何操作，直到出现外部中断为止。当外部中断发生时，进栈保护的 IP 值就是 WAIT 指令所在单元的地址。所以中断结束后又返回到等待状态，直到 TEST 引脚上信号有效（低电平），CPU 才退出等待，执行下一条指令。

#### 3. 总线锁定前缀 LOCK

LOCK 是一个单字节的前缀，可以加在任何指令前面，使 CPU 在执行这条指令期间保持输出引脚 LOCK 有效（低电平），即将总线封锁使其他控制器不能控制总线，直到该指令执行完后，总线封锁解除。在多处理器系统中，可以用这个前缀来对共享资源进行控制。

#### 4. 处理器交权指令 ESC

ESC 指令将浮点指令交给浮点运算协处理器执行。协处理器的运算指令是与 CPU 的指令组合在一个指令流中运行的，所有的协处理器运算指令的前几位编码，对于 CPU 来说就是一条 ESC 指令。当执行 ESC 指令时，80x86/Pentium CPU 就把控制权交给协处理器，以完成协处理器运算指令的操作。

### 3. 空操作指令 NOP

NOP 指令的作用是使 CPU 完成一次空操作，仅使指令指针 IP 加 1，不做任何其他操作，空耗一个指令执行周期。与 HLT 指令不同，NOP 指令执行后，CPU 继续执行其后的指令。
