## 1.1.4 微型计算机性能的评估指标

> [!abstract] 性能评估需要综合多项指标
> 一台微型计算机性能的优劣是由其系统结构、指令系统、硬件组成、外部设备、软件配备是否齐全等因素决定的。只有综合各项指标才能正确评价与衡量计算机性能的高低。

主要评估指标如下：

| 序号 | 指标 | 主要反映的维度 |
| :---: | :--- | :--- |
| 1 | CPU 字长 | 一次能处理的二进制位数 |
| 2 | 内存储器与高速缓存 | 存储容量与访问速度 |
| 3 | CPU 指令执行时间 | 运算速度 |
| 4 | 系统总线的传输速率 | I/O 带宽 |
| 5 | 优化的内部结构 | 微体系结构技术 |
| 6 | I/O 设备配备情况 | 外设性能 |
| 7 | 软件配备情况 | 系统软件齐全度 |

### 1. CPU 字长

> [!abstract] 字长的定义
> 计算机的字长决定了计算机内部一次可以处理的二进制代码的位数。它决定着计算机通用寄存器、加法器、数据总线等部件的位数，因此直接影响硬件成本。

一般地，一台计算机的字长由下列一个或两个条件所决定：

1. ALU 一次能处理的二进制编码位数；
2. 数据总线的宽度；
3. 内存储器的字长。

> [!important] 字长与精度、速度的关系
> - 字长越长，一个字能表示的数据精度越高。
> - 在完成同样精度的运算时，字长较长的计算机比字长短的计算机速度快。
> - 为了兼顾精度与硬件成本，有些计算机允许采用变字长运算。

> [!info] 内部字长与外部总线宽度可不一致
> 一般情况下，CPU 内部数据总线宽度和计算机对外的数据总线宽度是一致的。早期有些 CPU 为了配合外设的总线宽度，设计内部字长和外部数据总线宽度不一致：
> - **Intel 8088**：内部总线宽度为 $16$，外部数据总线宽度是 $8$ 位，称为"准 $16$ 位"芯片；
> - **Pentium**：字长为 $32$ 位，但访问内存的外部数据总线宽度为 $64$ 位，以提高访问速度。

### 2. 内存储器与高速缓存

> [!info] 内存储器容量
> 内存储器容量是衡量存储二进制信息量的重要指标，通常用字节表示（如 $64 \text{ MB}$）。容量大小根据用途、成本及价格等因素选择。

> [!abstract] 内存储器的速度指标
> 内存储器的速度可用下列指标衡量：
> - **访问延迟**：单次访问所需的时间；
> - **周期时间**：连续两次独立访问所需的最小时间间隔；
> - **带宽**：持续传输数据的速率。
>
> 存取周期表示存储器连续启动两次独立访问所需的最小时间间隔，不能与单次访问延迟或持续传输带宽混为一谈；不同 SRAM、DRAM 及其工作方式的指标差异很大，应结合器件和接口时序分析。

> [!abstract] 高速缓存（Cache）原理
> CPU 与主存之间存在明显的速度差距，高速缓存利用程序访问的**时间局部性**和**空间局部性**降低平均访存延迟。
> - **时间局部性**：最近访问过的数据很可能再次被访问；
> - **空间局部性**：地址相近的数据很可能在近期被访问。

> [!warning] 缓存容量不是唯一因素
> 缓存容量只是影响性能的因素之一，命中率、延迟、相联度、替换策略、层级组织与一致性开销同样重要；容量增大也可能带来更长访问延迟和更高面积、功耗成本。

### 3. CPU 指令执行时间

> [!info] 指令执行时间的意义
> 指令执行时间的长短反映 CPU 运算速度的快慢。因为执行不同的指令所需的时间不同，就产生了如何评估计算速度的问题。常用的方法有如下两种。

#### 1. 指令执行的平均速度值

> [!abstract] MIPS
> 根据不同类型指令在计算过程中出现的频繁程度乘上不同系数，可求得统计平均值，这里的运算速度指平均速度 **MIPS**（Millions of Instruction Per Second），以百万条指令/秒为单位。
>
> 其全名为 Drytone V1.1 MIPS，即运行一个名为 Drytone 的测试程序，测试性能规范到 VAX1.1 计算机。如 486DX 性能相当于每秒执行 2700 万条指令的 VAX1.1 计算机，称为 27VAX MIPS。

#### 2. CPU 的主频

> [!important] 主频
> 主频是 CPU 的时钟频率，很大程度上决定了计算机的运算速度，是 CPU 的标称频率。

| CPU 型号 | 时钟频率范围 |
| :--- | :--- |
| 80486 | $33 \sim 66 \text{ MHz}$ |
| Pentium | $60 \sim 133 \text{ MHz}$ |
| Pentium MMX | $230 \text{ MHz}$ 以上 |

> [!info] 倍频技术
> 在 80486 CPU 出现后，由于 CPU 频率不断提高，外接设备的速度却受到工艺的限制，不能承受更高的频率，因此限制了 CPU 频率的进一步提高，出现了**倍频技术**：
> - 使 CPU 内部工作频率变为外部频率的倍数；
> - 可通过提升倍频而达到提升主频的目的。

> [!warning] 频率提升的瓶颈
> 当主频升至 $3 \text{ GHz}$ 后，功耗和散热等因素使得频率不适宜进一步提升。为了优化处理器的工作速度，采用的解决办法是**多核设计**。

### 4. 系统总线的传输速率

> [!abstract] 总线传输速率
> 系统总线的有效传输速率会直接影响 I/O 性能，其上限与每次传输的数据宽度、传输频率、编码开销、协议开销和通道数量有关。
> - 并行 PCI 的理论峰值可用"总线频率 × 总线宽度"估算，例如 33 MHz、32 位 PCI 的峰值约为 $133\ \text{MB/s}$；
> - PCI Express（PCIe）采用点到点串行链路，带宽必须同时注明**代际、通道数和方向**；不能仅凭"PCIe ×16"给出一个跨代通用数值；
> - 实际吞吐量通常低于理论峰值。

> [!tip] 带宽计算口径
> 比较总线性能时，应先确认数值表示单向还是双向、原始传输率还是有效载荷速率，以及使用 MB/s（$10^6$ B/s）还是 MiB/s（$2^{20}$ B/s）。

### 5. 优化的内部结构

CPU 体系的发展过程中采用了许多新技术，优化了其软件、硬件结构，提升了 CPU 的整体性能。

> [!important] 两项关键优化技术
> - **指令流水线结构**：提升系统的并行处理能力，加快了指令执行时间；
> - **高速缓存（Cache）结构**：加快了 CPU 访问数据和指令的速度。
>
> 详见 [[01-03 软件系统与指令执行过程#3. 指令的流水线执行|指令的流水线执行]]。

高端 CPU 还往往在其设计实现中进行了多方面、多层次的优化，显著提升了其性能：

| 优化技术 | 说明 |
| :--- | :--- |
| 超流水线 | CPU 内部的流水线超过通常的 $5 \sim 6$ 步，如 Pentium CPU 的流水线为 $14$ 步。流水线设计的步数越多，完成一条指令的速度越快 |
| 超标量 | CPU 中有 $1$ 条以上的流水线，并且每个时钟周期内可以完成一条以上的指令 |
| 乱序执行 | 高端 CPU 进行动态处理的核心技术 |
| 分支预测执行 | 取指同时寻找将要执行的指令，提前判读并执行有可能需要的指令 |

> [!info] 动态处理
> 动态处理采用多路分流预测算法，在取指同时寻找将要执行的指令。该技术可加速向处理器传送任务，通过提前判读并执行有可能需要的指令的方式提高执行速度，可使高端 CPU 的处理能力得到充分的发挥，从而提升系统性能。

### 6. I/O 设备配备情况

> [!info] I/O 设备对综合性能的影响
> 在现代计算机系统中，I/O 设备配备的性能对整体系统的综合表现具有重大的影响。在微型计算机系统中，外存储器容量、打印机型号、显示屏幕的分辨率，都是 I/O 设备选用中要考虑的问题。

### 7. 软件配备情况

> [!info] 软件配备
> 软件是计算机系统不可缺少的重要组成部分，配备是否齐全直接关系到计算机性能的好坏、效率的高低。这里主要指操作系统、高级语言和汇编语言的种类、应用软件等。
