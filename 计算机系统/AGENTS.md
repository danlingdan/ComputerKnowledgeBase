# 计算机系统局部规则

- 明确内容所在抽象层：数字逻辑、微体系结构、指令集、操作系统、运行时、编译器或应用；跨层结论说明接口边界。
- 区分机制与策略、规范与实现、逻辑视图与物理实现、用户态与内核态；不要把某一系统实现写成普遍规律。
- 操作系统内容说明资源对象、系统状态、不变量、并发条件、权限和失败路径；调度或置换算法同时记录评价指标。
- 系统代码注明 Linux、Windows 或其他平台，注明 POSIX、Win32、ABI/API、编译器、架构和权限要求中的适用项。
- 并发示例说明共享状态、同步关系、可能的竞态、死锁条件和退出清理；未经验证不要声称线程安全。
- 体系结构内容注明位宽、字节序、存储层级、指令集或处理器代际；性能数据必须附测试条件或来源。
- 编译原理内容区分语言规范、编译阶段、中间表示、优化前提和目标平台，并链接形式语言与体系结构先修知识。
- 导入课程长篇原稿时先作为课程记录保留，再提炼稳定主题笔记；不要直接拆分、覆盖或大幅重写原稿。
